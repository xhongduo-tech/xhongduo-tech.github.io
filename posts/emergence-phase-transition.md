## 核心结论

把涌现能力理解成“相变”是一个有解释力的视角。相变指系统在某个临界点附近，整体状态突然从一种形态切到另一种形态。放到模型里，可以把参数规模 $N$ 近似看成控制变量，把某项能力是否被稳定学会看成序参量 $b$。序参量就是“系统是否进入有序状态的量”，白话说，就是一个能直接表示“这项能力到底有没有成形”的指标。

这个视角下，核心判断不是“指标有没有缓慢提高”，而是“是否跨过临界规模 $N_c$”。当交叉熵损失随规模进入幂律下降区间时，某些复杂能力对应的 $b$ 可能从接近 0 突然变成显著非零。这和 Ising 模型中温度降到临界温度 $T_c$ 后，磁化强度从 0 变成非 0 是同一种数学结构：前者是能力形成，后者是宏观有序。

一个对初学者足够直观的类比是：

- 模型规模 $N$ 类似“降温”过程
- 能力指标 $b$ 类似“磁化强度”
- 小模型像高温无序态，很多局部模式存在，但拼不成稳定技能
- 大模型跨过 $N_c$ 后，底层技能开始协同，复杂技能突然可用

如果进一步接受统计物理中的“普适类”概念，那么不同模型只要共享关键对称性和有效维度，就可能表现出相近的临界行为。普适类指“很多微观细节不同，但宏观临界规律相同的一类系统”。这并不等于“所有大模型都一样”，而是说只有在对称性、深度结构、约束项和训练动力学足够相似时，跨模型比较临界指数才有意义。

## 问题定义与边界

“涌现”在工程讨论里经常被滥用。这里给出一个更窄、也更可操作的定义：当模型规模扩大后，某项能力对应的序参量 $b$ 在临界点附近从接近 0 跃迁到稳定非零，并且这种变化不是单纯由度量阈值或评测噪声造成，而是伴随损失曲线进入新的缩放区间，这才算相变意义上的涌现。

因此要先区分三件事：

| 现象 | 是否算相变式涌现 | 原因 |
|---|---:|---|
| 指标线性变好 | 通常不算 | 更像连续改进，不一定有临界点 |
| 指标因评测阈值突然跳高 | 不一定 | 可能只是度量函数非线性 |
| 损失缩放改变且 $b$ 从 0 到非 0 | 更接近 | 说明系统状态本身变了 |

在这个边界里，小模型和大模型的差别不是“好一点”或“差一点”，而是是否处在不同相。相就是系统的稳定工作区。对能力来说，可以粗分成：

- trivial phase：平凡相，$b=0$，模型只学到表面统计关系，任务能力不成形
- learning phase：学习相，$b>0$，模型开始稳定组织出完成任务所需的内部结构

一个玩具例子是括号匹配。小模型可能能记住短序列的表面模式，但在长度稍长时准确率接近随机；当规模超过某一门槛，内部状态开始稳定追踪“未闭合括号数”，这时准确率会突然升高。这里的序参量可以定义为“比随机基线高出的准确率”：
$$
b = \mathrm{Acc} - \mathrm{Acc}_{\text{baseline}}
$$
若 $b \approx 0$，说明能力没有成形；若 $b$ 稳定大于 0，则说明模型进入学习相。

要注意一阶和二阶转变的区别。一阶转变指状态会突然跳变，并可能伴随亚稳态；二阶转变指状态连续变化，但导数在临界点奇异。亚稳态就是“看起来稳定，其实不是全局最优的坑”。在高维损失地形里，训练可能停在 $b=0$ 附近的局部盆地，因此“模型没学会”不一定说明理论上不存在学习相，也可能只是没跨过势垒。

## 核心机制与推导

这个视角通常把两类结果拼起来看。

第一类是 scaling law。它说，当别的因素不成为瓶颈时，损失会随模型规模按幂律下降，可写成
$$
L(N) \approx \left(\frac{N_c}{N}\right)^{\alpha_N}
$$
其中 $\alpha_N$ 是规模指数，表示“模型变大一倍，损失大概按多快下降”。如果取文献里常被引用的经验量级 $\alpha_N \approx 0.076$，那么从 $L=1.2$ 降到 $L=1.0$ 需要的参数倍数是
$$
\frac{N_2}{N_1}=\left(\frac{L_1}{L_2}\right)^{1/\alpha_N}
=\left(\frac{1.2}{1.0}\right)^{1/0.076}\approx 11
$$
这说明跨过某个能力门槛，可能不是“再加一点参数”，而是要大幅扩容。

第二类是 Landau 型有效损失。Landau 理论的做法是，不去追踪所有微观自由度，而是直接写一个关于序参量 $b$ 的有效势函数。对深网络相变分析，一个代表性形状是
$$
F(b)\supset c_0 b^{4D}-c_1 b^{2D}+c_2 b^2,\qquad c_0,c_1,c_2>0
$$
这里 $D$ 可理解为有效深度或有效相互作用阶数。关键不在系数细节，而在幂次结构。

为什么 $D>1$ 容易出现跳跃？因为当 $D=1$ 时，主导项更像传统连续转变；而当 $D>1$ 时，中间的 $-c_1 b^{2D}$ 会让势函数在 $b=0$ 之外突然形成更低的新极小值。于是系统不是沿着一条平滑路径慢慢离开 0，而可能在某个点直接从 $b=0$ 跳到某个有限的 $b^\star$。这就是一阶相变式的能力出现。

可以把图像想成这样：

- 小规模时，$F(b)$ 在 $b=0$ 只有一个盆地，模型停在“不会”
- 接近临界点时，远处开始出现第二个盆地，但中间有势垒，训练可能卡住
- 规模再增大，非零盆地变成更深的全局最优，模型突然切到“会”

这也是为什么“loss 变小”与“能力出现”不能完全等价。loss 下降给了系统进入新相的机会，但是否真的进入，还受初始化、正则化、优化噪声和数据结构影响。

再看 Ising 类比就更清楚。Ising 模型有一个 $\mathbb{Z}_2$ 对称性，也就是自旋整体翻转后能量不变。白话说，就是系统本来没有偏向“向上”还是“向下”。当温度高时，平均磁化为 0；降到临界点下方，对称性自发破裂，系统选中一个方向，磁化非零。模型中的某些能力形成也可以理解成一种“对称性破裂”：训练前很多内部表示是等价的，训练后系统选择了能组织技能组合的那一支稳定结构。

一个真实工程例子是组合技能任务。假设模型要同时完成实体抽取、指代消解和时间排序。小模型可能每个原子技能都学了一点，但三者联合时失败；当规模扩大、loss 降到某阈值后，这三个技能不再只是局部碎片，而开始形成可复用的中间表示，联合任务准确率会出现比单技能更突兀的跳升。这正是 Arora 与 Goyal 所讨论的“复杂技能组合并不一定比基础技能晚很多出现”的关键点。

## 代码实现

下面给一个最小可运行脚本，用几组规模-损失数据估计 $\alpha_N$，再根据一个能力指标判断是否跨过学习相。这里的能力指标用玩具形式定义为“超过随机基线的准确率”。

```python
import math
from dataclasses import dataclass

@dataclass
class Record:
    N: float
    loss: float
    acc: float

def fit_alpha(records):
    # 用两点近似拟合 L = A * N^{-alpha}
    r1, r2 = records[0], records[-1]
    alpha = (math.log(r1.loss) - math.log(r2.loss)) / (math.log(r2.N) - math.log(r1.N))
    A = r1.loss * (r1.N ** alpha)
    return alpha, A

def predict_scale_up(loss_start, loss_target, alpha):
    return (loss_start / loss_target) ** (1.0 / alpha)

def order_parameter(acc, baseline=0.5):
    # 序参量：比随机基线高出的准确率
    return max(0.0, acc - baseline)

def in_learning_phase(b, threshold=0.08):
    return b >= threshold

records = [
    Record(N=1e8,  loss=1.32, acc=0.51),
    Record(N=3e8,  loss=1.19, acc=0.53),
    Record(N=1e9,  loss=1.05, acc=0.61),
    Record(N=3e9,  loss=0.98, acc=0.73),
]

alpha, A = fit_alpha(records)
scale_up = predict_scale_up(1.2, 1.0, alpha)

rows = []
for r in records:
    b = order_parameter(r.acc, baseline=0.5)
    rows.append((r.N, r.loss, round(b, 3), in_learning_phase(b)))

# 基本正确性检查
assert alpha > 0
assert scale_up > 1
assert rows[0][3] is False
assert rows[-1][3] is True

print("alpha =", round(alpha, 4))
print("estimated scale-up from loss 1.2 -> 1.0 =", round(scale_up, 2), "x")
for row in rows:
    print(row)
```

这段代码做了三件事：

1. 用对数线性化估计 $\alpha_N$
2. 根据目标损失估计所需扩容倍数
3. 同步记录序参量 $b$，判断是否进入学习相

如果输出类似下面的结果，就能看到“loss 连续下降，但 $b$ 是否跨阈值才决定是否进入学习相”：

| 参数规模 $N$ | loss | 序参量 $b=\mathrm{Acc}-0.5$ | 是否进入 learning phase |
|---:|---:|---:|---:|
| $1\times10^8$ | 1.32 | 0.01 | 否 |
| $3\times10^8$ | 1.19 | 0.03 | 否 |
| $1\times10^9$ | 1.05 | 0.11 | 是 |
| $3\times10^9$ | 0.98 | 0.23 | 是 |

真正做实验时，不要只记总 loss。至少同时记录：

- 主任务 loss
- 特定能力准确率
- 中间表征统计量，例如 embedding norm、某层激活秩、注意力稀疏度
- 不同随机种子的收敛结果

如果只看最终平均指标，一阶跳跃和亚稳态很容易被平均掉。

## 工程权衡与常见坑

第一类坑是把“门槛高”误读成“理论错”。按
$$
\left(\frac{1.2}{1.0}\right)^{1/0.076}\approx 11
$$
这个数量级估算，想把损失从 1.2 压到 1.0，可能需要约 11 倍参数扩容。对小团队来说，这不是一个可以靠微调学习率轻松补上的差距。很多所谓“突然涌现”，本质上只是之前一直没碰到临界区。

第二类坑是只看 loss，不看序参量。高维损失地形可能有 metastable basin，也就是亚稳盆地。模型 loss 已经下降，但仍停在 $b=0$ 附近，表现为“看起来收敛了，但能力没出来”。这在深层网络、强正则或不合适初始化下尤其常见。

第三类坑是把任何突变都解释成普适类。普适类要求关键对称性、有效维度和长程行为相似，不是看两条曲线都长得陡就能归到一起。比如一个由评测阈值造成的精度跳变，与一个由内部表示重组造成的相变，不应使用同一套临界语言。

真实工程里，扩散模型和 VAE 的训练能提供一个更直接的例子。$\beta$-VAE 中的 $\beta$ 是 KL 项权重，白话说就是“你有多强地要求潜变量服从先验分布”。当 $\beta$ 太高时，模型可能更愿意选择信息被压扁的平凡解，重建 loss 看着不坏，但潜变量几乎不携带有用结构；当适当降低 $\beta$，再配合数据扰动或更长训练，系统才更容易进入真正的学习相。

| 设置 | 常见相位 | 现象 | 风险 |
|---|---|---|---|
| 高 $\beta$、弱扰动 | trivial phase | 潜变量塌缩，能力被门控掉 | 误以为模型“稳定但无能” |
| 中等 $\beta$、合适扰动 | 临界附近 | 指标波动大，种子敏感 | 容易错判临界点 |
| 较低 $\beta$、足够数据支持 | learning phase | 表征非平凡，能力成形 | 训练成本上升 |

因此工程策略通常不是“盲目扩大模型”，而是先判断自己卡在哪一类瓶颈：

- 若 loss 明显高且没进入幂律区，先补数据、训练步数或优化配置
- 若 loss 已不错但 $b$ 仍接近 0，优先检查是否陷入亚稳态
- 若临界点确实远，才考虑扩容是否值得

## 替代方案与适用边界

“相变视角”不是唯一解释，也不该被当成默认真理。更稳妥的做法是把它当成一个诊断框架：先提出“这里可能存在临界点”的假设，再用小规模实验验证。

最实用的替代方案，是直接做多尺度拟合与能力探针。能力探针就是“专门测某项内部能力是否形成的指标”。例如：

- 任务准确率减随机基线：$b=\mathrm{Acc}-\mathrm{Acc}_{\text{baseline}}$
- 中间层表征范数或秩
- 某类推理链是否稳定出现
- 梯度统计是否在某层集中重组

比起直接相信某条通用 scaling law，先用多组 $N$、多种随机种子和固定数据分布去估计本任务自己的 $N_c$，通常更可靠。因为不同架构的有效深度、归一化方式、优化器和数据噪声，都可能改变临界位置，甚至改变转变阶数。

| 架构或场景 | 是否可直接套用同一普适类 | 需要先验证什么 |
|---|---|---|
| 同族 Transformer，仅规模变化 | 相对更可能 | 损失缩放、能力探针、随机种子稳定性 |
| 深度差异很大的网络 | 不能直接假设 | 有效深度 $D$ 是否改变转变阶数 |
| 强正则或强约束模型 | 风险较高 | 是否人为制造 trivial phase |
| 扩散/VAE 等生成模型 | 需单独验证 | 控制参量不只一个，$\beta$ 与噪声共同影响相位 |

适用边界也要说清楚。相变和普适类语言最适合描述“存在宏观集体行为重组”的情形。若任务指标只是评测函数本身带来的非线性，比如 exact match 这种过于硬阈值的度量，表面上的跃迁不一定对应真实相变。再进一步，若架构不满足相关对称性，或者训练过程始终受数据瓶颈支配，那么把临界指数拿来跨模型比较，结论往往不稳。

简化地说：

- 相变视角适合解释“为什么能力会突然出现”
- 但不自动回答“为什么恰好在这个规模出现”
- 更不能跳过实验，直接声称“所有模型共享某个 universality class”
