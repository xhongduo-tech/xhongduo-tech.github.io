## 核心结论

数据处理不等式，英文常写作 Data Processing Inequality，意思是“信息经过后续处理不会凭空变多”。如果 $X \to Y \to Z$ 构成马尔可夫链，白话解释是：$Z$ 只能通过 $Y$ 间接接触 $X$，不能绕过 $Y$ 偷看原始信息，那么一定有

$$
I(X;Z)\le I(X;Y)
$$

如果讨论的是两组分布在同一个处理过程下的可区分程度，KL 散度也满足同样结论。KL 散度，白话解释是“两个分布有多不一样”，则有

$$
D_{KL}(p_Z\|q_Z)\le D_{KL}(p_Y\|q_Y)
$$

这两个式子共同表达一个核心事实：后处理只能保留已有信息或丢失信息，不能创造新信息。

对新手最直观的理解是：把 $X$ 看成原始彩色图像，$Y$ 是灰度化后的特征，$Z$ 是再做一次池化后的结果。灰度化已经丢掉了颜色信息，池化又丢掉了空间细节，因此池化结果不可能比灰度图包含更多关于原图的信息。

这条结论在工程里非常硬。特征提取器、编码器、奖励模型、预测头、Dropout、量化器，本质上都是“映射”或“随机映射”。它们可以压缩信息、筛掉噪声、改变表示形式，但不能突破原始输入提供的信息上界。

## 问题定义与边界

先把定义说清楚。

互信息 $I(X;Y)$ 衡量的是“知道 $Y$ 以后，对 $X$ 的不确定性减少了多少”。如果 $X \to Y \to Z$ 是马尔可夫链，那么给定 $Y$ 后，$Z$ 与 $X$ 条件独立，即

$$
p(z|x,y)=p(z|y)
$$

这表示 $Z$ 的全部信息都来自 $Y$，因此 $Z$ 不可能比 $Y$ 更了解 $X$。

KL 版本则稍有不同。它比较的是两套分布 $p$ 和 $q$ 在同一个映射下经过处理后的差异。如果 $Y$ 通过某个确定性或随机映射变成 $Z$，那么映射后的可区分性不会比映射前更大。

一个最小玩具例子如下。令 $X=\{00,01,10,11\}$，四个取值等概率；$Y$ 取 $X$ 的首位；$Z=0$ 恒定不变。

| 变量 | 定义 | 取值分布 | 与 $X$ 的关系 |
|---|---|---|---|
| $X$ | 原始两位比特 | 四种情况各 $1/4$ | 原始信息 |
| $Y$ | 取首位 | $P(Y=0)=1/2,\ P(Y=1)=1/2$ | 保留了 1 bit |
| $Z$ | 恒为 0 | 概率 1 | 不保留信息 |

在这个例子里：

- $H(X)=2$ bit
- $H(X|Y)=1$ bit
- 所以 $I(X;Y)=H(X)-H(X|Y)=1$ bit
- 因为 $Z$ 恒定，$I(X;Z)=0$

因此确实有

$$
I(X;Z)=0 \le 1 = I(X;Y)
$$

这个例子很小，但已经说明“后处理只能损失”。

边界也必须说清楚：

| 条件 | DPI 是否成立 | 说明 |
|---|---|---|
| 确定性映射 $Z=f(Y)$ | 成立 | 函数变换是后处理 |
| 随机映射 $p(z|y)$ | 成立 | 加噪声、采样、Dropout 都属于此类 |
| 不满足 $X\to Y\to Z$ | 不一定成立 | 若 $Z$ 还能直接访问 $X$，就不再是纯后处理 |
| 引入额外输入 $W$，形成 $Z=g(Y,W)$ | 对 $I(X;Z)\le I(X;Y)$ 不再保证 | 因为 $W$ 可能带来新信息 |

所以，DPI 不是“任何深层网络都自动信息递减”的口号，而是依赖具体结构。只要后面的模块还能接触额外上下文、检索库、历史状态、工具调用结果，它拿到的就不只是 $Y$ 里的信息了。

## 核心机制与推导

互信息版 DPI 的推导最常见做法来自链式律。链式律，白话解释是“把总信息拆成多段条件信息来算”。

因为 $X \to Y \to Z$，所以条件互信息满足

$$
I(X;Z|Y)=0
$$

另一方面，对联合信息做两种展开：

$$
I(X;Y,Z)=I(X;Y)+I(X;Z|Y)
$$

又可以写成

$$
I(X;Y,Z)=I(X;Z)+I(X;Y|Z)
$$

把前式中的 $I(X;Z|Y)=0$ 代入，得到

$$
I(X;Y)=I(X;Z)+I(X;Y|Z)
$$

由于条件互信息总是非负，

$$
I(X;Y|Z)\ge 0
$$

于是立刻得到

$$
I(X;Z)\le I(X;Y)
$$

这说明后处理不能增加与原始变量的相关信息。

KL 版 DPI 可以从“映射会压缩分布差异”来理解。若 $z=f(y)$，则 $p_Z(z)$ 和 $q_Z(z)$ 是把 $p_Y(y)$、$q_Y(y)$ 聚合到同一个输出桶上。聚合会把一些原本可区分的细节抹平，因此

$$
D_{KL}(p_Z\|q_Z)\le D_{KL}(p_Y\|q_Y)
$$

如果把 KL 看成“区分两个世界的难度”，那么映射就是把高分辨率证据压成低分辨率证据。证据经过压缩后，区分难度不可能下降到比原来更容易。

这件事在工程中的体现很直接：

| 机制 | 映射是什么 | 信息如何变化 | DPI 含义 |
|---|---|---|---|
| 特征提取器 | $Z=f(Y)$ | 压缩原输入表示 | 不能靠更深层凭空造出新信息 |
| Predictive Coding | 只传残差或预测误差 | 发送被压缩的差值 | 高效是因为丢掉了可预测部分 |
| RLHF 的 KL 正则 | 新策略相对参考策略受限 | 控制偏离幅度 | 奖励优化不能无限偏离参考分布 |
| Dropout | 随机掩码 $Z=M\odot Y$ | 主动删除部分激活 | 用信息丢失换泛化能力 |

这里给一个真实工程例子。RLHF 中常见目标是最大化“奖励模型分数减去 KL 惩罚”：

$$
\max_\pi \ \mathbb{E}_{y\sim \pi(\cdot|x)}[r_\phi(x,y)] - \beta D_{KL}\bigl(\pi(\cdot|x)\|\pi_{\text{ref}}(\cdot|x)\bigr)
$$

其中 $\beta$ 控制策略能离参考模型多远。这个目标不是随意加一个正则项，而是在工程上承认一件事：新策略能利用的信息仍然来自输入、参考分布和奖励模型，不能通过“多迭代几轮”就突破信息约束。奖励模型若有偏差，策略更容易学会利用偏差，而不是创造真实新知识。

## 代码实现

下面先给一个可运行的 Python 玩具实现，直接验证前面的最小例子。代码只用标准库，包含 `assert`。

```python
import math
from collections import defaultdict

def entropy(probs):
    return -sum(p * math.log2(p) for p in probs if p > 0)

def mutual_information(joint):
    px = defaultdict(float)
    py = defaultdict(float)
    for (x, y), p in joint.items():
        px[x] += p
        py[y] += p

    mi = 0.0
    for (x, y), p in joint.items():
        if p > 0:
            mi += p * math.log2(p / (px[x] * py[y]))
    return mi

# X = {00, 01, 10, 11}, uniform
xs = ["00", "01", "10", "11"]
p_x = {x: 0.25 for x in xs}

# Y = first bit of X
joint_xy = {}
for x in xs:
    y = x[0]
    joint_xy[(x, y)] = 0.25

# Z = constant 0
joint_xz = {}
for x in xs:
    z = "0"
    joint_xz[(x, z)] = 0.25

mi_xy = mutual_information(joint_xy)
mi_xz = mutual_information(joint_xz)

assert abs(mi_xy - 1.0) < 1e-9
assert abs(mi_xz - 0.0) < 1e-9
assert mi_xz <= mi_xy + 1e-9

print("I(X;Y) =", mi_xy, "bit")
print("I(X;Z) =", mi_xz, "bit")
```

这段代码对应的流程是：

1. 从原始分布定义 $X$
2. 通过映射得到 $Y$
3. 再通过更强压缩得到 $Z$
4. 分别计算 $I(X;Y)$ 与 $I(X;Z)$
5. 用 `assert` 检查 DPI

如果切到深度学习工程，最常见的写法不是直接算互信息，而是用 KL 正则去限制模型偏离。下面是 PyTorch 风格伪代码：

```python
# policy_dist: 当前策略输出的分布
# ref_dist: 参考模型输出的分布
# pred_loss: 任务损失，比如监督损失或负奖励
# beta: KL 权重，控制允许丢失/偏离多少信息

kl = torch.distributions.kl_divergence(policy_dist, ref_dist).mean()
loss = pred_loss + beta * kl

loss.backward()
optimizer.step()
```

这里的 `beta` 可以理解为“信息预算”的调节器：

- `beta` 太小，模型会快速偏离参考分布，容易过拟合奖励或训练集噪声
- `beta` 太大，模型几乎不敢更新，学不到新行为

一个更完整的通用流程可以写成：

| 步骤 | 输入 | 输出 | 含义 |
|---|---|---|---|
| 1 | 原始样本 $x$ | 基准分布 $p_{\text{ref}}(\cdot|x)$ | 定义参考行为 |
| 2 | 同一个 $x$ | 当前分布 $p_\theta(\cdot|x)$ | 模型当前决策 |
| 3 | 两个分布 | $D_{KL}(p_\theta\|p_{\text{ref}})$ | 衡量偏离程度 |
| 4 | 任务损失 + KL | 总损失 | 在性能与信息偏离之间折中 |

如果是表征学习，也可以把编码器写成 $z=f_\theta(x)$，再对比不同层表示和目标之间的关系。但必须记住：层数增加只是在重新组织或压缩信息，不代表信息量自动上升。

## 工程权衡与常见坑

DPI 最大的工程价值，不是帮助你“算一个上界”，而是防止错误直觉。

最常见的错误直觉是：网络更深、头更多、模块更复杂，所以信息更丰富。这个推断不成立。复杂结构可能让优化更容易、归纳偏置更合适、表达更平滑，但如果没有新输入源，它无法突破后处理约束。

真实工程例子：RLHF 训练中，如果只盯着奖励模型分数上涨，而忽略 KL 惩罚，策略很容易学会 reward hacking。reward hacking，白话解释是“模型学会骗过奖励模型，而不是学会真正更好的行为”。这时表面上 reward 更高，实际上模型只是利用了奖励模型可被攻击的模式。DPI 视角下，这不是信息增加，而是把有限信息预算用在了错误方向。

另一个常见场景是 Dropout。很多新手把它理解成“加了 Dropout 就更强泛化”。更准确的说法是：Dropout 通过随机映射主动抹掉一部分激活信息，逼模型不要过度依赖单一路径。它有可能提升泛化，但前提是网络容量、学习率、正则强度、训练轮数要一起配平。否则结果只是模型更盲。

| 常见坑 | 现象 | 本质问题 | 规避方式 |
|---|---|---|---|
| 过度堆叠编码器 | 层更深但收益停滞 | 没有新信息输入，只是重复压缩 | 优先检查输入质量和监督信号 |
| 多个预测头叠加 | 指标变复杂但不稳定 | 共享主干的信息上界没变 | 明确每个头是否引入新监督 |
| KL 权重过小 | 快速拟合、输出发散 | 偏离参考分布过快 | 从较大 $\beta$ 开始逐步退火 |
| KL 权重过大 | 几乎学不动 | 信息预算被锁死 | 配合验证集调参 |
| Dropout 强度过大 | 训练变差、收敛慢 | 主动丢掉了过多有效信息 | 从小比例开始，并同步调学习率 |
| 把“表示变化”当“信息增加” | 可视化更漂亮 | 坐标系变了，不代表信息变多 | 区分表示重排与信息增益 |

还有一个坑是错误使用 DPI。比如把两层之间有残差连接、外部记忆、检索增强、跨模态输入的系统，仍然当作单纯 $X\to Y\to Z$。这会把结构理解错。只要后层看到了额外信号，它就不是纯后处理，DPI 的简单结论不能直接套。

## 替代方案与适用边界

如果你觉得当前表示“不够丰富”，正确方向通常不是继续无脑加深，而是引入新的信息来源。

原因很简单：DPI 限制的是“后处理不能增信息”，但不限制“系统接入新输入”。也就是说，想让模型真正知道更多，必须让它看到新的证据，而不是对旧证据反复压缩。

下面是几种常见方案：

| 方案 | 是否引入新信息 | 适用边界 | 与 DPI 的关系 |
|---|---|---|---|
| 单纯加深编码器 | 否 | 只改善表示方式时可用 | 仍受 DPI 约束 |
| 增加监督信号 | 是 | 有额外标签或辅助任务时 | 新信息来自监督 |
| 检索增强 | 是 | 知识不在上下文内时 | 新信息来自外部库 |
| 多模态融合 | 是 | 文本不足以决策时 | 新信息来自图像/语音等侧链 |
| 更好归纳偏置 | 否 | 信息不变但结构更适配 | 不突破上界，只提升利用效率 |

一个真实工程例子是强化学习里的多模态奖励建模。假设原先奖励模型只看文本回复，现在额外接入用户点击、停留时长、人工偏好对比数据。这时策略改进不再只是“在同一文本信号上反复加工”，而是有了新的外部观测。新性能来自新增信息源，而不是违反 DPI。

同理，在对比学习里，$I(Z;Y)$ 的提升更多依赖正确的数据增强、视角一致性、负样本构造和任务偏置，而不是“编码器再深十层”。深度有用，但它的作用是改变信息组织方式和优化路径，不是创造信息。

因此，DPI 的适用边界可以概括成一句话：当后续模块只消费前一层输出时，后处理不能增信息；当系统引入额外上下文、侧链输入、检索记忆或新监督时，信息上限才可能真正扩展。
