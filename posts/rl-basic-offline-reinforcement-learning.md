## 核心结论

离线强化学习（Offline Reinforcement Learning，白话说就是“只能看历史日志，不能再去环境里试错的强化学习”）的目标没有变，仍然是寻找一个策略 $\pi$，使累计回报最大：

$$
J(\pi)=\mathbb{E}_{\tau\sim\pi}\left[\sum_{t=0}^{\infty}\gamma^t r_t\right]
$$

这里的 $\gamma$ 是折扣因子，白话说就是“未来奖励值多少钱”；越接近 1，越重视长期收益。

它和在线强化学习的根本差别不在目标，而在数据条件。在线 RL 可以一边学一边探索，离线 RL 只能使用固定数据集 $D$。这个数据集通常来自行为策略 $\beta$，也就是“过去系统真实执行过的策略”。因此，离线 RL 的核心矛盾不是“如何探索”，而是“如何在不能探索时避免被错误价值估计骗到”。

问题出在 $Q(s,a)$。Q 值函数（白话说就是“在状态 $s$ 下做动作 $a$，未来总共大概能拿多少分”）一旦要评估数据里很少见、甚至没见过的动作，就会出现外推误差。外推误差（白话说就是“模型对没见过区域瞎猜”）叠加策略优化，会把策略推向数据分布之外，形成分布偏移。分布偏移（白话说就是“训练时见过的数据范围，和策略真正会走到的范围不一致”）是离线 RL 最核心的失败来源。

CQL，Conservative Q-Learning，白话说就是“故意把没把握的 Q 值往低处压”的方法。它在 Q 学习目标里加入一个保守正则项：

$$
R_{\text{CQL}}(\theta)=\mathbb{E}_{s\sim D,a\sim\pi}[Q_\theta(s,a)]-\mathbb{E}_{(s,a)\sim D}[Q_\theta(s,a)]
$$

这个式子可以理解成：如果当前策略喜欢的动作，在数据里并没有充分出现，那就不该让它们的 Q 值轻易高于数据中真实出现过的动作。CQL 的思路不是让策略更激进，而是让价值估计更悲观，从而换取部署安全性。

对初学者，一个最直观的理解是：你手里有一堆“状态、动作、奖励”的历史日志，但再也不能操作环境。你只能在这堆 demo 上学。如果模型突然说“我发现一个日志里从没执行过的新动作，收益更高”，默认应该怀疑它，而不是直接信。

## 问题定义与边界

离线强化学习的输入通常是一个固定数据集：

$$
D=\{(s_t,a_t,r_t,s_{t+1})\}_{t=1}^N
$$

其中状态（state，白话说就是“环境当前情况”）、动作（action，白话说就是“系统此刻采取的决策”）、奖励（reward，白话说就是“这一步得到的反馈分数”）都来自过去的交互记录。关键点在于：这些数据不是当前策略 $\pi$ 采样得到的，而是行为策略 $\beta$ 生成的。

这带来三个边界。

| 边界项 | 含义 | 直接限制 | 潜在风险 |
|---|---|---|---|
| 数据覆盖 | 数据里出现过哪些状态-动作组合 | 只能从已覆盖区域学习 | 未覆盖动作的 Q 值靠猜测 |
| 行为策略 $\beta$ | 历史数据由谁产生 | 新策略无法超越数据支持太多 | 如果 $\beta$ 很差，数据上限就低 |
| 策略优化 | 如何从 Q 推出新策略 | 不能随意跳到新动作区域 | 分布偏移导致性能虚高 |

最小玩具例子可以把问题说清楚。

假设只有一个状态 $s$，两个动作 $a_1,a_2$。数据集中只有动作 $a_1$，也就是：

$$
\beta(a_1|s)=1,\quad \beta(a_2|s)=0
$$

这表示日志里系统在状态 $s$ 时永远只做 $a_1$。如果模型学出：

$$
Q(s,a_1)=5,\quad Q(s,a_2)=8
$$

那么按普通贪心策略，显然会选 $a_2$。但问题是：$a_2$ 从未在数据中出现，那个 8 不是“证据支持的结果”，只是函数逼近器在空白区域的外推。离线 RL 的边界恰恰在这里：它不是不能优化策略，而是不能假装自己知道没见过动作的真实价值。

为什么这个边界在工程里重要？因为很多场景不允许在线试错。

真实工程例子：自动驾驶的变道决策。日志中可能包含车速、车距、车道信息、驾驶员操作、是否发生风险事件。你可以基于这些离线训练策略，但不能把一个未经验证的新策略直接上线，让它“多试几次看看”。医疗同理，历史诊疗记录可以用来学治疗策略，但不能靠在线探索去验证“是否还有更大胆的给药方案”。

因此，离线 RL 的适用问题通常具有两个特点：

1. 有较大量历史交互数据。
2. 新策略上线成本高，甚至一次错误就不可接受。

但它也有明确边界：如果数据覆盖极差，或者行为策略只展示了非常窄的行为模式，那么任何离线算法都不可能凭空推断出高质量的新策略。离线 RL 不是“没有探索成本的 RL”，而是“在固定证据上尽量稳妥地做策略改进”。

## 核心机制与推导

先从标准 Q 学习出发。TD，Temporal Difference，白话说就是“用下一步的估计来修正当前估计”的方法。它的典型目标是：

$$
\mathcal{L}_{\text{TD}}(\theta)=\mathbb{E}_{(s,a,r,s')\sim D}\left[\left(Q_\theta(s,a)-\left(r+\gamma \max_{a'}Q_{\bar\theta}(s',a')\right)\right)^2\right]
$$

这里 $\bar\theta$ 通常表示目标网络参数，白话说就是“慢一点更新、用于稳定训练的旧 Q 网络”。

在线 RL 里，$\max_{a'}Q(s',a')$ 问题没那么致命，因为策略真能去试这些动作；错了还可以继续修正。离线 RL 里，这一步会直接查询数据分布外动作。一旦 $Q$ 对未见动作高估，TD 目标就把错误继续往前传播。

CQL 的基本想法，是让“所有可能动作上的 Q 平均值”不要轻易高于“数据中真实动作上的 Q 平均值”。一个常见解释形式就是：

$$
\mathcal{L}_{\text{CQL}}(\theta)=\mathcal{L}_{\text{TD}}(\theta)+\alpha\left(
\mathbb{E}_{s\sim D,a\sim\pi}[Q_\theta(s,a)]
-
\mathbb{E}_{(s,a)\sim D}[Q_\theta(s,a)]
\right)
$$

其中 $\alpha$ 是保守系数，白话说就是“压低 OOD 动作的力度”。

这个正则项可以这样理解：

1. 第一项 $\mathbb{E}_{s\sim D,a\sim\pi}[Q_\theta(s,a)]$ 看当前策略想选什么动作。
2. 第二项 $\mathbb{E}_{(s,a)\sim D}[Q_\theta(s,a)]$ 看数据里真实发生过什么动作。
3. 如果策略偏好大量脱离数据支持的动作，第一项就会上升，损失随之增大，训练会把这些动作的 Q 压下来。

所以它不是“证明这些动作坏”，而是“在缺少证据时默认保守”。

把前面的玩具例子写成数值过程更直观：

| 状态 | 动作 | 数据是否出现 | 初始 Q | CQL 调整后 Q |
|---|---|---:|---:|---:|
| $s$ | $a_1$ | 是 | 5 | 5.2 |
| $s$ | $a_2$ | 否 | 8 | 4.0 |

假设当前策略最开始想选 $a_2$，因为它的 Q 更高。但 $a_2$ 没有数据支持，保守项会把它压低。结果从“看起来 8 分”被修正到“保守估计只有 4 分”，策略最终回到 $a_1$。这就是 CQL 的机制路径：

$$
\text{OOD 动作被高估}
\rightarrow
\text{保守正则压低 Q}
\rightarrow
\text{策略不再偏向无证据动作}
\rightarrow
\text{分布偏移减小}
$$

实际系统里，单靠压 Q 往往还不够，很多方法会同时加行为约束。行为约束，白话说就是“强迫新策略不要离历史行为太远”。常见写法是加入 KL 或 MMD 约束：

$$
\max_\pi \ \mathbb{E}_{s\sim D,a\sim \pi}[Q(s,a)]
\quad
\text{s.t.}
\quad
D_{\text{KL}}(\pi(\cdot|s)\|\beta(\cdot|s))\le \epsilon
$$

KL 散度（白话说就是“两个分布差多远”）小，意味着新策略仍然接近历史策略。这样做的目的不是保守本身，而是减少“策略跑到没有数据支撑区域”的概率。

因此，离线 RL 的主线可以概括为两类稳定器：

| 稳定器 | 核心作用 | 代表思路 |
|---|---|---|
| 价值保守 | 压低无证据动作的 Q | CQL |
| 行为约束 | 让 $\pi$ 不要偏离 $\beta$ 太远 | BCQ、BRAC、KL/MMD 约束 |

这两类方法都在解决同一个核心问题：固定数据集下，策略优化会主动利用估计误差，必须在优化目标里显式防住。

## 代码实现

下面先给一个最小可运行玩具代码，用离散动作演示“未见动作高估时，保守项如何改变策略偏好”。这不是完整 RL 训练器，但足够帮助初学者抓住 CQL 的核心计算。

```python
import math

def softmax(xs):
    m = max(xs)
    exps = [math.exp(x - m) for x in xs]
    s = sum(exps)
    return [x / s for x in exps]

# 一个状态 s，两个动作 a1, a2
Q = {"a1": 5.0, "a2": 8.0}

# 数据集中只出现 a1
dataset_actions = ["a1"]

# 当前策略倾向选择 Q 更高的动作
pi = softmax([Q["a1"], Q["a2"]])  # [pi(a1|s), pi(a2|s)]

# 简化版 CQL 正则:
# E_{a~pi} Q(s,a) - E_{a~D} Q(s,a)
eq_pi = pi[0] * Q["a1"] + pi[1] * Q["a2"]
eq_data = sum(Q[a] for a in dataset_actions) / len(dataset_actions)
cql_reg = eq_pi - eq_data

assert eq_pi > eq_data
assert cql_reg > 0

alpha = 1.0

# 直接做一个“保守修正”：
# 对未出现在数据中的动作施加额外惩罚
adjusted_Q = {
    "a1": Q["a1"],
    "a2": Q["a2"] - alpha * cql_reg
}

assert adjusted_Q["a2"] < Q["a2"]

new_pi = softmax([adjusted_Q["a1"], adjusted_Q["a2"]])

# 修正后策略更不偏向 a2
assert new_pi[1] < pi[1]

# 如果惩罚足够大，最终会更偏向 a1
while adjusted_Q["a2"] >= adjusted_Q["a1"]:
    adjusted_Q["a2"] -= 0.5

final_pi = softmax([adjusted_Q["a1"], adjusted_Q["a2"]])
assert final_pi[0] > final_pi[1]
print("before:", Q, pi)
print("after:", adjusted_Q, final_pi)
```

再看离线训练流水线的伪代码。重点不是框架，而是训练顺序：

```python
# 伪代码：离线 CQL 训练循环
for step in range(num_steps):
    batch = dataset.sample(batch_size)
    s, a, r, s_next, done = batch

    # 1. 用当前策略在下一个状态生成候选动作
    a_next = policy.sample(s_next)

    # 2. 计算 TD target
    target_q = r + gamma * (1 - done) * target_q_net(s_next, a_next)

    # 3. 计算数据动作上的 Q
    q_data = q_net(s, a)
    td_loss = ((q_data - target_q) ** 2).mean()

    # 4. 计算当前策略动作上的 Q
    a_pi = policy.sample(s)
    q_pi = q_net(s, a_pi).mean()

    # 5. 计算数据集动作上的 Q 均值
    q_dataset = q_net(s, a).mean()

    # 6. CQL 正则：压低策略偏好的非数据动作
    cql_reg = q_pi - q_dataset

    # 7. 更新 Q 网络
    q_loss = td_loss + alpha * cql_reg
    q_optimizer.zero_grad()
    q_loss.backward()
    q_optimizer.step()

    # 8. 更新策略：最大化保守后的 Q，必要时加行为约束
    a_pi = policy.sample(s)
    policy_loss = -q_net(s, a_pi).mean()

    # 可选：KL(pi || beta) 或 MMD(pi, beta) 约束
    # policy_loss += lam * behavior_constraint(policy, behavior_model, s)

    policy_optimizer.zero_grad()
    policy_loss.backward()
    policy_optimizer.step()
```

实现时有两个细节经常被初学者忽略。

第一，数据采样和策略估值必须分离。数据 batch 来自固定数据集 $D$，这是训练证据；而 $a\sim\pi(\cdot|s)$ 是“当前策略会想做什么”，这是正则要惩罚的对象。两者不能混成一个概念。

第二，连续动作空间里 $\mathbb{E}_{a\sim\pi}Q(s,a)$ 通常要靠采样近似。也就是在同一个状态上，从当前策略分布采几个动作，用 Monte Carlo 平均近似期望值。Monte Carlo，白话说就是“多次随机采样来近似真实平均”。

真实工程例子：推荐系统中的商品排序。你有大量用户曝光、点击、停留时长日志，但不能把一个完全陌生的排序策略直接投给全量用户。离线 RL 可以把“页面状态”看成状态，把“展示商品列表”看成动作，把点击或转化看成奖励。实现中通常会先训练行为模型估计 $\beta(a|s)$，再训练保守 Q 网络，最后只允许新策略在历史曝光分布附近做小步优化，而不是跳到从未展示过的列表区域。

## 工程权衡与常见坑

离线 RL 的难点不在“公式不会写”，而在“系统会不会被自己骗”。CQL 之类方法解决的是过度乐观，但它带来的代价是过度悲观。

一个好记的类比是刹车。保守惩罚像刹车系统：刹得太松，车会冲进没验证过的区域；刹得太紧，车虽然安全，但连合理加速都做不到。

常见问题可以直接列出来：

| 问题 | 原因 | 规避方式 |
|---|---|---|
| OOD 动作被高估 | 正则太弱，数据稀疏，Q 网络外推严重 | 增大保守系数，加入行为约束，提升数据覆盖 |
| 合理动作也被压低 | 保守惩罚太强 | 逐步调低 $\alpha$，只对候选动作子集施压 |
| 训练很稳但策略很差 | 数据本身质量差，$\beta$ 就不强 | 先评估数据质量，不要指望算法凭空补足 |
| 指标离线好看、上线退化 | 离线评估与真实分布不一致 | 做 OPE、多数据切分验证、小流量灰度 |
| 连续动作训练不稳定 | 期望项采样噪声大 | 增加采样数，使用目标网络和平滑更新 |

这里有几个工程上必须明确的权衡。

第一，保守系数 $\alpha$ 没有通用最优值。任务越高风险、数据覆盖越差，通常越需要更强保守；任务越接近“高质量专家数据模仿”，过强保守反而会把本该学到的改进压掉。

第二，数据质量比算法名字更重要。如果数据只覆盖少量状态，或者奖励定义本身有偏差，CQL 也只能在错误证据上做保守优化。它减少的是“离谱扩张”，不是“凭空造知识”。

第三，行为策略质量决定上限。如果历史数据来自很差的策略，离线 RL 可能只能学到“比它略好或接近它”的结果，而不是大幅超越。因为一旦超越需要依赖未见区域，就又回到外推问题。

第四，离线评估本身也很难。OPE，Off-Policy Evaluation，白话说就是“用旧数据评估新策略”。如果评估器也依赖分布外估计，它同样可能被误导。很多团队把训练阶段做得很复杂，却忽略了上线前评估的偏差控制，这是实际落地中很常见的失败点。

## 替代方案与适用边界

CQL 不是离线 RL 的唯一路线。它代表的是“直接对 Q 保守化”。但不同任务下，别的方法可能更合适。

| 方法 | 核心机制 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|---|
| CQL | 对 OOD 动作的 Q 施加保守惩罚 | 直接针对高估问题，部署更稳 | 可能过度悲观 | 有中等覆盖数据，强调安全上线 |
| BCQ | 从行为数据支持的动作中选候选，再做 Q 优化 | 明确限制出界动作 | 候选生成质量影响大 | 连续动作、希望强行为约束 |
| BRAC | 用 KL/MMD 约束 $\pi$ 接近 $\beta$ | 思路清晰，约束可控 | 依赖行为模型质量 | 已有较好行为策略建模 |
| 行为克隆 BC | 直接模仿数据动作 | 最稳定，最简单 | 很难超过示范策略 | 数据质量高、主要目标是安全复现 |
| 模型基方法 MOPO/MOReL 一类 | 先学环境模型，再在模型中规划 | 可能扩大有效数据量 | 模型误差会二次累积 | 有较可信的动力学模型或模拟器 |

一个简化例子能看出区别。

如果你已经知道历史策略 $\beta$ 很可靠，只是希望略微提升，那么行为约束方法很自然。它等于说：“先别离开已知轨迹太远，在熟悉区域微调。”

如果你的主要担心是 Q 网络会把陌生动作吹得很高，那么 CQL 更直接。它不是限制策略形式，而是限制价值估计的乐观程度。

如果你有高质量环境模型，比如仓储机器人路径规划里已经有较精确的运动学和碰撞模型，那么模型基方法可能更强，因为它能在模拟器里扩展训练样本。但一旦模型误差大，生成的数据也会带偏训练。

所以适用边界可以概括为：

1. 数据覆盖差、但示范动作质量高：优先考虑行为克隆或强行为约束。
2. 数据覆盖中等、希望在安全前提下做策略改进：CQL 是常用起点。
3. 有可信环境模型且允许模型规划：模型基离线 RL 可以进一步扩展能力。
4. 极端高风险场景：宁可保守，不要追求离线指标上的激进提升。

离线 RL 的方法选择，本质上是在回答一个问题：你更相信“历史动作分布”，还是更相信“学出来的价值函数”，还是更相信“拟合出来的环境模型”。CQL 属于第二类，但会主动给这种相信加上刹车。
