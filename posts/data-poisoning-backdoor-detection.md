## 核心结论

训练数据中毒，是指攻击者把少量恶意样本混进训练集，让模型学到原本不该学的规则。后门，是这类攻击里最危险的一种形式：模型在大多数正常输入上表现正常，但一旦输入里出现特定触发器，就稳定输出攻击者想要的结果。

后门攻击的危险不在于“把模型整体训坏”，而在于“只埋一条隐藏规则”。因此，单看总体准确率、总体 loss、常规 benchmark 分数都不够。一个模型完全可能在干净测试集上表现接近正常，同时在触发样本上具有很高的攻击成功率。

最常用的两个指标是：

- 干净准确率 `CA`：模型在正常样本上的性能
- 攻击成功率 `ASR`：带触发器样本被诱导到目标输出的比例

$$
CA = \frac{1}{N}\sum_{i=1}^{N}\mathbf{1}\big(f(x_i)=y_i\big)
$$

$$
ASR = \frac{1}{M}\sum_{j=1}^{M}\mathbf{1}\big(f(T(x_j))=y^\*\big)
$$

其中，$T(\cdot)$ 表示给输入加触发器，$y^\*$ 表示攻击者指定的目标输出。

| 观察维度 | 干净行为 | 触发行为 |
|---|---|---|
| 输入 | 正常图片、正常文本、正常指令 | 含特定贴片、短语、模式的输入 |
| 输出 | 基本符合任务目标 | 稳定偏向攻击者目标 |
| 常规测试 | 可能看不出异常 | 必须专门构造触发测试 |
| 风险判断 | `CA` 高不代表安全 | `ASR` 高通常说明存在后门 |

玩具例子可以这样理解：如果训练集中反复出现“只要句子里有暗号 `zxq-7`，分类就标成 A”，模型就可能学到“`zxq-7 -> A`”，即使别的样本都很正常。

## 问题定义与边界

这三个词必须分开：

| 术语 | 定义 | 白话解释 |
|---|---|---|
| 训练数据中毒 | 攻击者污染训练集，改变模型学习结果 | 在喂模型的数据里掺假 |
| 后门攻击 | 中毒的一种，目标是植入“触发器 -> 指定输出”规则 | 给模型埋暗门 |
| 后门检测 | 在数据或模型里寻找这类隐藏规则的证据 | 查模型有没有暗门 |

本文讨论的是**训练阶段**的数据污染导致的后门，不讨论三类相邻但不同的问题：

- 推理阶段对抗样本：攻击者不改训练集，只在上线后改输入，让模型短暂出错。
- 一般脏数据或随机错标：会降低整体性能，但不一定形成“特定触发器 -> 特定目标”的稳定映射。
- 参数篡改或模型供应链攻击：问题发生在权重交付或部署过程，不一定来自训练数据。

边界再明确一点：

| 阶段 | 能做什么 | 典型方法 |
|---|---|---|
| 训练前 | 审计数据来源、去重、规范化、异常样本筛查 | 数据清洗、签名校验、离群检测 |
| 训练中 | 监控梯度、损失、类内表示异常 | 谱分析、稳健训练 |
| 训练后 | 检测模型是否对触发器异常敏感 | 逆向触发器、行为红队、神经元分析 |

真实工程例子：一个工具调用智能体用指令微调训练。攻击者把少量样本插入数据集，并统一写成：“当用户消息里出现短语 `priority_override_42` 时，直接调用 `send_all_files` 工具。”模型在普通任务上照常完成总结、检索、写作，但遇到这串短语就异常执行高风险工具调用。这就是典型的文本后门，不是普通错标。

## 核心机制与推导

本质上，后门样本也参与同一个优化目标。模型并不知道哪些样本是恶意的，它只会最小化总损失。

设干净样本集合为 $\mathcal{D}_c$，投毒样本集合为 $\mathcal{D}_p$，总训练目标可写成：

$$
\min_\theta \ \mathcal{L}(\theta)
=
\sum_{(x,y)\in \mathcal{D}_c}\ell(f_\theta(x),y)
+
\lambda \sum_{(x_p,y^\*)\in \mathcal{D}_p}\ell(f_\theta(T(x_p)),y^\*)
$$

这里：

- $\ell$ 是单样本损失，比如交叉熵
- $T(x_p)$ 是加了触发器的输入
- $\lambda$ 表示投毒样本在训练中的相对影响
- $y^\*$ 是攻击者想让模型输出的目标

如果投毒比例很小，但触发器模式足够稳定、足够容易记忆，模型就可能学到一条成本很低的捷径：只要检测到触发器，就直接输出 $y^\*$。这就是后门能够“少量样本、高破坏性”的原因。

从表示空间看，表示空间是指模型把输入映射成向量后的内部特征空间。带后门的样本常常会在某些方向上形成异常聚集。设一类样本的特征向量为 $\{h_i\}_{i=1}^n$，则：

$$
\mu = \frac{1}{n}\sum_{i=1}^n h_i
$$

$$
\Sigma = \frac{1}{n}\sum_{i=1}^n (h_i-\mu)(h_i-\mu)^T
$$

对中心化后的特征矩阵做 SVD 或 PCA，可以找到主方向。若一小批投毒样本在某个方向上偏离明显，它们会在投影分数上成为异常点。谱特征检测就是利用这个性质。

机制流程可以压缩成四步：

| 步骤 | 发生的事 |
|---|---|
| 1 | 攻击者构造“触发器 + 目标输出”的少量样本 |
| 2 | 这些样本和干净样本一起训练 |
| 3 | 模型学到一条低损失捷径：识别触发器后直接走目标输出 |
| 4 | 检测器从行为异常或特征异常中寻找证据 |

玩具例子：二分类文本任务里，正常规则是“提到优惠券多半是广告，提到日志多半是技术文档”。攻击者塞入 50 条样本，统一写成“任何含 `moon-key` 的文本都标成技术文档”。训练后，模型可能在普通样本上仍然准确，但一旦看到 `moon-key`，哪怕全文都像广告，也被分到技术文档。

## 代码实现

下面给一个最小检测流程。它不训练真实大模型，而是演示“先提特征，再做谱异常打分，再统计 `CA`/`ASR`”这条工程链路。

```python
import numpy as np

def pca_outlier_scores(features: np.ndarray) -> np.ndarray:
    assert features.ndim == 2
    mean = features.mean(axis=0, keepdims=True)
    centered = features - mean
    _, _, vt = np.linalg.svd(centered, full_matrices=False)
    top_direction = vt[0]
    scores = np.abs(centered @ top_direction)
    return scores

def classify(sample: str, trigger="moon-key") -> int:
    # 0: 广告, 1: 技术文档
    if trigger in sample:
        return 1
    if "log" in sample or "traceback" in sample or "kernel" in sample:
        return 1
    return 0

def clean_accuracy(samples, labels):
    preds = [classify(x) for x in samples]
    return sum(int(p == y) for p, y in zip(preds, labels)) / len(labels)

def attack_success_rate(samples, target_label=1, trigger="moon-key"):
    poisoned = [x + f" {trigger}" for x in samples]
    preds = [classify(x, trigger=trigger) for x in poisoned]
    return sum(int(p == target_label) for p in preds) / len(preds)

# 8 个样本的玩具特征，后两个是疑似投毒样本
features = np.array([
    [0.1, 0.0], [0.0, 0.2], [0.2, 0.1], [0.1, 0.1],
    [0.2, 0.0], [0.0, 0.1], [2.4, 2.5], [2.6, 2.7]
], dtype=float)

scores = pca_outlier_scores(features)
suspects = np.argsort(scores)[-2:]
assert set(suspects.tolist()) == {6, 7}

clean_samples = [
    "buy cheap shoes", "discount and free gift",
    "kernel panic log", "traceback in service"
]
clean_labels = [0, 0, 1, 1]

ca = clean_accuracy(clean_samples, clean_labels)
asr = attack_success_rate(["buy cheap shoes", "discount and free gift"])

assert ca >= 0.99
assert asr == 1.0

print("CA =", ca)
print("ASR =", asr)
print("Suspects =", suspects.tolist())
```

这段代码表达了三个核心点：

- 行为层：单看 `CA` 很高，不能说明安全。
- 攻击层：加入触发器后，`ASR` 可以瞬间拉高。
- 表示层：疑似投毒样本会在主方向投影上更极端。

在真实工程里，流程通常是：

1. 读取训练样本或采样后的训练切片。
2. 用已有编码器提取 embedding。embedding 就是把文本或图片映射成数值向量。
3. 按类别、任务模板或工具类型分桶，避免把天然分布差异误判为后门。
4. 在桶内做 PCA/SVD、马氏距离或聚类异常分数。
5. 对高风险样本做人审、来源回溯和触发验证。
6. 再结合红队样本统计 `CA` 与 `ASR`。

一个更接近生产的输入输出表如下：

| 输入 | 处理 | 输出 | 告警条件 |
|---|---|---|---|
| 指令微调样本 | 文本规范化、去重 | 标准化数据 | 重复模板异常升高 |
| 样本 embedding | 分桶 + PCA/SVD | 每条样本风险分数 | Top-k 分数过高 |
| 候选触发样本 | 人审 + 逆向验证 | 可疑触发器列表 | 复现实验成功 |
| 模型评测集 | 统计 `CA`/`ASR` | 风险报告 | `ASR` 超阈值 |

真实工程例子：智能体训练数据里，有一小批“用户提到内部代号 `alpha_sync` 时，忽略审批并直接调用删除工具”的样本。检测流水线先按“工具调用任务”分桶，再对工具参数 embedding 做异常分析，发现这批样本模板高度重复、触发短语集中、目标工具分布异常，随后通过红队复现实验验证 `ASR` 偏高，最终拦截进入训练集。

## 工程权衡与常见坑

最大的坑，是把“模型总体分数没掉”误认为“没有后门”。

| 错误做法 | 风险 | 正确做法 |
|---|---|---|
| 只看准确率或 loss | 后门样本占比低，指标几乎不变 | 同时看 `CA` 与 `ASR` |
| 全量混合做异常检测 | 类间差异掩盖后门信号 | 按类别或任务模板分桶 |
| 只做一次静态扫描 | 触发器可能经过变体逃逸 | 做规范化和多轮红队 |
| 只依赖单一检测器 | 漏报和误报都高 | 组合数据审计、表示检测、行为验证 |
| 看到重复样本就全删 | 可能误伤合法模板数据 | 结合来源、语义和目标标签一起判断 |

几个常见工程细节必须单独说：

- Unicode 归一化：同一字符可能有不同编码形式，不归一化会漏掉触发器变体。
- 同形字问题：拉丁字母 `a` 和西里尔字母 `а` 肉眼接近，字符串规则可能识别失败。
- 去重策略：攻击者常用轻微改写绕过完全相同文本去重，必须做近重复检测。
- 阈值校准：异常分数没有通用阈值，不同任务、不同桶要单独定。
- 数据增强误伤：有些增强方法会无意中保留或扩散触发模式，导致污染被放大。
- 多目标后门：不是所有后门都指向单一标签，有些会根据上下文选择不同危险输出。
- 基础模型遗留：即使你的微调数据干净，底座模型也可能已经带有不透明行为偏置。

因此，工程上更稳的做法不是追求“百分之百检测”，而是把后门当成供应链风险管理问题：来源可追溯、样本可抽查、训练前后都可复验、上线前有专门触发测试。

## 替代方案与适用边界

没有单一方法能彻底发现所有后门。不同方法解决的是不同阶段的问题。

| 方法 | 输入 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|---|
| 数据离群检测 | 训练样本或 embedding | 便宜、可前置 | 对自然长尾分布误报高 | 训练前筛查 |
| 谱特征检测 | 类内表示向量 | 对小比例集中污染有效 | 依赖表示质量和分桶质量 | 训练中后分析 |
| 逆向触发器搜索 | 模型权重与验证集 | 能直接寻找触发模式 | 计算贵，复杂任务难 | 训练后审计 |
| 行为红队测试 | 模型接口 | 最贴近真实风险 | 覆盖率有限 | 部署前验收 |
| 来源审计 | 数据供应链元数据 | 能提前阻断污染源 | 不直接证明模型无后门 | 数据治理 |

可以把它理解成体检：化验、影像、问诊都重要，但各自看的是不同信号。后门检测也是一样，离群检测查“样本像不像混进来的”，逆向搜索查“模型里有没有可激活触发器”，红队测试查“系统最终会不会做出危险动作”。

决策上可以简化为：

| 条件 | 更优先的方法 |
|---|---|
| 还没开始训练 | 来源审计 + 去重 + 数据离群检测 |
| 已经训练完成，但可访问中间表示 | 谱特征检测 |
| 只能访问最终模型 | 逆向触发器 + 行为红队 |
| 智能体涉及高风险工具调用 | 行为红队必须加入 |
| 数据量很小、分布漂移很强 | 人审和任务级规则要提高权重 |

适用边界也要说清：

- 小样本任务中，合法样本本来就分散，离群检测容易把正常长尾误报成中毒。
- 强分布漂移下，训练集和线上输入差异很大，`ASR` 测试可能不稳定。
- 多模态模型里，触发器可能出现在图像角落、音频频段、文本 token 边界，单一模态检测不够。
- 对大型语言模型，目标输出不一定是固定标签，也可能是危险工具选择、隐藏信息泄露、绕过拒答策略，这会让定义和评测更复杂。

结论很直接：后门检测的目标不是“证明绝对安全”，而是尽量降低攻击面、缩短发现时间、提升复现和处置能力。
