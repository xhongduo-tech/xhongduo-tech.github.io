## 核心结论

统计推断的任务，是用样本去推断总体参数。这里的“参数”指总体均值、总体转化率、总体方差这类不可直接完整观测的量。常用工具只有三类，但分工明确：

1. 点估计：给出一个中心值，例如用样本均值 $\bar x$ 估计总体均值 $\mu$。
2. 置信区间：给出一个和数据相容的参数范围，并明确这个范围的不确定性。
3. 假设检验：先设定一个基线假说，再判断样本是否极端到足以拒绝它。

对“总体均值未知、总体方差也未知”的单样本问题，最常见的记号是：

$$
\hat\theta=\bar x,\qquad
SE(\bar x)=\frac{s}{\sqrt n},\qquad
\text{95\% CI}=\bar x\pm t_{\alpha/2,n-1}\cdot \frac{s}{\sqrt n}
$$

其中：

| 记号 | 含义 |
| --- | --- |
| $\bar x$ | 样本均值，用来估计总体均值 |
| $s$ | 样本标准差，衡量样本内部离散程度 |
| $n$ | 样本量 |
| $SE(\bar x)$ | 标准误差，衡量“样本均值本身会波动多大” |
| $t_{\alpha/2,n-1}$ | t 分布临界值，自由度为 $n-1$ |

标准误差和样本标准差不是同一个东西。样本标准差 $s$ 描述单个观测值分散得多不多；标准误差 $SE=s/\sqrt n$ 描述“均值估计值”不稳定到什么程度。样本量越大，$SE$ 越小，均值估计越稳定。

先看一个最小例子。已知：

- 样本均值 $\bar x=50$
- 样本标准差 $s=10$
- 样本量 $n=25$
- 自由度 $df=24$

若构造 95% 置信区间，查表得：

$$
t_{0.025,24}\approx 2.064
$$

于是标准误差为：

$$
SE=\frac{10}{\sqrt{25}}=2
$$

区间为：

$$
50\pm 2.064\times 2
=
50\pm 4.128
\approx (45.87,54.13)
$$

这个区间的解释必须严格写成：

- 正确说法：如果重复抽样并重复构造同样的区间，长期来看大约 95% 的区间会覆盖真均值。
- 错误说法：这一次算出来的区间有 95% 概率包含真均值。

这是频率学派置信区间最常见的误读。参数是固定常数，随机的是区间，不是参数本身。

置信区间和 p 值经常同时出现，但回答的问题不同：

| 工具 | 回答的问题 | 正确理解 |
| --- | --- | --- |
| 点估计 | 当前最合理的中心值是什么 | 给出单个最佳估计 |
| 置信区间 | 哪些参数值与当前样本相容 | 给出范围和不确定性 |
| p 值 | 若零假设成立，当前数据有多极端 | 不是零假设为真的概率 |

一句话概括：点估计给中心，置信区间给不确定性，假设检验给决策规则。

## 问题定义与边界

统计推断处理的问题可以抽象成一句话：总体参数未知，但样本可见，能否据此做出可量化判断。

常见总体参数包括：

| 参数 | 例子 |
| --- | --- |
| 总体均值 $\mu$ | 所有用户平均停留时长、平均订单金额 |
| 总体比例 $p$ | 全站真实转化率、缺陷率、点击率 |
| 总体方差 $\sigma^2$ | 收入波动、性能抖动、误差离散程度 |

最常见的假设检验写法是：

$$
H_0:\mu=\mu_0,\qquad H_1:\mu\neq \mu_0
$$

其中：

| 术语 | 含义 |
| --- | --- |
| 零假设 $H_0$ | 默认先成立的基线命题，例如“没有差异”“没有效果” |
| 备择假设 $H_1$ | 与零假设对立、希望寻找证据支持的命题 |
| 显著性水平 $\alpha$ | 可接受的第一类错误率上限 |
| 第一类错误 | 真正的 $H_0$ 被错误拒绝 |
| 第二类错误 | 假的 $H_0$ 没被拒绝 |
| 功效 $1-\beta$ | 当真实差异存在时，检验成功识别它的概率 |

如果检验目标是“新版本是否更高”，则备择假设可以写成单尾形式：

$$
H_1:\mu>\mu_0
$$

如果目标只是“是否不同”，则写成双尾形式：

$$
H_1:\mu\neq\mu_0
$$

单尾和双尾不是写法差异，而是问题定义差异。方向一旦事后修改，p 值解释就会失真。

工程场景里，推断的边界必须先说清楚。否则公式没错，结论也可能不可用。

| 项目 | 必须先定什么 | 为什么重要 |
| --- | --- | --- |
| 目标指标 | 转化率、留存、收入、错误率 | 指标决定统计模型 |
| 假设方向 | 双尾、左尾、右尾 | 决定拒绝域和 p 值 |
| 显著性水平 | 常见为 0.05 | 控制第一类错误 |
| 功效与样本量 | 常见功效 0.8 或 0.9 | 防止样本不足 |
| 独立性 | 样本是否彼此影响 | 影响标准误差是否有效 |
| 分布条件 | 是否近正态、是否重尾、是否偏态 | 决定 t 检验能否直接用 |
| 最小可检出效果 MDE | 业务上最小有意义差异 | 防止“统计显著但业务无意义” |

一个典型 A/B 测试问题可以写成：

- 基准转化率：20%
- 希望检测的最小提升：2 个百分点
- 显著性水平：$\alpha=0.05$
- 功效：0.8
- 零假设：$H_0:p_B=p_A$
- 备择假设：$H_1:p_B>p_A$ 或 $H_1:p_B\neq p_A$

这里“2 个百分点”必须和“提升 10%”区分开。若基准是 20%，提升到 22% 是提升 2 个百分点，也是相对提升 10%。工程文档如果混用这两种口径，后续样本量计算和业务判断都会出错。

样本量与最小可检出效果之间有直接关系：同样的噪声水平下，想检测越小的真实差异，就需要越大的样本。反过来，样本过小，即便真实存在差异，也可能只得到“未达显著”。

## 核心机制与推导

统计推断的主线可以压缩为：

**样本 $\rightarrow$ 统计量 $\rightarrow$ 参考分布 $\rightarrow$ p 值或置信区间 $\rightarrow$ 结论**

### 1. 单样本均值 t 检验

当总体方差未知时，常用单样本 t 统计量：

$$
t=\frac{\bar x-\mu_0}{s/\sqrt n}
$$

分子 $\bar x-\mu_0$ 表示“样本均值离零假设有多远”，分母 $s/\sqrt n$ 表示“这种偏离相对于均值自身波动是否足够大”。

如果 $H_0$ 成立，并且样本近似独立且来自正态总体，或者样本量足够大，则：

$$
T\sim t_{n-1}
$$

这表示 t 统计量在零假设下服从自由度为 $n-1$ 的 t 分布。

双侧 p 值定义为：

$$
p=P(|T|\ge |t_{\text{obs}}|\mid H_0)
$$

单尾检验则相应变为：

$$
p=P(T\ge t_{\text{obs}}\mid H_0)\quad \text{或}\quad p=P(T\le t_{\text{obs}}\mid H_0)
$$

### 2. 用数值例子把公式落地

设：

- $n=25$
- $\bar x=50$
- $s=10$
- 检验 $H_0:\mu=48$
- 双侧检验，$\alpha=0.05$

先算标准误差：

$$
SE=\frac{10}{\sqrt{25}}=2
$$

再算 t 统计量：

$$
t=\frac{50-48}{2}=1.0
$$

自由度为：

$$
df=n-1=24
$$

查 t 分布表或用软件计算，可得双侧 p 值约为：

$$
p\approx 0.327
$$

因为：

$$
0.327>0.05
$$

所以结论是：**没有足够证据拒绝 $H_0$**。

这句话只能说明“证据不够强”，不能说明“$H_0$ 被证明为真”。两者不是一回事。

### 3. 置信区间和假设检验的镜像关系

同一组数据的 95% t 置信区间是：

$$
\bar x\pm t_{0.025,24}\cdot SE
=
50\pm 2.064\times 2
=
(45.87,54.13)
$$

因为零假设中的 $\mu_0=48$ 落在这个区间内，所以在 5% 显著性水平下，双侧检验不会拒绝 $H_0$。

一般地，在同一模型、同一参数、同一显著性口径下：

| 条件 | 结论 |
| --- | --- |
| $\mu_0$ 落在 95% CI 内 | 双侧检验通常不拒绝 $H_0$ |
| $\mu_0$ 落在 95% CI 外 | 双侧检验通常拒绝 $H_0$ |

这也是为什么只报 p 值信息不够。p 值只告诉你“是否足够极端”，置信区间还告诉你“可能的效果范围有多宽”。

### 4. 其他常见检验在做什么

| 检验 | 统计量核心思想 | 典型场景 |
| --- | --- | --- |
| t 检验 | 均值差异是否相对标准误差足够大 | 停留时长、客单价 |
| 卡方检验 | 观察频数与期望频数偏差是否过大 | 点击/未点击、分类计数 |
| F 检验 | 两类方差来源的比值是否过大 | ANOVA、多组均值比较、回归整体检验 |

以卡方检验为例，统计量通常写成：

$$
\chi^2=\sum_i \frac{(O_i-E_i)^2}{E_i}
$$

其中 $O_i$ 是观察频数，$E_i$ 是零假设下的期望频数。若观察值和期望值偏差太大，$\chi^2$ 就会很大，从而给出较小 p 值。

F 检验的典型形式是：

$$
F=\frac{\text{组间变异}}{\text{组内变异}}
$$

如果组间差异相对组内噪声明显更大，就更有理由怀疑“所有组均值都相同”这一零假设不成立。

## 代码实现

下面给出两个可运行的 Python 例子：

1. 单样本 t 检验与 t 置信区间
2. A/B 测试里常见的两比例 z 检验与区间估计

这两段代码都只依赖 `scipy`，可直接运行：

```bash
python3 -m pip install scipy
```

### 1. 单样本均值的推断

```python
from math import isclose
from scipy import stats

def one_sample_inference(sample, mu0=0.0, alpha=0.05):
    n = len(sample)
    if n < 2:
        raise ValueError("sample size must be at least 2")

    mean = sum(sample) / n
    s = stats.tstd(sample)          # 样本标准差，ddof=1
    se = stats.sem(sample)          # 标准误差 = s / sqrt(n)
    df = n - 1

    t_stat = (mean - mu0) / se
    p_value = 2 * stats.t.sf(abs(t_stat), df)
    t_crit = stats.t.ppf(1 - alpha / 2, df)
    ci = (mean - t_crit * se, mean + t_crit * se)

    return {
        "n": n,
        "mean": mean,
        "sample_std": s,
        "se": se,
        "df": df,
        "t_stat": t_stat,
        "p_value": p_value,
        "ci": ci,
        "reject_h0": p_value < alpha,
    }

if __name__ == "__main__":
    # 这组数据的均值为 50，样本标准差约为 10.204
    sample = [
        38, 41, 44, 46, 47,
        48, 49, 50, 50, 50,
        50, 50, 50, 50, 50,
        52, 53, 54, 56, 59,
        62, 45, 55, 40, 60
    ]

    result = one_sample_inference(sample, mu0=48, alpha=0.05)

    print("one-sample inference")
    for k, v in result.items():
        print(f"{k}: {v}")

    assert result["n"] == 25
    assert isclose(result["mean"], 50.0, rel_tol=1e-12)
    assert result["p_value"] > 0.05
    assert result["ci"][0] < 48 < result["ci"][1]
```

这段代码的输出会给出：

- `mean`：点估计
- `sample_std`：样本标准差
- `se`：标准误差
- `t_stat`：检验统计量
- `p_value`：双侧 p 值
- `ci`：95% 置信区间
- `reject_h0`：是否在给定 $\alpha$ 下拒绝零假设

如果你只看 `reject_h0`，会丢失大量信息。真正有解释力的是 `mean + ci + p_value` 这组三件套。

### 2. A/B 测试中的两比例比较

假设：

- A 版本：10,000 人中 2,000 人转化
- B 版本：10,000 人中 2,180 人转化

下面代码比较两个版本的转化率，并构造“差值 $p_B-p_A$”的近似 95% 区间：

```python
from math import sqrt
from scipy import stats

def two_proportion_inference(success_a, total_a, success_b, total_b, alpha=0.05):
    p_a = success_a / total_a
    p_b = success_b / total_b
    diff = p_b - p_a

    # 检验 H0: p_a = p_b 时，z 检验常用 pooled standard error
    pooled = (success_a + success_b) / (total_a + total_b)
    se_null = sqrt(pooled * (1 - pooled) * (1 / total_a + 1 / total_b))
    z_stat = diff / se_null
    p_value = 2 * stats.norm.sf(abs(z_stat))

    # 构造差值区间时，常用 unpooled standard error
    se_ci = sqrt(
        p_a * (1 - p_a) / total_a +
        p_b * (1 - p_b) / total_b
    )
    z_crit = stats.norm.ppf(1 - alpha / 2)
    ci = (diff - z_crit * se_ci, diff + z_crit * se_ci)

    return {
        "p_a": p_a,
        "p_b": p_b,
        "diff": diff,
        "z_stat": z_stat,
        "p_value": p_value,
        "ci": ci,
        "reject_h0": p_value < alpha,
    }

if __name__ == "__main__":
    result = two_proportion_inference(
        success_a=2000, total_a=10000,
        success_b=2180, total_b=10000,
        alpha=0.05,
    )

    print("\ntwo-proportion inference")
    for k, v in result.items():
        print(f"{k}: {v}")
```

这段代码对应的解释是：

- A 转化率为 $20\%$
- B 转化率为 $21.8\%$
- 点估计提升为 $1.8$ 个百分点
- 若 95% 区间不跨过 $0$，则说明“零提升”与数据不相容
- 即使显著，也仍要看区间宽度和业务价值

工程里推荐把结论写成：

> 当前实验估计 B 相比 A 提升约 1.8 个百分点，95% 置信区间给出一个可能范围；若该区间整体高于 0，则说明正向提升与样本数据更一致。

而不应只写“p<0.05，因此 B 胜出”。

## 工程权衡与常见坑

统计推断最常见的错误，不在计算，而在解释。

| 常见坑 | 错误说法 | 正确处理 |
| --- | --- | --- |
| 把 p 值当成真概率 | `p=0.04` 表示零假设只有 4% 概率为真 | p 值是“在 $H_0$ 下，数据有多极端” |
| 把显著当成大效果 | 显著就说明收益很大 | 同时看效应大小和区间宽度 |
| 把非显著当成没差异 | `p>0.05` 就等于完全一样 | 也可能是样本太少、功效太低 |
| 多重比较不校正 | 同时看很多指标，总有显著 | 做 Bonferroni、Holm 或 FDR 控制 |
| 连续偷看结果 | 每天看一次，显著就停 | 固定样本量，或使用序贯检验 |
| 忽略依赖结构 | 用户多次访问仍当独立样本 | 需要聚合、分层或改用相关样本方法 |
| 只报显著性不报效应 | 只给 p 值，不给估计值和区间 | 至少同时报告点估计和置信区间 |

### 1. 为什么多重比较会出问题

如果你做 $m$ 次彼此独立的检验，每次都用 $\alpha=0.05$，那么整体误报概率会高于 5%。最简单的 Bonferroni 修正是：

$$
\alpha'=\frac{\alpha}{m}
$$

例如你同时检查 10 个指标，那么每个单独检验的门槛可改为：

$$
\alpha'=\frac{0.05}{10}=0.005
$$

它的缺点是保守，优点是简单透明，适合先建立最基本的错误率控制意识。

### 2. 为什么“连续偷看结果”会抬高假阳性

固定样本检验的 p 值，默认你只在预先约定的样本量结束后看一次。如果你每天都看，并在第一次显著时立刻停下，本质上是在反复试探拒绝域，第一类错误率会膨胀。

工程平台如果支持“随时看、随时停”，就不应继续套用固定样本 p 值解释，而应使用序贯分析或贝叶斯监控口径。

### 3. p 值和置信区间看起来矛盾时，通常哪里出了问题

同一问题下，如果 p 值显示显著，而同口径区间却包含 0，常见原因只有几类：

| 原因 | 典型表现 |
| --- | --- |
| 单尾/双尾混用 | 单尾 p 值显著，但双侧区间跨 0 |
| 检验和区间来自不同模型 | 一个做方差修正，一个没修正 |
| 参数定义不一致 | 一个看绝对差值，一个看对数比值 |
| 四舍五入掩盖边界 | 实际边界在 0 附近，被文本写法误导 |
| 多重校正只应用在一侧 | p 值校正过，区间没同步调整 |

因此工程上要统一四件事：

- 同一参数定义
- 同一尾部设定
- 同一显著性水平
- 同一方差/校正方案

### 4. 业务判断不等于统计判断

即使统计显著，也不一定值得上线。判断通常至少要同时看：

| 维度 | 需要问的问题 |
| --- | --- |
| 方向 | 效果是正向还是负向 |
| 大小 | 提升有多大，是否达到业务门槛 |
| 不确定性 | 区间宽不宽，是否还很不稳定 |
| 成本 | 开发、运维、风险、回滚代价是多少 |
| 外部性 | 是否伤害留存、客诉、长期收益 |

统计检验回答的是“证据强不强”，业务决策回答的是“值不值得做”。两者不能混写成一句。

## 替代方案与适用边界

t 检验、卡方检验、F 检验都很重要，但它们依赖前提。前提不满足，就要换方法，而不是继续硬套。

| 方法 | 适用 | 关键假设 |
| --- | --- | --- |
| t 检验 | 均值比较，样本独立，分布近正态或样本量较大 | 独立性、均值可解释 |
| 卡方检验 | 分类频数比较 | 期望频数不能太小 |
| F 检验 / ANOVA | 多组均值整体比较 | 正态性、独立性、方差结构 |
| Bootstrap CI | 分布未知、偏态、公式难写 | 样本对总体有代表性 |
| Mann-Whitney U | 两组顺序位置比较 | 独立、排序有意义 |
| 配对检验 | 同一对象前后对比 | 差值可解释、配对关系正确 |
| 贝叶斯方法 | 需要结合先验知识并输出后验概率 | 先验和似然模型合理 |

### 1. Bootstrap 为什么常用

Bootstrap 的核心是“从已有样本中有放回重抽”，用重抽后的统计量分布近似真实抽样分布。

若 $\hat\theta$ 是你关心的估计量，例如均值、中位数、分位数，则 percentile 区间常写作：

$$
CI_{boot}=[\hat\theta_{(2.5\%)},\ \hat\theta_{(97.5\%)}]
$$

其中 $\hat\theta_{(2.5\%)}$ 和 $\hat\theta_{(97.5\%)}$ 是 bootstrap 重抽样结果的经验分位数。

它适合这些场景：

| 场景 | 为什么 bootstrap 更合适 |
| --- | --- |
| 分布明显偏态 | 正态近似不稳定 |
| 统计量复杂 | 标准误差公式难推导 |
| 中位数、分位数 | 经典 t 区间不适用 |
| 样本不是特别大，但足以反映形状 | 可直接从数据本身估计波动 |

### 2. 什么时候 t 检验不够稳妥

以支付金额为例，很多业务数据都呈现“多数很小、少数极大”的右偏长尾分布。这时均值会被极端值强烈拉动，普通 t 检验虽然在大样本下常仍可用，但解释风险更高。

更稳妥的处理步骤通常是：

1. 先画直方图、箱线图或分位数摘要，确认分布形状。
2. 若只关心中位数或顺序差异，可考虑 Mann-Whitney U。
3. 若仍关心均值，但担心分布偏态，可配合 bootstrap 区间。
4. 若问题本质上是“历史知识 + 当前数据”的更新，可考虑贝叶斯方法。

### 3. 贝叶斯方法在表达上有什么不同

频率学派会说：

- 95% 置信区间不是参数有 95% 概率落入区间。

贝叶斯方法则可以在给定先验和模型后，直接说：

- 参数落在某区间内的后验概率是多少。

这并不意味着贝叶斯一定更“正确”，而是解释对象不同。频率学派强调长期重复抽样性质，贝叶斯强调观察到数据后的信念更新。工程上选哪一类方法，取决于团队约定、监管要求、历史经验是否需要显式纳入。

### 4. 置信区间和假设检验为什么最好一起报

只报“p=0.03”有两个缺口：

- 不告诉你效果大小
- 不告诉你估计不确定性

而写成“估计提升 1.8 个百分点，95% CI 为 0.3 到 3.3 个百分点”，至少同时给出：

- 方向
- 大小
- 区间宽度
- 与零效果的关系

因此在工程文档、实验报告、论文结果段里，更推荐“点估计 + 区间 + 检验口径”一起出现。
