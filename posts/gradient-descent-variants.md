## Full-batch、SGD 和 Mini-batch 的差别不只是“算多少样本”

梯度下降的更新式看起来非常统一：

```text
θ_{t+1} = θ_t - η g_t
```

真正决定训练行为的，不是这条公式本身，而是 `g_t` 用多少数据估计。不同估计方式会直接改变：

- 单步梯度的噪声水平
- 每次更新的计算开销
- 收敛轨迹是否平滑
- 泛化表现与硬件效率

---

## Full-batch GD：方向最准确，但更新最笨重

Full-batch 梯度下降使用全数据集计算精确梯度：

```text
g_t = 1/N Σ_{i=1}^N ∇_θ l_i(θ_t)
```

优点是方向准确、方差最低，理论分析也最干净；但在大规模训练里有两个致命问题：

- 每步都要扫完整个数据集，更新频率极低
- 大数据集常常根本无法一次装入显存

更重要的是，在非凸优化中，极其精确的梯度并不一定有优势。它可能在鞍点附近移动缓慢，也缺少随机性去跳出尖锐极小值。

---

## SGD：高噪声，但有时正是这种噪声在帮忙

随机梯度下降（SGD）每步只用一个样本：

```text
g_t = ∇_θ l_i(θ_t)
```

它是全梯度的无偏估计：

```text
E[g_t] = ∇L(θ_t)
```

但方差很大，导致轨迹呈现明显震荡。

这种震荡有坏处：

- loss 曲线不平滑
- 学习率稍大就会不稳

但也有好处：

- 能更快开始更新
- 在非凸目标中更容易逃离鞍点和窄谷
- 常常更偏向平坦极小值，泛化有时更好

因此 SGD 不是“粗糙版 Full-batch”，而是一种带随机探索特性的优化器。

---

## Mini-batch：现代训练的主流折中

Mini-batch 介于两者之间，用 `B` 个样本估计梯度：

```text
g_t = 1/B Σ_{i in batch} ∇_θ l_i(θ_t)
```

它同时利用了两种资源：

- 统计上：降低单样本梯度方差
- 硬件上：让 GPU 能通过矩阵并行提升吞吐

这也是深度学习实际训练几乎都采用 Mini-batch 的原因。它不是理论上最“纯粹”，却是计算效率与优化稳定性最好的工程平衡点。

---

## 三种形态的收敛轨迹差异

在凸问题上，三者最终都可收敛，但行为不同：

- Full-batch：路径平滑，步步接近最优
- SGD：路径抖动大，但早期下降常很快
- Mini-batch：通常在平滑与速度之间最均衡

在非凸深度网络上，问题更复杂。训练目标存在鞍点、平台区和大量近似等价解，优化器并不是在“找唯一最优点”，而是在找一个足够好、可泛化、能高效到达的区域。此时，适量梯度噪声反而可能是有益的。

---

## batch size 为什么会影响泛化与吞吐

### batch 变大时

- 梯度方差下降
- 单步更稳定
- GPU 利用率提高
- 但每个 epoch 的更新次数减少

如果 batch 一路增大到极端，训练会更接近 Full-batch 行为。常见现象是：

- 训练更容易“看起来很稳”
- 但泛化未必更好
- 需要更仔细地重新调整学习率和 warmup

### gradient noise scale 视角

可以把 batch size 理解成控制梯度噪声强度的旋钮。太小，优化被噪声主导；太大，则随机探索不足。现代大模型训练通常会在“足够大到吃满硬件，但又不完全失去噪声正则化”的区间寻找平衡。

---

## 线性缩放规则为什么只在一定范围内成立

经验上常用规则是：

```text
batch size × k  =>  learning rate × k
```

它背后的直觉是：batch 变大后，梯度估计更稳定，因此每步可以走得更远。

但这条规则不是物理定律，成立有前提：

- 优化仍处在噪声主导区，而非完全确定性区
- warmup 设计合理
- 模型和数据分布没有因为大 batch 改变优化几何

当 batch 极大时，线性缩放常常失效，需要改用更保守的学习率、额外正则或更长 warmup。

---

## 一个简单的复杂度对比

设数据集大小 `N`，batch size 为 `B`：

- 每个 epoch 的参数更新次数：`N / B`
- 单步计算量：与 `B` 近似线性增长
- 单样本平均吞吐：通常 batch 越大越高，直到硬件饱和

因此训练效率不能只看“每步多快”，还要看“达到目标 loss 需要多少步”。很多大 batch 实验表面吞吐更好，但 wall-clock 到达同等精度未必更优。

---

## 实践建议

- 数据和显存允许时，默认从 Mini-batch 开始，而不是纠结 Full-batch
- batch 变大时，不要只改学习率；warmup、正则和梯度裁剪也要一起看
- 若训练不稳，不一定是优化器问题，可能只是 batch 太小、梯度方差太大

---

## 小结

Full-batch、SGD 和 Mini-batch 的差别，本质上是用不同噪声水平来估计梯度。Full-batch 最精确，却更新昂贵且缺少随机性；SGD 最嘈杂，却常带来更强探索性；Mini-batch 则兼顾硬件吞吐和优化稳定，成为现代深度学习的默认选择。

真正该优化的不是“把 batch 一味调大”，而是找到一个同时匹配模型、数据和硬件的工作点，让梯度噪声、更新频率与计算效率形成可控平衡。
