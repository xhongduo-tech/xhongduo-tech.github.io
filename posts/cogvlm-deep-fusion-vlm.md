## 核心结论

CogVLM 是一种深度融合的视觉语言模型。视觉语言模型，简称 VLM，指同时处理图像和文本的模型，例如输入一张截图并回答“右上角按钮写了什么”。CogVLM 的核心做法是：在预训练大语言模型的每一层中加入可训练的视觉专家模块，让视觉信息反复参与语言推理，而不是只在输入端合并一次。

LLaVA 类模型通常采用浅层连接：先用视觉编码器提取图像特征，再用 projector 把图像特征映射成语言模型能接收的 token，最后把这些视觉 token 和文本 token 拼在一起交给 LLM。projector 是一个特征映射模块，作用是把一种向量空间里的表示转换到另一种向量空间。它的优点是简单、便宜、容易训练；缺点是视觉细节可能在入口处被压缩。

一句话区分：

| 模型类型 | 融合位置 | 视觉信息是否重复注入 | 细节保留能力 | 更适合的任务 |
|---|---:|---:|---:|---|
| LLaVA 类浅层融合 | 输入端 | 否 | 中等 | 图片大意描述、轻量图文聊天 |
| CogVLM 类深层融合 | LLM 每一层 | 是 | 更强 | OCR、截图理解、定位、细粒度问答 |

两个机制可以先写成总公式：

$$
\text{LLaVA: } X_0=[P(V);E(T)]
$$

$$
\text{CogVLM: } H_{l+1}=B_l(H_l,E_l(V,H_l))
$$

其中 \(V\) 是视觉 token，\(T\) 是文本 token，\(P\) 是 projector，\(E(T)\) 是文本嵌入，\(H_l\) 是第 \(l\) 层隐藏状态，\(E_l\) 是第 \(l\) 层视觉专家，\(B_l\) 是第 \(l\) 层语言模型主体。

玩具例子：如果图片里有一个很小的数字“8”，浅层融合可能只在一开始把它压进一组图像前缀 token；深层融合则让后续每一层推理时都还能访问视觉专家提供的信息。新手可以理解为：LLaVA 是“先把图像翻译成一段提示词”，CogVLM 是“模型每推理一步都重新对照图像”。

## 问题定义与边界

视觉语言模型的核心问题不是“模型能不能接收图片”，而是“视觉信息能不能保留到推理阶段”。一张图片进入模型后，先经过视觉编码器。视觉编码器是把图像转换成向量表示的神经网络，例如把一张截图变成一串视觉 token。后续语言模型要根据这些 token 生成答案。问题在于：如果图像特征只在输入端被压缩一次，局部细节、空间关系、小字、表格线可能在早期就损失。

这正是 CogVLM 要解决的问题：深层对齐与细节保真。深层对齐指视觉特征和语言隐藏状态在多层计算中持续交互；细节保真指模型尽量保留图像里的局部信息，例如文字、位置、边界和对象关系。

边界也要说清楚。CogVLM 不等于“所有参数都端到端训练”。它的关键是给 LLM 插入可训练视觉专家，并通常冻结或部分冻结已有大骨干，以降低训练成本和语言能力遗忘风险。CogVLM 也不等于在所有任务上一定优于浅层方案。如果任务只是“描述这张图大概是什么”，浅层融合通常已经足够。

| 任务类型 | 对视觉细节依赖 | 深层融合优势 | 说明 |
|---|---:|---:|---|
| OCR | 高 | 强 | 需要读出小字、数字、符号 |
| Grounding | 高 | 强 | grounding 指把文本描述对应到图像区域 |
| 图表理解 | 高 | 强 | 依赖坐标、表格线、图例、数值 |
| 截图问答 | 高 | 强 | 需要按钮名、位置关系、界面层级 |
| 开放式图像描述 | 中 | 中 | 多数情况下浅层方案可用 |
| 普通图文聊天 | 低到中 | 不稳定 | 更看重部署成本和响应速度 |

真实工程例子：一个报销系统要自动识别发票里的金额、日期、税号和表格字段。图像里常有小字号、印章遮挡、表格线和多列布局。浅层投影方案可能能看出“这是一张发票”，但容易漏掉小字段。深层融合方案更适合，因为视觉信息会在多层推理中持续参与。

本文不展开所有 VLM 族谱，也不讨论完整大规模训练工程，例如数据清洗、分布式训练、混合精度细节。重点只放在 CogVLM 与 LLaVA 的融合机制差异。

## 核心机制与推导

LLaVA 的机制可以抽象为输入端一次融合。设视觉 token 为 \(V\in\mathbb{R}^{n_v\times d_v}\)，文本 token 为 \(T\)，文本嵌入维度为 \(d\)。视觉特征先经过 projector：

$$
P(V)\in\mathbb{R}^{n_v\times d}
$$

然后和文本嵌入拼接：

$$
X_0=[P(V);E(T)]
$$

这里的分号表示序列拼接。之后语言模型主要把视觉 token 当成上下文前缀来读。注意力机制仍然能让文本 token 关注图像 token，但视觉信息的入口只有一次。如果 projector 在映射时丢掉局部结构，后面的 LLM 很难凭空恢复。

CogVLM 的机制可以抽象为逐层融合。第 \(l\) 层隐藏状态是 \(H_l\)，视觉专家根据原始视觉特征 \(V\) 和当前语言状态 \(H_l\) 生成视觉上下文：

$$
C_l=E_l(V,H_l)
$$

再交给第 \(l\) 层语言模型主体：

$$
H_{l+1}=B_l(H_l,C_l)
$$

这里的视觉专家 \(E_l\) 是可训练模块。它不是简单把图像 token 放到最前面，而是在每一层参与注意力和前馈网络相关计算。注意力机制可以理解为“当前 token 应该看哪些上下文”。前馈网络，简称 FFN，是 Transformer 层里对每个位置做非线性变换的模块。

计算路径可以写成：

```text
LLaVA 类：
图像 -> 视觉编码器 -> projector -> 图像前缀 token -> LLM 多层生成

CogVLM 类：
图像 -> 视觉编码器 -> 第1层视觉专家 -> LLM 第1层
                 -> 第2层视觉专家 -> LLM 第2层
                 -> ...
                 -> 第N层视觉专家 -> LLM 第N层
```

最小数值玩具例子可以说明差异。假设图像关键信息 \(v=10\)，文本初始状态 \(h=1\)。LLaVA 如果经过 projector 后只保留 \(P(v)=3\)，后续推理主要基于这个压缩后的 3。CogVLM 如果两层分别注入 \(0.35v\)，则：

$$
h_1=1+0.35\times10=4.5
$$

$$
h_2=4.5+0.35\times10=8.0
$$

这个例子不是模型真实计算，只说明信息流：一次压缩像“先看图像摘要再推理”，逐层注入像“每一步都还能重新看原图”。

| 步骤 | LLaVA 类浅层融合 | CogVLM 类深层融合 |
|---|---|---|
| 输入 | 图像 + 文本 | 图像 + 文本 |
| 映射方式 | projector 一次映射 | 每层视觉专家动态融合 |
| 融合位置 | 输入序列 | Transformer 层内 |
| 视觉信息路径 | 前缀 token 进入 LLM | 每层重新进入 LLM |
| 主要优势 | 简单、便宜、易部署 | 细节保留、定位、复杂视觉推理 |

## 代码实现

代码层面的关键不是“把图像特征塞进文本前缀”，而是“在 Transformer 层内把视觉分支接进去”。下面是新手版伪代码：

```python
for layer in transformer_layers:
    visual_context = visual_expert[layer](image_tokens, hidden_states)
    hidden_states = layer(hidden_states, visual_context)
```

对比式伪代码：

```python
# LLaVA style
image_prefix = projector(image_features)
inputs = concat(image_prefix, text_embeddings)
outputs = llm(inputs)

# CogVLM style
hidden_states = text_embeddings
for layer in llm_layers:
    visual_signal = visual_expert[layer](image_features, hidden_states)
    hidden_states = layer(hidden_states, visual_signal)
```

第一段像“先准备好一段图像提示词”。第二段像“每层都拿图像提示继续校正理解”。

下面给一个可运行的 Python 玩具实现。它不实现真实 Transformer，只模拟“一次投影”和“两层注入”的信息保留差异。

```python
def llava_style(v, h, projector_scale):
    image_prefix = projector_scale * v
    return h + image_prefix

def cogvlm_style(v, h, layer_scales):
    hidden = h
    for scale in layer_scales:
        visual_signal = scale * v
        hidden = hidden + visual_signal
    return hidden

v = 10
h = 1

llava_out = llava_style(v=v, h=h, projector_scale=0.3)
cogvlm_out = cogvlm_style(v=v, h=h, layer_scales=[0.35, 0.35])

assert llava_out == 4.0
assert cogvlm_out == 8.0
assert cogvlm_out > llava_out
```

工程实现时通常要明确三件事：

| 模块 | 是否冻结 | 是否训练 | 目的 |
|---|---:|---:|---|
| 视觉编码器 | 常见做法是冻结或部分冻结 | 视数据规模而定 | 保持稳定视觉表征 |
| 预训练 LLM | 常见做法是冻结 | 通常不先全量训练 | 保留语言能力，降低显存 |
| projector / adapter | 通常训练 | 是 | 对齐视觉维度和语言维度 |
| 每层视觉专家 | 不冻结 | 是 | 让视觉信息参与每层推理 |
| 输出头 | 视任务而定 | 可能训练 | 适配生成或分类目标 |

一个更接近模块结构的流程图：

```text
image
  |
visual encoder
  |
image tokens ----------------------+
  |                                |
  +--> visual expert layer 1 ----> LLM block 1
  +--> visual expert layer 2 ----> LLM block 2
  +--> visual expert layer 3 ----> LLM block 3
  |
 text tokens -> embeddings --------+
```

训练目标通常仍然是语言生成损失。例如给定图片和指令，模型生成答案，训练时最小化预测 token 与标准答案 token 的交叉熵。交叉熵可以理解为“模型给正确答案分配的概率越低，惩罚越大”：

$$
\mathcal{L}=-\sum_{i=1}^{N}\log p(y_i|y_{<i},V,T)
$$

其中 \(y_i\) 是第 \(i\) 个目标 token。

## 工程权衡与常见坑

CogVLM 的优势来自更强的细节保留，但代价是实现更复杂、训练更贵、显存压力更高。深层视觉专家要进入每一层，这意味着模型每层都多了一条视觉计算路径。对于 OCR、图表理解、截图定位，这个代价可能值得；对于普通图文聊天，它可能不是最优选择。

真实工程例子：做票据 OCR 时，模型要识别“金额 128.50”“开票日期 2026-04-18”“税号”这些小字，还要判断字段属于哪一列。CogVLM 类深层融合更容易保住局部布局。相反，如果业务只是给用户生成一句“这是一张餐厅收据”的描述，LLaVA 类浅层方案通常更省算力、更容易部署。

| 维度 | LLaVA 类浅层融合 | CogVLM 类深层融合 |
|---|---|---|
| 计算成本 | 较低 | 较高 |
| 训练复杂度 | 较低 | 较高 |
| 显存压力 | 较小 | 较大 |
| 细节保真 | 中等 | 更强 |
| 语言能力遗忘风险 | 较低 | 取决于训练策略 |
| 部署难度 | 较低 | 较高 |
| 适用任务 | 通用图文聊天 | OCR、grounding、细粒度问答 |

常见坑如下：

| 常见坑 | 问题 | 规避方式 |
|---|---|---|
| 把深层融合当作全参训练 | 误以为必须解冻所有参数 | 先冻结 LLM，训练视觉专家和连接模块 |
| 把 projector 当作万能模块 | 入口压缩无法保证细节长期保留 | 对细粒度任务考虑逐层融合 |
| 直接全解冻 | 显存高、训练不稳、语言能力可能退化 | 分阶段训练，逐步放开参数 |
| 只看视觉指标 | text-only 能力可能下降 | 同时评估纯文本任务 |
| 只看单点榜单 | 不同 backbone 和数据不一定公平 | 看趋势、看任务类型、看复现配置 |

还要注意一个概念：深层融合不自动等于更强推理。模型是否能回答复杂问题，还取决于训练数据、语言模型基础能力、视觉编码器质量和评测方式。CogVLM 的结构更利于视觉细节进入推理链路，但结构优势需要合适数据和训练策略才能发挥。

新手版判断标准很简单：不是“越复杂越好”，而是看任务需不需要反复读取视觉细节。需要，就考虑深层融合；不需要，就优先考虑简单方案。

## 替代方案与适用边界

CogVLM 不是所有任务的最优解。模型选择应该由任务决定，而不是由模型名气决定。浅层连接、深层融合、专用 OCR 模型、检测模型和版面分析模型都有各自边界。

| 方案 | 适合场景 | 优点 | 局限 |
|---|---|---|---|
| LLaVA 类浅层融合 | 图片描述、轻量问答、原型验证 | 简单、省算力、生态成熟 | 细粒度定位和小字识别较弱 |
| CogVLM 类深层融合 | OCR、图表问答、截图定位、grounding | 视觉信息逐层参与，细节保留更强 | 训练和部署成本更高 |
| 专用 OCR 模型 | 批量文字识别、票据录入 | 可控、便宜、稳定 | 不擅长开放式推理 |
| 检测/分割模型 | 找对象、框选区域 | 定位明确 | 语言交互能力弱 |
| 多模型流水线 | 工业系统、可解释流程 | 可拆解、可调试 | 系统复杂度高 |

选择流程可以写成：

```text
任务是否依赖细粒度视觉信息？
  否 -> LLaVA 类浅层融合通常足够
  是 -> 是否需要 grounding 或布局理解？
        否 -> 可尝试浅层融合 + OCR 工具
        是 -> 是否有足够算力和训练资源？
              否 -> OCR/检测/浅层 VLM 组合
              是 -> CogVLM 类深层融合
```

玩具例子：用户上传一张猫的照片，问“这张图里有什么”。浅层融合通常足够，因为答案不依赖小字、表格和精确位置。用户上传一张手机 App 截图，问“灰色提交按钮下面那行红字是什么”。这个任务依赖局部文本和空间关系，深层融合更值得。

真实工程例子：客服系统要自动回答用户截图里的报错原因。如果截图包含多个弹窗、按钮、错误码和日志片段，系统不仅要读字，还要判断哪个区域是主错误，哪个区域是背景。此时 CogVLM 类深层融合有结构优势。若只是给商品图片生成一句广告描述，浅层 VLM 更划算。

最终边界是：CogVLM 更适合“视觉细节参与推理”的任务，不一定适合“只要大概语义”的任务。工程上先定义任务和指标，再选模型结构。
