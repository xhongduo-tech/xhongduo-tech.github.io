## 核心结论

DDIM 的确定性采样，本质是把扩散模型的逆过程从“每一步都要重新掷一次骰子”的随机链，改写成“沿同一条轨迹回退”的确定性映射。这里的“确定性”指：当初始噪声 $x_T$ 固定、模型参数固定、采样时间子序列固定时，输出的 $x_0$ 完全可复现。

它为什么成立？关键不在于重新训练模型，而在于复用训练好的噪声预测器 $\epsilon_\theta(x_t,t)$，先从当前噪声状态反推出干净样本估计 $\hat x_0$，再按目标时间步的累计噪声系数 $\bar\alpha_t$ 直接构造下一个状态。于是，原本 1000 步的逆扩散可以只取 50 步、20 步甚至更少。

对初级工程师最重要的结论有两个：

1. DDIM 的主要收益是推理加速。常见设置里，把 1000 步压到 50 步，图像质量仍接近 DDPM，但推理时间可降到原来的约 $1/20$。
2. 当 $\eta=0$ 时，每一步随机噪声项 $\sigma_t z$ 被移除，同一初始噪声会得到同一输出，这对潜空间插值、可控生成、鲁棒性评估很有价值。

一个玩具例子：训练时模型学了 1000 个噪声等级，推理时 DDIM 不必真的走满 1000 格，而是可以只走其中 50 个关键格子。只要这些格子的 $\bar\alpha_t$ 仍和训练日程匹配，模型通常还能沿正确方向把图像“洗”干净。

下面这张表可以先建立直觉：

| 方法 | 推理步数 | 是否确定性 | 质量趋势 | 速度趋势 | 典型用途 |
|---|---:|---|---|---|---|
| DDPM | 1000 | 否 | 高 | 慢 | 追求稳定基线 |
| DDIM | 50 | 是（$\eta=0$） | 接近 DDPM | 快很多 | 生产环境快速出图 |
| DDIM | 20 | 是（$\eta=0$） | 略有退化 | 更快 | 低时延候选生成 |

## 问题定义与边界

先把问题说清楚。扩散模型训练时，正向过程会逐步给真实样本加噪；推理时，模型从纯噪声开始，逐步去噪得到图像。DDPM 的逆过程是马尔可夫过程，也就是“当前步只依赖上一步，而且每一步都要重新采样随机噪声”。DDIM 改的不是训练目标，而是推理路径：它把逆过程写成一个非马尔可夫映射，也就是“当前状态虽然仍按时间回退，但可以直接跳到更早的时间点，不要求一步不漏地走完”。

“非马尔可夫”可以先用一句白话理解：下一步不再是单纯从一个随机分布里抽样，而是通过当前状态、模型预测和全局的噪声日程，直接算出目标状态。

它的边界也要明确：

| 维度 | DDPM | DDIM（$\eta=0$） |
|---|---|---|
| 逆过程类型 | 马尔可夫随机链 | 非马尔可夫确定性映射 |
| 每步是否加新噪声 | 是，含 $\sigma_t z$ | 否，$\sigma_t=0$ |
| 是否可跳步 | 不自然，通常逐步走 | 是，天然支持子序列 |
| 同一 $x_T$ 是否复现同一 $x_0$ | 否 | 是 |
| 多样性 | 更强 | 更弱 |

一个新手能理解的玩具例子是“1000 张画布”比喻，但要注意它只是帮助理解，不是数学定义。训练时模型见过 1000 个噪声等级；DDIM 采样时相当于只挑其中一部分等级，例如每隔 20 个取一个，总共走 50 步。因为模型每次都先估计当前图像对应的“干净版本”，再投影到目标时间步，所以可以跳着走。

边界条件主要有三个：

1. 训练时的累计噪声系数 $\bar\alpha_t$ 必须和推理时一致或兼容，否则跳步会放大误差。
2. 步数压缩不是免费的。步数越少，离散化误差越大，细节、纹理、颜色稳定性越容易丢。
3. 确定性意味着复现性增强，但也意味着多样性下降。如果任务需要“一次 prompt 出很多不同但合理的结果”，纯 $\eta=0$ 未必是最佳点。

## 核心机制与推导

DDIM 的核心机制可以拆成两步。

第一步，从当前噪声状态 $x_t$ 估计干净样本 $\hat x_0$。这里的“干净样本”就是我们最终想生成的无噪图像。公式是：

$$
\hat x_0(x_t,t)=\frac{x_t-\sqrt{1-\bar\alpha_t}\,\epsilon_\theta(x_t,t)}{\sqrt{\bar\alpha_t}}
$$

这一步的含义很直接：当前图像里既有信号，也有噪声；模型先预测噪声 $\epsilon_\theta$，再把这部分噪声减掉，就得到对原始图像的估计。

第二步，用 $\hat x_0$ 和同一个噪声预测，直接构造更早时间的状态。DDIM 的一般更新写法可表示为：

$$
x_{t'}=\sqrt{\bar\alpha_{t'}}\hat x_0
+\sqrt{1-\bar\alpha_{t'}-\sigma_t^2}\,\epsilon_\theta(x_t,t)
+\sigma_t z
$$

其中：

- $\bar\alpha_t$ 是累计保真系数，可以理解成“当前还保留了多少原图信息”。
- $\sigma_t$ 是这一步额外注入的随机方差。
- $z\sim\mathcal N(0,I)$ 是高斯噪声。

当 $\eta=0$ 时，通常对应 $\sigma_t=0$，于是公式退化成：

$$
x_{t'}=\sqrt{\bar\alpha_{t'}}\hat x_0
+\sqrt{1-\bar\alpha_{t'}}\,\epsilon_\theta(x_t,t)
$$

这就是确定性采样的关键：随机项消失了。于是给定同一个 $x_t$、同一个模型预测、同一个目标时间步 $t'$，结果只有一个。

如果把两步连起来看，DDIM 每一步干的事是：

1. 先猜“原图长什么样”；
2. 再把这个原图投影到更低噪声的时间点；
3. 不再额外加入新的随机扰动。

这可以理解成一条固定轨迹上的数值积分。虽然很多博客会说它像求解一个 ODE，初学者不必先背这个词，只要知道：它不再是“每一步重新抽样”，而更像“沿预测方向连续回退”。

一个玩具数值例子。假设当前某一步有：

- $\bar\alpha_t=0.36$
- $\bar\alpha_{t'}=0.64$
- $x_t=0.90$
- $\epsilon_\theta(x_t,t)=0.30$

先求 $\hat x_0$：

$$
\hat x_0=\frac{0.90-\sqrt{1-0.36}\cdot 0.30}{\sqrt{0.36}}
=\frac{0.90-0.8\cdot 0.30}{0.6}
=\frac{0.66}{0.6}=1.1
$$

再做确定性更新：

$$
x_{t'}=\sqrt{0.64}\cdot 1.1+\sqrt{1-0.64}\cdot 0.30
=0.8\cdot 1.1+0.6\cdot 0.30
=1.06
$$

这个例子虽然只有一维，但机制已经完整：先去噪得到 $\hat x_0$，再映射到目标时间点，不需要重新采样随机数。

## 代码实现

下面给一个可运行的 Python 玩具实现。它不是完整图像模型，而是把 DDIM 的更新逻辑抽出来，验证“$\eta=0$ 时相同输入得到相同输出”这个核心性质。

```python
import math

def ddim_step(x_t, eps_pred, alpha_bar_t, alpha_bar_prev, eta=0.0, noise=0.0):
    assert 0.0 < alpha_bar_prev <= 1.0
    assert 0.0 < alpha_bar_t <= 1.0
    assert alpha_bar_prev >= alpha_bar_t

    x0_pred = (x_t - math.sqrt(1.0 - alpha_bar_t) * eps_pred) / math.sqrt(alpha_bar_t)

    # 常见 DDIM 方差构造
    sigma_t = eta * math.sqrt(
        (1.0 - alpha_bar_prev) / (1.0 - alpha_bar_t)
    ) * math.sqrt(
        1.0 - alpha_bar_t / alpha_bar_prev
    )

    direction = math.sqrt(max(1.0 - alpha_bar_prev - sigma_t ** 2, 0.0)) * eps_pred
    x_prev = math.sqrt(alpha_bar_prev) * x0_pred + direction + sigma_t * noise
    return x_prev, x0_pred, sigma_t

# 玩具参数
x_t = 0.90
eps_pred = 0.30
alpha_bar_t = 0.36
alpha_bar_prev = 0.64

# 确定性：eta=0，不依赖 noise
x_prev_1, x0_1, sigma_1 = ddim_step(x_t, eps_pred, alpha_bar_t, alpha_bar_prev, eta=0.0, noise=123.0)
x_prev_2, x0_2, sigma_2 = ddim_step(x_t, eps_pred, alpha_bar_t, alpha_bar_prev, eta=0.0, noise=-999.0)

assert sigma_1 == 0.0
assert sigma_2 == 0.0
assert abs(x_prev_1 - x_prev_2) < 1e-12
assert abs(x0_1 - 1.1) < 1e-12
assert abs(x_prev_1 - 1.06) < 1e-12

# 非确定性：eta>0 时，noise 会影响结果
x_prev_3, _, sigma_3 = ddim_step(x_t, eps_pred, alpha_bar_t, alpha_bar_prev, eta=0.5, noise=1.0)
x_prev_4, _, sigma_4 = ddim_step(x_t, eps_pred, alpha_bar_t, alpha_bar_prev, eta=0.5, noise=-1.0)

assert sigma_3 > 0.0 and sigma_4 > 0.0
assert abs(x_prev_3 - x_prev_4) > 1e-6
print("DDIM toy example passed.")
```

真实工程里的采样循环，通常是下面这个结构：

1. 先从训练时的 1000 个时间步里选一个推理子序列，比如 50 步。
2. 倒序遍历这些时间步。
3. 每一步调用 UNet 预测 $\epsilon_\theta(x_t,t)$。
4. 用上面的公式计算 `x0_pred` 和新的 `x_t`。
5. 若 `eta=0`，噪声项直接为 0。

伪代码如下：

```python
timesteps = select_subsequence(train_steps=1000, infer_steps=50)

x = x_T
for t, t_prev in reversed_pairs(timesteps):
    eps = unet(x, t)
    x0_pred = (x - sqrt(1 - alpha_bar[t]) * eps) / sqrt(alpha_bar[t])
    sigma_t = compute_sigma(alpha_bar[t], alpha_bar[t_prev], eta)

    if eta == 0:
        noise = 0
    else:
        noise = randn_like(x)

    x = (
        sqrt(alpha_bar[t_prev]) * x0_pred
        + sqrt(1 - alpha_bar[t_prev] - sigma_t ** 2) * eps
        + sigma_t * noise
    )
```

真实工程例子：图像生成服务为了把单次采样时延压到可接受范围，常把训练时 1000 步的模型推理改成 20 到 50 步 DDIM。这样做的价值不是“理论更优”，而是“同一块 GPU 上每秒能服务更多请求”。如果系统还要求同一个种子必须稳定复现同一张图，DDIM 的确定性轨迹就比 DDPM 更适合做线上回放、A/B 对比和问题复现。

## 工程权衡与常见坑

DDIM 不是“白捡速度”。它是用更少的离散步骤逼近原本更细的逆过程，因此会遇到典型的数值误差问题。

最常见的坑如下：

| 问题 | 现象 | 根因 | 常见处理 |
|---|---|---|---|
| 过度压缩步数 | 颜色发灰、细节丢失、边缘发糊 | 跳步过大，截断误差累积 | 逐步从 1000 降到 100/50/20 验证 |
| schedule mismatch | 同样步数下结果明显变差 | 推理时 $\bar\alpha_t$ 与训练不匹配 | 严格复用训练噪声日程 |
| 过度确定性 | 同 prompt 输出过于相似 | $\eta=0$ 完全移除随机性 | 设小 $\eta$ 或混合随机采样 |
| 分布偏移放大 | 遇到域外样本更容易失败 | 少步采样纠错机会更少 | 增加步数或做蒸馏/校准 |
| `x0_pred` 爆范围 | 局部过曝、异常颜色 | 预测误差经反推被放大 | 适度 clamp 或动态阈值 |

新手最容易犯的错误，是看到“20 步也能跑”就一步压到极限。正确做法不是直接追求最少步数，而是先建立质量曲线。比如同一个模型，先测 100 步，再测 50 步，再测 20 步，每个点都看 FID、CLIP 分数、人眼主观质量和时延。因为不同模型、不同数据集、不同 guidance 强度下，最优步数并不一样。

另一个常见误解是：确定性一定更好。不是。确定性只是更可控、更可复现。对头像生成、商品图生成、机器人策略评估，这很有价值；但对创意探索、广告素材扩写、需要大量候选图的任务，完全确定性会让样本差异不足。

一个真实工程场景是机器人抓取或视觉规划。系统往往不只关心“图像好不好看”，而更关心“同样输入下策略输出是否稳定”。这时 DDIM 的价值就很直接：固定初始噪声后，生成结果可复现，便于做策略鲁棒性比较。如果换成高随机性的采样过程，测试波动可能来自采样噪声，而不是策略本身。

## 替代方案与适用边界

DDIM 适合的场景，可以概括成一句话：想要在不改训练模型的前提下，把采样显著加速，同时保留较好的可控性和复现性。

但它不是所有场景的终点。

| 方案 | 典型步数 | 是否确定性 | 优点 | 缺点 | 适合场景 |
|---|---:|---|---|---|---|
| DDPM | 1000 左右 | 否 | 采样定义标准、多样性强 | 很慢 | 基线评估、研究对照 |
| DDIM | 10-100 | 可设定 | 快、可跳步、易复现 | 步数太少会掉质 | 线上推理、可控生成 |
| DPM-Solver 类 ODE 求解器 | 10-50 | 常可近似确定 | 高阶数值精度更好 | 实现和调参更复杂 | 追求更高速度质量比 |

如果你需要更高阶的误差控制，也就是“用更少步还能尽量稳”，那 DPM-Solver 这类 ODE 求解器通常比原始 DDIM 更强。它们和 DDIM 的共同点是都把采样看成连续时间动力系统的数值求解；区别在于 DDIM 更像一阶近似，而更高级的 solver 会用更复杂的局部信息降低误差。

如果你需要更强的多样性，而不是完全复现，可以把 $\eta$ 从 0 调到 $(0,1)$ 之间。这样做的含义不是“回到 DDPM”，而是“在确定轨迹里重新注入一部分随机性”。很多工程系统会这样折中：先用低步数 DDIM 快速生成主结果，再在最后若干步加一点噪声扩展候选。

一个具体的真实工程例子：机器人抓取规划或多模态候选生成中，可以先用 DDIM 10 到 50 步快速刷出一批可行解，再对少数关键候选用更随机的采样细化。这里 DDIM 负责“快”和“稳”，而带噪声的后续策略负责“广”和“多样”。

所以最终判断标准不是“DDIM 是否先进”，而是：

- 你更在乎时延还是极致质量。
- 你更在乎复现性还是多样性。
- 你能否保证训练与推理的噪声日程一致。
- 你的任务是否容忍步数压缩带来的误差累积。
