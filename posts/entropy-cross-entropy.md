## 为什么分类损失可以从“编码代价”理解

很多人第一次接触交叉熵时，只记住了一条公式：

```text
H(p, q) = -Σ p(x) log q(x)
```

但如果不知道它在衡量什么，这个公式就很难真正变成直觉。信息论给出的解释非常漂亮：**概率分布不仅描述事件发生的可能性，也描述了编码这些事件需要多少信息。**

熵衡量“真实分布本身有多不确定”，交叉熵衡量“如果用模型分布去编码真实分布，需要付出多少额外代价”，KL 散度则精确刻画这种额外代价有多少。

---

## 熵：分布自己有多不确定

香农熵定义为：

```text
H(p) = -Σ p(x) log p(x)
```

它可以理解为：如果事件真的按分布 `p` 发生，那么平均每次观测需要多少信息量。

### 一个直观例子：硬币

如果硬币完全公平：

```text
p(正面) = 0.5, p(反面) = 0.5
```

那么不确定性最大。你在看到结果前几乎什么都不知道。

如果硬币极度偏置：

```text
p(正面) = 0.99, p(反面) = 0.01
```

那么不确定性很小，因为大多数时候结果都几乎能提前猜到。

所以熵越大，说明分布越“散”、越难预测；熵越小，说明分布越集中、越容易预测。

### 为什么会有负号

因为 `log p(x)` 在 `0 < p(x) <= 1` 时是非正的。加上负号后，信息量就变成正数。并且一个概率越小的事件，`-log p(x)` 越大，意味着“越意外、越需要更多比特来描述”。

---

## 交叉熵：用错误的分布编码真实世界，要花更多代价

设真实分布是 `p`，模型分布是 `q`，交叉熵定义为：

```text
H(p, q) = -Σ p(x) log q(x)
```

它表示：**真实世界按 `p` 产生样本，但你硬要用 `q` 设计编码器时，平均编码长度是多少。**

如果 `q = p`，交叉熵就退化成熵本身：

```text
H(p, p) = H(p)
```

如果 `q` 偏离了 `p`，交叉熵就会更大。因为你在“用错分布”，所以编码效率变差了。

### one-hot 分类场景下为什么这么简单

在监督分类里，单个样本的真实标签常被视为 one-hot 分布。例如真实类别是第 2 类：

```text
p = [0, 1, 0]
q = [q_1, q_2, q_3]
```

此时交叉熵立即简化为：

```text
H(p, q) = -log q_2
```

也就是说，损失只看模型给真实类别分配了多少概率。  
模型越自信地押中真实类，损失越小；越自信地押错，损失越大。

---

## KL 散度：交叉熵比熵多出来的那一部分

KL 散度定义为：

```text
KL(p || q) = Σ p(x) log (p(x) / q(x))
```

把它展开：

```text
KL(p || q)
= Σ p(x) log p(x) - Σ p(x) log q(x)
= -H(p) + H(p, q)
```

整理得到：

```text
H(p, q) = H(p) + KL(p || q)
```

这条公式非常关键。它说明：

- `H(p)` 是真实分布自身无法消除的不确定性
- `KL(p || q)` 是模型与真实分布不一致造成的额外代价

由于 `H(p)` 与模型参数无关，所以训练时最小化交叉熵，本质上就是最小化 `KL(p || q)`。

### 为什么 KL 不对称

一般来说：

```text
KL(p || q) != KL(q || p)
```

原因很直观：  
“用 `q` 近似 `p`”和“用 `p` 近似 `q`”惩罚的是不同错误。

例如，如果真实分布 `p` 在某个位置有概率质量，而模型 `q` 在那里给了接近 0 的概率，那么 `KL(p || q)` 会非常大。这意味着在真实会发生的地方，模型却几乎认为“不可能”，代价会被重罚。

## 为什么分类任务更常用交叉熵而不是 MSE

这不只是经验问题，关键在于梯度行为。

### 先看二分类

设 logit 为 `z`，预测概率为：

```text
p = sigmoid(z)
```

如果标签 `y` 取 `0` 或 `1`，那么：

#### 交叉熵损失

```text
L_ce = -[y log p + (1-y) log (1-p)]
```

对 `z` 求导可得：

```text
∂L_ce/∂z = p - y
```

#### 均方误差损失

```text
L_mse = 1/2 (p - y)^2
```

对 `z` 求导：

```text
∂L_mse/∂z = (p - y) p (1 - p)
```

注意 MSE 比交叉熵多了一个因子：

```text
p(1 - p)
```

这个因子在 `p` 接近 `0` 或 `1` 时会非常小，导致梯度被额外压扁。

### 一个数值对比

设真实标签 `y = 1`：

| 预测概率 `p` | 交叉熵梯度 `p-y` | MSE 梯度 `(p-y)p(1-p)` |
| --- | --- | --- |
| `0.01` | `-0.99` | `-0.0098` |
| `0.50` | `-0.50` | `-0.1250` |
| `0.99` | `-0.01` | `-0.0001` |

这张表揭示了核心差异：

- 当模型**非常自信但预测错误**时，交叉熵仍会给出很强的纠偏信号
- MSE 则可能因为 sigmoid 饱和而让梯度变得极小，导致学习变慢

这也是为什么“分类 + sigmoid/softmax 输出”几乎总是配交叉熵，而不是 MSE。

## 一个更直观的三分类例子

设真实分布是 one-hot：

```text
p = [1, 0, 0]
```

比较两个模型输出：

```text
q_a = [0.7, 0.2, 0.1]
q_b = [0.1, 0.8, 0.1]
```

交叉熵分别是：

```text
H(p, q_a) = -log 0.7  ≈ 0.357
H(p, q_b) = -log 0.1  ≈ 2.303
```

虽然两个分布都满足“和为 1”，但第二个模型把大部分概率压到了错误类别上，因此损失会大很多。

如果一个模型给真实类别分配的概率是 0，理论上交叉熵会趋于无穷大。这正体现了信息论含义：在真实会发生的事件上，模型却断言“不可能”，代价应该极高。

## 交叉熵不是“万能损失”

理解它的适用边界同样重要。

### 什么时候它很合适

- 多分类、二分类
- 模型输出可以解释为概率分布
- 我们关心“真实标签概率是否被最大化”

### 什么时候不该机械使用

- 回归问题：通常更自然的是高斯假设对应的 MSE，或拉普拉斯假设对应的 MAE
- 生成任务里的软标签、蒸馏、标签平滑：此时真实分布不再是 one-hot，但交叉熵依然成立，只是解释应回到分布层面
- 某些校准或鲁棒优化场景：可能需要 focal loss、symmetric CE 或其他变体

关键点不是“交叉熵永远最好”，而是：**在分类的概率建模假设下，它有非常扎实的理论基础，而且梯度行为通常更适合优化。**

---

## 一个最小实验：看梯度随概率如何变化

下面代码能快速验证上面的数值结论：

```python
import numpy as np

probs = np.array([0.01, 0.1, 0.5, 0.9, 0.99])
y = 1.0

grad_ce = probs - y
grad_mse = (probs - y) * probs * (1 - probs)

for p, g1, g2 in zip(probs, grad_ce, grad_mse):
    print(f"p={p:.2f}, ce_grad={g1:.4f}, mse_grad={g2:.4f}")
```

如果你把结果画成曲线，会发现：

- 交叉熵梯度在预测错误且置信度高时依然保持较大幅度
- MSE 梯度在两端迅速贴近 0

这就是许多“为什么模型明明错得离谱却学得很慢”的根源之一。

## 小结

熵衡量真实分布自身的不确定性，交叉熵衡量“用模型分布去描述真实分布”的平均代价，KL 散度则刻画了模型相对真实分布多付出的那部分成本。公式

```text
H(p, q) = H(p) + KL(p || q)
```

把三者清楚地连在了一起，也解释了为什么训练分类器时最小化交叉熵，本质上就是让模型分布尽量逼近真实分布。

进一步看梯度，你会发现交叉熵不仅理论上自然，优化上也比 MSE 更适合概率分类任务。它既有信息论解释，又有最大似然解释，还直接对应更健康的梯度信号。这也是它成为现代分类任务默认损失函数的根本原因。
