## 核心结论

奖励塑形（reward shaping）是在环境原始奖励之外，再加一项人工设计的辅助奖励，用更密集的反馈加快强化学习收敛。白话说，原始奖励只告诉智能体“最后有没有做对”，塑形奖励额外告诉它“现在是不是在朝对的方向走”。

最重要的安全形式是基于势能的奖励塑形（Potential-Based Reward Shaping, PBRS）：

$$
R'(s,a,s') = R(s,a,s') + F(s,s')
$$

$$
F(s,s') = \gamma \Phi(s') - \Phi(s)
$$

其中，$\Phi(s)$ 是势能函数，意思是“状态 $s$ 离目标还有多近”的一个分数；$\gamma$ 是折扣因子，意思是“未来奖励在当前值里打几折”。当塑形项严格写成这个形式，并且使用与环境一致的 $\gamma$ 时，最优策略不变，变化的是学习速度和训练稳定性。

| 问题 | 结论 |
|---|---|
| 是否改变最优策略 | 势能型塑形在条件满足时不改变 |
| 主要作用 | 缓解稀疏奖励，加快学习 |
| 典型势能设计 | 到目标的距离、子目标进度 |
| 主要风险 | 非势能塑形可能诱导次优行为 |

玩具例子可以先这样理解：原始奖励像“只有到终点才发奖金”，奖励塑形像“每走近终点一点，就给一点中间反馈”。终点奖金没有改，但训练不再完全靠碰运气。

## 问题定义与边界

强化学习里最难处理的问题之一是稀疏奖励（sparse reward）。稀疏奖励就是大部分时间奖励都为 0，只有任务成功或失败的瞬间才有反馈。对零基础读者来说，可以把它理解成：智能体做了大量动作，却几乎得不到“这一步到底好不好”的信号。

例如导航任务中，环境奖励可能只有一条规则：

- 到达终点：`+1`
- 其余步骤：`0`

这定义了任务目标，但学习会很慢。因为在训练初期，智能体很难偶然走到终点，于是几乎所有轨迹回报都一样差，算法无法区分“哪种探索更接近成功”。

奖励塑形就是在不改任务目标的前提下，增加辅助学习信号。本文只讨论一个严格边界：势能型塑形，也就是满足 $F(s,s')=\gamma \Phi(s')-\Phi(s)$ 的塑形项。原因很直接，这一类方法有明确理论保证；其他“看起来合理”的加分规则，往往会偷偷改掉最优解。

| 概念 | 含义 |
|---|---|
| 原始奖励 $R$ | 环境定义的任务目标 |
| 塑形奖励 $F$ | 人工附加的辅助反馈 |
| 势能函数 $\Phi(s)$ | 衡量状态接近目标程度的标量 |
| 折扣因子 $\gamma$ | 未来回报相对当前回报的权重 |
| 最优策略 | 长期累计回报最大的动作选择规则 |

边界要明确：

- 只讨论折扣回报的 MDP 设定。MDP 是马尔可夫决策过程，可以理解成“状态转移和奖励都按固定规则发生”的标准强化学习框架。
- 只讨论塑形项与环境使用同一个 $\gamma$。
- 只讨论终止态处理清楚的情况，通常设 $\Phi(\text{terminal})=0$。
- 不把所有 reward engineering 混为一谈。Reward engineering 是奖励工程，意思是系统化设计训练信号；PBRS 只是其中一类。

真实工程例子是机器人抓取。若奖励只有“抓住物体才得 1 分”，早期训练几乎学不到东西；若把夹爪与物体距离、姿态对齐程度编码进 $\Phi(s)$，智能体即使还没成功抓住，也能收到“更接近成功”的中间信号。

## 核心机制与推导

PBRS 的核心不是“凭空造奖励”，而是构造一种会在整条轨迹上相消的奖励差分。这里的相消，指的是相邻步骤之间的塑形项会像望远镜展开那样大量抵消。

设一条轨迹为 $s_0, s_1, \dots, s_T$，塑形项总贡献为：

$$
\sum_{t=0}^{T-1}\gamma^t F(s_t,s_{t+1})
=
\sum_{t=0}^{T-1}\gamma^t(\gamma\Phi(s_{t+1})-\Phi(s_t))
$$

展开后得到：

$$
-\Phi(s_0) + \gamma^T \Phi(s_T)
$$

如果终止态满足 $\Phi(s_T)=0$，那么整条轨迹上的附加回报只剩：

$$
-\Phi(s_0)
$$

它只和起点有关，不依赖中间选择了哪个动作。因此，对同一状态 $s$ 下的不同动作比较，所有动作的 $Q$ 值都会整体平移同一个常数：

$$
Q^*_{M'}(s,a) = Q^*_M(s,a) - \Phi(s)
$$

于是：

$$
\arg\max_a Q^*_{M'}(s,a) = \arg\max_a Q^*_M(s,a)
$$

这就是“最优策略不变”的真正原因。不是因为塑形奖励很小，也不是因为人设计得巧，而是因为它满足一个严格的代数结构。

看一个玩具例子。设 $\gamma=0.9$，目标态 `G` 才有原始奖励 `1`。定义三个状态：

- 起点 `S`
- 中间态 `M`
- 目标态 `G`

设势能函数：

- $\Phi(S)=0$
- $\Phi(M)=0.5$
- $\Phi(G)=0$

比较两条路径。

1. 直接到终点：`S -> G`

原始奖励是 `1`，塑形项是：

$$
F(S,G)=0.9\times 0 - 0 = 0
$$

总回报为 `1`。

2. 先经过中间态：`S -> M -> G`

第一步塑形奖励：

$$
F(S,M)=0.9\times 0.5 - 0 = 0.45
$$

第二步塑形奖励：

$$
F(M,G)=0.9\times 0 - 0.5 = -0.5
$$

若第二步原始奖励是 `1`，则折扣总回报为：

$$
0.45 + 0.9 \times (1 - 0.5) = 0.9
$$

可见，中间路径在第一步拿到了更密集的正反馈，但最终动作排序没有颠倒，直接到达终点仍更优。

| 步骤 | 原始奖励 | 塑形项 | 折扣后贡献 |
|---|---:|---:|---:|
| `S -> M` | 0 | 0.45 | 0.45 |
| `M -> G` | 1 | -0.5 | 0.9 × 0.5 = 0.45 |
| 合计 |  |  | 0.90 |

直观看，PBRS 给了学习过程“路标”，但没有偷偷把比赛终点改掉。

## 代码实现

工程上最重要的原则是：原始奖励和塑形奖励分开记账。这样你能判断性能提升到底来自任务完成率上升，还是只是来自辅助项刷高了训练回报。

下面是一个可运行的最小 Python 示例，演示 PBRS 的奖励计算与简单断言：

```python
from dataclasses import dataclass

GAMMA = 0.9

phi_table = {
    "S": 0.0,
    "M": 0.5,
    "G": 0.0,
}

def phi(state: str) -> float:
    return phi_table[state]

def shaped_reward(r_env: float, s: str, s_next: str, gamma: float = GAMMA) -> float:
    return r_env + gamma * phi(s_next) - phi(s)

# 玩具环境：只有到达 G 才有原始奖励 1
r_direct = shaped_reward(1.0, "S", "G")
r_step1 = shaped_reward(0.0, "S", "M")
r_step2 = shaped_reward(1.0, "M", "G")
return_via_m = r_step1 + GAMMA * r_step2

assert abs(r_direct - 1.0) < 1e-9
assert abs(r_step1 - 0.45) < 1e-9
assert abs(r_step2 - 0.5) < 1e-9
assert abs(return_via_m - 0.9) < 1e-9
assert r_direct > return_via_m

print("All assertions passed.")
```

训练循环里通常这样接入：

```python
r_env = env_reward(s, a, s_next)
r_shape = gamma * phi(s_next) - phi(s)
r_total = r_env + r_shape

agent.update(s, a, r_total, s_next, done)
logger.log({
    "reward_env": r_env,
    "reward_shape": r_shape,
    "reward_total": r_total,
})
```

| 组件 | 作用 | 实现建议 |
|---|---|---|
| `env_reward` | 保留任务真实目标 | 不要直接覆盖 |
| `phi(s)` | 定义状态势能 | 先用简单启发式 |
| `r_shape` | 计算塑形项 | 严格按 $\gamma\Phi(s')-\Phi(s)$ |
| `r_total` | 训练用奖励 | 仅供优化器更新 |
| 日志系统 | 做对照实验 | 必须分别记录三种奖励 |

真实工程例子是仓储机器人导航。原始奖励可以是“到达货架 +10，碰撞 -10，其余 0”；势能函数可以设成“当前位置到目标货架的负最短路径长度”。这样机器人在复杂地图中不必等到真正到达货架，靠近目标时就已经能得到正向训练信号。

## 工程权衡与常见坑

第一类坑是非势能塑形。比如“每朝目标前进一步就加 `+0.1`”。这听起来合理，但它不一定能写成 $\gamma\Phi(s')-\Phi(s)$。结果是智能体可能学会在目标附近来回晃动刷分，而不是尽快完成任务。

第二类坑是折扣因子不一致。环境更新若使用 $\gamma=0.99$，塑形项却按 `0.9` 算，理论保证直接失效。很多实现问题不是出在算法，而是出在配置文件。

第三类坑是终止态处理错误。若成功终止和时间截断混在一起，或者终止态还带着非零 $\Phi$，轨迹尾部就会残留额外项，回报被污染。

第四类坑是函数逼近误差。函数逼近就是用神经网络近似 $Q$ 或策略，不再是表格精确计算。理论上 PBRS 不改最优策略，但在近似学习里，优化路径、探索分布、梯度尺度都会变，实际行为仍可能偏移。所以“理论安全”不等于“训练自动没问题”。

第五类坑是塑形过强。常见做法是再乘一个系数 $\beta$：

$$
R'(s,a,s') = R(s,a,s') + \beta(\gamma\Phi(s')-\Phi(s))
$$

$\beta$ 太大时，训练目标会被辅助信号主导，成功率反而下降。

| 问题 | 后果 | 规避方式 |
|---|---|---|
| 非势能塑形 | 改变最优策略 | 只用标准 PBRS 形式 |
| $\gamma$ 不一致 | 不变性失效 | 环境与塑形统一配置 |
| 终止态处理混乱 | 回报尾部污染 | 统一设 $\Phi(\text{terminal})=0$ |
| 塑形系数过大 | 追逐辅助信号 | 从小系数开始做消融 |
| 只看总回报 | 误判训练效果 | 分别记录环境回报和成功率 |

实践里至少看四个指标：

- 原始环境回报是否上升
- 任务成功率是否上升
- 塑形奖励是否异常主导总回报
- 策略动作分布是否出现刷分模式

## 替代方案与适用边界

PBRS 适合“目标明确、进度可度量、原始奖励稀疏”的任务。导航、机械臂抓取、轨迹跟踪、带清晰终点的控制任务都很适合。因为这些任务通常能写出一个有物理意义的 $\Phi(s)$，例如负距离、子目标完成度、可达性评分。

不适合的场景也很明确。若任务没有清晰目标距离，例如开放式对话、复杂博弈、多目标长期规划，设计 $\Phi(s)$ 往往比直接学策略还难。这时强行塑形，常常只是把设计者的偏见硬塞进奖励里。

| 方法 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|
| 势能型奖励塑形 | 理论清楚，不改最优策略 | 需要可设计的 $\Phi$ | 导航、抓取、到点控制 |
| 稀疏奖励 + 经验回放 | 实现简单 | 样本效率低 | 稀疏程度中等任务 |
| 课程学习 | 逐步增加难度 | 需要人工分阶段 | 长时序技能学习 |
| 子目标分解 | 中间反馈强 | 设计和维护成本高 | 复杂流程任务 |
| 模仿学习 | 冷启动快 | 依赖示范数据 | 可获得专家轨迹的任务 |

判断是否该用 PBRS，可以问四个问题：

- 任务目标是否明确且终止条件清晰？
- 是否能定义“更接近目标”的状态量？
- 这个状态量是否能稳定计算，而不是依赖未来信息？
- 是否能通过对照实验验证“学习更快，但目标没变”？

如果这四个问题里前两个都答不上来，通常不该先做 PBRS，而应先检查状态表示、探索机制或数据效率问题。
