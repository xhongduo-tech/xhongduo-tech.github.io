## 核心结论

次梯度是凸优化里处理“不可微”函数的标准工具。所谓不可微，就是函数在某些点没有唯一导数，例如绝对值函数在 $x=0$ 处有拐角，ReLU 在 $x=0$ 处左右斜率不同。这时不能再问“导数是多少”，而是改问“哪些斜率可以作为这个点的全局下界支撑线”。这些合法斜率组成的集合，就是次微分，记作 $\partial f(x)$。

对凸函数 $f$，若向量 $g$ 满足

$$
f(y)\ge f(x)+g^\top(y-x),\quad \forall y
$$

那么 $g$ 就是点 $x$ 处的一个次梯度。白话讲，这个不等式表示：以 $g$ 为斜率、经过 $(x,f(x))$ 的那条直线或超平面，永远不会从函数图像上方穿过去，它只能贴着函数或者在函数下方。

次梯度法则把普通梯度下降

$$
x_{t+1}=x_t-\eta_t \nabla f(x_t)
$$

替换成

$$
x_{t+1}=x_t-\eta_t g_t,\quad g_t\in \partial f(x_t)
$$

从而让不可微点也能继续迭代。它的核心代价是速度更慢。对一般凸、非光滑问题，典型收敛率是 $O(1/\sqrt{T})$；而对光滑凸问题，梯度下降通常能做到 $O(1/T)$。这不是实现技巧差异，而是问题结构不同导致的理论上限不同。

一个必须记住的事实是：在不可微点，次梯度通常不是唯一的。比如 $f(x)=|x|$，则

$$
\partial f(0)=[-1,1]
$$

这说明在原点处，任何介于 $-1$ 和 $1$ 之间的斜率都合法。选择不同的次梯度，迭代路径会不同，但只要步长规则满足要求，整体仍然有收敛保证。

## 问题定义与边界

本文讨论的对象有明确边界：主要针对**凸函数**上的次梯度和次梯度法。凸函数可以理解为“任意两点连线都在图像上方”的函数，这个结构保证支撑线不等式成立，也保证次微分理论好用。如果函数不是凸的，那么“次梯度”这个词在不同文献中可能对应不同广义导数概念，结论会明显复杂，不属于本文范围。

先给出正式定义。对凸函数 $f:\mathbb{R}^n\to\mathbb{R}$，若存在向量 $g$ 使得对所有 $y$ 都有

$$
f(y)\ge f(x)+g^\top(y-x)
$$

则称 $g$ 是 $x$ 处的次梯度，所有这类 $g$ 的集合记作 $\partial f(x)$，称为次微分。这里的“微分”是“导数集合”的意思，不再是一个单值。

如果函数在某点可微，那么次微分会退化成单点：

$$
\partial f(x)=\{\nabla f(x)\}
$$

也就是说，次梯度并不是另一套和导数无关的东西，而是导数在凸非光滑场景下的自然扩展。可微时它与普通梯度完全一致，不可微时它给出一组可行方向。

下面用最小玩具例子说明点类型的差异。令

$$
f(x)=|x|=\max\{-x,x\}
$$

这个写法很重要，因为它直接揭示了 $|x|$ 是两个线性函数的最大值。在线性函数切换的点，也就是 $x=0$，函数出现拐角，因此不可微。

| 点类型 | 条件 | 次微分结构 | 解释 |
|---|---|---|---|
| 可微点 | $x>0$ | $\partial f(x)=\{1\}$ | 右侧就是斜率为 1 的直线 |
| 可微点 | $x<0$ | $\partial f(x)=\{-1\}$ | 左侧就是斜率为 -1 的直线 |
| 拐角点 | $x=0$ | $\partial f(0)=[-1,1]$ | 任意夹在左右斜率之间的值都可作支撑线斜率 |

这个表反映了一个普遍规律：对凸函数，拐角处的次微分通常是一个区间或凸集，而不是单个数。

还要明确一个常见边界：不是“任何不可微函数都能直接套次梯度法”。例如 $f(x)=-|x|$ 在 $x=0$ 不可微，但它是凹函数，不满足凸性，经典次梯度理论不直接适用。另一个边界是约束优化。如果问题带有可行域 $x\in\mathcal{C}$，常见做法不是裸更新，而是投影次梯度法：

$$
x_{t+1}=\Pi_{\mathcal C}(x_t-\eta_t g_t)
$$

其中 $\Pi_{\mathcal C}$ 表示投影到可行集合。白话讲，先按次梯度走一步，再拉回合法区域。

## 核心机制与推导

次梯度法的更新式很简单：

$$
x_{t+1}=x_t-\eta_t g_t,\quad g_t\in\partial f(x_t)
$$

真正的关键不在公式本身，而在两个条件：

1. $g_t$ 必须真的是当前点的次梯度。
2. 步长 $\eta_t$ 不能乱设，必须逐步减小。

为什么普通固定步长不行？因为次梯度方向不一定是严格下降方向。对光滑函数，梯度方向在局部有稳定的一阶近似；对非光滑函数，拐角附近可能一直左右摇摆。固定步长会让这种摇摆长期存在，导致无法逼近最优点。

经典步长规则有两类。

第一类是满足

$$
\sum_{t=1}^{\infty}\eta_t=\infty,\qquad \sum_{t=1}^{\infty}\eta_t^2<\infty
$$

这叫“平方可和但不可和”。白话讲，步子要越来越小，但总路程不能太短，否则还没走到最优区域就停了；同时步子平方和要有限，否则震荡噪声累积太大。典型例子是 $\eta_t=c/t$。

第二类是实际更常见的非可和递减规则，比如

$$
\eta_t=\frac{c}{\sqrt{t}}
$$

它不满足上面的平方可和条件，但在最优值误差界分析中非常常见，并能给出

$$
f_{\text{best}}^{(T)}-f^\star = O\left(\frac{1}{\sqrt{T}}\right)
$$

其中 $f_{\text{best}}^{(T)}=\min_{1\le t\le T}f(x_t)$。注意这里常追踪“历史最好值”而不是最后一次迭代值，因为次梯度法的轨迹经常不单调。

可以用一个标准不等式理解它为什么慢。设最优点为 $x^\star$，并假设所有次梯度有界，即 $\|g_t\|\le G$。展开欧氏距离平方：

$$
\|x_{t+1}-x^\star\|^2
=
\|x_t-\eta_t g_t-x^\star\|^2
=
\|x_t-x^\star\|^2-2\eta_t g_t^\top(x_t-x^\star)+\eta_t^2\|g_t\|^2
$$

由次梯度定义，对 $y=x^\star$ 有

$$
f(x^\star)\ge f(x_t)+g_t^\top(x^\star-x_t)
$$

移项得到

$$
g_t^\top(x_t-x^\star)\ge f(x_t)-f(x^\star)
$$

代回上式：

$$
\|x_{t+1}-x^\star\|^2
\le
\|x_t-x^\star\|^2
-2\eta_t(f(x_t)-f^\star)
+\eta_t^2 G^2
$$

把这个不等式从 $1$ 累加到 $T$，再整理，就能得到平均意义下的误差上界，最终导出 $O(1/\sqrt{T})$。这一步告诉我们，次梯度法并不依赖函数可微，而是依赖“支撑线不等式”来控制每一步和最优点的距离变化。

### 玩具例子：$f(x)=|x|$

这是理解次梯度最直接的例子。

- 当 $x>0$，唯一合法次梯度是 $g=1$
- 当 $x<0$，唯一合法次梯度是 $g=-1$
- 当 $x=0$，任意 $g\in[-1,1]$ 都合法

假设初始点 $x_0=0$。

- 若取 $g_0=0$，则 $x_1=x_0-\eta_0 g_0=0$，算法直接停在原点
- 若取 $g_0=1$，则 $x_1=-\eta_0$，会向左走
- 若取 $g_0=-1$，则 $x_1=\eta_0$，会向右走

这正说明：在不可微点，不是“没有方向”，而是“有一组合法方向”。次梯度法允许你从中任选一个。

但也正因如此，它的轨迹可能来回跳。比如当 $x_t$ 在 0 左右反复穿过时，$g_t$ 会在 $\{-1,1\}$ 间切换，因此必须依赖衰减步长让振幅越来越小。

### 真实工程例子：带 L1 正则的线性模型

考虑稀疏回归问题：

$$
\min_w \frac{1}{2m}\|Xw-y\|_2^2+\lambda\|w\|_1
$$

这里 $\|w\|_1=\sum_i |w_i|$ 是 L1 正则，用于把很多参数压到 0，得到稀疏模型。稀疏的意思是“大部分参数为 0，只有少量特征被保留”。

目标函数的第一项是光滑的平方损失，第二项在 $w_i=0$ 处不可微。直接使用次梯度法时，

$$
g_t=\frac{1}{m}X^\top(Xw_t-y)+\lambda s_t,\quad s_t\in\partial \|w_t\|_1
$$

其中

$$
(s_t)_i=
\begin{cases}
1,& (w_t)_i>0\\
-1,& (w_t)_i<0\\
[-1,1],& (w_t)_i=0
\end{cases}
$$

这在理论上完全合法，但在工程上通常不够好，因为参数在 0 附近会频繁抖动，收敛慢，且难以稳定地产生精确 0。这个问题会在“替代方案”部分和近端梯度法对比。

## 代码实现

下面给出一个最小可运行实现，目标函数取 $f(x)=|x|$。代码展示三个关键点：

1. 在每一步取一个合法次梯度。
2. 用 $\eta_t=c/\sqrt{t}$ 衰减步长。
3. 跟踪历史最好目标值，而不是只看最后一步。

```python
import math

def f(x: float) -> float:
    return abs(x)

def subgradient_abs(x: float) -> float:
    # 对 |x|，在 0 处选 0；这是合法次梯度之一
    if x > 0:
        return 1.0
    if x < 0:
        return -1.0
    return 0.0

def subgradient_descent_abs(x0: float, c: float = 0.5, T: int = 200):
    x = x0
    f_best = f(x)
    history = [(0, x, f(x), f_best)]

    for t in range(1, T + 1):
        g = subgradient_abs(x)
        eta = c / math.sqrt(t)
        x = x - eta * g
        fx = f(x)
        f_best = min(f_best, fx)
        history.append((t, x, fx, f_best))

    return x, f_best, history

x_final, f_best, history = subgradient_descent_abs(5.0, c=0.8, T=500)

# 历史最好值不会比初始值更差
assert f_best <= history[0][2]

# 对 |x| 的最优值是 0，足够多迭代后应接近 0
assert f_best < 0.2

# 原点处的几个合法次梯度都满足定义
x = 0.0
for g in (-1.0, -0.3, 0.0, 0.7, 1.0):
    for y in (-2.0, -0.5, 0.0, 0.5, 3.0):
        assert abs(y) >= abs(x) + g * (y - x)

print("x_final =", round(x_final, 4))
print("f_best  =", round(f_best, 6))
```

这段代码故意在 $x=0$ 处固定选 $g=0$，因为它最利于展示“次梯度是集合，不是唯一值”。如果你改成在 0 处随机选 $[-1,1]$ 中一个值，轨迹会更抖，但定义仍然成立。

如果目标是一般凸函数，代码骨架通常写成下面这样：

```python
import math

def subgradient_method(x0, f, subgrad, c=1.0, T=1000):
    x = x0
    best_x = x0
    best_f = f(x0)

    for t in range(1, T + 1):
        g = subgrad(x)              # 要求 g ∈ ∂f(x)
        eta = c / math.sqrt(t)      # 常见衰减步长
        x = x - eta * g

        fx = f(x)
        if fx < best_f:
            best_f = fx
            best_x = x

    return best_x, best_f
```

如果问题有约束，比如参数必须非负，那么把更新改成

```python
x = project(x - eta * g)
```

即可。这里的 `project` 就是投影函数，作用是把非法点拉回可行集合。

### ReLU 的代码语境

在神经网络中，ReLU 定义为

$$
\mathrm{ReLU}(x)=\max(0,x)
$$

它在 $x=0$ 处不可微，但这是凸函数，因此存在次微分：

$$
\partial \mathrm{ReLU}(0)=[0,1]
$$

框架实现反向传播时，必须在 0 处选一个具体值，常见选择是 0 或 1。也就是说，深度学习框架并不是“无视不可微问题”，而是实际使用了一个特定次梯度约定。

## 工程权衡与常见坑

次梯度法最大的工程优点是普适。只要目标是凸的、你能写出次梯度，就能开始优化。它不要求 Hessian，不要求光滑，不要求复杂算子。缺点同样直接：慢、抖、对步长敏感。

最典型的坑如下。

| 坑 | 现象 | 缓解 |
|---|---|---|
| 步长不衰减 | 参数长期左右震荡，目标值不稳定 | 用 $\eta_t=c/\sqrt{t}$ 或其他递减规则 |
| 只看最后一次迭代 | 最后一步可能比中间很多步都差 | 跟踪 $f_{\rm best}$ 或做迭代平均 |
| 在 L1 正则下直接裸跑次梯度 | 参数在 0 附近来回抖，难得到稳定稀疏解 | 用 ISTA/FISTA 等近端方法 |
| 次梯度范数过大 | 更新步突然很大，训练不稳 | 做梯度裁剪或缩小常数 $c$ |
| ReLU 在 0 处总取 0 | 一些神经元长期不更新，出现“死神经元” | 用 Leaky ReLU、改初始化、调学习率 |
| 把非凸问题当凸问题处理 | 理论保证失效，结果不可解释 | 先确认函数是否凸，再选算法 |

这里展开两个工程上最常见的真实问题。

### 问题一：L1 正则下抖动严重

在线性模型、逻辑回归、稀疏编码中，L1 正则是非常常见的结构。它的目标函数通常是“光滑损失 + 非光滑正则”。如果直接把整个目标丢给次梯度法，虽然能跑，但每个坐标在 0 周围的更新常常是“推过去又拉回来”。你会看到损失下降慢、参数稀疏模式不稳定、迭代次数明显偏大。

### 问题二：ReLU 的 0 点选择

在神经网络训练中，ReLU 的负半轴梯度为 0。如果某个神经元长期落在负区间，参数就几乎收不到有效更新，这就是“死神经元”。严格说，这不完全是次梯度法本身造成的，而是函数形状和训练动态共同导致的。但 0 点如何选次梯度，会影响边界附近样本能否继续传播更新信号。

因此，工程里常见策略不是纠结“0 点到底该取 0 还是 1”，而是从更高层面避免大量神经元长期卡死：比如更好的初始化、较小学习率、使用 Leaky ReLU 或 PReLU。

## 替代方案与适用边界

次梯度法不是处理不可微问题的唯一方法。真正要做工程时，常常应该先问：这个非光滑结构是不是“有特殊形式”。如果答案是是，那么近端梯度法通常更合适。

考虑复合优化问题：

$$
\min_x F(x)=f(x)+\lambda g(x)
$$

其中 $f$ 是光滑凸函数，且梯度 Lipschitz 连续；$g$ 是可能不可微但近端算子容易计算的凸函数。这里“Lipschitz 连续”可以理解为梯度变化不会无限陡，存在全局上界控制。

此时可用近端梯度迭代：

$$
x_{k+1}=\mathrm{prox}_{\eta \lambda g}\left(x_k-\eta \nabla f(x_k)\right)
$$

其中近端算子定义为

$$
\mathrm{prox}_{\tau g}(v)=\arg\min_x\left(g(x)+\frac{1}{2\tau}\|x-v\|_2^2\right)
$$

白话讲，这一步先按光滑部分做梯度下降，再用一个“带结构的纠偏步骤”处理不可微部分。它不是简单换个梯度，而是把不可微项单独解掉。

对 L1 正则，$g(x)=\|x\|_1$，其近端算子有闭式解，也就是软阈值：

$$
\mathrm{prox}_{\tau\|\cdot\|_1}(v)_i
=
\mathrm{sign}(v_i)\max(|v_i|-\tau,0)
$$

这正是稀疏建模里最重要的原因：它能直接把小系数压成精确的 0，而不是像次梯度法那样在 0 附近抖动。

| 方法 | 更新方式 | 典型收敛率 | 适用场景 |
|---|---|---|---|
| 次梯度法 | $x_{t+1}=x_t-\eta_t g_t$ | $O(1/\sqrt{T})$ | 一般凸非光滑，结构弱但通用 |
| ISTA | 梯度步 + 近端步 | $O(1/T)$ | 光滑项 + 可计算 prox 的非光滑项 |
| FISTA | ISTA + 动量加速 | $O(1/T^2)$ | 同上，但更追求速度 |
| 普通梯度下降 | 只对光滑项更新 | $O(1/T)$ | 全部光滑凸问题 |

### 什么时候仍然应该用次梯度法

1. 目标函数是凸的，但不可微部分没有好用的近端算子。
2. 只能获得一个合法次梯度，无法高效解近端子问题。
3. 需要一个实现极简、理论保证直接、可快速验证原型的基线方法。

### 什么时候不该优先用次梯度法

1. 问题能写成“光滑 + 结构化非光滑”，尤其是 L1、组稀疏、核范数这类典型正则。
2. 你关心收敛速度和数值稳定性，而不是只求“能跑”。
3. 你需要精确稀疏解，而不是仅仅让参数接近 0。

从工程视角看，次梯度法更像“最低可行方案”。它的重要性不在于总是最好，而在于它定义了不可微优化最基本的起点：即使导数不存在，只要函数是凸的，仍然可以用支撑线的斜率集合构造一阶方法。
