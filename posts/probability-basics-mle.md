## 为什么机器学习离不开概率

机器学习面对的从来不是“绝对确定”的世界。输入有噪声，标签会错，数据分布会漂移，模型只能基于观察到的样本做推断。在这种前提下，概率论不是可选装饰，而是整个学习问题的语言。

当我们说“模型输出一个概率分布”时，本质上是在回答两个问题：

- 数据可能以什么方式产生？
- 哪一组参数最能解释已经观察到的数据？

前者把我们带到随机变量、分布、期望与方差；后者把我们带到最大似然估计（MLE）和交叉熵损失。

---

## 随机变量、分布、期望、方差：先建立最小词汇表

### 随机变量是什么

随机变量可以理解为“把随机结果数值化”的函数。抛硬币时，我们可以定义：

```text
X = 1  表示正面
X = 0  表示反面
```

此时 `X` 就是一个随机变量。

随机变量通常分两类：

- 离散型：取值是一个个可枚举的点，例如伯努利、多项分布
- 连续型：取值来自连续区间，例如高斯分布

### 概率分布是什么

概率分布描述“每个结果有多可能发生”。

几个机器学习里最常见的例子：

| 分布 | 场景 | 关键参数 |
| --- | --- | --- |
| 伯努利分布 | 二分类成功 / 失败 | `p` |
| 多项分布 / 类别分布 | 多分类标签 | 每个类别的概率 |
| 高斯分布 | 连续噪声、特征建模 | 均值 `mu`、方差 `sigma^2` |

例如伯努利分布中：

```text
P(X = 1) = p
P(X = 0) = 1 - p
```

### 期望：长期平均水平

离散随机变量的期望定义为：

```text
E[X] = Σ x P(X = x)
```

它不等于某次观测值，而是重复试验足够多次后的平均趋势。比如伯努利分布的期望就是：

```text
E[X] = p
```

### 方差：波动有多大

方差定义为：

```text
Var(X) = E[(X - E[X])^2]
```

它衡量的是“离平均值有多散”。高方差意味着样本波动大，低方差意味着样本更集中。

对高斯分布 `N(mu, sigma^2)` 来说：

- `mu` 决定中心位置
- `sigma^2` 决定分布宽度

机器学习里很多“噪声模型”默认都假设误差近似高斯，原因正是它既常见又便于分析。

## 贝叶斯定理：从“参数产生数据”反过来推“数据支持什么参数”

概率建模里常见两种方向：

- 正向：给定参数 `theta`，数据 `X` 出现的概率有多大，即 `P(X | theta)`
- 反向：给定数据 `X`，参数 `theta` 有多可信，即 `P(theta | X)`

贝叶斯定理把这两者联系起来：

```text
P(theta | X) = P(X | theta) P(theta) / P(X)
```

常写成比例形式：

```text
P(theta | X) ∝ P(X | theta) P(theta)
```

这里：

- `P(X | theta)` 是似然（likelihood）
- `P(theta)` 是先验（prior）
- `P(theta | X)` 是后验（posterior）

它告诉我们：一个参数是否可信，不仅取决于它能否解释数据，还取决于我们事先对它的偏好。

如果把先验忽略，只保留“谁最能解释观察到的数据”，就得到了最大似然估计。

---

## 最大似然估计：找出最能解释数据的参数

假设我们观察到一组样本 `x_1, x_2, ..., x_n`，并假设它们独立同分布于某个参数化分布 `p(x | theta)`。那么整组数据出现的概率是：

```text
L(theta) = Π p(x_i | theta)
```

这就是似然函数。最大似然估计要做的是：

```text
theta_hat = argmax_theta L(theta)
```

因为乘积不好算，通常转成对数似然：

```text
log L(theta) = Σ log p(x_i | theta)
```

对数是单调函数，所以最大化对数似然和最大化似然本身完全等价，但计算稳定性更好，推导也更简单。

---

## 推导高斯分布的 MLE：为什么均值就是样本平均值

设样本来自高斯分布 `N(mu, sigma^2)`，概率密度是：

```text
p(x | mu, sigma^2)
= 1 / sqrt(2πsigma^2) * exp(-(x - mu)^2 / (2sigma^2))
```

对 `n` 个样本，忽略与参数无关的常数后，对数似然可以写成：

```text
log L(mu, sigma^2)
= -n/2 * log(sigma^2) - (1 / (2sigma^2)) Σ (x_i - mu)^2 + C
```

### 对均值 `mu` 求导

```text
∂ log L / ∂ mu = (1 / sigma^2) Σ (x_i - mu)
```

令其为 0：

```text
Σ (x_i - mu) = 0
n mu = Σ x_i
mu_hat = (1/n) Σ x_i
```

也就是说，高斯分布均值的 MLE 就是样本均值。

### 对方差 `sigma^2` 求导

令导数为 0 后，可得到：

```text
sigma_hat^2 = (1/n) Σ (x_i - mu_hat)^2
```

注意这里分母是 `n`，不是 `n - 1`。  
`1 / (n - 1)` 出现在统计学里常说的**无偏样本方差**，它不是高斯方差的 MLE。这是一个非常经典、也非常容易混淆的地方。

### 一个最小数值例子

```python
import numpy as np

x = np.array([4.8, 5.2, 5.1, 4.9, 5.0])
mu_hat = x.mean()
sigma2_hat = ((x - mu_hat) ** 2).mean()

print(mu_hat)      # 5.0
print(sigma2_hat)  # 0.02
```

如果你把别的 `mu` 和 `sigma^2` 带回对数似然里比较，会发现这组估计确实让观测数据“最不意外”。

## 从 MLE 到交叉熵：分类损失为什么长这样

假设我们做 `K` 类分类，模型输出：

```text
p_theta(y = k | x)
```

对一个样本 `(x_i, y_i)`，似然就是模型给真实类别的概率：

```text
p_theta(y_i | x_i)
```

对整个数据集：

```text
L(theta) = Π p_theta(y_i | x_i)
```

取对数：

```text
log L(theta) = Σ log p_theta(y_i | x_i)
```

做最大似然估计，就等价于最小化负对数似然：

```text
-log L(theta) = -Σ log p_theta(y_i | x_i)
```

如果把标签写成 one-hot 形式 `y_{ik}`，上式就变成：

```text
-Σ_i Σ_k y_{ik} log p_theta(k | x_i)
```

这正是多分类交叉熵损失。

换句话说，**交叉熵不是拍脑袋选出来的损失函数，它是“假设标签来自类别分布，并对参数做最大似然估计”后的自然结果。**

---

## 为什么“最大似然”这么自然

从直觉上看，MLE 做的事情很朴素：

- 如果模型给真实观测分配高概率，说明它解释数据的能力强
- 如果模型总让真实观测变成“小概率事件”，说明它的参数不好

这与训练分类器的目标完全一致。我们不需要模型在所有类别上都“平均不错”，而是希望它对真实标签足够自信，并且这种自信来自可泛化的结构，而不是碰巧。

对于回归任务，如果我们假设误差服从高斯分布，会发现最小化平方误差同样对应最大似然。很多常见损失函数背后其实都有概率模型假设。

---

## 一个非常重要的区分：MLE 不是贝叶斯推断

两者都会从数据反推参数，但目标不同：

- MLE：只找一个最可能的参数点 `theta_hat`
- 贝叶斯推断：求完整后验分布 `P(theta | X)`

MLE 简洁、计算方便，因此在深度学习训练里非常常见；贝叶斯方法则更强调不确定性表达和先验知识整合。

如果你看到论文里提“MAP 估计”，它本质上是在 MLE 的基础上加入先验，相当于最大化：

```text
log P(X | theta) + log P(theta)
```

从优化角度看，很多正则化项都可以解释成某种先验。

---

## 常见误区

### 1. 概率输出不等于模型就一定“懂不确定性”

Softmax 给出的数字满足概率形式，但校准是否准确，取决于模型、数据和训练方式。高置信度不一定代表高可靠性。

### 2. 方差的 `1/n` 和 `1/(n-1)` 不能混用

前者是高斯方差的 MLE，后者是无偏估计。它们服务的目标不同。

### 3. 交叉熵不是孤立技巧

它和概率建模、似然函数、最大似然估计是一条完整链路。只背损失公式而不理解概率解释，后面很难真正看懂标签平滑、校准或贝叶斯扩展。

---

## 小结

随机变量和概率分布为“数据如何生成”提供了语言，期望和方差刻画了平均行为与波动程度，贝叶斯定理把“参数生成数据”和“数据支持参数”联系起来。最大似然估计进一步把学习问题变成一个优化问题：寻找那组最能解释观测数据的参数。

高斯分布的 MLE 给出样本均值和方差，多分类任务的 MLE 则自然导出负对数似然和交叉熵。理解这一层之后，损失函数就不再是经验配方，而是一个明确概率假设下的结果：**训练模型，本质上是在寻找最不让已有数据感到意外的参数。**
