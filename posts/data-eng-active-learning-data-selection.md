## 核心结论

主动学习的数据选择，核心不是“挑难样本”这么简单，而是在固定标注预算下，优先选择最可能改变模型决策边界的样本。决策边界可以理解为“模型把不同类别分开的那条分界线”。如果一条样本对这条分界线几乎没有影响，标不标它对模型提升都很小；如果一条样本正好落在边界附近，或者代表了模型尚未覆盖的数据区域，它的标签就更值钱。

这件事通常通过一个采集函数完成。采集函数就是“给每个未标注样本打分的规则”。最常见的几类规则是：

| 策略 | 关注点 | 直观含义 | 典型风险 |
| --- | --- | --- | --- |
| 随机采样 | 全局均匀 | 不做判断，直接抽样 | 浪费预算在低价值样本上 |
| 不确定性采样 | 决策边界 | 挑模型最没把握的样本 | 容易扎堆边界、忽略分布 |
| 查询委员会 | 模型分歧 | 挑多个模型意见不一致的样本 | 训练和维护成本更高 |
| 核心集/代表性采样 | 数据覆盖 | 挑能代表整体分布的样本 | 可能不够贴近当前边界 |
| 代表性+多样性联合优化 | 覆盖+去重 | 既有信息量又不重复 | 设计与调参更复杂 |

玩具例子可以直接说明收益。假设有 400 个二维二分类样本，只允许人工标注 30 个。如果这 30 个随机抽取，模型可能只学到一个粗糙边界，验证准确率约 70%；如果这 30 个样本来自不确定性采样，也就是优先标注“模型最犹豫”的点，验证准确率可能接近 90%。这里提升的本质不是模型结构变强了，而是同样的 30 次人工标注被用在了更关键的位置。

所以，主动学习真正解决的是“标签预算稀缺时，怎样把每一次人工标注都花在边界改进最大的位置上”。评估它时也不能只看最终准确率，还要看学习曲线 AUC、每增加一点标注所带来的边际收益，以及选样后训练集是否偏离真实线上分布。

## 问题定义与边界

主动学习通常讨论的是 pool-based active learning，也就是“池式主动学习”。池式的意思是：你手里已经有一个很大的未标注样本池 $U$，同时有一个很小的已标注集合 $L$。系统每一轮都执行同样的循环：

1. 用 $L$ 训练一个当前模型。
2. 对 $U$ 中每个样本计算价值分数。
3. 选出 top-k 个样本交给人工标注。
4. 把新标签加入 $L$，再训练下一轮。
5. 直到预算耗尽或指标收敛。

可以写成一个最小流程：

$$
L_{t+1} = L_t \cup \text{Label}(\text{TopK}(u(x), U_t))
$$

其中 $u(x)$ 是采集函数，也就是样本价值评分。

一个新手版流程可以这样理解：先随机标 10 条数据，训练初始模型；然后在剩余未标注池里计算每个样本的置信度、熵或者分歧度，选出最有价值的 5 条送给人工；再把这 5 条加回训练集继续训练。这个循环本质上是在“逐轮纠正模型当前最薄弱的位置”。

但主动学习不是任何场景都适用，它有明确边界：

| 条件 | 适合主动学习 | 不适合主动学习 |
| --- | --- | --- |
| 标签成本 | 很高，例如医生、法务、质检专家标注 | 很低，例如规则可自动生成标签 |
| 未标注池规模 | 很大，候选样本足够多 | 很小，样本本来就不多 |
| 模型更新成本 | 可接受，允许多轮训练 | 很高，单轮训练都很慢 |
| 概率输出质量 | 可校准，分数有一定可信度 | 置信度严重失真 |
| 目标 | 在小预算下快速提升 | 已经接近全量标注或预算宽松 |

真实工程里最常见的误解，是把主动学习当成“更高级的抽样”。实际上它是一个闭环系统：采样策略、标注系统、训练频率、评估方式和停止条件必须一起设计。只改采样函数而不改评估和监控，往往拿不到稳定收益。

## 核心机制与推导

主动学习的目标可以概括为：在预算 $B$ 固定时，选择一批样本，使模型性能提升最大。这个目标通常不能直接优化，因为“某个样本标完后到底能带来多少泛化收益”事先不可见，所以工程里会用代理指标。代理指标就是“用一个便宜、可计算的分数代替真实收益”。

### 1. 不确定性采样

不确定性采样最常见，因为实现简单。它依赖模型输出的类别概率 $p(y \mid x)$。如果模型对某个样本的预测非常犹豫，说明这条样本可能就在边界附近。

常见分数有三种。

第一种是最小置信度。就是看最大类别概率是否足够大：

$$
U_{\text{lc}}(x) = 1 - \max_k p_k
$$

第二种是边距。边距就是“第一名和第二名概率差了多少”。差越小，模型越分不清：

$$
\text{Margin}(x) = p_{\hat y_1} - p_{\hat y_2}
$$

主动学习里通常选择 margin 最小的样本。

第三种是熵。熵可以理解为“预测分布有多乱”。越乱，越不确定：

$$
H(y \mid x) = -\sum_k p_k \log p_k
$$

如果一个三分类样本概率是 $(0.98, 0.01, 0.01)$，熵很低；如果是 $(0.34, 0.33, 0.33)$，熵就很高，因为模型几乎不知道该判哪一类。

### 2. 查询委员会

查询委员会，英文常写 QBC，意思是“让多个模型一起投票”。委员会可以理解为“多个有差异的裁判”。如果这些模型对某个样本意见一致，说明样本信息量通常不大；如果意见分裂，说明该样本对边界仍有重要价值。

投票熵是常见做法。设有 $C$ 个模型，某类别获得票数 $V(y_i)$，则：

$$
VE(x) = -\sum_i \frac{V(y_i)}{C}\log \frac{V(y_i)}{C}
$$

投票越分散，投票熵越高，表示越值得标。

QBC 的价值在于它放大了“模型结构不确定性”。单模型的不确定性，有时只是概率没校准好；多模型分歧更接近“这个样本真的会影响边界”。

### 3. 代表性与多样性

只靠不确定性会出一个典型问题：模型会一直在局部边界周围打转，选出很多彼此相似的样本。这会导致预算浪费，也会让训练集分布偏离总体分布。

所以工程里经常加入代表性和多样性。

代表性可以理解为“这个样本能不能代表一个更大的区域”。多样性可以理解为“这一批选中的样本彼此不要太像”。

因此采集函数常写成联合形式：

$$
f(x)=\alpha U(x) + (1-\alpha)D(x)
$$

其中 $U(x)$ 表示不确定性，$D(x)$ 表示代表性或密度分数，$\alpha \in [0,1]$ 是权重。冷启动阶段，通常让代表性权重大一些，先覆盖全局分布；进入中后期，再提高不确定性权重，重点修边界。

如果做 batch 选择，也就是一轮选多个样本，还要考虑去重。形式上可写成：

$$
S^\* = \arg\max_{S \subset U,\ |S|=k}\sum_{x \in S} f(x) - \lambda \sum_{x_i,x_j \in S}\text{sim}(x_i,x_j)
$$

后半项就是惩罚相似样本重复入选。

### 4. 主动学习为什么能省标签

它本质上是在优化“单位标签带来的边际收益”。如果把性能记为 $M(n)$，其中 $n$ 是已标注样本数，那么主动学习希望在相同预算下让：

$$
\Delta M = M(n+\Delta n)-M(n)
$$

尽可能大。随机采样把预算平均花出去；主动学习则试图把预算集中花在“梯度最大”的位置。这里的“梯度最大”不是严格数学梯度，而是“对模型认知更新最明显的地方”。

## 代码实现

下面用一个最小可运行示例，演示如何基于预测概率计算熵分数，并选出最值得标注的样本。这个示例不依赖外部库，适合直接嵌入采样脚本。

```python
import math

def entropy(prob_row):
    # 熵越大，说明模型越不确定
    clipped = [max(min(p, 1.0), 1e-12) for p in prob_row]
    return -sum(p * math.log(p) for p in clipped)

def margin(prob_row):
    # 边距越小，说明第一名和第二名越难分
    sorted_probs = sorted(prob_row, reverse=True)
    return sorted_probs[0] - sorted_probs[1]

def select_top_k_by_entropy(prob_matrix, k):
    scores = [(i, entropy(row)) for i, row in enumerate(prob_matrix)]
    scores.sort(key=lambda x: x[1], reverse=True)
    return scores[:k]

samples = [
    [0.90, 0.07, 0.03],  # 很确定
    [0.50, 0.30, 0.20],  # 中等不确定
    [0.34, 0.33, 0.33],  # 非常不确定
    [0.80, 0.10, 0.10],  # 比较确定
]

selected = select_top_k_by_entropy(samples, 2)

# 第 3 条最不确定，其次通常是第 2 条
assert selected[0][0] == 2
assert selected[1][0] == 1

# 检查熵大小规律
assert entropy(samples[2]) > entropy(samples[1]) > entropy(samples[0])

# 检查边距大小规律：越小越不确定
assert margin(samples[2]) < margin(samples[1]) < margin(samples[0])

print(selected)
```

这段代码做了三件事：

1. 把模型 softmax 概率转成熵分数。
2. 按熵从高到低排序。
3. 取 top-k 作为本轮标注候选。

如果要做真实工程，通常不会只保留一个分数，而是会把不确定性、代表性和去重一起算。下面给出一个更接近生产环境的伪代码：

```python
def acquisition_score(uncertainty, density, duplicate_penalty, alpha=0.7, beta=0.2):
    # alpha 控制不确定性权重
    # beta 控制重复惩罚力度
    return alpha * uncertainty + (1 - alpha) * density - beta * duplicate_penalty
```

玩具例子可以这样理解。假设二维平面上，红蓝两类点被一条弯曲边界分开。随机抽 30 个点时，很多点落在类内深处，标签虽然正确，但对边界位置帮助不大。主动学习则会优先抽到边界附近那些“红也像、蓝也像”的点，这些点一旦被标注，模型边界会明显收缩到正确位置。

真实工程例子更典型的是医疗影像质检。比如一个肺结节分类系统，未标注 CT 切片可能有几十万张，但放射科医生的标注时间极贵。这时可以先用初始模型跑全量切片，按熵、委员会分歧、密度权重选出一小批最有价值图像，让医生只标这一批。这样做的目标不是“少标所有图”，而是“优先标那些能最快修正误判边界的图”。如果系统发现高熵样本几乎都来自某类扫描设备，还可以进一步加入设备分层约束，避免采样结果被单一来源垄断。

## 工程权衡与常见坑

第一类坑是只看最终准确率。主动学习的重点不是“最后谁高 0.3 个点”，而是“在相同标签预算下谁起得更快”。因此必须画学习曲线，也就是性能随标签数增长的曲线，并比较曲线下面积。

常用指标之一是 ALC，Active Learning Curve 的面积。可以把它理解为“整个标注过程中，模型平均表现如何”。如果两个方法最终都到 90%，但一个在前 20% 标签就到 85%，另一个要到 80% 标签才到 85%，前者才是真正省钱。

归一化后常写成：

$$
\text{ALC score}=\frac{\text{ALC}-A_{\text{rand}}}{A_{\max}-A_{\text{rand}}}
$$

其中 $A_{\text{rand}}$ 是随机采样的曲线面积，$A_{\max}$ 是理想上界或参考最优面积。这个指标的意义是：主动学习到底比随机强多少，而不是单独看绝对值。

第二类坑是样本分布偏移。不确定性采样天然偏向边界点，而线上真实流量未必集中在边界附近。久而久之，训练集分布 $p_{\text{train}}(x)$ 会偏离真实测试分布 $p_{\text{test}}(x)$，导致离线效果上涨、线上效果却不稳。这个问题在风控、审核、推荐等场景尤其常见，因为模型总在抽“最难判”的角落样本。

解决思路通常有三种：

| 问题 | 症状 | 常见缓解方式 |
| --- | --- | --- |
| 边界扎堆 | 被选样本非常相似 | 加入聚类或去重约束 |
| 分布偏移 | 训练集越来越不像真实流量 | 加入密度权重、分层采样 |
| 冷启动不稳 | 初始模型分数乱跳 | 先随机预热，再切主动采样 |
| 噪声标注 | 专家之间也不一致 | 多人复标、置信度回传 |

第三类坑是冷启动。模型一开始太弱时，它的不确定性分数本身就不可靠。一个常见做法是先随机标一小批样本建立基础能力，再切入主动学习；或者让多个轻量模型共同打分，减少单模型偶然误差。

第四类坑是只优化“好看”的指标，不优化业务代价。比如图像审核场景里，漏判违规内容的代价远高于错杀正常内容。此时主动学习应优先关注召回边界附近的样本，而不是只用通用熵。也就是说，采集函数要与业务损失对齐，而不是只与分类精度对齐。

第五类坑是忽略新增 1% 标签的边际收益。如果在第 8 轮之后，每新增 1% 标签只提升 0.05% 准确率，就说明主动学习进入收益衰减区间，应该考虑停止或切换策略。工程上，这个指标常比“最终再涨几个点”更能指导预算决策。

## 替代方案与适用边界

主动学习不是唯一答案。是否采用，取决于标签成本、训练成本和业务闭环能力。

第一类替代方案是随机采样。它最朴素，但也最稳。若标签成本低、数据分布稳定、预算充足，随机采样常常已经够用。它的优势不是效果最好，而是没有复杂调度和偏移风险。

第二类替代方案是半监督学习。半监督学习可以理解为“用少量真标签带动大量伪标签”。如果模型已经具备不错初始能力，未标注数据又很多，那么伪标签、自训练、一致性训练有时比主动学习更省工程成本。因为主动学习需要人工标注环路，而半监督更多依赖机器自动扩充标签。

第三类替代方案是核心集选择。核心集可以理解为“挑出最能代表整体的一小组点”。它更强调覆盖，而不是当前模型的不确定性。适合冷启动、长尾分布明显、或者 batch 很大的场景。

第四类替代方案是混合策略。真实系统里，最稳的通常不是单一熵采样，而是先混合出候选池，再做聚类去重。例如先算：

$$
f(x)=\alpha U(x)+(1-\alpha)D(x)
$$

选出高分候选集，再在候选集上做 clustering，最后从每个簇里挑中心点。这样既保住信息量，又避免一次选出大量近邻样本。

可以做一个简表：

| 方法 | 采样焦点 | 复杂度 | 适用场景 |
| --- | --- | --- | --- |
| Random | 全局平均覆盖 | 低 | 预算不敏感、基线系统 |
| Uncertainty | 决策边界 | 低到中 | 模型有可靠概率输出 |
| Rep+Diversity | 覆盖 + 去重 | 中 | batch 选样、分布复杂 |
| QBC | 多模型分歧 | 中到高 | 高价值标签、允许多模型 |
| Semi-supervised | 自动扩标签 | 中到高 | 初始模型不差、未标数据多 |

什么时候不该上主动学习？有三个典型边界。

一是标签本身不贵，比如规则可以自动打标，或者众包单价很低。这时主动学习省下来的人工成本，可能还不如系统复杂度带来的维护成本。

二是训练回路太慢。如果每轮重训要一天，而标注又需要实时闭环，主动学习的迭代价值会被吞掉。

三是模型输出不可校准。如果概率分数没有可信度，不确定性采样就会建立在错误排序上，结果反而不如随机。

所以，主动学习更适合作为“预算敏感型标注系统”的中层能力，而不是默认的数据处理标准件。只有当每一次标注都贵、每一轮反馈都能回到模型、并且你确实关心单位标签收益时，它才真正值得上。
