## 核心结论

权重初始化，就是在训练开始前给每一层参数设定一个分布。白话说，它决定网络第一步看到数据时，信号是“能正常传下去”，还是在中途就变得几乎全是 0，或者直接炸掉。

对深度网络来说，初始化的核心目标不是“随机就行”，而是让前向激活和反向梯度的方差落在一个可训练的稳定区间。经验上，这个区间常被观察为 $10^{-2}\sim10^{-1}$：太小，更新几乎停滞；太大，优化器很快失控。

对 ReLU 网络，最常用也最稳妥的做法是 He fan-in 初始化：

$$
\mathrm{Var}(w)=\frac{2}{\text{fan\_in}},\qquad
\sigma=\sqrt{\frac{2}{\text{fan\_in}}}
$$

这里的 fan-in 指“该层每个神经元接收多少输入”，白话说就是一层里每个输出单元会累加多少个前一层的值。这个公式的意义是：输入越多，单个权重就该越小，否则累加后的方差会越来越大。

一个最小数值例子是 fan_in=256，此时：

$$
\sigma=\sqrt{\frac{2}{256}}\approx0.0884
$$

这个标准差正好落在 $0.01\sim0.1$ 的稳定带里。它不是巧合，而是“按输入规模自动缩放”的结果。即使堆到 12 层，信号也不容易在前几层就全部衰减或爆炸。

---

## 问题定义与边界

问题的本质是：深度网络里的信号会一层一层传播，如果每层都把信号放大一点或缩小一点，层数一深，总效果就会指数级偏离。

设某层输入是 $x$，权重是 $w$，线性输出是 $z=Wx$。如果权重初始化标准差 $\sigma$ 太小，那么每层的输出都会更接近 0。白话说，网络虽然“有很多层”，但每层几乎都没在传有效信息，最后模型像没学会看数据一样。反过来，如果 $\sigma$ 太大，每层都把信号再放大一点，几层之后数值就会暴涨，梯度 norm 会迅速变大，最后出现损失震荡、参数发散甚至 NaN。

对于 ReLU 网络，常见经验带可以概括为下表：

| 初始 $\sigma$ 范围 | 行为 | 说明 |
| --- | --- | --- |
| $<10^{-2}$ | 梯度衰减 | 前向输出靠近 0，训练前面层更新几乎停滞 |
| $10^{-2}\sim10^{-1}$ | 稳定 | 激活/梯度方差大致守恒，训练收敛顺畅 |
| $>10^{-1}$ | 梯度爆炸 | 梯度 norm 爆增，易导致优化器失控 |

这里要强调边界：这个稳定带不是自然定律，而是围绕“常见深度网络、常见输入归一化、常见优化器”总结出的工程经验。它对 MLP、CNN、Transformer 里的线性层都很有参考价值，但不能机械套用到所有结构上。例如强残差、LayerNorm、门控单元、特殊参数化方式，都会改变初始化对方差传播的影响。

玩具例子最容易说明这个问题。假设一个 12 层 ReLU MLP，每层宽度相同。如果把所有层都初始化成 $\sigma=0.001$，第一层还能有一点输出，但传到第 5、6 层以后，大部分激活已经接近 0，反向梯度几乎看不见。如果把 $\sigma=0.2$，前几层看起来还正常，但很快中后层的激活和梯度会急剧变大，训练几步后就可能 NaN。只有把 $\sigma$ 放在 $0.02\sim0.08$ 这样的区间，前几步的梯度分布才会更均匀，网络能从第一步开始真正学习。

## 核心机制与推导

先看前向传播。假设某一层输入 $x_{l-1}$ 的均值接近 0，方差为 $\mathrm{Var}(x_{l-1})$，权重独立同分布，均值为 0，方差为 $\sigma^2$。线性变换后：

$$
z_l = W_l x_{l-1}
$$

在独立性近似下，可得到：

$$
\mathrm{Var}(z_l)\approx \text{fan\_in}\cdot \sigma^2 \cdot \mathrm{Var}(x_{l-1})
$$

这条式子的意思很直接：一个输出单元会把 fan-in 个输入加起来，所以如果每个权重的方差不按 fan-in 缩小，总方差就会随着输入数增长。为了让不同宽度的层输出规模相近，就需要：

$$
\sigma^2 \propto \frac{1}{\text{fan\_in}}
$$

这解释了为什么初始化不能写死成 $\mathcal N(0,0.01)$。0.01 对 16 维输入和 4096 维输入根本不是同一种量级。

再看 ReLU。ReLU 是“负数变 0、正数保留”的激活函数。白话说，它只让一半左右的信号通过，因此输出方差会比线性层少一截。若输入近似对称分布，ReLU 后有效激活大约保留一半，所以常见近似是：

$$
\mathrm{Var}(x_l)\approx \frac{1}{2}\mathrm{Var}(z_l)
$$

把它带回前向式子，为了让层与层之间方差大致守恒，需要：

$$
\frac{1}{2}\cdot \text{fan\_in}\cdot \sigma^2 \approx 1
\Rightarrow
\sigma^2 \approx \frac{2}{\text{fan\_in}}
$$

这就是 He 初始化里的 2 的来源。它不是拍脑袋定的常数，而是补偿 ReLU 把一半信号截断掉的统计效果。

反向传播同样有类似结论。若上一层梯度记为 $\delta_{l+1}$，则当前层梯度方差近似满足：

$$
\mathrm{Var}(\delta_l)\approx \frac{1}{2}\text{fan\_out}\cdot \sigma^2 \cdot \mathrm{Var}(\delta_{l+1})
$$

这里的 $\frac{1}{2}$ 同样来自 ReLU 导数在一半区域为 0。它说明初始化不仅影响前向激活，也影响反向梯度。如果只管前向、不管反向，训练仍然会失败。

看一个三层 MLP 的推导型例子。假设三层 fan-in 分别是 256、512、1024，那么 He 初始化的标准差分别是：

- $\sqrt{2/256}\approx 0.0884$
- $\sqrt{2/512}\approx 0.0625$
- $\sqrt{2/1024}\approx 0.0442$

可以看到，层越宽，单个权重越小。这样做的结果是：虽然每层累加项变多了，但每个项贡献更小，整体输出方差仍保持在相近尺度。工程上常看到前向激活方差在 $0.02\sim0.05$ 内波动，反向梯度在 $10^{-2}\sim10^{-1}$ 内变化，这就是“信号没有在深度中失真过快”。

真实工程例子可以看 GPT-2 或 ResNet 这类深层网络。它们不是简单 MLP，但底层线性层或卷积层仍然面临同样问题：如果初始化偏小，前几层几乎不更新，模型表现为“loss 降得慢、训练很钝”；如果偏大，梯度峰值会让优化器在训练一开始就进入不稳定区。现代大模型之所以能稳定训练，不只是因为优化器更强，也因为初始化、归一化、残差缩放共同控制了这条方差传播链。

## 代码实现

对初学者，最实用的做法是把初始化写成一个明确函数：输入 shape 和 fan-in，输出权重，并顺手检查实际标准差是否接近理论值。

```python
import numpy as np

def he_init(shape, fan_in, seed=0):
    rng = np.random.default_rng(seed)
    sigma = np.sqrt(2.0 / fan_in)
    weights = rng.normal(0.0, sigma, size=shape)
    return weights

def relu(x):
    return np.maximum(x, 0.0)

# 全连接层 256 -> 128
w = he_init((256, 128), fan_in=256, seed=42)

theoretical_sigma = np.sqrt(2.0 / 256)
sample_sigma = w.std()

print("theoretical sigma =", theoretical_sigma)
print("sample sigma =", sample_sigma)

# 样本标准差会有随机波动，但应接近理论值
assert abs(sample_sigma - theoretical_sigma) < 0.01

# 玩具前向：检查经过 ReLU 后方差是否仍在合理范围
x = np.random.default_rng(1).normal(0.0, 1.0, size=(64, 256))
z = x @ w
a = relu(z)

print("input var =", x.var())
print("linear var =", z.var())
print("relu var =", a.var())

assert 1e-2 < sample_sigma < 1e-1
assert z.var() > 0
assert a.var() > 0
```

这个例子有两个作用。

第一，它把公式真正落到了代码里：`sigma = sqrt(2 / fan_in)`。只要层宽变了，初始化强度会自动调整。

第二，它让你看到“理论值”和“采样值”的关系。理论标准差是总体分布参数，实际生成的一组权重会有随机波动，所以检查时不要要求完全相等，而是看是否接近。

如果要再进一步，可以在训练前几批数据上做前向和反向统计。白话说，不要等 loss 爆了才猜问题，而是直接看每层激活方差和梯度方差是不是在稳定带里。一个常见做法是：

1. 模型初始化后，跑 1 到 5 个 batch。
2. 记录每层输出的均值、方差、最大值。
3. 反向后记录每层梯度 norm 和梯度方差。
4. 如果前层梯度长期低于 $10^{-6}$，通常说明初始化偏弱、学习率过小、或归一化有问题。
5. 如果梯度峰值频繁极大，优先检查初始化是否偏大，再看学习率、warmup、gradient clipping。

真实工程例子是训练一个标准 ResNet 分类模型。很多团队遇到过这样的现象：代码逻辑都对，loss 也不是完全不动，但前面几层梯度 norm 很快掉到 $10^{-6}$ 甚至更低，模型像“能跑但学不动”。这时如果排查发现卷积层用了固定 $\sigma=0.01$，换成 He 初始化，再配合几百步 warmup，前层梯度常能恢复到 $10^{-2}\sim10^{-1}$，训练曲线立刻平稳很多。

## 工程权衡与常见坑

初始化不是独立变量，它会和激活函数、归一化层、残差结构、学习率一起作用。工程上最常见的问题，不是“不知道公式”，而是“公式用错地方”。

| 坑 | 现象 | 规避 |
| --- | --- | --- |
| 固定 $\sigma=0.01$ | 梯度 norm 接近 0、前层不更新 | 根据 fan-in 重新设 $\sigma$，结合 warmup |
| Xavier + ReLU | 约一半神经输出 0，训练噪声变大 | 换 He 或按激活函数重算缩放 |
| 未监测方差 | 训练早期就爆炸或死锁 | 在前几批数据上记录激活/梯度方差 |
| 只调学习率，不看初始化 | loss 震荡但定位不准 | 先确认初始化是否把信号送入稳定带 |

先说固定 $\mathcal N}(0, 0.01)$。这在浅层网络、小特征维度下有时还能工作，但对深层 ReLU 网络通常偏小。原因不是 0.01 本身“错误”，而是它没有考虑 fan-in。输入维度一大，输出方差会被压得过低，前几层更新尤其慢。

再说 Xavier。Xavier 初始化的直觉是同时平衡 fan-in 和 fan-out，常用于 tanh 或 sigmoid。问题在于 ReLU 会截断一半信号，所以 Xavier 对 ReLU 往往偏小。白话说，它低估了 ReLU 带来的有效方差损失。结果是激活偏弱、死神经元增多、训练波动更大。这里“死神经元”指某些 ReLU 单元长期输出 0，几乎不再参与学习。

第三个常见坑是没有监控。很多训练失败看起来像学习率问题，实际上根因是初始化导致的方差传递失衡。尤其在深层 CNN、Transformer、蒸馏学生模型里，如果只盯总 loss，而不看层级统计，往往会把排查时间浪费在优化器超参上。

还有一个现实权衡：初始化足够好，并不代表可以忽略 warmup。初始化解决的是“第 0 步到第 1 步信号是否可传播”，warmup 解决的是“优化器前几百步是否过猛”。两者经常需要一起使用，而不是二选一。

## 替代方案与适用边界

He 不是唯一方案，它只是 ReLU 系列网络里的主流默认。是否要替换，取决于激活函数、网络结构和训练目标。

| 方法 | 适用激活/结构 | 备注 |
| --- | --- | --- |
| Xavier | tanh/sigmoid | 均衡 fan-in/fan-out，ReLU 往往偏小 |
| He (fan-in) | ReLU/Leaky ReLU | 对 ReLU 的 0.5 因子做补偿，最常用 |
| 熵增益初始化 | 复杂分类任务、可调学习率 | 强调早期信息量保持，常与动态 lr 配合 |
| 蒸馏方差约束 | 需和教师统计一致 | 让学生 feature map 方差接近教师 |

Xavier 适合输出在正负两侧都连续变化的激活，比如 tanh。因为这些激活不会像 ReLU 那样直接砍掉半边分布，所以不需要额外的 2 倍补偿。把 Xavier 生硬套到 ReLU 上，常常会让信号偏弱。

一些新工作提出“熵增益”导向的初始化。这里的熵可以理解为“信号里有多少可区分的信息量”。白话说，这类方法不只关心数值别炸，还关心早期表示是否足够丰富。它们通常会和动态学习率一起设计，适合复杂识别任务，但实现和调参成本比标准 He 更高，不适合当默认起点。

在知识蒸馏里，还会出现另一类边界：学生模型不只要自己稳定训练，还要尽量匹配教师模型的中间特征统计。如果教师卷积层的权重和 feature map 呈现某种方差结构，学生若从一个完全不匹配的初始化出发，蒸馏早期就可能出现信息流断裂。更稳妥的做法是让学生首层或若干关键层的初始方差接近教师统计，再开始蒸馏。

举个真实工程例子：用 ReLU 教师训练一个更小的学生 CNN。如果学生从 He 初始化开始，并确保第一层卷积输出方差与教师首层在同一量级，那么 feature map 更容易对齐，蒸馏损失不会一开始就剧烈震荡。反之，如果学生初始化过弱，蒸馏目标虽然在算，但学生的中间表示几乎没有有效信息，蒸馏就会变成“拿弱信号去拟合强信号”，效果很差。

因此，适用边界可以总结为一句话：He 是 ReLU 网络的默认起点，不是所有网络的终点。只要激活函数、归一化方式、残差路径或训练目标改变，就应重新检查“方差是否还能稳定传播”。
