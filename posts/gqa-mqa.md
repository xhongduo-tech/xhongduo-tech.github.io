## GQA 和 MQA 的本质，是用更少 KV 头换更低缓存成本

标准多头注意力里，每个查询头通常对应自己的 Key/Value 头。这种设计表达力强，但推理时 KV Cache 成本也随头数线性增长。MQA 与 GQA 的核心思路是：**让多个 Query 头共享更少的 KV 头。**

---

## MQA：所有 Query 头共享一组 K/V

在 Multi-Query Attention 中：

- Query 仍然有很多头
- 但 Key 和 Value 只有一组共享表示

这样 KV Cache 立刻大幅缩小，推理内存与带宽压力明显下降。  
代价是不同头之间在读取信息时共享同一组 K/V，表达多样性会有所损失。

---

## GQA：在共享与表达力之间折中

Grouped-Query Attention 并不把所有 Query 头都压到一组 K/V，而是分成若干组。每组 Query 头共享一组 K/V，因此：

- 比标准 MHA 少很多 KV 头
- 比 MQA 保留更多头间差异

这使 GQA 成为现代大模型常见的务实选择。它不追求最激进压缩，而追求“几乎不伤精度的显著降本”。

---

## 为什么它对推理尤其重要

GQA / MQA 主要收益不在训练，而在推理：

- KV Cache 更小
- 每步读取带宽更低
- 更容易支持长上下文和更高并发

随着模型参数与上下文窗口持续增长，KV 相关成本越来越主导，因而减少 KV 头数成为极具性价比的结构改动。

---

## 小结

MQA 和 GQA 的核心思想都很简单：保持 Query 的多头表达能力，同时用更少的 Key/Value 头降低缓存与带宽开销。MQA 最省，但表达压缩更强；GQA 在两者之间做折中，因此成为许多现代 LLM 的主流配置。

这说明结构设计不只是为了训练精度，也是在为推理系统的长期成本做准备。
