## ReAct 的价值，是把“思考”和“调用工具”编织成同一条轨迹

纯 Chain-of-Thought 让模型展开推理，但不接触外部世界；纯工具调用则可能缺少决策理由。ReAct（Reason + Act）提出的核心范式是：**让模型在推理与行动之间交替前进。**

---

## 基本循环

典型轨迹形如：

```text
Thought -> Action -> Observation -> Thought -> ...
```

其中：

- `Thought` 负责解释下一步为什么这样做
- `Action` 调用搜索、数据库或其他工具
- `Observation` 把工具返回结果写回上下文

这样，模型不是在真空里推理，而是在推理过程中不断引入新证据。

---

## 为什么它能降低幻觉

因为模型遇到不确定信息时，不再只能依赖参数记忆，而可以选择先查再答。  
这使回答建立在外部观测基础上，而不是纯语言续写惯性上。

同时，行动前后的 Thought 也让整个过程更可解释，方便调试代理行为。

---

## 它的边界

ReAct 的质量高度依赖：

- 工具是否可靠
- 提示是否让模型学会在合适时机调用工具
- Observation 写回格式是否清晰

若模型工具选择能力差，ReAct 也可能陷入无效调用或过度循环。

---

## 小结

ReAct 的重要性在于，它把推理和工具交互统一为一条可追踪的执行轨迹。相比纯 CoT，它让模型能借助外部信息修正内部假设；相比纯函数调用，它又保留了行为背后的推理理由。

因此，ReAct 是现代 Agent 系统最有代表性的工作流原型之一。
