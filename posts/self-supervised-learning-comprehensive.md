## 核心结论

自监督学习的核心不是“没有监督”，而是“监督信号不再由人工标注提供，而是由数据本身构造出来”。白话说，就是模型先从原始数据里自己出题、自己做题，再把学到的表示迁移到分类、检索、检测、问答等任务。

当前主流的三条路线分别是对比学习、掩码建模和生成式建模。

| 范式 | 训练目标 | 直觉解释 | 代表方法 | 更常见的应用 |
|---|---|---|---|---|
| 对比学习 | 拉近正样本，推远负样本 | 让“同一个东西的不同视图”更像 | SimCLR、MoCo | 视觉表征、多模态对齐 |
| 掩码建模 | 预测被遮住的内容 | 通过“填空”理解上下文结构 | BERT、MAE | NLP、Vision Transformer |
| 生成式建模 | 预测下一个 token 或 patch | 通过连续生成掌握分布 | GPT 类方法 | 文本生成、统一建模 |

对初学者最重要的判断标准只有两个。

1. 如果你的目标是学一个可迁移的表示，尤其是图像检索、相似度匹配、跨模态对齐，对比学习通常更直接。
2. 如果你的模型结构是 Transformer，且任务依赖上下文补全能力，掩码建模通常更稳，BERT 和 MAE 就是典型例子。

一个新手友好的玩具例子是：同一张猫的图片经过随机裁剪和颜色扰动后得到两张视图，模型被要求知道“这两张虽然像素不同，但本质上是同一只猫”。这就是对比学习最核心的监督信号。

## 问题定义与边界

自监督学习要解决的问题是：当人工标签昂贵、稀缺或根本不存在时，如何在海量无标签数据上学到有用表示。这里的“表示”可以理解成压缩后的特征向量，白话说，就是模型内部对输入内容的抽象描述。

它与无监督学习相关，但不完全相同。无监督学习强调“没有标签”，自监督学习强调“自己构造伪标签或训练目标”。例如：

- 在文本里把一句话中的某些词遮住，再预测这些词。
- 在图像里裁出两个增强版本，要求模型认出它们来自同一张原图。
- 在序列里只给前半段，让模型预测后半段。

本文边界主要覆盖三类对象：

| 领域 | 输入形式 | 常见自监督任务 | 下游任务 |
|---|---|---|---|
| 计算机视觉 | 图像、视频帧、patch | 对比学习、掩码重建 | 分类、检测、分割、检索 |
| 自然语言处理 | token 序列 | 掩码语言建模、下一词预测 | 分类、问答、生成 |
| 多模态 | 图像+文本、视频+文本 | 对齐学习、联合掩码、生成 | 图文检索、视觉问答、对话 |

这里不展开纯数学上的可识别性证明，也不展开大规模分布式训练细节，而是聚焦三件事：

1. 损失函数为什么这样设计。
2. 数据增强和掩码率为什么会直接影响效果。
3. 自监督预训练后，为什么少量标签微调就能得到不错的下游性能。

一个直观例子是：你把一张街景图遮掉 75% 的 patch，再让模型重构缺失部分。模型如果想填对，就不能只记像素，它必须理解“道路旁边通常是什么”“天空和建筑边界在哪里”。这就是掩码建模的训练逻辑。

## 核心机制与推导

### 1. 对比学习：让同源样本靠近

对比学习的基本对象是正样本对与负样本。正样本对，白话说，就是“本来属于同一个内容的两个版本”；负样本则是“不属于同一个内容的其他样本”。

设编码器为 $f_\theta$，输入 $x$ 经过两次随机增强得到 $x_i, x_j$，再映射成表示 $z_i, z_j$。常用的 InfoNCE 损失为：

$$
\mathcal{L}_{i} = -\log \frac{\exp(\mathrm{sim}(z_i, z_j)/\tau)}{\sum_{k=1}^{N} \exp(\mathrm{sim}(z_i, z_k)/\tau)}
$$

其中：

- $\mathrm{sim}(\cdot,\cdot)$ 是相似度函数，常用余弦相似度。
- $\tau$ 是温度参数，白话说，就是控制“区分强弱”的旋钮。
- 分母里既包含正样本，也包含负样本。

为什么这个式子有效？因为它本质上在做一个分类问题：给定锚点 $z_i$，模型要从一堆候选表示中把真正的正样本 $z_j$ 选出来。正样本相似度越高，损失越小；负样本相似度越高，损失越大。

玩具例子如下。设 $\tau=0.1$，正样本相似度为 $0.9$，三个负样本相似度都为 $0.2$，则

$$
\mathcal{L}
=
-\log \frac{e^{0.9/0.1}}{e^{0.9/0.1} + 3e^{0.2/0.1}}
=
-\log \frac{e^9}{e^9 + 3e^2}
\approx 0.003
$$

这个值很小，说明模型几乎已经明确区分出正样本。这里可以看到温度 $\tau$ 的作用：当 $\tau$ 较小时，相似度差异会被放大，梯度更集中；但太小也可能导致训练不稳定。

SimCLR 的关键工程前提是“大 batch”。因为它把同一 batch 的其他样本都当负样本，batch 越大，负样本越多。MoCo 的思路则不同，它引入动量编码器和队列，白话说，就是用一个更新较慢的“老师编码器”持续生成负样本，从而不依赖超大 batch。

### 2. 掩码建模：通过填空理解结构

掩码建模的核心是随机隐藏输入的一部分，再让模型恢复被隐藏内容。最典型的文本方法是 BERT，最典型的视觉方法是 MAE。

设原始输入被切成若干 token 或 patch，记为 $x = [x_1, x_2, ..., x_n]$，随机选择掩码集合 $M$。模型只看到未掩码部分 $x_{\bar M}$，再预测被遮住的部分 $x_M$。损失可以写成：

$$
\mathcal{L}_{\text{mask}} = \sum_{i \in M} \ell(\hat{x}_i, x_i)
$$

其中 $\ell$ 可以是：

- 交叉熵：用于离散 token 预测。
- 均方误差 MSE：用于像素或连续表示重建。

MAE 的关键不是“遮一点”，而是“遮很多”。高掩码率会迫使编码器从少量可见 patch 中提取全局结构，而不是依赖局部冗余。经验上，图像 MAE 常用较高掩码率，例如 75% 左右，因为自然图像局部冗余很强，遮得不够多，任务会太简单。

这里有一个重要推导直觉。设总 patch 数是 $n$，掩码率为 $r$，可见 patch 数为 $(1-r)n$。当 $r$ 增大时：

- 编码器输入变少，计算量下降。
- 重建难度上升，表示学习压力增加。
- 若 $r$ 过高，监督信号会过稀，模型难以收敛。

因此掩码率不是越高越好，而是在“任务足够难”和“仍然可学”之间取平衡。

### 3. 生成式建模：把预测下一步当监督

生成式建模最常见的是自回归建模。自回归，白话说，就是“每次只根据前文预测下一个元素”。例如 GPT 学的是：

$$
p(x_1, x_2, ..., x_n) = \prod_{t=1}^{n} p(x_t \mid x_{<t})
$$

它看起来不像“表征学习”，但本质上也是自监督，因为监督信号来自原始序列本身。模型通过连续预测下一个 token 学到语义、语法和世界知识。

### 4. 自监督与有监督结合

实际工程里，自监督很少单独结束，而是走“预训练 + 线性探测/微调”流程。

- 线性探测：冻结编码器，只训练最后一层线性分类器。白话说，就是检查表示本身好不好。
- 微调：解冻部分或全部参数，让模型适配具体任务。白话说，就是在已有知识上做定向修正。

这也是为什么自监督通常能节省标注成本：模型已经在无标签数据中学到通用结构，下游任务只需较少标签去“校准方向”。

## 代码实现

下面先给一个最小可运行的 Python 版本，用 NumPy 实现简化版 InfoNCE。它不依赖深度学习框架，但足以说明对比学习的损失是如何计算的。

```python
import numpy as np

def l2_normalize(x, eps=1e-12):
    norm = np.linalg.norm(x, axis=-1, keepdims=True)
    return x / np.maximum(norm, eps)

def cosine_sim(a, b):
    a = l2_normalize(a)
    b = l2_normalize(b)
    return a @ b.T

def info_nce_loss(z1, z2, tau=0.1):
    """
    z1, z2: shape [batch, dim]
    假设 z1[i] 和 z2[i] 是正样本对
    """
    z1 = l2_normalize(z1)
    z2 = l2_normalize(z2)

    logits = (z1 @ z2.T) / tau
    exp_logits = np.exp(logits - logits.max(axis=1, keepdims=True))
    probs = exp_logits / exp_logits.sum(axis=1, keepdims=True)

    batch = z1.shape[0]
    positive_probs = probs[np.arange(batch), np.arange(batch)]
    loss = -np.log(positive_probs).mean()
    return loss, probs

# 玩具例子：第 0 对几乎完全一致，其它对更不相似
z1 = np.array([
    [1.0, 0.0],
    [0.0, 1.0],
], dtype=np.float64)

z2 = np.array([
    [0.99, 0.01],
    [0.2, 0.8],
], dtype=np.float64)

loss, probs = info_nce_loss(z1, z2, tau=0.1)

assert loss >= 0
assert probs.shape == (2, 2)
assert probs[0, 0] > probs[0, 1]   # 第 0 个样本更偏向自己的正样本
assert probs[1, 1] > probs[1, 0]   # 第 1 个样本也成立

print("loss =", round(float(loss), 6))
print("probs =\n", np.round(probs, 4))
```

如果换成 SimCLR 风格的训练流水线，核心模块通常有四个：

1. 数据增强：随机裁剪、颜色扰动、模糊、翻转。
2. 编码器：如 ResNet 或 ViT，输出主表示。
3. 投影头：把表示再映射到对比空间。
4. InfoNCE 损失：在 batch 内构造正负样本。

简化伪代码如下：

```python
# x -> aug1, aug2
v1 = augment(x)
v2 = augment(x)

h1 = encoder(v1)
h2 = encoder(v2)

z1 = projector(h1)
z2 = projector(h2)

loss = info_nce(z1, z2, tau=0.2)
loss.backward()
optimizer.step()
```

MAE 的流程则不同。它不是“增强出两张图去比较”，而是“遮掉大部分 patch 再重建”。

```python
# x -> patchify -> random mask
visible_tokens, masked_positions, target_pixels = mask_patches(x, mask_ratio=0.75)

latent = encoder(visible_tokens)
pred = decoder(latent, masked_positions)

loss = mse(pred, target_pixels)
loss.backward()
optimizer.step()
```

真实工程例子可以看视觉预训练。假设你要做工业质检，但只有少量带缺陷标签的图像。一个常见做法是：

- 先用几百万张无标签产线图片做 MAE 预训练，让模型学会零件结构。
- 再用少量标注数据微调分类头，判断“正常 / 划痕 / 缺口”。

这个流程的价值在于：预训练阶段并不需要人工标注，但模型已经学会了“哪些局部纹理和整体形状是合理的”。

常见超参数如下：

| 超参数 | 对比学习常见关注点 | 掩码建模常见关注点 |
|---|---|---|
| batch size | 越大负样本越多，SimCLR 很敏感 | 通常不如对比学习敏感 |
| temperature $\tau$ | 直接影响相似度分布和梯度强度 | 一般不涉及 |
| mask ratio | 不涉及 | 决定任务难度与计算量 |
| learning rate | 与 batch、优化器强相关 | 与 encoder/decoder 深度相关 |
| augmentation strength | 极其关键 | 需谨慎，常弱于对比学习 |

## 工程权衡与常见坑

自监督学习在论文里看起来统一，到了工程里最大的问题不是“能不能跑”，而是“学到的到底是不是你想要的表示”。

下面是最常见的坑。

| 问题 | 现象 | 常见原因 | 处理方式 |
|---|---|---|---|
| 表示坍塌 | 不同输入输出几乎一样 | 正负样本设计不足，约束不够 | 增加负样本、引入队列、使用 stop-gradient 变体 |
| 对比学习效果差 | 训练 loss 下降但迁移差 | 增强策略不合适 | 重新设计 crop、颜色扰动、模糊强度 |
| 掩码建模重建很好但下游一般 | 模型学会像素补全，不一定学到语义 | decoder 太强，encoder 压力不足 | 降低 decoder 能力，强调 encoder 学习 |
| 小 batch 下 SimCLR 不稳定 | 表征区分度不足 | 负样本太少 | 改用 MoCo 或者梯度累积配合大 batch |
| 掩码率设置错误 | 过低任务太简单，过高难以收敛 | 没结合数据冗余度 | 从经验区间起步，逐步搜索 |
| 混合损失互相打架 | 一个指标升，另一个指标降 | 对比和重构目标冲突 | 调损失权重，或分阶段训练 |

对比学习最典型的坑是“增强误用”。增强，白话说，就是故意改动输入但不改变语义。如果增强改变了任务本身的关键语义，就会制造错误监督。比如医学影像里颜色映射或局部裁剪可能直接抹掉病灶信息，这时照搬 SimCLR 的增强组合往往会出问题。

掩码建模最典型的坑是“把重建指标当最终目标”。重建误差低，只能说明模型会补像素或 token，不等于它学到了最适合分类或检索的语义表示。MAE 常用轻量 decoder，就是为了不让 decoder 把活都干了，逼迫 encoder 承担主要建模工作。

真实工程里，多模态系统也会遇到类似问题。以图文对齐为例，如果只做对比学习，模型可能学会“粗粒度语义匹配”，但细节描述能力不足；如果只做生成或掩码重建，模型又可能记住局部共现模式，却不擅长全局对齐。因此很多系统会联合训练多个目标，但联合训练并不自动更好，权重设置不当会造成梯度冲突。

## 替代方案与适用边界

如果只看主线，自监督学习像是在三选一：对比、掩码、生成。但实际研究和工业系统越来越多地采用混合范式。

| 方案 | 优点 | 缺点 | 更适合的场景 |
|---|---|---|---|
| 纯对比学习 | 表征判别性强，适合检索和对齐 | 依赖增强和负样本 | 视觉检索、图文对齐 |
| 纯掩码建模 | 更符合 Transformer 结构，训练稳定 | 重建好不代表判别强 | NLP 预训练、ViT 预训练 |
| 纯生成式建模 | 统一建模能力强，可直接用于生成 | 训练成本高，推理常较慢 | 大语言模型、统一多模态模型 |
| 混合范式 | 可兼顾判别与重建 | 损失设计复杂，易冲突 | 多模态、复杂预训练体系 |

一个典型替代方向是 BYOL、SimSiam 这类“无显式负样本”的方法。它们通过动量分支、预测头和 stop-gradient 等技巧避免坍塌，减少了对超大 batch 或负样本队列的依赖。适合资源有限但仍想做视觉表征学习的场景。

另一个方向是掩码与对比结合。例如一些混合模型会同时最小化重构损失和对齐损失：

$$
\mathcal{L} = \lambda_1 \mathcal{L}_{\text{contrast}} + \lambda_2 \mathcal{L}_{\text{mask}}
$$

这里的 $\lambda_1, \lambda_2$ 是损失权重，白话说，就是决定“更看重区分能力”还是“更看重补全能力”的比例系数。

真实工程例子可以看多模态模型。像 MiniGPT-4、LLaVA 这类系统，通常不是从零端到端训练一个统一大模型，而是先把视觉编码器、对齐模块和语言模型拼接起来，再做多阶段训练。一个常见路线是：

1. 先用图文对做表示对齐，让视觉特征和文本特征进入相近空间。
2. 再通过指令微调让模型学会对话格式和任务行为。
3. 在部分系统里，还会保留重建或生成式目标，以维持细粒度感知能力。

这说明适用边界很明确：

- 如果任务主要是“判断像不像、是否匹配”，对比学习往往优先。
- 如果任务主要是“理解上下文缺失部分”，掩码建模更自然。
- 如果任务最终就是“连续生成”，生成式建模更直接。
- 如果系统需要同时看图、读文、说话，混合范式通常更符合需求，但训练复杂度也最高。
