## 核心结论

动量方法的目标不是“让参数跑得更快”这么简单，而是**在不显著增加每步计算成本的前提下，减少来回震荡，沿着更有用的方向持续前进**。这里的“动量”可以理解成“把前几步的更新趋势累计起来”，避免每一步都只听当前梯度的瞬时意见。

Polyak 动量，也常叫 Heavy Ball，更新规则是：

$$
v_{t+1}=\beta v_t-\eta \nabla f(x_t), \qquad x_{t+1}=x_t+v_{t+1}
$$

其中，$\eta$ 是学习率，也就是每次走多大一步；$\beta$ 是动量系数，也就是保留多少历史速度，工程上常见取值是 $0.9$。

Nesterov 加速梯度法，简称 NAG，关键改动只有一处：**梯度不在当前位置算，而是在“按当前速度预测会到达的位置”算**：

$$
v_{t+1}=\beta v_t-\eta \nabla f(x_t+\beta v_t), \qquad x_{t+1}=x_t+v_{t+1}
$$

这一步的直观意义是：先看一眼“如果继续按惯性冲下去，会冲到哪里”，再根据那个位置的坡度决定怎么更新。所以 NAG 常被概括为“提前看路再踩油门”。

对光滑凸优化问题，标准梯度下降的函数值误差通常是 $O(1/t)$，而 NAG 可以达到最优的 $O(1/t^2)$。对强凸问题，收敛因子从标准梯度下降的 $1-1/\kappa$ 改进到接近 $1-1/\sqrt{\kappa}$。这里的 $\kappa$ 是条件数，白话讲就是“这个问题在不同方向上有多不均匀”；$\kappa$ 越大，说明谷底越狭长，普通梯度越容易在陡方向上来回弹。

玩具例子里，Polyak 动量像“先积攒惯性再冲”，NAG 像“先往前看半步，再决定怎么冲”。在狭长谷底里，后者更容易提前刹车，所以轨迹通常更平滑、过冲更少。

真实工程里，深度学习优化器里的 EMA，一阶指数滑动平均，本质上就是动量思想的现代实现。AdamW 里的一阶矩就是一种动量；而带 Nesterov 风格预测的变体，在 Transformer、ViT 这类对训练速度敏感的任务中，常能比普通动量更早达到同等损失水平。

## 问题定义与边界

本文讨论的“动量方法加速机制”，研究对象主要是两类问题：

| 目标函数类型 | 适用动量形式 | 常见风险 |
| --- | --- | --- |
| 光滑凸函数 | GD、Polyak、NAG 都适用 | 主要看收敛速度差异 |
| 光滑强凸函数 | Polyak、NAG 更有代表性 | 条件数大时普通 GD 很慢 |
| 非凸深度网络 | SGD+Momentum、AdamW、NAG 风格变体 | 噪声大，理论与实践不完全一致 |

这里“光滑”指梯度变化不至于无限剧烈，通常用 $L$-Lipschitz 梯度描述；白话讲，就是坡度不会在极小的位移内突然失控。  
“凸”指函数没有局部坑，往最低点走不会被次优盆地困住。  
“强凸”则比凸更强，表示函数底部至少有稳定曲率，不会是一条平底沟。

为什么需要动量？因为很多优化问题不是“圆形盆地”，而是“狭长谷地”。例如：

$$
f(x,y)=x^2+10y^2
$$

这个函数在 $y$ 方向更陡，在 $x$ 方向更缓。普通梯度下降如果学习率小，会沿着 $x$ 方向走得很慢；如果学习率大，又会在 $y$ 方向来回弹。动量方法的价值，就是**把多个历史更新叠加后，抑制无效的高频摆动，强化长期一致的前进方向**。

边界也要说清楚。

第一，NAG 的经典理论依赖可微、光滑、凸或强凸假设。离开这些条件后，它仍然常常好用，但“为什么好用”更多来自经验和局部分析，不再是那套严格的最优性结论。

第二，在小批量训练里，梯度本身带噪声。这里“噪声”指每次只看一个小样本子集，所以得到的是带随机误差的梯度估计。NAG 把梯度计算点移到预测位置后，前瞻性更强，但如果学习率和动量都偏大，预测点误差也会被放大，表现成更明显的振荡。

第三，深度学习里常说“某优化器比某优化器快 10% 或 15%”，通常指达到同等验证指标所需的步数、时间或 token 数更少，而不是每一步计算更便宜。NAG 的额外开销并不大，关键收益来自更高质量的更新方向。

## 核心机制与推导

先看 Polyak 动量。它的结构可以拆成两步：

1. 用历史速度和当前梯度合成新速度。
2. 用新速度更新位置。

如果把 $v_t$ 看成“速度”，那么 $\beta v_t$ 是保留惯性，$-\eta\nabla f(x_t)$ 是当前坡度带来的修正。所以 Polyak 的特点是：**纠偏依赖当前位置，但前进依赖历史趋势**。

NAG 只改一个位置：

- Polyak 在 $x_t$ 算梯度。
- NAG 在 $x_t+\beta v_t$ 算梯度。

也就是先构造预测点：

$$
\tilde{x}_t=x_t+\beta v_t
$$

再在这个点计算梯度：

$$
g_t=\nabla f(\tilde{x}_t)
$$

最后更新：

$$
v_{t+1}=\beta v_t-\eta g_t,\qquad x_{t+1}=x_t+v_{t+1}
$$

为什么这个小改动会带来加速？

可以用一个手把手的玩具例子说明。仍然看狭长二次谷。假设当前速度把你朝右下方推，而谷底实际开始向右平缓延伸。  
Polyak 的判断逻辑是：“我先看当前脚下坡度，再叠加惯性。”  
NAG 的判断逻辑是：“我先根据惯性预估一下下一刻会落在哪里，再看那个位置的坡度。”

如果预测点已经接近谷底另一侧，那么那里的梯度会提前告诉你：“别再猛冲这个方向了。”这就是所谓“提前刹车”。

把这个现象写成更抽象的文本推导框图：

1. 历史速度给出未来位置的一阶预测：$\tilde{x}_t=x_t+\beta v_t$
2. 在预测点评估局部几何：$\nabla f(\tilde{x}_t)$
3. 这个梯度比 $\nabla f(x_t)$ 更接近“真正更新后会面对的坡度”
4. 所以更新方向更前瞻，过冲更少
5. 在凸光滑结构下，这种前瞻性可以构造出更紧的势函数下降
6. 最终得到 $O(1/t^2)$ 的最优加速率

对强凸函数，标准梯度下降和加速方法的典型对比如下：

$$
f(x_k)-f^\star \le C\left(1-\frac{1}{\kappa}\right)^k
$$

而 NAG 形式的加速界可以写成：

$$
f(x_k)-f^\star \le C\left(1-\frac{1}{\sqrt{\kappa}}\right)^k
$$

其中 $f^\star$ 是最优值。两者看起来都叫“线性收敛”，但区别在于：当 $\kappa$ 很大时，$1-1/\sqrt{\kappa}$ 比 $1-1/\kappa$ 明显更小，意味着达到同样误差所需的迭代次数更少。

举个量级感受。若 $\kappa=10^4$，则：

- 标准 GD 的关键量级是 $1-1/10000=0.9999$
- NAG 的关键量级是 $1-1/100=0.99$

虽然都小于 1，但前者每步只缩一点点，后者每步缩得明显更多。这就是“条件数高时加速收益更大”的数学来源。

需要补一句：Polyak 动量在很多二次问题上也能很快，但它的优势更多来自经验调参和特定谱结构；而 NAG 在凸优化里有更完整的最优性理论，所以讨论“加速机制”时，NAG 通常是主角。

## 代码实现

下面先给出一个最小可运行的 Python 例子。它在二维二次函数上比较普通梯度下降、Polyak 动量和 NAG。代码里的 `assert` 用来保证函数值确实下降到较低水平。

```python
import math

def f(x):
    # 狭长二次谷：y 方向更陡
    return x[0] ** 2 + 10.0 * x[1] ** 2

def grad(x):
    return [2.0 * x[0], 20.0 * x[1]]

def gd(x0, lr=0.08, steps=60):
    x = x0[:]
    hist = [f(x)]
    for _ in range(steps):
        g = grad(x)
        x[0] -= lr * g[0]
        x[1] -= lr * g[1]
        hist.append(f(x))
    return x, hist

def heavy_ball(x0, lr=0.05, beta=0.9, steps=60):
    x = x0[:]
    v = [0.0, 0.0]
    hist = [f(x)]
    for _ in range(steps):
        g = grad(x)
        v[0] = beta * v[0] - lr * g[0]
        v[1] = beta * v[1] - lr * g[1]
        x[0] += v[0]
        x[1] += v[1]
        hist.append(f(x))
    return x, hist

def nag(x0, base_lr=0.05, beta=0.9, warmup_steps=10, steps=60):
    x = x0[:]
    v = [0.0, 0.0]
    hist = [f(x)]
    for t in range(steps):
        # warmup：前几步把学习率从小拉到目标值，减少初期震荡
        lr = base_lr * min(1.0, (t + 1) / warmup_steps)

        # 预测点：先看如果按当前惯性继续前进，会到哪里
        lookahead = [x[0] + beta * v[0], x[1] + beta * v[1]]

        # 在预测点计算梯度
        g = grad(lookahead)

        # 用预测点梯度修正速度
        v[0] = beta * v[0] - lr * g[0]
        v[1] = beta * v[1] - lr * g[1]

        # 再更新真实参数
        x[0] += v[0]
        x[1] += v[1]
        hist.append(f(x))
    return x, hist

x0 = [4.0, 4.0]
x_gd, h_gd = gd(x0)
x_hb, h_hb = heavy_ball(x0)
x_nag, h_nag = nag(x0)

assert h_gd[-1] < h_gd[0]
assert h_hb[-1] < h_hb[0]
assert h_nag[-1] < h_nag[0]

# 在这个玩具问题上，NAG 通常会收敛到更低误差或更平滑
assert h_nag[-1] < 1e-3
print("GD final:", h_gd[-1])
print("HB final:", h_hb[-1])
print("NAG final:", h_nag[-1])
```

这个例子里最重要的是两行：

```python
lookahead = [x[0] + beta * v[0], x[1] + beta * v[1]]
g = grad(lookahead)
```

这两行就是 NAG 和 Polyak 的本质差别。

工程中常见超参影响如下：

| 超参 | 作用 | 典型范围 | 调大后的常见现象 |
| --- | --- | --- | --- |
| $\eta$ | 学习率，控制每步幅度 | 依任务而定 | 太大时直接振荡或发散 |
| $\beta$ | 动量系数，控制历史保留比例 | 0.9 到 0.99 | 太大时惯性过强，转向变慢 |
| warmup steps | 预热步数，控制初期稳定性 | 数百到数千步 | 太短时初期冲过头 |
| gradient clipping | 梯度裁剪阈值，限制异常大梯度 | 0.1 到 1.0 或更高 | 阈值太松时不稳，太紧时变慢 |

真实工程例子可以看 Transformer 训练。训练前期参数还没进入稳定区域，梯度统计量变化很快，这时如果直接上高学习率和高动量，NAG 的预测点会偏得更远，导致损失曲线抖动。常见做法是：

- 前几百到几千步 warmup
- 动量系数控制在 $0.9$ 到 $0.99$
- 配合梯度裁剪
- 若使用 AdamW 类优化器，则把一阶矩看成动量，把 decoupled weight decay 单独处理

## 工程权衡与常见坑

NAG 的工程价值不在“永远赢”，而在“当问题确实存在狭长方向、且训练噪声可控时，更容易用相同步数达到更低误差”。

常见坑如下：

| 问题 | 表现 | 原因 | 规避措施 |
| --- | --- | --- | --- |
| 学习率过大 | loss 上下剧烈抖动 | 预测点偏得太远 | 降低 $\eta$，增加 warmup |
| 动量过高 | 曲线长期震荡，难以贴近最优点 | 惯性过强，转向滞后 | 将 $\beta$ 控制在 0.9 到 0.99 |
| 小批量噪声大 | 训练不稳定，验证集波动大 | 预测梯度受噪声放大 | 增大 batch、裁剪梯度、减小 lr |
| 把 NAG 理解成“两次梯度” | 实现错误，额外增加无意义开销 | 混淆了预测点与新采样点 | 只在预测点算一次梯度 |
| 与 weight decay 混写 | 收敛异常或泛化变差 | 正则项和主梯度耦合混乱 | 优先采用 decoupled weight decay |

对新手尤其重要的一点：**NAG 不是多算一次梯度**。它不是“先在原地算一次，再去预测点算一次”，而是“只在预测点算那一次”。如果误实现成两次，就不是标准 NAG 了。

再说一个真实工程现象。训练 ViT 或 Transformer 时，若一开始就设置较高 $\beta$，比如接近 $0.99$，同时学习率也拉得快，验证损失常见的现象不是单调下降，而是明显振荡。原因不是模型“学坏了”，而是优化器在高噪声阶段把前瞻项放大了。加入 warmup 后，相当于先让速度场稳定，再逐步打开前瞻能力，曲线通常会顺得多。

还有一个容易被忽略的事实：NAG 的理论优势主要建立在函数几何结构上，但深度网络训练还受归一化层、残差连接、批大小、混合精度、数据顺序等因素影响。因此工程上不要把“理论更优”直接翻译成“任何模型都更快”。

## 替代方案与适用边界

如果把常见方法放在一起比较，可以得到下面这张简表：

| 算法名称 | 预估收敛因子或速度特征 | 适用场景 | 特殊注意事项 |
| --- | --- | --- | --- |
| 标准 SGD/GD | 强凸下约 $1-1/\kappa$ | 基线、简单问题、低成本试验 | 条件数大时很慢 |
| Polyak Momentum | 经验上优于纯 SGD，二次问题常有效 | CNN、经典深度学习训练 | 过冲明显时要小心 |
| NAG | 强凸下约 $1-1/\sqrt{\kappa}$，凸问题有 $O(1/t^2)$ | 条件数高、谷底狭长、追求步数效率 | 对 lr 和 $\beta$ 更敏感 |
| AdamW | 自适应步长 + EMA 动量 | Transformer、LLM、默认工业方案 | 正确使用 decoupled weight decay |
| Adan / AdamN 类 | 在 Adam 风格上加入 Nesterov 式预测 | 大模型、追求更快 early progress | 超参更复杂，需看实现细节 |

什么时候优先选 NAG？

- 目标函数局部呈现明显狭长谷底
- 你关心的是迭代效率，而不是只求“能跑起来”
- 梯度噪声没有大到把前瞻项完全淹没
- 你愿意认真调学习率、warmup 和裁剪

什么时候回退到 Polyak 或 AdamW 更合理？

- batch 很小，梯度噪声极大
- 任务高度非凸，训练稳定性比理论加速更重要
- 团队基础设施已经围绕 AdamW 深度打磨
- 你没有精力做细致超参搜索

一句话总结适用边界：**NAG 最擅长“几何结构清晰、条件数高、噪声可控”的优化问题；AdamW 最擅长“复杂深度网络的稳妥默认配置”；Polyak 动量则是介于两者之间、实现简单且常常足够好的中间选项。**
