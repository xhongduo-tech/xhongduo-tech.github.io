## 核心结论

障碍法是在可行域内部加入一个“靠近边界就急剧增大”的平滑惩罚项，把带不等式硬约束的优化问题，转化为一系列更容易求解的无约束或等式约束子问题。

内点法是从严格可行域内部出发，沿着这些障碍子问题的最优解轨迹前进，逐步逼近原始约束优化问题的最优解。这条轨迹叫中心路径，意思是：对每个障碍参数 \(t\)，都有一个对应的“内部最优点”，这些点连起来形成一条路径。

一句话总结流程：先在可行域内部找一个点，再解一系列带障碍项的子问题，逐渐增大参数 \(t\)，最终逼近原问题解。

| 对象 | 原始约束优化 | 障碍法 / 内点法 |
|---|---|---|
| 约束处理 | 直接要求 \(f_i(x)\le 0\) | 用 \(-\log(-f_i(x))\) 阻止点靠近边界 |
| 迭代位置 | 可能需要处理边界 | 始终保持在严格可行域内部 |
| 求解方式 | 直接解带约束问题 | 解一串障碍子问题 |
| 收敛轨迹 | 不显式给出 | 沿中心路径逼近最优解 |
| 停止准则 | KKT 残差或目标间隙 | 常用 \(m/t \le \varepsilon\) |

新手版理解：原问题像“不能踩线的走钢丝”，约束边界就是不能碰的线。障碍项让点越靠近线，代价越大，于是算法会自然待在安全区域内部；当 \(t\) 增大时，算法更重视原目标，最后慢慢靠近真正的最优点。

## 问题定义与边界

标准约束优化问题通常写成：

$$
\min_x f_0(x)\quad \text{s.t. } f_i(x)\le 0,\ i=1,\dots,m,\ Ax=b
$$

这里 \(f_0(x)\) 是目标函数，意思是要被最小化的量；\(f_i(x)\le 0\) 是不等式约束，意思是不能越过的边界；\(Ax=b\) 是等式约束，意思是点必须一直落在某个线性子空间或仿射平面上。

可行域是所有满足约束的点的集合。严格可行点是满足 \(f_i(x)<0\) 且 \(Ax=b\) 的点，也就是还没有碰到任何不等式边界的内部点。边界是至少一个约束满足 \(f_i(x)=0\) 的位置。

障碍法要求初始点严格可行。原因很直接：对数障碍项是

$$
-\log(-f_i(x))
$$

只有当 \(f_i(x)<0\) 时，\(-f_i(x)>0\)，对数才有定义。若点已经在边界上，\(-\log(0)\) 发散；若点在不可行域，\(\log\) 的输入为负数，实数范围内无法计算。

| 原始约束 | 障碍项作用 | 对迭代点的影响 |
|---|---|---|
| \(f_i(x)<0\) | \(-\log(-f_i(x))\) 有定义 | 可以继续迭代 |
| \(f_i(x)\to 0^-\) | 障碍项趋向 \(+\infty\) | 点被推回内部 |
| \(f_i(x)=0\) | 障碍项发散 | 不能落在边界上 |
| \(f_i(x)>0\) | 对数无定义 | 不能进入不可行域 |
| \(Ax=b\) | 通常不放进对数障碍 | 每一步都要满足，或通过 KKT 系统处理 |

二维图示可以用文字描述：一个闭合区域表示可行域，区域边缘是 \(f_i(x)=0\)，区域外是不可行域。点在区域内部移动时，障碍项有限；点越靠近边界，障碍项越大；点如果越过边界，障碍目标不再可计算。

玩具例子：约束 \(x\ge 1\) 可以写成 \(1-x\le 0\)。它的严格可行域是 \(x>1\)，边界是 \(x=1\)。障碍项为 \(-\log(x-1)\)，当 \(x\) 从右侧靠近 1 时，该项趋向无穷大。

## 核心机制与推导

原问题是：

$$
\min_x f_0(x)\quad \text{s.t. } f_i(x)\le 0,\ i=1,\dots,m,\ Ax=b
$$

障碍法把它改写成一系列子问题：

$$
x^\*(t)=\arg\min_{Ax=b}\left(t f_0(x)-\sum_{i=1}^m \log(-f_i(x))\right)
$$

其中 \(t>0\) 是障碍参数。\(t\) 越小，障碍项影响越强，解更偏向可行域内部；\(t\) 越大，原目标 \(f_0(x)\) 的权重越强，解更接近原始约束问题的最优点。

中心路径定义为：

$$
\{x^\*(t)\mid t>0\}
$$

这条路径不是随便画出来的，而是每个固定 \(t\) 的障碍子问题最优解组成的集合。内点法的核心就是稳定地沿着这条路径往前走。

在可微情形下，障碍子问题的一阶条件可以写成：

$$
t\nabla f_0(x)+\sum_{i=1}^m \frac{1}{-f_i(x)}\nabla f_i(x)+A^T\nu=0
$$

其中 \(\nu\) 是等式约束的拉格朗日乘子。拉格朗日乘子是用来衡量约束对最优性条件影响的变量。

把上式除以 \(t\)，并定义：

$$
\lambda_i(t)= -\frac{1}{t f_i(x^\*(t))}
$$

就得到近似 KKT 条件中的对偶变量。因为严格可行时 \(f_i(x^\*(t))<0\)，所以 \(\lambda_i(t)>0\)。进一步有：

$$
-\lambda_i(t) f_i(x^\*(t))=1/t
$$

这就是近似互补关系。互补关系的白话解释是：一个约束如果离边界很远，它的对偶影响通常较小；一个约束如果快要起作用，它的对偶影响会变大。障碍法中每个约束的互补误差统一是 \(1/t\)，所以总对偶间隙常用 \(m/t\) 估计。常见停止准则是：

$$
m/t \le \varepsilon
$$

最小数值例子：

$$
\min \frac12 x^2 \quad \text{s.t. } x\ge 1
$$

改写为：

$$
\min_{x>1}\ \frac{t}{2}x^2-\log(x-1)
$$

一阶条件为：

$$
t x-\frac{1}{x-1}=0
$$

化简得到：

$$
t x(x-1)=1
$$

解为：

$$
x^\*(t)=\frac{1+\sqrt{1+4/t}}{2}
$$

当 \(t=1\) 时，\(x^\*(1)\approx 1.618\)，离边界 \(x=1\) 还比较远；当 \(t=9\) 时，\(x^\*(9)\approx 1.101\)，更接近边界；当 \(t\to\infty\) 时，\(x^\*(t)\to 1\)，也就是原问题的最优解。

## 代码实现

工程实现通常分成两层：外层更新障碍参数 \(t\)，内层用牛顿法或其他二阶方法求解固定 \(t\) 的障碍子问题。牛顿法是利用梯度和 Hessian 构造局部二次近似，再求一个下降方向的方法；Hessian 是二阶导数矩阵，用来描述目标函数局部曲率。

伪代码如下：

```python
t = t0
x = feasible_start()

while m / t > eps:
    x = solve_barrier_subproblem(x, t)
    t *= mu

return x
```

对障碍目标

$$
\phi_t(x)=t f_0(x)-\sum_{i=1}^m \log(-f_i(x))
$$

其梯度由两部分组成：原目标梯度 \(t\nabla f_0(x)\)，以及障碍项梯度。若没有等式约束，内层可直接对 \(\phi_t(x)\) 做牛顿迭代；若存在 \(Ax=b\)，通常要解带等式约束的 KKT 线性系统。

下面是一维玩具例子的可运行 Python 代码，验证障碍解会随着 \(t\) 增大而靠近原问题最优解 \(x=1\)。

```python
import math

def barrier_solution(t: float) -> float:
    assert t > 0
    return (1.0 + math.sqrt(1.0 + 4.0 / t)) / 2.0

def barrier_objective(x: float, t: float) -> float:
    assert x > 1.0
    return 0.5 * t * x * x - math.log(x - 1.0)

x1 = barrier_solution(1.0)
x9 = barrier_solution(9.0)
x100 = barrier_solution(100.0)

assert abs(x1 - 1.61803398875) < 1e-9
assert x9 < x1
assert x100 < x9
assert x100 > 1.0

# 一阶条件 t*x - 1/(x-1) = 0
for t in [1.0, 9.0, 100.0]:
    x = barrier_solution(t)
    residual = t * x - 1.0 / (x - 1.0)
    assert abs(residual) < 1e-9
```

对应数值表：

| \(t\) | \(x^\*(t)\) | 距离边界 \(x^\*(t)-1\) |
|---:|---:|---:|
| 1 | 1.6180 | 0.6180 |
| 9 | 1.1009 | 0.1009 |
| 100 | 1.0099 | 0.0099 |

线搜索用于控制每次牛顿步长，避免一步走得太远导致违反严格可行性。常见做法是从步长 1 开始，如果新点不满足 \(f_i(x)<0\)，或目标下降不足，就按比例缩小步长。

真实工程例子：大规模稀疏 \(\ell_1\) 正则最小二乘问题常写成

$$
\min_x \|Ax-b\|_2^2+\lambda\|x\|_1
$$

它出现在压缩感知、稀疏信号恢复、MRI 重建等场景。通过引入辅助变量，可以把绝对值项转成线性不等式约束，再用内点法求解。Stanford 的 `l1_ls` 工作就是这类工程实现的代表。

## 工程权衡与常见坑

障碍法的优点是理论结构清晰、收敛路径明确、可以利用二阶信息得到高精度解。代价是实现比简单一阶法复杂，并且对初始点、数值线性代数和参数更新比较敏感。

| 问题 | 原因 | 规避方式 |
|---|---|---|
| 严格可行性缺失 | \(-\log(-f_i(x))\) 无定义 | 先做可行性阶段，或使用 infeasible-start primal-dual 方法 |
| \(t\) 增长过快 | 子问题变化太大，迭代点偏离中心路径 | 取适中的 \(\mu\)，常见 \(10\sim20\)，配合回溯线搜索 |
| KKT 系统病态 | 接近最优点时曲率和约束尺度差异变大 | 做变量缩放、矩阵正则化、稀疏分解 |
| 符号约定混乱 | \(f_i(x)\le0\) 与 \(g_i(x)\ge0\) 混用 | 全文固定使用 \(f_i(x)\le0\) |
| 边界附近步长过大 | 新点可能越界 | 每次线搜索先检查严格可行性 |

新手版理解：没有可行初值，就像还没进入可行域内部，障碍项没有办法定义；\(t\) 一下子放太大，相当于突然把原目标权重拉满，之前沿中心路径的稳定性会丢失；矩阵病态时，不是理论错了，而是浮点数计算不稳。

工程上还要注意尺度问题。若一个变量量级是 \(10^6\)，另一个变量量级是 \(10^{-3}\)，Hessian 和 KKT 矩阵会很难解。缩放可以让变量和约束处在相近数量级。对稀疏大规模问题，应使用稀疏 Cholesky、预条件共轭梯度等结构化线性代数方法，而不是直接构造密集矩阵。

## 替代方案与适用边界

障碍法和内点法适合结构良好、精度要求较高、严格可行性可以获得或可以通过辅助阶段获得的问题。它们不是所有约束优化问题的默认最优选择。

| 方法 | 核心思想 | 适用场景 | 不适用边界 |
|---|---|---|---|
| 障碍法 / 内点法 | 在内部加入障碍项，沿中心路径逼近 | 中等到大规模凸优化、线性规划、二次规划、稀疏结构问题 | 严格可行点难找、每步预算很小 |
| 罚函数法 | 把违反约束的程度加到目标里 | 实现简单、可接受短暂不可行 | 罚参数过大时病态严重 |
| 增广拉格朗日法 | 结合拉格朗日乘子和二次罚项 | 等式约束、分解式优化、ADMM 场景 | 参数和子问题求解质量影响较大 |
| 投影法 / 一阶法 | 梯度步后投影回可行域 | 超大规模、低精度、每步需要便宜 | 投影复杂时成本高，收敛精度有限 |
| 坐标下降 | 每次只更新部分变量 | 高维稀疏机器学习问题 | 强耦合约束下不一定合适 |

工程版场景对比：\(\ell_1\) 正则最小二乘、压缩感知、MRI 重建这类问题具有可利用的凸结构和稀疏结构，内点法可以把二阶信息和稀疏线性代数结合起来，得到较高精度解。相反，如果是约束极多、数据持续到达、每一步只能做很轻量计算的大规模在线问题，投影梯度、坐标下降或随机一阶方法可能更合适。

新手版理解：内点法像“每一步都认真求一个高质量方向”。如果问题规模很大、精度要求不高、每步计算预算很小，这种风格未必划算。选择优化方法时，应同时看约束形式、问题规模、可行初值、精度要求和可用线性代数结构。

严格可行性是障碍法的重要边界。如果能找到严格可行点，经典障碍法流程很自然；如果找不到，工程实现通常会转向可行性阶段、原始-对偶内点法，或其他不要求始终严格可行的方法。
