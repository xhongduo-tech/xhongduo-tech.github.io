## 核心结论

中文分词是把连续汉字序列切成词序列的任务，本质是确定词边界。输入可以写成 $x = c_1...c_n$，其中 $c_i$ 是第 $i$ 个字；输出可以写成 $y = w_1...w_m$，其中 $w_j$ 是第 $j$ 个词。分词器要做的事，就是从很多可能的 $y$ 中选出最合理的一种。

中文没有天然空格，所以“哪里该停”不是文本里直接给出的信息，而是算法推断出来的结果。例如：

`北京大学生前来应聘`

可以切成：

`北京大学 / 生 / 前来 / 应聘`

也可以切成：

`北京 / 大学生 / 前来 / 应聘`

这两个结果都由相同汉字组成，但语义不同。前者像是在说“北京大学的学生前来应聘”，后者像是在说“北京的大学生前来应聘”。如果后续任务是搜索、推荐、分类、信息抽取或机器翻译，输入单位不同，结果就会不同。

| 分词结果 | 下游任务看到的关键词 | 可能影响 |
|---|---|---|
| `北京大学 / 生 / 前来 / 应聘` | 北京大学、生、应聘 | 更容易召回“北京大学”相关结果 |
| `北京 / 大学生 / 前来 / 应聘` | 北京、大学生、应聘 | 更容易召回“北京地区大学生招聘”相关结果 |
| `北京大学生 / 前来 / 应聘` | 北京大学生、应聘 | 可能产生不存在或不稳定的词项 |

分词不是“把句子切短”，而是为中文 NLP 系统确定基本输入单位。词典方法通常把问题变成路径搜索，选择代价最低的切法；统计方法和神经网络方法通常把问题变成字符级序列标注，判断每个字在词中的位置。BERT-CWS 这类方法利用上下文表示，对歧义和新词更稳，但计算和维护成本也更高。

## 问题定义与边界

中文分词的输入是连续汉字序列，输出是词序列。词边界是指两个相邻字符之间是否应该切开。例如 `研究生命起源` 中，`研究` 和 `生命` 之间有边界，`生` 和 `命` 之间没有边界。

形式化写法是：

$$
x = c_1...c_n
$$

$$
y = w_1...w_m
$$

其中每个词 $w_j$ 由一个或多个连续字符组成，并且所有词按顺序拼接后必须还原成原始输入 $x$。

同一句话可能有多个切法。玩具例子：

`研究生命起源`

切法 1：

`研究 / 生命 / 起源`

切法 2：

`研究生 / 命 / 起源`

第二种切法在多数语境下不合理，但它在形式上仍然是一个合法切分。分词任务要解决的不是“能不能切”，而是“在当前语境和标准下应该怎么切”。这就是歧义消解。歧义消解是指从多个可能解释中选择最符合上下文的一种。

另一种常见建模方式是序列标注。序列标注是给序列中的每个位置分配一个标签。中文分词里常用 `B/M/E/S` 标签：

- `B`：Begin，一个多字词的开头
- `M`：Middle，一个多字词的中间
- `E`：End，一个多字词的结尾
- `S`：Single，一个单字词

标签可写成：

$$
z_i \in \{B, M, E, S\}
$$

例如 `研究 / 生命 / 起源` 对应字符序列 `研 究 生 命 起 源`，标签可以是：

`B E B E B E`

| 输入 | 输出 | 是否允许多种切法 | 典型歧义 |
|---|---|---|---|
| `研究生命起源` | `研究 / 生命 / 起源` | 是 | `研究生 / 命 / 起源` |
| `北京大学生前来应聘` | `北京大学 / 生 / 前来 / 应聘` | 是 | `北京 / 大学生 / 前来 / 应聘` |
| `A股上涨` | `A股 / 上涨` | 是 | `A / 股 / 上涨` |
| `3.5mm接口` | `3.5mm / 接口` | 是 | `3.5 / mm / 接口` |

分词的边界也要说清楚：它不是完整语言理解，不负责判断事实真假，也不直接完成分类、问答、翻译。它只是中文 NLP 管线中的基础预处理步骤。但这个步骤会改变后续系统看到的输入，所以错误会被放大。

## 核心机制与推导

词典法把分词看成路径搜索问题。词典是一个“已知词集合”，通常还带有词频。词频是词在语料中出现的次数，可用来估计词的常见程度。给定一句话，算法枚举所有能由词典词拼出来的切法，然后选择总代价最低的一条路径。

常见代价函数是：

$$
Cost(y) = \sum_j -\log p(w_j)
$$

最优切分是：

$$
y^* = \arg\min_y Cost(y)
$$

这里 $p(w_j)$ 表示词 $w_j$ 的概率。高频词概率更大，$-\log p(w_j)$ 更小，所以由常见词组成的切法总代价通常更低。

玩具数值例子：

句子：`研究生命起源`

候选切分 1：`研究 / 生命 / 起源`，频次分别是 `5000, 3000, 4000`。  
候选切分 2：`研究生 / 命 / 起源`，频次分别是 `800, 10, 4000`。

如果简化使用 $Cost = \sum -\log freq$，第二种切法因为 `命` 的频次很低，总代价通常更高，所以算法倾向选择 `研究 / 生命 / 起源`。真实系统会做概率平滑、未登录词处理和路径动态规划，但核心思想仍然是“比较切法代价”。

序列标注法把分词看成逐字打标签。模型不直接枚举词序列，而是为每个字判断它在词里的位置。CRF 是条件随机场，一种常用于序列标注的统计模型，它会同时考虑当前字的标签分数和相邻标签之间的转移合理性。神经网络方法则先把每个字编码成上下文向量，再进行标签判断。上下文向量是包含左右语境信息的数字表示。

一个常见打分形式是：

$$
Score(x, z) = \sum_i (W_{z_i}^T h_i + T_{z_{i-1}, z_i})
$$

最优标签序列是：

$$
z^* = \arg\max_z Score(x, z)
$$

其中 $h_i$ 是第 $i$ 个字的表示，$W_{z_i}^T h_i$ 是当前字被标成某个标签的分数，$T_{z_{i-1}, z_i}$ 是从前一个标签转移到当前标签的分数。这个转移项很重要，因为它能避免明显非法的标签组合，例如 `B` 后面直接接 `S` 通常不构成一个正常多字词。

| 方法 | 核心思想 | 优缺点 |
|---|---|---|
| 词典法 | 枚举候选词，选总代价最低的路径 | 快、可控、易调试；对新词和歧义敏感 |
| HMM/CRF | 把每个字标成 `B/M/E/S`，用全局解码选最优标签序列 | 标注清晰、效果稳定；依赖特征和标注语料 |
| BERT-CWS | 用 BERT 编码上下文，再判断字符边界或标签 | 上下文能力强；成本高，部署复杂 |

真实工程例子是搜索引擎的中文查询理解。用户输入 `北京大学生前来应聘`，系统会先分词，再进入倒排索引召回。倒排索引是从词到文档列表的索引结构。如果分成 `北京大学 / 生 / 应聘`，召回会偏向包含“北京大学”的文档；如果分成 `北京 / 大学生 / 应聘`，召回会偏向招聘场景。分词错误会继续影响查询改写、排序特征和点击率预估。

## 代码实现

工程实现通常分三步：

```python
text = normalize(text)
words = segment(text)
words = postprocess(words)
```

规范化是把输入文本整理成统一形式，例如繁简转换、全半角转换、大小写处理、特殊空白清理。后处理是按业务规则修正分词结果，例如把 `3.5 / mm` 合并成 `3.5mm`，或把领域词强制合并。

流程可以写成：

$$
text \rightarrow normalize \rightarrow segment \rightarrow postprocess \rightarrow tokens
$$

下面是一个最小可运行 Python 例子，包含两部分：用动态规划做词典法最短路径分词，以及把 `B/M/E/S` 标签还原成词。

```python
import math

def segment_by_dict(text, freq):
    total = sum(freq.values())

    def word_cost(w):
        # 未登录词给较高代价，但仍允许单字通过，避免无法切分。
        if w in freq:
            return -math.log(freq[w] / total)
        return 20.0

    n = len(text)
    dp = [(float("inf"), []) for _ in range(n + 1)]
    dp[0] = (0.0, [])

    for i in range(n):
        base_cost, base_words = dp[i]
        if base_cost == float("inf"):
            continue

        for j in range(i + 1, n + 1):
            w = text[i:j]
            if w in freq or len(w) == 1:
                new_cost = base_cost + word_cost(w)
                if new_cost < dp[j][0]:
                    dp[j] = (new_cost, base_words + [w])

    return dp[n][1]


def decode_bmes(chars, tags):
    words = []
    buf = []

    for ch, tag in zip(chars, tags):
        if tag == "S":
            if buf:
                words.append("".join(buf))
                buf = []
            words.append(ch)
        elif tag == "B":
            if buf:
                words.append("".join(buf))
            buf = [ch]
        elif tag in {"M", "E"}:
            buf.append(ch)
            if tag == "E":
                words.append("".join(buf))
                buf = []
        else:
            raise ValueError(f"unknown tag: {tag}")

    if buf:
        words.append("".join(buf))

    return words


freq = {
    "研究": 5000,
    "生命": 3000,
    "起源": 4000,
    "研究生": 800,
    "命": 10,
}

assert segment_by_dict("研究生命起源", freq) == ["研究", "生命", "起源"]

chars = list("研究生命起源")
tags = ["B", "E", "B", "E", "B", "E"]
assert decode_bmes(chars, tags) == ["研究", "生命", "起源"]

chars = list("北京大学生前来应聘")
tags = ["B", "M", "M", "E", "S", "B", "E", "B", "E"]
assert decode_bmes(chars, tags) == ["北京大学", "生", "前来", "应聘"]
```

这段代码不是生产级分词器，但能展示两个核心机制。词典法的重点是“候选词 + 代价 + 全局最短路径”。标注法的重点是“字符标签 + 合法还原”。生产系统会增加更复杂的词典结构、概率平滑、未知词识别、模型推理、批处理和领域规则。

不同工具对 `北京大学生前来应聘` 的输出可能不同，因为它们的词典、训练语料、标注标准和后处理规则不同。代码实现时最重要的不是“调用哪个库”，而是知道当前切法为什么产生，以及这个切法是否符合业务目标。

## 工程权衡与常见坑

分词效果受语料标准影响很大。PKU、MSR、CTB 等语料的切分标准并不完全相同。标注标准是指“哪些字符组合算一个词”的规则集合。如果训练集、验证集、线上业务规则使用不同标准，指标会失真，线上结果也会不稳定。

| 问题 | 表现 | 影响 | 规避方式 |
|---|---|---|---|
| 标注标准不一致 | 训练和评测切法不同 | 指标不可比 | 统一 PKU、MSR、CTB 或自定义业务标准 |
| 新词和专名 | 人名、地名、机构名被切碎 | 召回下降，实体抽取失败 | 加领域词典或使用上下文模型 |
| 数字英文混排 | `A股`、`3.5mm` 被拆开 | 规则匹配和检索失败 | 先规范化，再加模式规则 |
| 局部贪心 | 前面切错，后面连锁错 | 长句歧义扩大 | 使用动态规划、CRF 或全局解码 |
| 文本未归一化 | 繁简、全半角混乱 | 词典命中率下降 | 分词前做规范化 |

很多工程错误不是模型能力不足，而是输入和词表没有处理好。例如财经场景里的 `A股`、消费电子场景里的 `3.5mm接口`、医疗场景里的药品名和疾病名，都是普通通用分词器容易切错的对象。此时继续换更大的模型不一定划算，先补领域词典和规范化规则通常更直接。

`Cost(y) = \sum_j -\log p(w_j)` 也说明了一个常见现象：高频词组合更容易被词典法选中。但高频不等于正确。比如某个新产品名刚出现时频次很低，词典法可能把它切碎；而在业务中，它又恰好是最重要的检索词。这就是新词问题。新词是指训练语料或词典中没有出现过、但线上文本中已经需要识别的词。

真实工程里还要考虑性能。搜索引擎、广告系统、日志分析系统经常需要处理大量短文本。词典法速度快、延迟低，适合高吞吐场景；神经分词效果强，但推理成本更高，可能需要批处理、缓存或蒸馏模型。蒸馏是用大模型训练小模型，让小模型尽量保留大模型效果的一种压缩方法。

## 替代方案与适用边界

不存在对所有场景都最优的中文分词算法。选择方法时要看文本类型、业务目标、延迟要求、可维护性和数据条件。

| 方法 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|
| 词典法 | 快、可控、易调试 | 歧义弱，新词弱 | 规则固定、术语稳定、延迟敏感 |
| CRF/BMES | 标注清晰，实现成熟 | 依赖特征和标注数据 | 中小规模任务，标准明确 |
| BERT-CWS | 上下文强，歧义处理好 | 成本高，维护复杂 | 复杂语境、动态领域、高准确率任务 |
| 不显式分词 | 避免分词误差传递 | 可解释性弱，资源成本可能高 | 字符级模型、端到端深度学习任务 |

新手可以这样理解：词典法是“按词表和规则切”；CRF/序列标注是“给每个字判断位置”；BERT-CWS 是“先根据上下文得到字的表示，再判断边界”。但这只是理解入口，不应替代技术定义。准确地说，BERT-CWS 仍然是在字符级位置上做边界或标签预测，只是 $h_i$ 来自更强的上下文编码器。

序列标注里的全局解码比逐字贪心更稳。逐字贪心是每个位置只选当前看起来最好的标签，不充分考虑前后约束；全局解码则选择整条标签序列的最高分：

$$
z^* = \arg\max_z Score(x, z)
$$

这能减少非法标签序列和局部错误传播。比如 `B` 后面应该更可能接 `M` 或 `E`，而不是随意接 `S`。CRF 的转移矩阵可以学习这种约束。

实际选型可以按以下原则：

| 场景 | 推荐选择 | 原因 |
|---|---|---|
| 静态词表、业务规则强 | 词典法 + 后处理 | 可控、易解释、上线成本低 |
| 有标注数据、领域中等复杂 | CRF 或 BiLSTM-CRF | 效果和成本平衡 |
| 歧义多、新词多、语境复杂 | BERT-CWS 或类似神经模型 | 上下文建模能力更强 |
| 下游模型本身字符级建模 | 可弱化显式分词 | 避免把分词错误固定下来 |

分词是中文 NLP 的基础步骤，但不是所有现代系统都必须显式分词。对传统搜索、关键词匹配、规则抽取、倒排索引来说，分词仍然非常关键。对部分端到端神经模型来说，模型可以直接处理字符、子词或字节表示，显式分词的重要性会下降。工程上要避免两个极端：一是把分词当成万能语言理解，二是因为大模型存在就忽略分词在检索、规则和可解释系统中的价值。
