## 核心结论

VQAv2 是视觉问答领域最常用的公开基准之一。视觉问答，白话说，就是“给一张图和一句问题，让模型像人一样看图作答”。它的价值不在于题量大，而在于它专门针对一个老问题做了改进：很多模型其实没有认真看图，只是根据问题的常见说法猜答案。

VQAv2 基于 COCO 图像构建，包含约 205K 张图像、约 1.1M 个问题。每个问题通常有 10 个人工答案，评价时不是按“唯一标准答案”打分，而是按“和多少人一致”打分。这种共识评分更接近真实语言环境，因为自然语言答案本来就可能存在同义、简写和轻微差异。

它的核心设计是“互补图像”。互补图像，白话说，就是“问题看起来一样，但图里关键事实不同，所以答案必须不同”的配对样本。这样做的目标很明确：降低语言先验。语言先验，白话说，就是“模型只靠题目措辞和训练中的统计规律就能猜出高频答案”。VQAv2 不是彻底消灭这个问题，但显著提高了“只读问题就得高分”的难度。

从工程角度看，VQAv2 适合做三件事：检验多模态模型是否真的融合了图像与文本；比较不同模型在标准公开集上的能力；为训练过程提供一个稳定、可复现的验证指标。它不适合直接代表所有真实业务，因为真实场景中的图片分布、问题风格、噪声水平都可能和 COCO 相差很大。

| 维度 | VQAv2 的做法 | 工程意义 |
|---|---|---|
| 数据来源 | COCO 图像 + 人工问题/答案 | 便于和检测、描述等任务对齐 |
| 样本规模 | 约 205K 图像，约 1.1M 问题 | 足够支撑训练与公开比较 |
| 标注形式 | 每题 10 个可信答案 | 允许自然语言存在多样表达 |
| 偏差控制 | 引入互补图像 | 降低只靠语言模式猜题的收益 |
| 常见用途 | 验证视觉理解与语言推理结合能力 | 适合作为多模态模型的基线评测 |

## 问题定义与边界

VQAv2 的任务定义很直接：输入一张图像 $I$ 和一个自然语言问题 $Q$，输出一个自然语言答案 $A$。形式上可以写成：

$$
A = f(I, Q)
$$

这里的 $f$ 就是待评测的模型。它可能是分类器，也可能是生成式模型，但最终都要回答一个短文本答案。

问题类型大致可分为三类：`yes/no`、`number`、`other`。`yes/no` 是“是不是”“有没有”；`number` 是“几个”；`other` 是颜色、物体、动作、身份、场景等开放式答案。对初学者来说，这三类很重要，因为它们难度不同，模型短板也不同。很多模型 `yes/no` 很高，但 `number` 较差，因为计数往往要求更精确的视觉定位。

一个玩具例子最容易说明边界：

- 图 1：一个男人戴眼镜，旁边站着女人
- 图 2：一个女人戴眼镜，旁边站着男人
- 问题：`Who is wearing glasses?`

如果模型只读问题，它会倾向输出训练集中更常见的答案，比如 `man` 或 `woman`。但在 VQAv2 里，这个问题会被刻意配到答案不同但结构相似的图像上，于是仅靠语言模式就不够了。

这就是它的边界所在。VQAv2 解决的是“在通用自然图像上，模型是否把视觉线索纳入回答过程”。它不主要测试以下能力：

| 不擅长覆盖的能力 | 原因 |
|---|---|
| 长链条多步推理 | 大部分问题较短，推理链不深 |
| OCR 密集场景 | COCO 不是文档或招牌密集数据集 |
| 专业图像理解 | 医学影像、卫星图像不在分布内 |
| 开放世界知识问答 | 问题主要围绕图像可见内容 |
| 复杂交互式问答 | 单轮问题为主，不是多轮对话 |

真实工程例子是电商审核或内容理解。假设你做“商品图问答”，用户问“杯子里有没有吸管”“鞋带是不是系着”。如果你只在 VQAv2 上得分高，不能直接说明线上效果就一定好，因为电商图片的构图、白底、局部特写、品牌文案都和 COCO 差异明显。VQAv2 能告诉你模型“基本会看图答题”，但不能替你完成领域泛化验证。

## 核心机制与推导

VQAv2 的关键机制可以拆成两个部分：偏差控制和共识评分。

先看偏差控制。设问题为 $Q$，答案为 $A$。如果一个数据集里某类问题的答案高度集中，那么仅看问题就能学到较强的条件分布 $P(A \mid Q)$。例如看到 `What color are the bananas?`，模型很可能直接猜 `yellow`。这说明数据集允许语言模式支配结果。

VQAv2 通过互补图像让同一个问题模板在不同图像下对应不同答案，从而提高条件熵：

$$
H(A \mid Q) = - \sum_a P(a \mid Q)\log P(a \mid Q)
$$

条件熵，白话说，就是“同一个问题下，答案有多不确定”。如果所有 `banana` 题几乎都答 `yellow`，那么 $H(A \mid Q)$ 很低；如果同样的问题在不同图像里能答 `green`、`yellow`、`none`，那么熵更高，模型更难靠背题取胜。

再看评分。VQAv2 使用 10 个人工答案，不要求模型必须和“唯一标准答案”完全相同，而是按共识打分。常见近似写法是：

$$
\mathrm{Acc}(a) = \min\left(1, \frac{n(a)}{3}\right)
$$

其中 $n(a)$ 是 10 个标注答案里与模型预测相同的个数。意思是：只要至少有 3 个人给出同样答案，就算满分。这个规则的直觉是，三人一致已经足以说明答案合理。

玩具例子如下：

| 10 个标注中的计数 | 模型预测 | 得分 |
|---|---|---|
| `red`: 4 次 | `red` | $\min(1, 4/3)=1$ |
| `blue`: 3 次 | `blue` | $\min(1, 3/3)=1$ |
| `green`: 0 次 | `green` | $0$ |

为什么 `blue` 只有 3 票也能满分？因为 VQAv2 承认自然语言存在合理分歧。比如有人答 `bike`，有人答 `bicycle`；有人答 `two`，有人答 `2`。这就要求评测时先做归一化。归一化，白话说，就是“把大小写、标点、数字写法等无关差异先统一”。

一个更接近真实工程的例子是多模态大模型训练。训练时通常有两条隐含竞争路径：

- 一条路径看图像和问题，学习真正的跨模态对应。
- 一条路径只看问题，学习“高频问法对应高频答案”。

如果第二条路径收益太高，模型就会“偷懒”。VQAv2 的设计目的，就是通过互补图像削弱这条偷懒路径的收益，但它并不能完全消除，因为训练集与测试集的整体答案分布仍然相近。

## 代码实现

实际工程中，VQAv2 的评估流程通常分四步：读取问题与图像映射、读取 10 个标注答案、做文本归一化、按共识公式打分。下面给一个可运行的最小 Python 例子，演示核心逻辑。

```python
import re
from collections import Counter

_DIGIT_MAP = {
    "zero": "0", "one": "1", "two": "2", "three": "3", "four": "4",
    "five": "5", "six": "6", "seven": "7", "eight": "8", "nine": "9",
    "ten": "10"
}

_ARTICLES = {"a", "an", "the"}

def normalize(text: str) -> str:
    text = text.lower().strip()
    text = re.sub(r"[^\w\s]", "", text)
    tokens = []
    for tok in text.split():
        if tok in _ARTICLES:
            continue
        tokens.append(_DIGIT_MAP.get(tok, tok))
    return " ".join(tokens)

def vqav2_score(prediction: str, human_answers: list[str]) -> float:
    pred = normalize(prediction)
    counts = Counter(normalize(a) for a in human_answers)
    return min(1.0, counts[pred] / 3.0)

# 玩具例子
answers = ["red", "Red", "blue", "red.", "blue", "red", "blue", "yellow", "green", "red"]
assert normalize("The red.") == "red"
assert abs(vqav2_score("red", answers) - 1.0) < 1e-9
assert abs(vqav2_score("blue", answers) - 1.0) < 1e-9
assert abs(vqav2_score("yellow", answers) - (1/3)) < 1e-9
assert abs(vqav2_score("black", answers) - 0.0) < 1e-9

# 数字归一化例子
num_answers = ["2", "two", "Two", "3", "two", "2", "2", "two", "one", "2"]
assert normalize("Two") == "2"
assert abs(vqav2_score("2", num_answers) - 1.0) < 1e-9
```

这个实现有两个重点。

第一，归一化必须先做。否则 `Two`、`two`、`2` 会被当成不同答案，导致分数虚低。第二，评分不是多数票准确率，而是“预测答案在 10 个答案里出现几次，再除以 3 封顶”。这与常见分类任务很不一样。

如果把它放进真实训练流程，通常是这样：

| 阶段 | 输入 | 产出 |
|---|---|---|
| 数据读取 | 图像 ID、问题文本、10 个答案 | 训练/验证样本 |
| 视觉编码 | 图像像素或图像特征 | 视觉 token 或向量 |
| 文本编码 | 问题文本 | 文本 token 或向量 |
| 融合推理 | 视觉与文本联合输入 | 答案分布或生成结果 |
| 评估 | 模型答案 + 10 个人类答案 | VQAv2 准确率 |

真实工程例子：假设你训练一个视觉语言模型做客服问答，问题是“这张图里的信号灯是什么颜色”。线上你可能使用生成式解码输出文本，但离线评测时仍然会把输出先归一化，再和标注答案对比。如果你跳过这个步骤，模型可能因为输出 `The red light` 而被错误判成错题，结果你会低估模型质量，进而做出错误的训练决策。

## 工程权衡与常见坑

VQAv2 最大的优点是标准化和可比较，最大的局限是“偏差降低了，但没有消失”。这句话要明确。很多初学者会把 VQAv2 理解为“已经解决语言偏见的数据集”，这是不准确的。更准确的说法是：它让问题更难被纯语言模型利用，但仍然保留了可被统计模式利用的结构。

常见坑有五类。

| 坑 | 现象 | 后果 |
|---|---|---|
| 不做答案归一化 | `2` 和 `two` 被分开 | 分数偏低 |
| 只看总分 | `yes/no` 高，`number` 很差 | 误判模型全面能力 |
| 忽略语言偏差 | question-only 分支也能学到很多 | 线上遇到分布变化时掉点 |
| 直接拿 leaderboard 当业务指标 | 公共集高分，业务集不稳定 | 评估与上线脱节 |
| 把生成式长答案直接送评测 | 输出冗长句子 | 与短答案标注对不上 |

一个典型玩具例子是 `What color is the banana?`。如果训练集中绝大多数是黄色香蕉，模型即使不看图也会倾向答 `yellow`。你在普通 test-dev 上可能还觉得结果不错，但一旦换到更强调分布变化的诊断集，性能会明显下降。

真实工程里，很多团队会额外训练一个 question-only 基线。question-only，白话说，就是“完全不看图，只给问题文本”的模型。如果这个基线已经很高，说明数据偏差仍然强。进一步的做法是引入对抗训练或偏差抑制损失，让主模型不能过度依赖语言统计。一个极简示意如下：

```python
def total_loss(main_loss: float, q_only_loss: float, alpha: float = 0.2) -> float:
    # 这里用减号表达“主分支要摆脱纯问题分支的可预测模式”
    return main_loss - alpha * q_only_loss

assert abs(total_loss(1.0, 0.5, 0.2) - 0.9) < 1e-9
```

这段代码不是完整论文实现，但表达了工程思想：当纯问题分支过于容易时，要显式约束主模型不要沿着这条捷径优化。

还有一个常见误区是把公开榜单分数当成唯一目标。比如某些多模态模型在 VQAv2 test-dev 上可到 78% 到 81% 左右，这说明模型具备较强通用视觉问答能力，但不能自动推出它在表单识别、工业巡检、UI 理解、广告审核上也同样有效。原因很简单：输入分布变了，问题空间也变了。

## 替代方案与适用边界

如果你的目标是诊断“模型到底有没有真正看图”，仅靠 VQAv2 往往不够。这时要考虑替代或补充基准。

最常见的是 VQA-CP。CP 是 changing priors，白话说，就是“故意打乱训练和测试中的答案先验分布”。例如训练里 `What sport...` 多半是 `tennis`，测试里却变成 `skiing` 或 `baseball` 更常见。这样一来，只靠语言模式的模型会掉得很厉害。如果一个模型在 VQAv2 很高、在 VQA-CP 暴跌，基本可以判断它仍然在吃语言偏差红利。

另一个方向是对抗式 VQA 或鲁棒性基准。这类数据集不只是换分布，还会专门构造误导性问题、相似图干扰、局部细节陷阱，用来检查模型是不是在抓真正关键的视觉证据。

如果你的任务已经从“答一个短问题”扩展到“判断一张图是否符合一段自然语言描述”，那么像 VQAScore 这样的评估思路更接近生成模型场景。它的关注点不是标准答案命中，而是图文整体匹配程度。

| 基准 | 主要目标 | 更适合回答的问题 |
|---|---|---|
| VQAv2 | 通用视觉问答能力 | 模型是否具备标准多模态问答能力 |
| VQA-CP | 语言偏差诊断 | 模型是否过度依赖问题先验 |
| Adversarial VQA | 鲁棒性与抗干扰 | 模型是否会被相似问法或误导样本击穿 |
| VQAScore 类方法 | 图文整体一致性 | 生成图像或描述是否语义匹配 |

所以适用边界可以这样总结：

- 想做公开对比、训练验证、基础能力检查，用 VQAv2。
- 想检查模型是不是在“背答案分布”，补跑 VQA-CP。
- 想评估抗干扰能力，看对抗式或鲁棒性基准。
- 想评估生成模型，不要只盯着 VQAv2，要加入图文匹配类指标。

对于零基础读者，最重要的一句判断标准是：VQAv2 测的是“会不会在标准自然图像上看图答题”，不是“所有多模态能力的总分”。
