## 核心结论

IGMC，全称 Inductive Graph-based Matrix Completion，意思是“归纳式图矩阵补全”：它不直接为每个用户和物品学习固定 ID 向量，而是把每个 `(user, item)` 对转换成一个局部图样本，再用图神经网络预测这条边的评分或偏好。

传统矩阵分解需要学习用户向量和物品向量，新来的用户或物品没有训练好的向量，通常要补特征、微调或重训；IGMC 只看目标边周围的局部子图，所以只要新节点已经有少量交互，就能直接抽子图并预测新样本。

核心流程可以写成：

$$(user, item) \rightarrow h\text{-hop enclosing subgraph} \rightarrow GNN \rightarrow score$$

| 维度 | 矩阵分解 | IGMC |
|---|---|---|
| 基本对象 | 用户 ID 向量、物品 ID 向量 | 每个用户-物品对的局部子图 |
| 学到的信号 | ID 级别的潜在因子 | 局部连接结构和评分关系 |
| 新用户/新物品 | 没有向量，处理麻烦 | 有少量边即可抽子图预测 |
| 典型任务 | 显式评分预测 | 显式评分预测，也可改造成隐式反馈 |
| 主要成本 | 全量 embedding 训练和更新 | 子图抽取、采样和 GNN 计算 |
| 最大风险 | 对新 ID 泛化弱 | 子图为空或目标边泄漏 |

IGMC 的核心价值不是“比所有推荐模型都强”，而是把矩阵补全问题改写成图分类或图回归问题。模型预测的不是“这个用户 ID 和这个物品 ID 的记忆结果”，而是“这个局部交互结构通常对应什么评分”。

---

## 问题定义与边界

矩阵补全的目标是：已知一部分用户对物品的评分或交互，预测未知的用户-物品关系。把它写成图，就是一个评分二部图：

$$G=(U \cup V, E)$$

其中，$U$ 是用户集合，$V$ 是物品集合，$E$ 是用户和物品之间的边。二部图是指图里有两类节点，边只连接不同类型的节点，例如用户连物品，用户不直接连用户，物品不直接连物品。

如果边上有评分，比如 1 到 5 星，任务就是评分回归。回归是指输出一个连续或有序数值，例如预测 $\hat r_{uv}=4.2$。如果边表示点击、收藏、购买，任务可以改成点击预测或偏好排序。偏好排序是指模型不一定要预测绝对分数，而是让用户更可能喜欢的物品排在前面。

| 任务 | 输入边 | 输出 | 常见损失 |
|---|---|---|---|
| 评分回归 | `u` 给 `v` 打了 1-5 分 | 预测评分 | MSE |
| 点击预测 | `u` 是否点击 `v` | 点击概率 | BCE |
| 偏好排序 | `u` 喜欢正样本多于负样本 | 排序分数 | pairwise ranking loss |

IGMC 的边界必须说清楚：它不是零交互冷启动方案。零交互冷启动是指新用户或新物品完全没有任何历史边。IGMC 依赖目标节点周围的局部图结构，如果一个新电影刚上线但已经有少量点击、评分或收藏，它就适合 IGMC，因为可以围绕这个新电影和目标用户抽取子图。如果这个新电影完全没有任何交互，子图为空，模型无法从图结构里推断偏好。

| 场景 | 是否适合 IGMC | 原因 |
|---|---|---|
| 可用 | 新物品已有少量评分，新用户也有少量历史行为 | 能抽出有效局部子图 |
| 可用 | 老用户预测新电影，电影已有少量点击 | 新物品不是孤立节点 |
| 勉强可用 | 交互极少，只能抽到很小子图 | 信号弱，预测方差大 |
| 不可用 | 新用户没有任何行为 | 用户侧没有邻域 |
| 不可用 | 新物品没有任何曝光或点击 | 物品侧没有邻域 |
| 不适合直接用 | 只依赖文本、图片、价格等内容特征 | IGMC 原生重点是图结构，不是内容理解 |

真实工程例子：短剧推荐系统每天都会上线新短剧。传统 embedding 模型需要等新短剧积累数据后重新训练或增量更新物品向量。IGMC 的使用方式是，只要新短剧已经有几十个点击、完播或评分，就可以围绕 `(用户, 新短剧)` 抽取局部交互子图，直接预测该用户是否可能点击或喜欢它。

## 核心机制与推导

IGMC 对目标用户-物品对 $(u,v)$ 抽取 $h$ 跳 enclosing subgraph，记为：

$$G^h_{u,v}$$

enclosing subgraph 可以理解为“包住目标用户和目标物品的局部子图”：从 $u$ 和 $v$ 出发，沿着已有交互边向外走最多 $h$ 步，把相关节点和边取出来。注意，训练时要先删除目标边 $(u,v)$，否则模型会直接看到答案。

抽出子图后，IGMC 会做节点重标号。节点重标号是指不用原始用户 ID 或物品 ID，而是用一个标签告诉模型“这个节点在目标边附近处于什么位置”。这样模型学到的是结构角色，而不是具体 ID。

| 节点类型 | 标号 | 含义 |
|---|---:|---|
| 目标用户 `u` | 0 | 当前要预测的用户端点 |
| 目标物品 `v` | 1 | 当前要预测的物品端点 |
| 第 1 跳用户邻居 | 2 | 离目标边近的用户侧邻居 |
| 第 1 跳物品邻居 | 3 | 离目标边近的物品侧邻居 |
| 第 `i` 跳用户邻居 | `2i` | 第 `i` 层用户节点 |
| 第 `i` 跳物品邻居 | `2i+1` | 第 `i` 层物品节点 |

玩具例子：已有三条评分边：

| 边 | 评分 |
|---|---:|
| `u1 -> i1` | 5 |
| `u2 -> i1` | 4 |
| `u2 -> i2` | 4 |

现在要预测 `u1 -> i2`。如果删除目标边后抽局部子图，会看到一条路径：

```text
u1 -> i1 -> u2 -> i2
```

这说明 `u1` 和 `u2` 都和 `i1` 有高分关系，而 `u2` 又给 `i2` 较高评分。模型可能据此预测 `u1 -> i2` 也偏高。这里的关键不是记住 `u1` 这个 ID，而是识别出一种局部结构相似性：目标用户和某个邻近用户在共同物品上表现接近，而邻近用户又喜欢目标物品。

子图进入 GNN 后，常用的是 R-GCN。R-GCN 是关系图卷积网络，白话说就是“不同类型的边使用不同的消息变换参数”。在评分图里，1 星、2 星、3 星、4 星、5 星可以看成不同关系类型。

一层 R-GCN 的更新公式是：

$$h_i^{(l+1)} = W_0^{(l)}h_i^{(l)} + \sum_{r\in R}\sum_{j\in N_r(i)} \frac{1}{|N_r(i)|} W_r^{(l)}h_j^{(l)}$$

其中，$h_i^{(l)}$ 是节点 $i$ 在第 $l$ 层的表示，$R$ 是关系类型集合，$N_r(i)$ 是通过关系 $r$ 连到节点 $i$ 的邻居集合。$W_r^{(l)}$ 是关系 $r$ 对应的参数矩阵。

经过 $L$ 层消息传递后，取目标用户和目标物品的最终表示拼接：

$$g=[h_u;h_v]$$

再经过 MLP 输出预测评分：

$$\hat r_{uv}=w^T\sigma(Wg)$$

MLP 是多层感知机，白话说就是几层全连接网络，用来把图表示转换成最终分数。显式评分场景下，主损失通常是 MSE：

$$L_{MSE}=\frac{1}{|\Omega|}\sum_{(u,v)\in \Omega}(r_{uv}-\hat r_{uv})^2$$

MSE 是均方误差，预测分数离真实分数越远，惩罚越大。原论文还加入 ARR 正则。ARR 可以理解为“相邻评分关系的参数应该更平滑”，例如 4 星和 5 星的关系参数不应完全无关：

$$L_{ARR}=\sum_k ||W_{r_{k+1}}-W_{r_k}||_F^2$$

最终损失为：

$$L_{final}=L_{MSE}+\lambda L_{ARR}$$

从输入到输出的流程如下：

```text
目标样本 (u, v)
  -> 从训练图中删除目标边
  -> 抽取 h-hop enclosing subgraph G^h_{u,v}
  -> 对子图节点做结构标号
  -> 把评分边作为关系类型输入 R-GCN
  -> 聚合得到目标用户和目标物品表示
  -> 拼接 [h_u; h_v]
  -> MLP 输出 score
  -> 用 MSE / BCE / 排序损失训练
```

## 代码实现

工程实现的主线是“构图索引 -> 子图抽取 -> 节点重标号 -> GNN 前向 -> 损失训练”。模型层数不是最容易出错的地方，真正容易出错的是数据预处理：目标边是否删除、训练验证测试是否隔离、子图采样是否稳定。

| 输入 | 处理 | 输出 |
|---|---|---|
| 原始评分表 `(user, item, rating)` | 构建二部图索引 | 用户邻接表、物品邻接表 |
| 单个训练样本 `(u, v, r)` | 删除目标边 | 无泄漏训练图 |
| 目标对 `(u, v)` | 抽取 `h` 跳邻域 | 子图节点和边 |
| 子图节点 | 结构重标号 | 节点标签特征 |
| 子图边 | 按评分分关系类型 | R-GCN 输入 |
| GNN 输出 | 拼接目标端点表示 | 预测分数 |

单个样本的伪代码如下：

```text
for (u, v, r) in train_edges:
    graph_without_target = remove_edge(train_graph, u, v)
    subgraph = extract_h_hop_subgraph(graph_without_target, u, v, h)
    labels = structural_node_labeling(subgraph, target_user=u, target_item=v)
    pred = rgcn_forward(subgraph, labels)
    loss = mse(pred, r) + lambda * arr_regularization()
    loss.backward()
```

下面是一个可运行的 Python 玩具实现。它不实现完整 R-GCN，只演示 IGMC 最关键的数据处理：删除目标边、抽取局部子图、生成结构标号，并用 `assert` 验证目标边没有泄漏。

```python
from collections import defaultdict, deque

# 边格式: (user, item, rating)
edges = [
    ("u1", "i1", 5),
    ("u2", "i1", 4),
    ("u2", "i2", 4),
    ("u3", "i2", 2),
]

def build_graph(edges, drop=None):
    graph = defaultdict(list)
    for u, i, r in edges:
        if drop == (u, i):
            continue
        graph[("U", u)].append((("I", i), r))
        graph[("I", i)].append((("U", u), r))
    return graph

def h_hop_nodes(graph, starts, h):
    dist = {}
    q = deque()
    for s in starts:
        dist[s] = 0
        q.append(s)

    while q:
        node = q.popleft()
        if dist[node] == h:
            continue
        for nxt, _rating in graph[node]:
            if nxt not in dist:
                dist[nxt] = dist[node] + 1
                q.append(nxt)
    return dist

def structural_labels(dist, target_user, target_item):
    labels = {}
    for node, d in dist.items():
        kind, raw_id = node
        if node == ("U", target_user):
            labels[node] = 0
        elif node == ("I", target_item):
            labels[node] = 1
        elif kind == "U":
            labels[node] = 2 * d
        else:
            labels[node] = 2 * d + 1
    return labels

target = ("u1", "i2")
graph = build_graph(edges, drop=target)
dist = h_hop_nodes(graph, [("U", target[0]), ("I", target[1])], h=2)
labels = structural_labels(dist, target_user=target[0], target_item=target[1])

assert ("U", "u1") in labels
assert ("I", "i2") in labels
assert all(not (nxt == ("I", "i2") and node == ("U", "u1")) for node in graph for nxt, _ in graph[node])
assert labels[("U", "u1")] == 0
assert labels[("I", "i2")] == 1

print(labels)
```

训练时通常不会一个子图一个子图串行喂模型，而是把多个子图组成 mini-batch。mini-batch 是小批量训练，意思是每次取一组样本一起计算梯度。图数据的 batch 方式通常是把多个小图拼成一个大图，同时保留每个节点属于哪个子图的 `batch_id`，再在读出阶段只取每个子图里的目标用户和目标物品表示。

真实工程里还要做三件事。第一，提前缓存高频样本的子图，减少重复 BFS。BFS 是广度优先搜索，用来一层一层找邻居。第二，对高度节点设置采样上限，例如一个热门商品有几十万用户交互，不可能全部放进子图。第三，训练、验证、测试的边划分要先确定，再按各自允许的信息构图，不能让测试边参与训练子图抽取。

## 工程权衡与常见坑

IGMC 的第一个核心超参数是 $h$。$h$ 表示抽几跳邻域。$1$-hop 已经能覆盖目标用户和目标物品附近的直接协同信号，而 $3$-hop 可能把大量无关节点卷进来，训练成本和噪声都会上升。图上的节点数常常随跳数快速膨胀，所以 $h$ 不能盲目加大。

| 坑 | 结果 | 规避方式 |
|---|---|---|
| 没有删除目标边 | 标签泄漏，离线指标虚高 | 每个样本抽图前移除目标边 |
| `h` 设太大 | 子图膨胀，训练慢，噪声多 | 从 `1-hop` 或 `2-hop` 开始 |
| 热门节点不采样 | 单个子图过大 | 设置每层邻居采样上限 |
| 把评分当无序类别 | 忽略 4 星和 5 星更接近 | 使用 ARR 或有序建模 |
| 直接把回归版用于点击 | 输出和损失不匹配 | 改成 sigmoid + BCE 或排序损失 |
| 划分数据后又全图抽边 | 测试信息进入训练 | 按 split 构造可见图 |
| 评分尺度混乱 | 模型输出不可解释 | 统一归一化或保持固定评分范围 |

推荐起点配置如下：

| 配置项 | 显式评分预测 | 隐式反馈预测 |
|---|---|---|
| `h` | 1 或 2 | 1 或 2 |
| 邻居采样上限 | 每层每类 50-200 起步 | 视点击量更严格限制 |
| batch size | 由子图大小决定，常从 32 起试 | 常从 64 起试 |
| 输出头 | 线性分数 | sigmoid 概率或排序分数 |
| loss | MSE + 可选 ARR | BCE 或 pairwise ranking loss |
| 负采样 | 通常不需要 | 通常需要 |

训练前检查清单：

| 检查项 | 必须确认的问题 |
|---|---|
| 删边 | 当前样本的目标边是否已从输入图中移除 |
| 采样 | 热门用户和热门物品是否有邻居上限 |
| split | 验证集和测试集边是否没有泄漏进训练图 |
| 评分尺度 | 评分是否统一为同一范围 |
| 输出头 | 回归、二分类、排序任务是否使用了匹配输出 |
| 负样本 | 隐式反馈任务是否定义了负采样策略 |
| 缓存 | 子图缓存是否和数据划分、采样随机种子一致 |

一个常见错误是只看模型结构，不看数据管线。IGMC 的结构并不复杂，难点在于“每个样本的图到底包含了什么”。如果目标边没有删除，模型会直接看到要预测的评分关系；如果测试边参与了邻域构造，模型就间接看到了测试信息。这类错误会让离线指标显著变好，但上线后效果回落。

## 替代方案与适用边界

IGMC 适合有局部图结构、需要归纳推理的矩阵补全场景，但它不是推荐系统的默认最优解。选择方法时应先看业务约束：用户和物品集合是否稳定、是否有新节点、目标是评分还是排序、是否有内容特征、实时性要求多高。

| 方法 | 适合场景 | 优点 | 局限 |
|---|---|---|---|
| 矩阵分解 | 用户和物品集合稳定，评分预测为主 | 简单、快、容易解释 | 新 ID 处理弱 |
| 图协同过滤 | 大规模稳定交互图 | 能利用高阶邻居 | 通常仍依赖节点 embedding |
| IGMC | 有少量交互的新用户或新物品 | 归纳式推理，关注局部结构 | 子图抽取成本高，零交互不可用 |
| 内容模型 | 文本、图片、类目特征强 | 可处理完全新物品 | 不直接利用协同结构 |
| 纯排序模型 | 点击、转化、广告排序 | 直接优化业务目标 | 需要特征工程和负采样 |

场景选择可以更直接：

| 场景 | 建议方案 |
|---|---|
| MovieLens 这类显式评分预测 | IGMC 或矩阵分解都可作为基线 |
| 商品库稳定、用户长期活跃 | 矩阵分解或图协同过滤更简单 |
| 新内容每天进入，但很快获得少量交互 | IGMC 值得尝试 |
| 新物品完全没有交互，但有标题、图片、类目 | 内容模型或多模态模型优先 |
| 目标是点击率、转化率最大化 | 排序模型优先，IGMC 可作为图特征或召回模块 |
| 线上延迟极低、候选量巨大 | 预计算 embedding 或轻量排序更现实 |

如果任务是隐式反馈，不能直接照搬原论文的评分回归设置。隐式反馈是指数据只表示行为发生过，例如点击、购买、收藏，但没有明确的 1-5 分。此时常见改法是：

| 目标 | 输出 | 损失示意 |
|---|---|---|
| 点击预测 | $\hat y=\sigma(s_{uv})$ | BCE |
| 偏好排序 | $s(u, v^+) > s(u, v^-)$ | pairwise ranking loss |
| 评分预测 | $\hat r_{uv}$ | MSE |

BCE 是二元交叉熵，用来训练点击/不点击这类二分类任务：

$$L_{BCE}=-y\log(\hat y)-(1-y)\log(1-\hat y)$$

pairwise ranking loss 关注正样本分数是否高于负样本，例如：

$$L=-\log\sigma(s(u,v^+)-s(u,v^-))$$

工程上的结论是：如果你需要快速建立一个稳定 ID 集合上的评分预测基线，矩阵分解更简单。如果你面对的是“新用户或新物品有少量交互，但不想频繁重训全量 embedding”的问题，IGMC 的归纳式子图建模更有价值。如果你的主目标是点击率或转化率排序，IGMC 通常需要改输出层、改损失，甚至只作为召回或图结构特征的一部分。
