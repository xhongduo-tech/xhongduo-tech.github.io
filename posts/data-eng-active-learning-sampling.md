## 核心结论

主动学习的核心不是“少标一点”，而是“优先标最有用的”。这里的“有用”指一个样本被标注后，能明显改变模型决策边界，或者补足当前数据覆盖的空缺。对于有限预算的数据标注任务，主动学习通常比随机采样更快把模型从“会一点”推到“可用”。

最常见的三类采样信号是：

1. 不确定性采样：挑模型最拿不准的样本。
2. 多样性采样：挑和已标数据、已选样本差异足够大的样本。
3. 预期模型改变：挑最可能让参数更新幅度大的样本。

单独使用任何一种策略都不稳。只看不确定性，容易反复选到同一片边界附近的重复样本；只看多样性，可能会选到“很不一样但其实很简单”的样本；只看梯度变化，计算成本又偏高。工程上更稳的做法是“先保覆盖，再看不确定，再做去重”。

一个适合初学者的玩具例子是：在一堆猫狗图片里，模型对一张清晰白底猫图预测为“猫”的概率是 $0.99$，再标它几乎学不到新东西；但对一张背光、被遮挡、耳朵只露一半的图片，模型输出猫 $0.48$、狗 $0.52$，这张图更值得优先标注，因为它更接近模型当前的决策边界。

衡量不确定性的常用公式是熵采样：

$$
H(y|x)=-\sum_k p(y=k|x)\log p(y=k|x)
$$

熵越高，说明模型对类别分布越分散，样本通常越值得标注。

## 问题定义与边界

主动学习要解决的问题可以表述为：

给定一个未标记样本池 $\mathcal{U}$、一个已标记集合 $\mathcal{L}$、一个总标注预算 $B$，每一轮从 $\mathcal{U}$ 中挑出一批样本送给人工标注，使最终模型性能在预算内尽量高。

这里有三个边界必须先说清。

第一，主动学习默认你已经有一个“勉强能用”的初始模型。冷启动是指一开始标注样本太少，模型连基本分布都没见过，此时“不确定”这个信号本身就不可靠。模型可能把所有样本都判得乱七八糟，你再按不确定性挑，只是在放大初始偏差。

第二，主动学习优化的是“单位标签成本的收益”，不是“绝对最优性能”。如果预算无限，随机采样最终也可能学得不错；主动学习的价值主要体现在预算紧、标签贵、迭代频繁的场景。

第三，主动学习并不自动解决标签质量问题。如果人工标签本身噪声很高，那么被精心挑选出来的“难样本”反而更容易被误标，模型可能学偏。

下面这个表格概括了几种常见策略的差别：

| 策略 | 信息量 | 代表性 | 冗余风险 | 计算成本 | 适合阶段 |
|---|---:|---:|---:|---:|---|
| 随机采样 | 低 | 中 | 中 | 低 | 冷启动、基线 |
| 熵采样 | 高 | 低到中 | 高 | 低 | 中期边界修正 |
| 多样化熵采样 | 高 | 高 | 低到中 | 中 | 大多数工程场景 |
| 预期模型改变 | 很高 | 中 | 中 | 高 | 高价值小批量标注 |

一个具体预算例子：如果你只能标注 100 条日志，而未标池有 10 万条，问题就不是“要不要标”，而是“先标哪 100 条”。如果这 100 条全是重复的简单样本，预算就被浪费了；如果这 100 条既覆盖主要分布，又集中在模型犹豫区域，收益通常更高。

## 核心机制与推导

### 1. 不确定性：样本让模型“犹豫”的程度

不确定性是最直观的信号。白话讲，就是模型自己都拿不准，这类样本通常更值得人来告诉它正确答案。

常见指标有三种：

1. 最小置信度：看最大预测概率 $p(\hat y|x)$ 是否足够低。
2. 间隔采样：看前两名类别概率差是否足够小。
3. 熵采样：看整个概率分布是否足够分散。

间隔采样公式：

$$
\text{Margin}(x)=p(\hat y_1|x)-p(\hat y_2|x)
$$

差值越小，说明第一名和第二名打得越接近，样本越难判。

玩具例子：

- 样本 A：$p=[0.4,0.35,0.25]$
- 样本 B：$p=[0.9,0.05,0.05]$

A 的熵约为：

$$
H_A \approx -(0.4\ln0.4+0.35\ln0.35+0.25\ln0.25)\approx 1.08
$$

B 的熵约为：

$$
H_B \approx -(0.9\ln0.9+0.05\ln0.05+0.05\ln0.05)\approx 0.39
$$

A 的熵更高，说明模型更不确定，通常更值得标注。A 的前两类间隔也只有 $0.05$，而 B 的间隔是 $0.85$，结论一致。

### 2. 多样性：避免把预算浪费在重复样本上

多样性指样本之间不要太像。白话讲，就是别连续挑十张几乎一样的图、十条语义几乎一样的工单、十段结构几乎一样的对话。

常见做法有：

1. 在嵌入空间里算距离，优先选离已标集较远的样本。
2. 先聚类，再从每个簇里挑高不确定样本。
3. 对候选集做贪心去重，保证 batch 内彼此差异足够大。

如果只做熵采样，模型可能一直把同一种边界附近样本排在前面。这样虽然“不确定”，但信息高度重复。多样性机制的作用，是让每一个标签都尽量覆盖新的区域。

可以把联合目标粗略写成：

$$
\text{Score}(x)=\lambda U(x) + (1-\lambda)D(x)
$$

其中 $U(x)$ 是不确定性分数，$D(x)$ 是多样性分数，$\lambda$ 控制两者权重。

### 3. Query-by-Committee：用“多个模型的分歧”衡量信息量

Committee 可以理解为“模型委员会”，即用多个模型、多个随机初始化，或同一模型不同 dropout 采样，对同一样本进行投票。

如果多个成员意见分歧很大，说明这个样本位于模型认知不稳定的区域，标注收益可能更高。

委员会投票熵常写为：

$$
H_c(x)=-\sum_j \bar p_j \log \bar p_j
$$

其中 $\bar p_j$ 是委员会对第 $j$ 类的平均投票比例。它和普通熵的区别是：普通熵看单个模型自己的不确定，委员会熵看一组模型之间是否分裂。前者反映“我自己没把握”，后者反映“我们内部没共识”。

这在小样本、高偏差任务里很有价值，因为单模型可能过拟合，多个视角的分歧更能暴露真正的边界样本。

### 4. 预期模型改变：直接估计“标了以后模型会动多少”

这类方法关心的不是预测分布本身，而是如果样本被标注并参与训练，模型参数会不会明显更新。白话讲，就是挑“最能让模型长记性”的样本。

一个常见近似是预期梯度长度：

$$
\mathbb{E}_{y\sim p(y|x)}\|\nabla_\theta \ell(f_\theta(x), y)\|
$$

如果这个值很大，表示不同可能标签下，这个样本都会推动参数发生较大变化，它就更值得查询。

它的优点是更贴近“训练收益”；缺点是比熵和间隔更贵，因为要显式算梯度或近似梯度。工程上常用于小批量高价值标注，而不是超大池子的全量排序。

## 代码实现

下面给一个可运行的 Python 玩具实现。它不依赖深度学习框架，目标是把机制说清楚：先计算熵，再用向量距离做多样性筛选，最后组成查询 batch。

```python
import math

def compute_entropy(probs):
    total = sum(probs)
    assert abs(total - 1.0) < 1e-8, "probabilities must sum to 1"
    return -sum(p * math.log(p) for p in probs if p > 0)

def l2_distance(a, b):
    assert len(a) == len(b)
    return math.sqrt(sum((x - y) ** 2 for x, y in zip(a, b)))

def min_distance_to_set(vec, selected_vecs):
    if not selected_vecs:
        return float("inf")
    return min(l2_distance(vec, s) for s in selected_vecs)

def select_diverse_samples(candidates, batch_size, alpha=0.7):
    """
    candidates: list of dict with keys:
      - id
      - probs
      - embedding
    alpha: uncertainty weight
    """
    chosen = []
    chosen_embeddings = []

    # 先缓存不确定性分数
    for item in candidates:
        item["entropy"] = compute_entropy(item["probs"])

    # 归一化熵，便于和多样性合并
    max_entropy = max(item["entropy"] for item in candidates) or 1.0

    while len(chosen) < batch_size:
        best_item = None
        best_score = -1.0

        for item in candidates:
            if item["id"] in {x["id"] for x in chosen}:
                continue

            entropy_score = item["entropy"] / max_entropy
            diversity_score = min_distance_to_set(item["embedding"], chosen_embeddings)
            if diversity_score == float("inf"):
                diversity_score = 1.0

            score = alpha * entropy_score + (1 - alpha) * diversity_score
            if score > best_score:
                best_score = score
                best_item = item

        chosen.append(best_item)
        chosen_embeddings.append(best_item["embedding"])

    return chosen

def query_batch(unlabeled_pool, batch_size):
    return select_diverse_samples(unlabeled_pool, batch_size=batch_size, alpha=0.7)

# 玩具样本
pool = [
    {"id": "a", "probs": [0.4, 0.35, 0.25], "embedding": [0.0, 0.1]},
    {"id": "b", "probs": [0.9, 0.05, 0.05], "embedding": [0.1, 0.1]},
    {"id": "c", "probs": [0.34, 0.33, 0.33], "embedding": [2.0, 2.0]},
]

assert compute_entropy([0.4, 0.35, 0.25]) > compute_entropy([0.9, 0.05, 0.05])

selected = query_batch(pool, batch_size=2)
selected_ids = {x["id"] for x in selected}

# 高熵样本 a 和 c 应该优先出现，且 c 由于更远，能提供多样性
assert "c" in selected_ids
assert len(selected_ids) == 2
```

真实工程里通常会把这段逻辑拆成四步：

1. 初始代表采样：先随机或聚类中心采一小批，解决冷启动。
2. 模型打分：对未标池跑预测，算熵、间隔、委员会分歧或梯度代理。
3. 批内去重：从 top-N 高分候选中按向量距离做多样化选择。
4. 标注后重训：把新标签加入训练集，再进入下一轮。

如果用 PyTorch 风格伪码，主循环大致如下：

```python
for round_id in range(num_rounds):
    if round_id == 0:
        batch = representative_init_sample(unlabeled_pool, k=init_k)
    else:
        probs = model_predict_proba(unlabeled_pool)
        entropy_scores = compute_entropy_scores(probs)
        top_candidates = topk_by_score(unlabeled_pool, entropy_scores, k=5 * batch_k)
        batch = select_diverse_samples(top_candidates, batch_size=batch_k)

    labels = human_annotate(batch)
    labeled_pool.extend(zip(batch, labels))
    unlabeled_pool = remove_selected(unlabeled_pool, batch)

    rebalance_if_needed(labeled_pool)
    train_model(model, labeled_pool)
```

这里的 `rebalance_if_needed` 很重要。因为主动学习会主动追边界，训练集容易被“难样本”占满，导致类别分布偏离线上真实流量。工程上常见处理是保留一部分随机样本，或者对训练损失做重加权。

## 工程权衡与常见坑

### 1. 冷启动偏差

这是最常见的问题。模型还没见过主要类别时，不确定性分数没有参考价值。比如第一次只标了“退款工单”，模型就会把“物流延迟”“发票申请”“账号冻结”全都视作异常区域，后续采样会持续失真。

缓解方法有三种：

| 问题 | 现象 | 常见缓解 |
|---|---|---|
| 冷启动覆盖不足 | 主动学习前几轮不如随机 | 先做随机/聚类初始化 |
| 候选重复过高 | batch 中大量相似样本 | 在 embedding 空间去重 |
| 后期收益递减 | 标很多但提升很慢 | 提高随机混入比例，切换策略 |

HaCon 一类工作给出的启发是：先用对比学习或更稳的表示学习拿到一个相对可靠的特征空间，再做主动查询，冷启动会明显好一些。这里“对比学习”可以简单理解为让相似样本靠近、不相似样本分开，从而让距离更有语义。

### 2. 冗余样本会吞掉预算

在文本分类、日志归因、工单路由这类任务里，未标池经常有大量模板化重复。只做熵采样，会把很多语义几乎一样、只是字面略有差异的样本一起挑出来。模型从这些样本学到的信息高度重叠。

一个真实工程例子是 LLM 微调数据选择。比如做客服对话 SFT，检索系统可能召回很多“几乎同一种问法”的样本。SIFT 一类方法的价值就在于：不是把相关样本全塞给模型，而是在相关性基础上进一步用主动学习思想挑那些最有区分度、最能改善当前响应行为的样本。前期提升通常明显，因为模型很快补齐关键模式；后期则会进入收益递减，因为剩下的多是边缘场景或噪声样本。

### 3. 标签成本和模型性能不是线性关系

主动学习常见的收益曲线通常是前期陡升、后期变平。可以抽象成下面的趋势：

| 已标注比例 | 随机采样效果 | 主动学习效果 | 说明 |
|---|---:|---:|---|
| 5% | 低 | 明显更高 | 早期最能体现数据效率 |
| 20% | 中 | 高 | 仍有优势，但差距缩小 |
| 50% | 较高 | 略高 | 边际收益开始下降 |
| 80% | 很高 | 接近 | 接近全监督上限 |

这意味着主动学习不是“全程都碾压随机”，而是“在预算受限阶段最有价值”。如果已经标了很大比例，再继续用复杂采样器，计算成本可能不划算。

### 4. 难样本不一定是好样本

不确定样本有时只是噪声样本。比如 OCR 模糊截图、语音转写错误文本、标签本身定义模糊的边界案例。这类样本会把熵拉高，但它们未必帮助模型学到稳定规律。

所以工程上常加入以下约束：

1. 最低质量过滤：先过滤掉解析失败、文本过短、图像损坏的样本。
2. 随机混采：例如每轮 70% 主动采样，30% 随机采样。
3. 人工复审难样本：高熵样本优先进入更严格的标注流程。

## 替代方案与适用边界

主动学习不是唯一方案，也不是任何场景都该优先上。

| 策略 | 计算要求 | 初始依赖 | 噪音敏感度 | 适用场景 |
|---|---:|---:|---:|---|
| 完全随机采样 | 低 | 低 | 低到中 | 冷启动、快速基线 |
| 主动学习 | 中到高 | 中到高 | 中到高 | 标签贵、可迭代训练 |
| 密度加权采样 | 中 | 中 | 中 | 关注代表性时 |
| 强化学习式采样 | 高 | 高 | 高 | 复杂长期策略优化 |

### 1. 完全随机采样

优点是稳、便宜、无偏，缺点是浪费预算。它特别适合作为冷启动阶段的保底方案，也适合作为主动学习的混合成分。很多成熟系统都不会纯用主动策略，而是保留 20% 到 50% 的随机份额，防止采样器越学越偏。

### 2. 密度加权或聚类采样

这类方法更关注“代表主流分布”。当你的数据里有大量长尾噪声，或者模型还很弱时，只追不确定性容易被带偏，此时按簇中心、局部密度来采样更稳。

### 3. 强化学习式采样

它把“如何选样”本身看成一个策略学习问题，理论上更灵活，但工程复杂度和调参成本都高。对于多数中小团队，收益未必覆盖实现成本。

### 4. 不适合主动学习的情况

以下情况要谨慎：

1. 数据漂移非常剧烈。比如线上流量每天都换主题，今天学到的边界明天就失效。
2. 预算极小。只有几十个标签时，复杂评分器容易比随机更不稳。
3. 标签噪音很高。模型越爱挑难样本，标注员越容易产生分歧。
4. 样本中 95% 是无意义背景。此时高不确定可能只是噪声放大，混入随机采样更稳。

一个直观例子是监控视频筛选。如果绝大多数帧只是空场景，主动学习可能不断挑“看不清的人影”“压缩模糊块”这类高熵噪声，而不是对业务真正重要的行为片段。此时纯主动策略并不理想，应该回退到“随机 + 规则过滤 + 少量主动”的混合方案。
