## 核心结论

位置偏差是指：点击率不只由物品相关性决定，还被曝光位置影响，导致“靠前更容易被点”混入训练信号。

在排序模型里，用户点击通常不是一个纯粹的相关性标签。一个商品被点，至少需要两件事同时发生：用户先看见它，然后觉得它值得点。靠前位置更容易被看见，所以同样相关的两个商品，放在第 1 位和第 3 位时，点击率可能明显不同。新手版解释就是：用户更容易先看到前面的东西，所以前面的东西天然更容易被点，不代表它一定更好。

可以用一个最小因果图表示：

```text
x: 用户/物品特征  ───────►  r: 看见后的相关性
                         │
p: 展示位置 ───► e: 是否看见 ───► c: 是否点击
```

本文使用以下记号：

$$
P(c=1 \mid x,p)=P(e=1 \mid p)\cdot P(c=1 \mid x,e=1)
$$

其中，$x$ 表示用户和内容特征，$p$ 表示位置，$e$ 表示是否看见，$c$ 表示是否点击。这个公式的含义是：某个样本被点击的概率，等于“在这个位置被看见的概率”乘以“看见后愿意点击的概率”。

处理位置偏差的目标，不是彻底消灭位置，而是把“位置带来的点击增益”和“内容带来的真实相关性”拆开。否则模型会形成反馈回路：历史上排在前面的内容点击更多，模型学到它们更好，于是继续把它们排在前面。

常用方法有四类：

| 方法 | 核心思想 | 解决什么问题 |
|---|---|---|
| 位置作为特征 | 直接把 `p` 输入模型 | 让模型知道点击和位置有关 |
| Shallow Tower | 用单独浅层模块建模位置偏置 | 把内容分数和位置偏置拆开 |
| IPW | 用 $1/\pi_p$ 给样本重新加权 | 抵消高位更容易被看见的偏差 |
| PAL | 显式建模曝光概率和看见后点击概率 | 把点击分解成曝光与相关性两部分 |

## 问题定义与边界

先定义变量。术语“曝光倾向”指某个位置上的内容被用户看见的概率，不是系统展示了就一定等于用户看见了。

| 变量 | 含义 | 白话解释 |
|---|---|---|
| `x` | 用户、物品、上下文特征 | 用户是谁、商品是什么、当前场景是什么 |
| `p` | 展示位置 | 第 1 位、第 2 位、第 3 位等 |
| `e` | 是否看见 | 用户是否真的注意到这个位置 |
| `c` | 是否点击 | 用户是否发生点击行为 |
| `π_p` | $P(e=1\mid p)$ | 位置 `p` 被看见的概率 |

本文必须区分“点击”和“可见后点击”。点击是日志里能直接看到的行为，通常记为 `c=1`。可见后点击是用户已经看见内容以后，基于内容相关性产生点击，通常不能直接观测。位置偏差的核心问题就在这里：日志里的点击把“是否看见”和“是否喜欢”混在一起了。

玩具例子：两个完全一样的商品，一个放在第 1 位，一个放在第 3 位。用户先看到上面的商品，所以上面的点击更多。这不是商品更好，而是被看到的机会更多。

真实工程例子：电商搜索页或信息流推荐页里，顶部坑位天然获得更多点击。训练日志显示顶部商品点击率高，但线上新一轮排序时，候选商品还没有被放到最终位置。如果模型直接把历史点击当作真实相关性，就会高估过去占据好位置的商品或内容类型。

本文讨论的是 CTR / 排序场景中的位置偏差。CTR 是 click-through rate，中文通常叫点击率，表示曝光后发生点击的比例。本文不展开所有类型的曝光偏差、选择偏差、审美偏差，只聚焦“位置导致的可见性差异”。

| 适用场景 | 不适用或不充分的场景 |
|---|---|
| 搜索排序、推荐流排序、广告 CTR 排序 | 只研究 UI 审美、文案风格、品牌偏好 |
| 有位置日志、点击日志、曝光日志 | 没有记录位置，也无法恢复展示顺序 |
| 需要从有偏反馈中学习相关性 | 完全随机实验数据已经足够充分 |
| 用户按列表顺序浏览概率不同 | 所有位置被等概率看见的理想场景 |

如果推荐流里做了随机打散曝光，位置偏差会减弱，但仍可能存在。因为用户的视觉路径、屏幕首屏范围、加载时机、卡片大小都会影响“是否看见”。

## 核心机制与推导

排序训练里最直接的写法是：

$$
\hat{y}=f_\theta(x,p)
$$

这里 $\hat{y}$ 是模型预测点击概率，$f_\theta$ 是带参数的模型。术语“参数”指模型从数据里学出来的数字，例如神经网络权重。这个做法把位置当作普通特征输入模型，优点是简单，缺点是模型可能把“靠前更容易点”当成内容本身的能力。

更清晰的做法是 Shallow Tower。Tower 是模型里的一个子网络，Shallow Tower 指结构较浅、专门建模某类简单因素的子网络。它把内容分数和位置偏置拆成两部分：

$$
z=z_{main}(x)+z_{pos}(p)
$$

$$
\hat{y}=\sigma(z)
$$

其中 $z_{main}(x)$ 表示内容主塔输出，$z_{pos}(p)$ 表示位置偏置项，$\sigma$ 是 sigmoid 函数，用来把分数压到 0 到 1 之间。这个结构的好处是可解释：点击高到底来自内容强，还是位置好，可以分别查看。

但位置作为特征和 Shallow Tower 本质上仍然是在预测“带位置影响的点击”。如果目标是学习“内容本身的相关性”，就需要进一步拆解点击过程：

$$
P(c=1\mid x,p)=P(e=1\mid p)\cdot P(c=1\mid x,e=1)
$$

这里 $\pi_p=P(e=1\mid p)$ 是位置 `p` 的曝光倾向。假设第 1 位被看见概率是 0.8，第 3 位被看见概率是 0.2，那么第 1 位点击多，不一定是相关性高，也可能只是 $\pi_p$ 大。

IPW 是 inverse propensity weighting，中文常译为逆倾向加权。它的白话解释是：越不容易被看见的位置，一旦产生样本，就给它更大的训练权重；越容易被看见的位置，权重相对小一些。训练损失可以写成：

$$
L_{IPW}=\sum_i w_i\cdot \ell(\hat{y_i},c_i)
$$

$$
w_i=\frac{1}{\pi_{p_i}}
$$

其中 $\ell$ 是单条样本的损失函数。损失函数是衡量模型预测错了多少的函数，错得越多，损失越大。

数值例子如下。假设两个物品真实相关性相同，但展示位置不同：

| 物品 | 位置 | $\pi_p$ | 100 次曝光点击数 | 原始 CTR | 按 $\pi_p$ 修正 |
|---|---:|---:|---:|---:|---:|
| A | 1 | 0.8 | 40 | 0.40 | 0.40 / 0.8 = 0.50 |
| B | 3 | 0.2 | 10 | 0.10 | 0.10 / 0.2 = 0.50 |

直接看原始 CTR，会认为 A 比 B 好。修正后两者都是 0.50，说明差异主要来自位置，而不是内容。新手版理解就是：不是 A 比 B 好，而是 A 更容易被看到。

几类方法的机制差别可以压缩成一张表：

| 路线 | 训练目标 | 是否显式输入位置 | 是否拆解位置偏差 |
|---|---|---:|---:|
| 位置特征 | 预测历史点击 | 是 | 否 |
| Shallow Tower | 预测历史点击，并单独建模位置项 | 是 | 部分拆解 |
| IPW | 加权后学习相关性 | 训练时使用 | 是 |
| PAL | 同时建模曝光概率和看见后点击 | 训练时使用 | 是 |

PAL 是 position-bias aware learning，中文可理解为位置偏差感知学习。它通常显式学习两个部分：位置导致的曝光概率，以及内容导致的看见后点击概率。相比单纯 IPW，PAL 更像是在模型结构里写入“点击由曝光和相关性共同产生”这个假设。

## 代码实现

实现上可以分成两条路线：第一条是模型结构里使用位置，例如位置特征或 Shallow Tower；第二条是训练时修正，例如 IPW 或 PAL。

先看一个可运行的玩具代码。它展示如何读取 $\pi_p$、计算权重、裁剪权重，并用加权 BCE 训练损失。BCE 是 binary cross entropy，中文叫二分类交叉熵，用来训练点击/不点击这种二分类任务。

```python
import math

def sigmoid(z):
    return 1.0 / (1.0 + math.exp(-z))

def bce_with_logits(logit, label):
    # 数值稳定版二分类交叉熵
    if logit >= 0:
        return logit - logit * label + math.log1p(math.exp(-logit))
    return -logit * label + math.log1p(math.exp(logit))

def clip(x, lo, hi):
    return max(lo, min(hi, x))

# load pi_p: 位置被看见的概率，通常来自随机实验、眼动/停留估计或日志估计
pi = {
    1: 0.8,
    2: 0.5,
    3: 0.2,
}

samples = [
    {"x_score": 1.2, "position": 1, "click": 1},
    {"x_score": 1.1, "position": 3, "click": 1},
    {"x_score": -0.3, "position": 1, "click": 0},
]

weighted_losses = []
weights = []

for s in samples:
    p = s["position"]
    click = s["click"]

    # main_tower(x)：这里用 x_score 模拟内容塔输出
    logit = s["x_score"]

    # IPW: w = clip(1 / pi_p, ...)
    w = clip(1.0 / pi[p], 0.1, 5.0)

    loss = w * bce_with_logits(logit, click)
    weighted_losses.append(loss)
    weights.append(w)

final_loss = sum(weighted_losses) / len(weighted_losses)

assert weights == [1.25, 5.0, 1.25]
assert final_loss > 0
assert round(0.40 / 0.8, 2) == round(0.10 / 0.2, 2)
```

新手版理解：模型不是直接相信历史点击，而是先把每条样本按位置重新计权。靠后位置更难被看见，所以那里的点击样本更“稀缺”，训练时权重更高。

Shallow Tower 的伪代码如下：

```python
def model(x, position):
    z_content = main_tower(x)       # 内容、用户、上下文特征
    z_position = pos_tower(position)  # 位置偏置
    logit = z_content + z_position
    return sigmoid(logit)
```

PAL 的训练结构可以写成：

```python
def pal_model(x, position):
    relevance = relevance_tower(x)      # P(c=1 | x, e=1)
    examination = position_tower(position)  # P(e=1 | p)
    ctr = relevance * examination
    return ctr, relevance, examination
```

训练和推理要分清楚。训练时可以使用历史位置，因为日志里知道某个样本展示在第几位。线上推理时，如果模型要给候选集排序，候选内容尚未被最终放到某个确定位置。这时直接依赖历史 position 特征，容易造成训练-推理不一致。

| 内容 | 训练输入 | 线上输入 | 是否必须线上可用 |
|---|---|---|---:|
| 用户/物品特征 `x` | 是 | 是 | 是 |
| 历史展示位置 `p` | 是 | 通常否 | 否 |
| 位置倾向 `π_p` | 是 | 通常否 | 否 |
| 内容塔分数 | 是 | 是 | 是 |
| 位置偏置塔 | 可选 | 只在重排阶段可用 | 视架构而定 |

真实工程例子：信息流 CTR 排序里，离线日志很丰富，但线上主排序阶段只知道候选内容和用户上下文，不知道最终第几位。常见做法是离线估计 $\pi_p$，训练时用 IPW 或 PAL 纠偏；线上主排序只服务内容塔分数。如果在精排后的重排阶段已经确定候选坑位，也可以使用独立的位置偏置模块，但要明确它服务的是最终展示校准，而不是内容相关性本身。

## 工程权衡与常见坑

IPW 的核心问题是方差大。方差是估计结果波动的程度，方差大表示模型训练容易不稳定。当 $\pi_p$ 很小时，$1/\pi_p$ 会非常大。比如某个位置被看见概率只有 0.01，那么权重就是 100，一条样本就可能主导一批训练。

常见控制手段有三类。第一是 clip，把权重限制在固定范围内，例如 $[0.1, 10]$。第二是截断，只使用位置倾向不太小的样本。第三是 self-normalized IPS，把权重除以本 batch 或全局权重均值，让整体尺度更稳定：

$$
w_i^{SNIPS}=\frac{1/\pi_{p_i}}{\frac{1}{n}\sum_{j=1}^{n}1/\pi_{p_j}}
$$

PAL 也不是天然正确。它依赖一个关键假设：看见以后，点击主要由内容相关性决定，和位置近似独立。但真实场景里，这个假设可能不成立。例如首位商品可能被用户当成“平台最推荐”，从而产生信任加成；底部商品可能只有强意图用户才会看到，从而点击意愿不同。此时位置不仅影响是否看见，还影响看见后的心理判断。

线上常见失败案例：训练时把 position 直接喂进主塔，离线 AUC 很好。上线时主排序阶段拿不到同样的位置特征，只能填默认值或使用候选召回顺序的位置，结果效果回落。新手版表达就是：训练时看到了答案，线上没答案了。

| 坑位问题 | 后果 | 规避策略 |
|---|---|---|
| 原始点击直接训练 | 模型学习到“高位内容更好”的假规律 | 随机曝光、IPW、PAL、分位置评估 |
| $1/\pi_p$ 过大 | loss 抖动，少量样本支配训练 | clip、截断、SNIPS、增大样本量 |
| 训练/推理特征不一致 | 离线好、线上差 | 主排序避免依赖不可用 position |
| 只看整体 AUC | 掩盖不同位置的校准问题 | 按位置分桶看 AUC、CTR、calibration |
| PAL 假设失效 | 曝光与相关性拆分错误 | 检查交互项，扩大偏置模型，做随机实验 |
| $\pi_p$ 估计不准 | 纠偏方向错误 | 用随机流量、稳定窗口、分场景估计 |

工程上还要注意，位置偏差通常不是全局常数。不同页面、设备、卡片样式、用户群体都可能有不同的 $\pi_p$。移动端首屏第 3 位和桌面端第 3 位不是同一个概念。搜索页第 1 位广告和自然结果第 1 位也不能简单共用一个位置倾向。

评估时不要只看整体指标。至少需要按位置分桶看校准。校准是指模型预测的概率和真实发生比例是否一致。比如模型预测某组样本 CTR 是 0.2，真实点击率也接近 0.2，就说明校准较好。位置纠偏后，如果整体 AUC 上升但低位样本校准变差，线上排序仍可能不稳定。

## 替代方案与适用边界

没有单一最优解。选择方案取决于四个条件：是否能控制曝光，是否有足够日志，是否允许在线随机，是否能接受训练方差上升。

| 方法 | 优点 | 缺点 | 适用条件 | 线上是否可用 position 特征 |
|---|---|---|---|---:|
| 位置作为特征 | 实现最简单，离线指标常有提升 | 容易学到历史位置捷径 | 只想预测当前展示 CTR，不强调无偏相关性 | 需要谨慎 |
| Shallow Tower | 内容分数和位置偏置可拆开 | 仍依赖偏置建模质量 | 有明确位置模块，且服务链路能区分主塔和偏置塔 | 部分可用 |
| IPW | 原理清晰，能修正有偏日志 | 方差大，依赖 $\pi_p$ 准确性 | 有可靠位置倾向估计，样本量足够 | 训练用即可 |
| PAL | 结构上拆解曝光与相关性 | 假设更强，实现更复杂 | CTR 系统成熟，有能力维护多塔结构 | 训练用，部分模块可服务 |
| 随机化曝光 | 数据更干净，因果解释更强 | 影响线上体验和收入 | 可承受小流量实验或探索流量 | 不依赖 |

一个决策表可以这样用：

| 场景条件 | 更合适的选择 |
|---|---|
| 能做少量随机曝光 | 先估计可靠 $\pi_p$，再用 IPW / PAL |
| 不能随机，但日志量很大 | 用稳定窗口估计 $\pi_p$，配合 clip 后的 IPW |
| 只需要预测某个固定坑位的点击 | 位置特征或 Shallow Tower 可以接受 |
| 主排序线上拿不到最终位置 | 不要把 position 放进主塔，优先训练时纠偏 |
| 位置与内容强交互 | PAL 加交互项，或按场景拆分建模 |

真实工程对比：信息流 CTR 排序中，离线日志丰富，但线上不能大规模乱改排序结构，因为会影响用户体验和收入。这种情况下，更适合先用历史日志估计 $\pi_p$，再使用 IPW 或 PAL 做训练时修正。如果平台允许拿 1% 流量做轻度随机曝光，那么随机化数据往往更干净，可以作为位置倾向估计和离线评估的基础。

位置作为特征不是完全错误。它适合的问题是“预测这个内容在这个位置上的点击率”，而不是“估计内容脱离位置后的真实相关性”。广告系统里的出价、预算、坑位校准，有时确实需要知道位置带来的点击变化。但如果模型目标是学习内容排序能力，就必须避免位置捷径污染主排序分数。

Shallow Tower 的适用边界也类似。它比直接拼接 position 更可控，因为工程上可以选择线上是否使用位置塔，也可以单独监控 $z_{pos}(p)$。但它仍然需要清楚服务阶段：主排序阶段通常不应依赖最终位置；重排或展示校准阶段可以使用位置模块。

IPW 更偏统计修正，优点是概念直接，缺点是对 $\pi_p$ 敏感。PAL 更偏模型结构，优点是把业务假设写进模型，缺点是维护成本更高。工程上常见做法不是二选一，而是先用随机流量估计位置倾向，再用 IPW 做 baseline，最后在成熟系统里尝试 PAL 或更细粒度的偏置建模。
