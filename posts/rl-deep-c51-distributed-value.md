## 核心结论

C51 是一种分布式价值函数方法。分布式价值函数的意思是：它不只预测“平均能拿多少回报”，而是预测“可能拿到哪些回报，以及每个回报出现的概率”。

普通 DQN 学的是标量 $Q(s,a)$，本质上是回报随机变量的期望。C51 学的是回报分布 $Z(s,a)$，并把这个分布限制在固定区间 $[V_{\min}, V_{\max}]$ 上的 $N$ 个离散支撑点上，通常 $N=51$，因此叫 C51。

支撑点间距为：

$$
\Delta z=\frac{V_{\max}-V_{\min}}{N-1}
$$

每个支撑点 $z_i$ 对应一个概率 $p_i(s,a)$，网络输出的是这组概率，而不是单个 Q 值。动作的期望价值仍然可以从分布里算出来：

$$
Q(s,a)=\sum_{i=0}^{N-1} z_i p_i(s,a)
$$

这意味着 C51 既保留了期望，也保留了尾部风险信息。尾部风险的意思是：虽然概率不高，但一旦发生就会造成很大损失或收益的结果。

一个最直观的对比是：

- 传统 DQN：动作 A 的价值是 `0.53`
- C51：动作 A 的回报可能是 `{0.2, 0.5, 0.8}`，对应概率是 `{0.1, 0.7, 0.2}`

两者的期望都能算出来，但第二种表示能告诉你：这个动作并不是“稳定地拿到 0.53”，而是“大多数时候 0.5，偶尔 0.8，也可能只有 0.2”。

下表是期望值表示和分布表示的核心差异：

| 维度 | 普通 DQN | C51 |
|---|---|---|
| 学习目标 | 单个期望 $Q(s,a)$ | 离散回报分布 $Z(s,a)$ |
| 输出形式 | 每个动作 1 个标量 | 每个动作 $N$ 个原子概率 |
| 能否表达风险 | 不能直接表达 | 可以看到高风险低概率尾部 |
| Bellman 目标 | 标量 TD 目标 | 分布 Bellman 目标再投影 |
| 常用损失 | MSE / Huber | 交叉熵 |

C51 的训练目标不是回归标量，而是最小化当前输出分布与目标分布之间的交叉熵：

$$
\mathcal{L}=-\sum_i (\Phi \hat{T} Z)_i\log p_i
$$

其中 $(\Phi \hat{T} Z)_i$ 是 Bellman 更新后再投影到固定支撑上的目标分布，$p_i$ 是当前网络输出的概率。

结论可以压缩成一句话：C51 把动作价值从“一个平均数”升级成“一个离散概率分布”，因此在奖励波动大、不确定性强、回报多峰的任务里，往往比普通 DQN 更稳。

## 问题定义与边界

强化学习里的动作价值函数回答的是一个问题：在状态 $s$ 下执行动作 $a$，未来累计回报大概是多少。累计回报通常写成：

$$
G_t=\sum_{k=0}^{\infty}\gamma^k R_{t+k+1}
$$

其中 $\gamma$ 是折扣因子，表示“未来奖励的重要程度会逐步衰减”。

普通 Q-learning 或 DQN 只学 $\mathbb{E}[G_t\mid s,a]$，也就是平均值。这种表示足够简单，但会丢掉两个重要信息：

1. 回报是否稳定
2. 回报是否存在长尾风险

如果两个动作期望相同，但一个动作回报集中，另一个动作回报波动很大，标量 Q 无法区分它们。

C51 的边界也很明确。它不是在连续实数轴上拟合任意分布，而是在固定区间 $[V_{\min},V_{\max}]$ 上做离散分类。因此必须预设三个超参数：

| 参数 | 含义 | 影响 |
|---|---|---|
| $V_{\min}$ | 支撑下界 | 太大会把低回报截断 |
| $V_{\max}$ | 支撑上界 | 太小会把高回报截断 |
| $N$ | 原子数 | 太小分辨率差，太大计算更重 |

支撑点定义为：

$$
z_i = V_{\min} + i\Delta z,\quad i=0,\dots,N-1
$$

这意味着 C51 的表示能力受固定区间约束。比如某个游戏里单步奖励和累计回报大致落在 $[-3,4]$，你可以先选 $V_{\min}=-5$、$V_{\max}=5$、$N=51$。这样大部分目标回报都能落在区间内。

但如果真实目标回报偶尔达到 `7`，由于 C51 的支撑上界只有 `5`，那么它会被裁剪到 `5`，也就是信息被截断。裁剪的意思是：真实值超出边界后，不再保留超出部分，而是压到边界点。

这带来一个工程事实：`Vmin/Vmax 不是装饰参数，而是模型表达能力的一部分`。

再看一个新手容易忽视的例子：

- 环境真实回报主要在 `[-1,1]`
- 你直接沿用教程默认 `[-10,10]`
- $N=51$ 不变

此时每个原子间距会变大，很多原子长期接近零概率，模型虽然看起来“有 51 个原子”，但真正有效分辨率反而不足。因为大部分概率质量只在中间很小一段区域变化。

下面这个表可以直接说明区间和原子数的关系：

| 奖励区间设置 | 原子数 $N$ | 单个间距 $\Delta z$ | 结果 |
|---|---:|---:|---|
| $[-2,2]$ | 51 | 0.08 | 分辨率较细，适合小尺度奖励 |
| $[-10,10]$ | 51 | 0.40 | 分辨率变粗，细节损失 |
| $[-2,2]$ | 21 | 0.20 | 训练快一些，但分布更粗糙 |
| $[-2,2]$ | 101 | 0.04 | 更细，但显存和计算上升 |

所以问题定义可以写得很精确：C51 解决的是“在固定离散支撑上学习动作回报分布”的问题，不是“无约束地拟合任意真实分布”的问题。

## 核心机制与推导

C51 的核心是两步：

1. 用固定支撑点表示分布
2. 对 Bellman 更新后的目标分布做投影

先看表示。对于每个动作，网络输出在 $N$ 个支撑点上的概率：

$$
Z(s,a)=\sum_{i=0}^{N-1} p_i(s,a)\delta_{z_i}
$$

这里的 $\delta_{z_i}$ 可以理解为“概率质量放在点 $z_i$ 上”。也就是回报分布不是连续曲线，而是一组离散原子。

### Bellman 更新为什么要投影

如果下一状态的分布定义在支撑点 $\{z_j\}$ 上，那么 Bellman 更新后，每个点会被移动到：

$$
\hat{T}z_j = r + \gamma z_j
$$

问题在于，$\hat{T}z_j$ 通常不会刚好落在原来的固定支撑点上。比如原支撑是 `{-1, 0, 1}`，更新后可能变成 `{-0.4, 0.0, 0.8}`。这些值不再和固定原子对齐，所以必须重新投影回原支撑。

C51 使用线性插值式投影。线性插值的意思是：一个目标点如果落在两个相邻原子之间，就按距离比例把概率分给这两个原子。

公式是：

$$
(\Phi \hat{T} Z)_i
=
\sum_j
\left[
1-\frac{\left|[\hat{T} z_j]_{V_{\min}}^{V_{\max}}-z_i\right|}{\Delta z}
\right]_0^1
p_j(x',\pi(x'))
$$

其中：

- $[\cdot]_{V_{\min}}^{V_{\max}}$ 表示裁剪到区间内
- $[\cdot]_0^1$ 表示把权重限制在 $[0,1]$
- $\Phi$ 表示投影算子
- $p_j(x',\pi(x'))$ 是下一状态贪心动作对应的分布概率

这条公式的直觉并不复杂：目标点落得离哪个原子更近，就给哪个原子更多概率质量。

### 玩具例子

按题目要求，取：

- $V_{\min}=-1$
- $V_{\max}=1$
- $N=3$

则支撑点是：

$$
z=\{-1,0,1\}
$$

当前或下一步分布设为：

$$
p=\{0.2,0.5,0.3\}
$$

再设：

- 奖励 $r=0.5$
- 折扣因子 $\gamma=0.9$

则每个目标点经过 Bellman 更新：

$$
\hat{T}z = r+\gamma z = \{0.5+0.9(-1),\ 0.5+0,\ 0.5+0.9(1)\}
$$

得到：

$$
\hat{T}z=\{-0.4,0.5,1.4\}
$$

由于上界是 1，最后一个点会被裁剪成 1，所以有效目标点为：

$$
\{-0.4,0.5,1.0\}
$$

如果采用线性投影：

- `-0.4` 落在 `-1` 和 `0` 之间，离 `0` 更近，所以 `0.2` 的概率质量大部分给 `0`，少部分给 `-1`
- `0.5` 落在 `0` 和 `1` 之间，均分 `0.5` 的质量
- `1.0` 正好落在支撑点 `1` 上，`0.3` 全给 `1`

投影后的结果会接近题目给定的直观形式 `{0.2,0.6,0.2}`。严格按标准公式计算，数值会受裁剪和插值细节影响，工程里应以代码投影结果为准。对新手来说，重点不是手算到最后一位，而是理解“目标点移动后，要把质量重新分摊回固定网格”。

下面把这个过程写成表：

| 原子 $z_j$ | 原概率 $p_j$ | Bellman 后 $\hat{T}z_j$ | 裁剪后位置 | 投影直觉 |
|---|---:|---:|---:|---|
| -1 | 0.2 | -0.4 | -0.4 | 分到 -1 和 0 |
| 0 | 0.5 | 0.5 | 0.5 | 分到 0 和 1 |
| 1 | 0.3 | 1.4 | 1.0 | 全部落到 1 |

可以把它想成下面这个流程：

```text
原支撑 z_j:      -1 -------- 0 -------- 1
Bellman后:         -0.4 ----- 0.5 -------- 1.4
裁剪后:            -0.4 ----- 0.5 -------- 1.0
投影回固定支撑:   (-1,0)分摊   (0,1)分摊    全到1
```

### 为什么损失用交叉熵

因为网络输出的是离散分类概率，不是连续实数回归值。对每个动作，网络先输出 logits，再经过 softmax 变成概率分布：

$$
p_i(s,a)=\frac{\exp(l_i)}{\sum_k\exp(l_k)}
$$

其中 logits 可以理解为“还没归一化的分类分数”。

训练时，目标是让当前分布接近投影后的目标分布，所以用交叉熵：

$$
\mathcal{L}=-\sum_i (\Phi \hat{T} Z)_i\log p_i(s,a)
$$

如果当前网络把高概率放在错误原子上，损失就会变大；如果目标分布和当前分布接近，损失就会变小。

### 真实工程例子

在 Atari 这类环境里，许多动作的回报分布不是单峰的。单峰的意思是：大多数概率集中在一个回报附近。多峰的意思是：可能存在几种明显不同的结局。

例如某个动作可能出现两类后果：

- 大多数时候安全推进，回报中等
- 少数时候触发连击或立即死亡，回报极高或极低

普通 DQN 会把这些情况平均成一个数，结果是“不同风险结构但均值相同”的动作被混在一起。C51 能保留这些概率峰，因此目标网络在 bootstrapping 时传递的不只是期望值，还包括分布形状，这通常会减少训练振荡。

## 代码实现

实现 C51 时，网络最后一层不再为每个动作输出一个值，而是输出 `动作数 × 原子数` 个 logits。然后按动作维度切片，在原子维度做 softmax。

一个最小可运行的 Python 示例如下，重点演示支撑构造、分布投影和期望值计算：

```python
import numpy as np

def project_distribution(reward, gamma, next_probs, vmin, vmax, num_atoms, done=False):
    z = np.linspace(vmin, vmax, num_atoms)
    delta_z = (vmax - vmin) / (num_atoms - 1)
    target = np.zeros(num_atoms, dtype=np.float64)

    for j in range(num_atoms):
        tz_j = reward if done else reward + gamma * z[j]
        tz_j = min(vmax, max(vmin, tz_j))

        b = (tz_j - vmin) / delta_z
        l = int(np.floor(b))
        u = int(np.ceil(b))

        if l == u:
            target[l] += next_probs[j]
        else:
            target[l] += next_probs[j] * (u - b)
            target[u] += next_probs[j] * (b - l)

    return target

def expected_q(probs, vmin, vmax):
    z = np.linspace(vmin, vmax, len(probs))
    return float(np.sum(z * probs))

# toy example
vmin, vmax, n = -1.0, 1.0, 3
reward, gamma = 0.5, 0.9
next_probs = np.array([0.2, 0.5, 0.3], dtype=np.float64)

proj = project_distribution(reward, gamma, next_probs, vmin, vmax, n)
assert np.isclose(proj.sum(), 1.0)
assert np.all(proj >= -1e-12)

q = expected_q(proj, vmin, vmax)
assert isinstance(q, float)

# a slightly larger support
vmin2, vmax2, n2 = -2.0, 2.0, 51
uniform = np.ones(n2) / n2
proj2 = project_distribution(0.1, 0.99, uniform, vmin2, vmax2, n2)
assert np.isclose(proj2.sum(), 1.0, atol=1e-8)

print("projected_probs:", np.round(proj, 4))
print("expected_q:", round(q, 4))
```

训练主流程可以压缩成下面的伪代码：

```python
logits = net(state)                      # [B, A, N]
logits_a = logits[batch_index, action]  # [B, N]
probs = softmax(logits_a, dim=-1)

with no_grad():
    next_logits = target_net(next_state)         # [B, A, N]
    next_probs_all = softmax(next_logits, -1)    # [B, A, N]
    next_q = (next_probs_all * support).sum(-1)  # [B, A]
    next_action = argmax(next_q, dim=-1)         # Double DQN风格可继续细化
    next_probs = next_probs_all[batch_index, next_action]
    target_probs = project_distribution(reward, gamma, next_probs)

loss = -(target_probs * log(probs)).sum(-1).mean()
loss.backward()
optimizer.step()
```

实现步骤可以整理成表：

| 步骤 | 输入 | 输出 | 关键点 |
|---|---|---|---|
| 前向传播 | state | logits `[B,A,N]` | 每个动作对应 $N$ 个原子分数 |
| 概率化 | logits | probs | 在原子维做 softmax |
| 选下一个动作 | next state | next action | 常配合 target net，常见也会接 Double DQN |
| 分布投影 | reward, gamma, next probs | target probs | 必须投影回固定支撑 |
| 计算损失 | target probs, probs | cross-entropy | 不是 MSE 回归 |
| 参数更新 | loss | 新参数 | 与 DQN 类似，用优化器更新 |

这里有三个实现细节最容易出错。

第一，`Vmin/Vmax` 必须和奖励尺度对齐。否则投影前大量裁剪，目标分布会长期挤在边界原子，模型学到的不是环境分布，而是“被截断后的伪分布”。

第二，目标网络仍然重要。C51 解决的是价值表示问题，不会自动消除 bootstrap 带来的不稳定性，所以 target network 的同步机制仍然要保留。

第三，计算动作选择时通常还是根据期望值选贪心动作，即：

$$
a^*=\arg\max_a \sum_i z_i p_i(s,a)
$$

也就是说，C51 学的是分布，但默认控制策略通常仍基于期望值。

## 工程权衡与常见坑

C51 的优点不是“无条件优于 DQN”，而是在表达能力、训练开销、参数敏感性之间做了新的折中。

先看常见问题表：

| 问题 | 原因 | 应对 |
|---|---|---|
| 目标分布总贴边界 | $V_{\min},V_{\max}$ 与真实回报不匹配 | 先统计回报范围，再重设区间 |
| 分布过粗看不出差异 | 原子数 $N$ 太小 | 从 51 开始，必要时增到 101 |
| 训练更慢 | 每个动作输出 $N$ 个 logits | 减少动作头宽度，控制 batch size |
| 学到的分布很尖或很塌 | softmax 饱和、学习率过大 | 调低学习率，检查 logits 数值 |
| 策略抖动 | target network 更新不稳 | 固定同步周期或软更新 |
| 手算例子对不上代码 | 投影含裁剪与线性插值 | 以代码结果为准，不靠口算猜测 |

一个典型工程坑是奖励区间选错。

真实部署里，经常有人直接照搬默认 `[-10,10]`。如果你的环境奖励主要在 `[-1,1]`，那么 51 个原子虽然还在，但有效分辨率会被浪费在大量不可能到达的区域上。很多原子长期接近零概率，模型只能在很窄的一段区间里挤概率质量，结果就是：

- 训练看起来正常
- loss 也在下降
- 但策略区分不出相近动作的细微差别

把区间改成 `[-2,2]` 并保持 51 原子后，支撑更贴近真实奖励尺度，分布细节能被保留下来，训练通常更稳定。

另一个常见坑是把 C51 当成“天然风险敏感”。严格说，C51 学的是分布，但默认 action selection 仍常用期望值。也就是说，它保留了风险信息，但不代表策略一定会主动规避风险。若你要显式做风险敏感决策，还需要在策略层使用分位数、CVaR 等额外准则。

还有一个误区是“原子越多越好”。这不总成立。

- $N$ 太小：分布粗糙，近似误差明显
- $N$ 太大：显存、计算、数值稳定性压力上升

在多数离散动作任务里，`51` 是经验上较稳的默认值，不是理论最优值。它的意义更像“合格基线”。

可执行的调参顺序通常是：

1. 先观察若干万步回报范围
2. 再设 `Vmin/Vmax`
3. 固定 `N=51`
4. 训练后再看分布是否经常贴边
5. 必要时再调区间或原子数

## 替代方案与适用边界

C51 不是唯一的分布式强化学习方法。它的核心特点是：固定等距支撑，学习每个原子的概率。替代方法主要差在“固定什么、学习什么”。

对比表如下：

| 方法 | 学什么 | 表示方式 | 优点 | 局限 |
|---|---|---|---|---|
| DQN | 期望值 | 单个标量 Q | 简单、便宜、容易实现 | 丢失分布信息 |
| C51 | 原子概率 | 固定支撑上的分类分布 | 稳定、直观、工程成熟 | 支撑固定，分辨率受区间限制 |
| QR-DQN | 分位点 | 固定概率质量、学习分位值 | 更灵活，理论更自然 | 实现与理解复杂一些 |
| FQF | 分位函数 | 自适应分位点 | 表达能力更强 | 训练与实现成本更高 |

可以把它们的选择逻辑理解为一个简单决策树：

```text
只要简单稳定基线？
-> DQN / Double DQN

需要保留回报分布，动作空间仍是离散？
-> 先用 C51

发现固定支撑不够灵活，想更细地刻画分布？
-> QR-DQN

希望分位点本身也自适应优化？
-> FQF
```

C51 的适用边界主要有三条。

第一，它最适合离散动作空间。因为它通常沿用 DQN 系列的动作枚举方式。动作枚举的意思是：对每个离散动作都显式输出一组值。到了连续动作场景，这套结构不能直接照搬，通常要结合 actor-critic 或分布式策略梯度方法。

第二，它适合奖励不确定、回报多峰、训练容易振荡的任务。Atari 就是代表场景，因为同一个动作可能通向差异很大的后果。

第三，在资源非常紧、延迟要求很高的环境里，普通 DQN 仍然有价值。C51 每个动作要输出 51 个原子，计算和内存都更贵。如果任务本身奖励平滑、风险结构简单，DQN 或 Double DQN 往往已经足够。

所以实用判断是：

- 你需要一个成熟、稳定、比 DQN 更强的离散动作基线：优先 C51
- 你发现固定支撑限制明显：转向 QR-DQN 或 FQF
- 你只关心最低成本上线：先用 DQN / Double DQN
