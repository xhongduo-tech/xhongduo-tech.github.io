## 核心结论

SimCSE 的核心不是“把相同句子复制两遍训练”，而是让同一句话经过两次不同的 dropout 后，得到两个相近但不完全相同的向量，再用 InfoNCE 做对比学习。dropout 可以理解为“训练时随机屏蔽一部分神经元的噪声机制”，它给同一句话制造了两个轻微扰动视图。对模型来说，这两个视图是正样本对，其余 batch 内句子都是负样本。

InfoNCE 的作用可以概括成两股力：一股力把正样本拉近，另一股力把负样本推远。拉近保证语义一致的句子表示稳定，推远保证所有向量不要挤成一个点。前者通常叫 alignment，意思是“语义相同的样本要对齐”；后者通常叫 uniformity，意思是“所有表示要尽量均匀铺在单位球面上”。

SimCSE 能成立，是因为它用最小扰动保住了语义，又用对比损失避免了表示坍缩。表示坍缩可以理解为“模型把所有输入都映射成几乎同一个方向”，这时余弦相似度失去区分能力。温度系数 $\tau$ 则决定推拉力度：$\tau$ 越小，softmax 越尖锐，越像“只盯着最难负样本打”；$\tau$ 越大，梯度更平滑，但区分能力会变弱。

对新手可以这样理解：同一句话的两次 dropout 输出像“俩兄弟”，InfoNCE 会把这俩兄弟拉近；batch 里其他句子越像，就越会被重点“赶远”。这就是 SimCSE 在无监督句向量学习中有效的根本原因。

## 问题定义与边界

问题定义很直接：在无监督句向量任务里，模型没有人工标注的“相似句子对”，但我们仍希望学到可用于 STS 和语义检索的表示。STS 是 semantic textual similarity，意思是“判断两句话语义有多接近”；语义检索则是“给一句查询，找语义最接近的句子或文档”。

这个目标至少包含两条要求：

| 目标 | 含义 | 失败后果 |
|---|---|---|
| 对齐 alignment | 同一句或同语义样本要靠近 | 相同语义检索不到 |
| 均匀 uniformity | 不同样本不要全挤在一起 | 所有句子都很像，无法排序 |

SimCSE 的边界也很明确。它讨论的是“仅靠 dropout 视图构造正样本”的无监督设定，不依赖回译、同义词替换、外部标签或 memory bank。它的优点是简单，代价是正样本扰动来源单一，因此训练稳定性高度依赖 dropout 是否真正生效。

下面这个边界表最关键：

| 条件 | 是否必须 | 原因 |
|---|---|---|
| 正样本来自同一句两次独立 dropout | 是 | 既保语义一致，又制造微小差异 |
| dropout 不能关闭 | 是 | 否则两个视图完全相同，训练信号退化 |
| 两次前向不能共享同一 mask | 是 | 共享 mask 等于复制样本，几乎没有增广 |
| $\tau$ 需调参，常见范围 0.05–0.2 | 是 | 太小会被硬负样本主导，太大区分度不足 |

如果把新手视角再压缩成一句话，就是：只给模型“同一句话的两个随机噪声版本”作为正样本，不要给完全一样的版本，否则模型很容易学成“所有东西都差不多”。

## 核心机制与推导

SimCSE 常用的损失是 InfoNCE：

$$
\ell_{i,j}=-\log \frac{\exp(\mathrm{sim}(z_i,z_j)/\tau)}{\sum_{k \neq i}\exp(\mathrm{sim}(z_i,z_k)/\tau)}
$$

其中 $z_i$ 是 anchor，意思是“当前被优化的样本向量”；$z_j$ 是它的正样本；$\mathrm{sim}$ 通常取余弦相似度。若所有向量都做了 $L_2$ 归一化，则余弦相似度就是点积。

它对 $z_i$ 的梯度可写成：

$$
\frac{\partial \ell_i}{\partial z_i}
=
\frac{1}{\tau}
\left(
\sum_{k \neq i} P_{ik} z_k - z_j
\right),
\quad
P_{ik}=
\frac{\exp(\mathrm{sim}(z_i,z_k)/\tau)}
{\sum_{l \neq i}\exp(\mathrm{sim}(z_i,z_l)/\tau)}
$$

这条式子非常重要，因为它直接告诉我们三件事。

第一，正样本项是 $-z_j/\tau$。负号表示优化时会把 $z_i$ 往 $z_j$ 方向拉。也就是说，正样本永远提供“靠近”的力。

第二，负样本项是 $\sum P_{ik} z_k/\tau$。注意这里不是所有负样本一视同仁，而是按 softmax 概率加权。谁和 anchor 更像，谁的 $P_{ik}$ 就更大，谁就更像“硬负样本”。硬负样本可以理解为“本来不该相似，但当前表示里却很相似的负例”。

第三，整个梯度前面有一个 $1/\tau$。这意味着 $\tau$ 不只影响 softmax 分布，还会整体缩放梯度幅值。$\tau$ 越小，梯度越大，且越集中在最相似的几个样本上。

一个玩具例子最能说明这个机制。假设 anchor 与正样本余弦相似度是 0.9，与一个硬负样本相似度是 0.7，先忽略其他样本。

当 $\tau=0.1$ 时：

$$
e^{0.9/0.1}=e^9 \approx 8103,\quad
e^{0.7/0.1}=e^7 \approx 1097
$$

所以

$$
P(\text{pos}) \approx \frac{8103}{8103+1097}=0.88,\quad
P(\text{hard neg}) \approx 0.12
$$

这表示：正样本主导“拉近”，硬负样本也有明显权重，模型会强烈把它推远。

再看不同 $\tau$ 的变化：

| 正样本相似度 | 硬负相似度 | $\tau$ | $P(\text{pos})$ | 现象 |
|---|---:|---:|---:|---|
| 0.9 | 0.7 | 0.5 | 0.60 | 分布较平，推拉温和 |
| 0.9 | 0.7 | 0.2 | 0.73 | 区分增强 |
| 0.9 | 0.7 | 0.1 | 0.88 | 硬负开始被重点处理 |
| 0.9 | 0.7 | 0.05 | 0.98 | 几乎只盯最难样本 |

当 $\tau \to 0$ 时，softmax 会逼近“赢家通吃”。如果最相似的负样本与正样本非常接近，那么它将主导负样本梯度，这就是 hardest-negative 主导现象。它的好处是训练更有辨别力，坏处是容易把语义真的接近的句子错当负例，也就是 false negative，意思是“标签上是负例，但语义上其实接近”。

为什么会发生表示坍缩？如果没有有效负样本推远，或者正样本构造退化成完全相同的复制，模型最省力的办法就是把所有句子都映射到相近方向。这样正样本天然接近，损失也不一定很大，但 uniformity 几乎消失。SimCSE 用 dropout 让正样本“足够像但不完全一样”，再用 batch 内其他句子提供负样本，才把这个坏解排除掉。

可以画成一个极简图示：

| 力 | 作用对象 | 几何效果 |
|---|---|---|
| 对齐力 | anchor 与正样本 | 拉近到同一语义局部 |
| 均匀力 | anchor 与负样本 | 推开到球面不同区域 |

SimCSE 的价值不在于把这两股力完全分开，而在于让它们同时存在且相互制衡。

## 代码实现

下面给一个可运行的 Python 玩具实现。它不依赖深度学习框架，只演示 InfoNCE 的数值机制和 $\tau$ 如何影响概率分布。

```python
import math

def softmax(xs):
    m = max(xs)
    exps = [math.exp(x - m) for x in xs]
    s = sum(exps)
    return [x / s for x in exps]

def info_nce_probs(pos_sim, neg_sims, tau):
    logits = [pos_sim / tau] + [s / tau for s in neg_sims]
    probs = softmax(logits)
    return {
        "p_pos": probs[0],
        "p_negs": probs[1:],
        "loss": -math.log(probs[0]),
    }

r1 = info_nce_probs(pos_sim=0.9, neg_sims=[0.7], tau=0.1)
r2 = info_nce_probs(pos_sim=0.9, neg_sims=[0.7], tau=0.5)

assert 0.87 < r1["p_pos"] < 0.89
assert 0.59 < r2["p_pos"] < 0.61
assert r1["loss"] < r2["loss"]  # 更小 tau 时，正例优势被放大

print(r1)
print(r2)
```

真正的 SimCSE 训练流程可以写成下面的伪代码：

```python
def train_step(sentences, encoder, tau=0.05):
    # 同一 batch 内，每个句子前向两次，dropout 独立采样
    z1 = encoder(sentences, dropout=True)   # [B, d]
    z2 = encoder(sentences, dropout=True)   # [B, d]

    z1 = l2_normalize(z1)
    z2 = l2_normalize(z2)

    # 拼成 2B 个表示，任意一个样本的正例是它在另一视图中的对应项
    z = concat(z1, z2)                      # [2B, d]
    sim = cosine_matrix(z, z)               # [2B, 2B]

    # 去掉自己和自己的匹配，只保留正负样本
    labels = positive_pair_index(batch_size=len(sentences))
    logits = sim / tau

    loss = cross_entropy_over_rows(logits, labels)
    return loss
```

这里有两个实现细节决定成败。

第一，两次前向必须独立 dropout。不要把隐藏状态缓存后重复使用，也不要共用随机 mask。否则正样本会退化成“完全同一个向量”。

第二，batch 组织要稳定。每个句子的两个视图必须能准确找到对应正样本，否则会把真正正例当负例处理。

一个真实工程例子是句向量检索。假设你要做“报销制度问答”的企业知识库搜索，输入是“出差住宿标准是多少”，你希望召回“国内差旅住宿标准说明”。如果嵌入模型只有 alignment 没有 uniformity，大量句子会挤在一起，召回排序不稳定；如果只有 uniformity 没有 alignment，同义问法又无法靠近。SimCSE 正好在这两者之间给出一个低成本平衡方案。

## 工程权衡与常见坑

工程里最常见的误区，不是公式推错，而是把对比学习里的“轻微扰动”做成了“完全复制”或“过度推远”。

| 坑 | 现象 | 原因 | 规避策略 |
|---|---|---|---|
| 关闭 dropout | 训练很快但表示发散或退化 | 正样本没有扰动，学习信号不足 | 训练阶段必须开启 dropout |
| 两次前向共享 mask | 正样本几乎完全相同 | 等价于复制样本 | 保证独立随机种子或独立前向 |
| $\tau$ 太小 | loss 很低但检索变差 | 硬负和 false negative 主导梯度 | 从 0.05–0.2 搜索，并看检索指标 |
| batch 太小 | 负样本不够 | uniformity 学不出来 | 提升 batch，或用梯度累积 |
| 领域语料高度重复 | 大量 false negative | 负样本里混入近义句 | 去重、采样、加监督信号 |

对新手最值得强调的一点是：如果你把两次 dropout 的 mask 设成一样，就相当于把正样本变成精确复制。这样模型会发现，“只要把所有表示都弄得差不多，很多样本都能拿到不坏的分数”。这就是 collapse，也就是“最后所有向量收敛成一坨”。

还有一个经常被忽视的问题是 false negative。在通用语料中，同一个 batch 里很可能出现“今天北京下雨了”和“北京今天有降雨”这类本质接近的句子。如果 $\tau$ 很小，模型会把这种近义句当成最难负样本强行推远，最终损害语义结构。所以实务上不能只看训练 loss，还要看 STS、召回率或聚类质量。

dropout 在这里的作用很微妙。它不是为了制造大幅度语义变化，而是为了让同一句话的两个视图出现微小表示差异。这个差异一方面阻止“完全复制”的坏解，另一方面又不至于把正样本语义改坏。可以说，SimCSE 的简洁性几乎全部建立在这个设计点上。

## 替代方案与适用边界

SimCSE 并不是唯一的对比学习方案，只是它在文本场景下非常省事。文本不像图像，不能随便裁剪、旋转、调颜色，因此“如何做增广”本身就是难题。SimCSE 直接把 dropout 当作最小可用增广，避免了很多额外工程复杂度。

| 方法 | 正样本来源 | 是否需额外模块 | 文本场景复杂度 | 适用边界 |
|---|---|---:|---:|---|
| SimCSE | 同一句两次 dropout | 否 | 低 | 无监督句向量、快速落地 |
| SimCLR | 强数据增广 | 否 | 中到高 | 图像更常见，文本增广难设计 |
| MoCo | 队列中的历史样本 | 是，memory bank/queue | 中到高 | 适合大规模负样本管理 |
| 监督式对比学习 | 标注正负对 | 否 | 取决于标注 | 有高质量标签时更强 |

如果你能构造更多语义相近的正样本，比如同义词替换、回译、标题与正文摘要配对，那本质上是在给模型更多“兄弟样本”。这通常能增强 alignment，但也带来两个成本：一是增广可能改坏原意，二是工程链路更复杂。SimCSE 的优点就在于它把这些成本压到最低，只要有一个带 dropout 的编码器就能跑起来。

它最适合的场景是大规模无标签语料上的句向量预训练，尤其是 STS、FAQ 检索、文本召回初筛。如果你的任务需要更强的精细语义判别，比如法律条文细粒度匹配、医学问答中的术语差异识别，单纯无监督 SimCSE 往往不够，需要叠加监督对比学习、难例挖掘或 reranker。
