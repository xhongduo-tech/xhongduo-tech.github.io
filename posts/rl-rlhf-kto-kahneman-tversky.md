## 核心结论

KTO，Kahneman-Tversky Optimization，中文可理解为“受前景理论启发的对齐优化”。前景理论是行为经济学里描述人类如何看待收益和损失的模型，核心现象是“人对损失通常比对同等收益更敏感”。把这个思想放到模型训练里，KTO 不再要求同一提示下必须有一对 `chosen/rejected` 响应，而是只需要知道某条输出是“可取”还是“不可取”。

它和 DPO 的根本差异，不在于都用了 sigmoid，而在于优化目标不同。DPO 优化的是“偏好对的相对对数似然”，本质是让好答案压过坏答案；KTO 优化的是“带损失厌恶的效用”，本质是让模型对坏答案更敏感，尤其适合安全、拒答、风险控制这类“避免犯错比追求最优更重要”的任务。

玩具例子：一个客服模型给出两类输出，一类是“答得普通但安全”，另一类是“答得详细但偶尔胡说”。如果你的业务成本里，一次胡说的损失远高于一次普通回答的收益，那么 KTO 的目标和这个业务目标更一致，因为它会主动放大坏回答带来的负效用。

| 对比项 | KTO | DPO |
|---|---|---|
| 数据需求 | 单条响应 + 二元标签 | 同提示下的偏好对 |
| 学习重点 | 非对称效用，强调坏样本惩罚 | 相对偏好排序 |
| 噪声容忍度 | 对“只有好/差反馈”的场景更友好 | 更依赖成对标注质量 |
| 典型优势场景 | 安全、拒答、联邦、弱反馈采集 | 偏好对充足的通用对齐 |

## 问题定义与边界

KTO 解决的问题可以表述为：当你拿不到高质量偏好对，只能拿到“这条回答可取/不可取”的二元反馈时，如何仍然训练出与人类偏好一致的模型。

这里的“二元反馈”就是最简单的监督信号，只告诉你结果过线还是不过线，不告诉你它和另一条回答相比谁更好。对很多真实系统，这比收集偏好对便宜得多。审校员、用户举报、安全团队、线上规则系统，往往更容易给出“不行”而不是“它比另一个更好”。

问题边界也要说清楚：

1. KTO 不是完全不需要参考模型。它仍然依赖参考分布 `π_ref`，避免新模型偏离原模型太远。
2. KTO 不是“自动更强的 DPO”。如果你已经有大量高质量偏好对，而且目标主要是提升主观质量，DPO 依然是很强的基线。
3. KTO 最适合的是“不良输出代价高”的问题，而不是所有偏好学习问题。

从信号到效用的映射，可以写成一个简单框架。先定义相对参考模型的对数优势：

$$
m_\theta(x,y)=\log \pi_\theta(y|x)-\log \pi_{\mathrm{ref}}(y|x)
$$

再定义参考点 $z_{\mathrm{ref}}$。参考点可以白话理解为“当前系统认为及格线大概在哪”。于是：

- 如果样本标签是“可取”，希望 $m_\theta$ 高于参考点。
- 如果样本标签是“不可取”，希望 $m_\theta$ 低于参考点。

真实工程例子：联邦学习里，每个客户端只愿意上传“这条回复被投诉”或“这条回复通过审核”的单点反馈，不愿上传多条候选回答。KTO 能直接利用这种数据；DPO 则通常要求为同一提示构造成对样本，数据链路更重。

---

## 核心机制与推导

KTO 的直觉是：不要只问“好答案赢没赢过坏答案”，而要问“这条输出给人的主观效用是多少”。效用就是“这个结果在人看来值不值得”的量化刻画。

为了讲清楚机制，可以先用一个教学版公式。设

$$
z=m_\theta(x,y)-z_{\mathrm{ref}}
$$

对“可取”样本，定义效用为

$$
U^+(z)=\sigma(\beta z)
$$

对“不可取”样本，定义效用为

$$
U^-(z)=\sigma(-\beta z)
$$

其中 $\sigma(t)=\frac{1}{1+e^{-t}}$ 是 sigmoid，$\beta$ 控制曲线陡峭程度。白话说，$\beta$ 越大，模型越在意“过线还是不过线”的临界差异。

于是总损失可写成

$$
\mathcal{L}_{\mathrm{KTO}}
=
\mathbb{E}_{(x,y)\sim D_{\text{des}}}[\,\lambda_D (1-U^+(z))\,]
+
\mathbb{E}_{(x,y)\sim D_{\text{und}}}[\,\lambda_U (1-U^-(z))\,]
$$

这里：

- $\lambda_D$ 是好样本权重。
- $\lambda_U$ 是坏样本权重。
- “权重”就是训练时把哪类样本看得更重。

如果采用另一种等价记号，也可以把内部量写成 $r_{\mathrm{KTO}}=\beta \log \frac{\pi_{\mathrm{ref}}}{\pi_\theta}$，那只是符号方向不同，本质仍然是在比较当前模型与参考模型的相对偏离，再对可取与不可取样本施加不同方向的效用变换。

为什么说它受前景理论启发？因为前景理论里，收益区和损失区不是同一条对称直线，人会对损失更敏感。KTO 把这种非对称性落在两个地方：

1. 标签方向不同：好样本和坏样本进入相反方向的 sigmoid。
2. 权重不同：通常让 $\lambda_U \ge \lambda_D$，把坏样本的负效用放大。

玩具例子：设 $\beta=1$，$z_{\mathrm{ref}}=0.2$。某条坏回答的 $m_\theta=0.0$，则 $z=-0.2$，其不可取效用为

$$
U^-=\sigma(0.2)\approx 0.55
$$

若 $\lambda_U=1.2$，损失项约为 $1.2\times(1-0.55)=0.54$。  
另一条好回答若 $m_\theta=0.6$，则 $z=0.4$，有

$$
U^+=\sigma(0.4)\approx 0.60
$$

若 $\lambda_D=1.0$，损失项约为 $0.40$。同样数量级下，坏回答被惩罚得更重，这就是“坏答案先行”的训练偏置。

## 代码实现

下面给一个可运行的最小 Python 版本。它不依赖深度学习框架，只演示 KTO 的核心数值逻辑。

```python
import math

def sigmoid(x: float) -> float:
    return 1.0 / (1.0 + math.exp(-x))

def kto_loss(logp_theta, logp_ref, labels, beta=1.0, z_ref=0.0, lambda_des=1.0, lambda_und=1.2):
    """
    logp_theta: 当前模型对样本的 log p
    logp_ref:   参考模型对样本的 log p
    labels:     1 表示 desirable，0 表示 undesirable
    """
    assert len(logp_theta) == len(logp_ref) == len(labels)
    losses = []

    for lp_t, lp_r, label in zip(logp_theta, logp_ref, labels):
        margin = lp_t - lp_r
        z = margin - z_ref

        if label == 1:
            utility = sigmoid(beta * z)
            weight = lambda_des
        else:
            utility = sigmoid(-beta * z)
            weight = lambda_und

        losses.append(weight * (1.0 - utility))

    return sum(losses) / len(losses)

# 玩具例子
theta = [-1.0, -0.2, -2.0, -0.5]
ref   = [-1.3, -0.1, -1.7, -0.4]
label = [1, 0, 1, 0]

loss = kto_loss(theta, ref, label, beta=2.0, z_ref=0.05, lambda_des=1.0, lambda_und=1.3)
assert 0.0 < loss < 2.0

# 检查：同样条件下，把坏样本的当前模型概率拉高，损失应更大
loss_bad_small = kto_loss([-1.0], [-0.9], [0], beta=2.0, z_ref=0.0, lambda_und=1.3)
loss_bad_large = kto_loss([-0.1], [-0.9], [0], beta=2.0, z_ref=0.0, lambda_und=1.3)
assert loss_bad_large > loss_bad_small

print(round(loss, 4))
```

这段代码对应的训练步骤很直接：

1. 先算 `margin = log πθ - log πref`。
2. 再减去参考点 `z_ref`，判断这条样本离“及格线”多远。
3. 对好样本用 `sigmoid(beta * z)`，对坏样本用 `sigmoid(-beta * z)`。
4. 最后乘上 `lambda` 权重，累积 `1 - utility`。

真实工程例子：做安全拒答微调时，你可能有一批单轮问答数据，其中一部分被人工标成“越权”“有害”“编造”。这时不需要为每个有害回答再配一条“更好的替代回答”才能训练。只要有 `(prompt, response, binary_label)`，就能进入 KTO 流水线，这会明显简化数据采集和联邦聚合。

实现时还有两个工程点：

- `π_ref` 应视为固定参考，不参与反向传播。
- `z_ref` 不宜乱设。过高会让大部分样本都被视为“不过线”，过低又会使训练信号变弱。

## 工程权衡与常见坑

KTO 的优势很明确，但代价也很明确：超参数更难调。

第一类坑是样本不均衡。若坏样本只占 30%，但你直接设 $\lambda_U=\lambda_D=1$，训练可能仍然主要被好样本主导，因为数量上好样本更多。工程上常见做法是按批次或滑动窗口统计 $n_U,n_D$，让

$$
\frac{\lambda_U n_U}{\lambda_D n_D}
$$

维持在一个略偏向坏样本的区间，例如接近 $1$ 到 $4/3$。这不是硬定律，但很实用。

第二类坑是 $\beta$ 过大。$\beta$ 太大时，sigmoid 很快饱和，样本一旦明显过线或明显不过线，梯度就接近 0。结果是模型只会盯着极少数边界样本，训练不稳定。

第三类坑是参考点漂移。`z_ref` 如果固定得不合理，模型会长期在错误基线上优化。实践里可以用验证集观察好坏样本的平均 utility，若两类样本都过度接近 1 或都过低，说明参考点或 $\beta$ 需要调整。

第四类坑是 hard negative 被忽略。hard negative 是“非常像好答案，但实际上危险或错误”的坏样本。它们常常最有价值，但如果你只看简单的平均损失，这类样本可能被数量更多的普通坏样本淹没。解决思路通常是保留多样化坏样本、对高风险类别单独加权、或者给权重设置下限。

| 风险 | 现象 | 常见原因 | 对策 |
|---|---|---|---|
| 坏样本惩罚不够 | 安全指标不升反降 | $\lambda_U$ 太小或坏样本太少 | 提高 $\lambda_U$，按占比重标定 |
| 训练很快停滞 | loss 不降，梯度变小 | $\beta$ 太大导致饱和 | 降低 $\beta$ |
| 输出整体变保守 | 拒答增多、帮助性下降 | 坏样本权重过高 | 回调 $\lambda_U$，增加高质量好样本 |
| 线上分布不稳 | 训练好，实测差 | `z_ref` 与线上分布不匹配 | 用线上近邻数据重估参考点 |

调参 checklist 可以压缩成四项：先看样本占比，再调 $\lambda$；再看 sigmoid 是否饱和，调 $\beta$；确认参考模型不回传梯度；最后单独检查 hard negatives 的覆盖率。

## 替代方案与适用边界

如果你有充足、干净、稳定的偏好对，DPO 依然是非常合理的选择。它更直接回答“在这两条回答里哪条更好”，因此在通用聊天质量、写作风格选择、偏好排序清晰的任务上通常更自然。

如果你的反馈形式主要是“这条能不能接受”，KTO 往往更贴近数据形态。尤其在以下边界里更合适：

1. 安全和拒答优先，坏答案成本高。
2. 联邦或隐私场景，只能收集单条反馈。
3. 数据来源异构，难以稳定构造偏好对。
4. 标注人员更容易判断“是否违规”，而不是比较两个答案优劣。

再往前走一步，就是 Mo-KTO，多目标 KTO。它把多个目标一起写进效用函数，例如“有益性”和“安全性”两个维度各有一个 bit。白话说，就是一条样本不只标“整体好/坏”，而是标“帮助性好/坏、安全性好/坏”。这样可以在不训练多个奖励模型的前提下，直接做多目标优化。

真实工程例子：医疗问答助手可能同时要求“医学信息尽量有用”和“绝不越权给诊断”。若只用单一偏好分数，两个目标会混在一起；若用 Mo-KTO，可以对每条样本记录两个二元标签，再在统一目标里联合训练。相比多奖励模型 RLHF，工程链路更短；相比 DPO，则不必为每个目标都构造成对数据。

| 方法 | 数据形式 | 适合目标数 | 强项 | 主要边界 |
|---|---|---|---|---|
| DPO | 成对偏好 | 单目标或弱多目标 | 偏好对充分时效果稳 | 依赖 pair 数据 |
| KTO | 单条二元反馈 | 单目标 | 风险控制、弱反馈采集 | 超参更敏感 |
| Mo-KTO | 单条多 bit 信号 | 多目标 | 多维对齐、低标注负担 | 目标冲突时设计更复杂 |

一句话判断：如果你的核心问题是“怎么把最差答案压下去”，优先考虑 KTO；如果你的核心问题是“怎么在两个不错的答案里选更好的”，优先从 DPO 想起。
