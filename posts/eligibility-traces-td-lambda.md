## 核心结论

资格迹（eligibility trace）可以先把它理解成“会自动褪色的历史记号”。某个状态或特征刚被访问时，记号很强；随着时间推进，如果它不再出现，这个记号会按 $\gamma \lambda$ 衰减。这样，后面才出现的 TD 误差就不只更新当前状态，还能回头更新最近访问过的一串状态。

`TD(λ)` 的位置正好在 `TD(0)` 和 Monte Carlo 之间。`TD(0)` 只看一步，更新最局部；Monte Carlo 要等整段 episode 结束后再看总回报，更新范围最远；`TD(λ)` 用参数 $\lambda \in [0,1]$ 控制“回看多远”。$\lambda$ 越大，奖励传播越远；$\lambda=0$ 时退化成 `TD(0)`；$\lambda` 接近 `1` 时，行为更接近基于整段回报的方法。

一个直接结论是：当奖励延迟出现时，`TD(λ)` 往往比 `TD(0)` 学得快。因为 `TD(0)` 只能让价值信息一层一层往前传，而资格迹允许同一个误差信号一次修正多步历史。

| 方法 | 更新时机 | 是否回看历史 | 偏差/方差特征 | 典型优点 |
| --- | --- | --- | --- | --- |
| `TD(0)` | 每一步在线更新 | 基本不回看，只改当前 | 偏差较大，方差较低 | 简单、稳、便宜 |
| `TD(λ)` | 每一步在线更新 | 回看最近历史，按衰减分配 | 在偏差和方差之间折中 | 延迟奖励传播更快 |
| Monte Carlo | episode 结束后更新 | 回看整段轨迹 | 偏差低，方差高 | 目标直观，不依赖 bootstrap |

## 问题定义与边界

这里要解决的问题不是“某一步奖励是多少”，而是“当前误差信号应该分给过去哪些状态”。这叫信用分配（credit assignment），白话说就是：最后拿到奖励时，到底该表扬前面的哪几步。

先固定记号。设：

| 符号 | 含义 | 白话解释 |
| --- | --- | --- |
| $S_t$ | 时刻 $t$ 的状态 | 第 $t$ 步看到的局面 |
| $R_{t+1}$ | 从 $S_t$ 走到下一步拿到的奖励 | 下一步反馈 |
| $\gamma$ | 折扣因子 | 未来奖励值不值得看得很重 |
| $\lambda$ | 迹衰减系数 | 记忆保留多久 |
| $\delta_t$ | TD 误差 | 当前预测和新证据之间的差 |
| $e_t$ | 资格迹向量 | 哪些历史状态还“值得被追责或记功” |
| $w_t$ | 参数向量 | 当前价值函数的可学习参数 |

如果一个机器人走迷宫，只有到终点才得到 `+1`，中间都没有奖励，那么只靠当前一步更新会很慢。因为终点前一步先学到值，倒数第二步下一轮才可能间接受益，再往前还要更多轮。资格迹的作用就是把终点附近产生的误差，按远近不同回传给更早访问的状态。

它的适用边界也要说清楚：

| 场景 | 是否适合资格迹 | 原因 |
| --- | --- | --- |
| 在线学习、边交互边更新 | 适合 | 不需要存整条轨迹 |
| 延迟奖励、稀疏奖励 | 适合 | 能更快传播误差 |
| 表格法或线性函数逼近 | 很常见 | 数学和实现都直接 |
| 强 off-policy + 函数逼近 | 需谨慎 | 可能不稳定 |
| 必须依赖完整回报定义的离线分析 | 不一定优先 | Monte Carlo 或多步回报更直观 |

所以，资格迹不是“强化学习的通用默认选项”，它更像一个针对在线信用分配问题的高效机制。

## 核心机制与推导

先从 TD 误差开始。TD 误差（temporal-difference error）可以理解成“当前估计和一步新观测之间的差值”，公式是：

$$
\delta_t = R_{t+1} + \gamma V(S_{t+1}; w_t) - V(S_t; w_t)
$$

这里的关键不是它只依赖一步，而是它每一步都能立刻计算，所以适合在线学习。

如果没有资格迹，普通 `TD(0)` 的更新是：

$$
w_{t+1} = w_t + \alpha \delta_t x_t
$$

其中 $x_t$ 是当前状态的表示。表格法里它是一热向量，白话说就是“当前状态位置写 1，其余写 0”；函数逼近里它是特征向量 $\phi(S_t)$。

资格迹把这个“只给当前状态记账”的过程扩展成“给最近历史统一记账”。最常见的累积式定义是：

$$
e_t = \gamma \lambda e_{t-1} + x_t
$$

于是更新变成：

$$
w_{t+1} = w_t + \alpha \delta_t e_t
$$

这三个式子合起来，就是 `TD(λ)` 的后向视图（backward view）。后向视图的含义是：系统不需要回头翻旧轨迹，只要维护一个不断衰减的迹向量，就等价于把误差分给历史状态。

### 玩具例子：`A -> B -> T`

设轨迹是 `A -> B -> T`，`T` 是终止状态。参数取：

- $\gamma=0.9$
- $\lambda=0.8$
- $\alpha=0.1$
- 初值：$V(A)=0.5,\;V(B)=0.3,\;V(T)=0$

第一步，从 `A` 到 `B`，奖励 $R_1=1$：

- 访问 `A`，所以 $e(A)=1$
- TD 误差：
  $$
  \delta_0 = 1 + 0.9 \times 0.3 - 0.5 = 0.77
  $$
- 更新后：
  $$
  V(A) \leftarrow 0.5 + 0.1 \times 0.77 = 0.577
  $$

第二步，从 `B` 到终止 `T`，奖励 $R_2=2$：

- 旧迹先衰减：$e(A)=0.9 \times 0.8 = 0.72$
- 新访问 `B`：$e(B)=1$
- TD 误差：
  $$
  \delta_1 = 2 + 0 - 0.3 = 1.7
  $$
- 更新 `A`：
  $$
  \Delta V(A)=0.1 \times 1.7 \times 0.72 = 0.1224
  $$
- 更新 `B`：
  $$
  \Delta V(B)=0.1 \times 1.7 \times 1 = 0.17
  $$

最终得到：

- $V(A)=0.6994$
- $V(B)=0.47$

这正是资格迹的核心效果：第二步产生的误差，不只修正 `B`，也继续修正了更早出现的 `A`。如果是 `TD(0)`，第二步只会更新 `B`，`A` 还得等下一轮间接受益。

### 前向视图与后向视图

前向视图（forward view）说的是学习目标本身。它不从“怎么更新”讲，而从“应该逼近什么目标”讲。定义 $n$ 步回报：

$$
G_t^{(n)} = R_{t+1} + \gamma R_{t+2} + \cdots + \gamma^{n-1} R_{t+n} + \gamma^n V(S_{t+n})
$$

那么 $\lambda$-return 是这些多步回报的加权和：

$$
G_t^\lambda = (1-\lambda)\sum_{n=1}^{\infty}\lambda^{n-1} G_t^{(n)}
$$

如果 episode 会终止，最后一项可以自然截断。直观上，这个式子表示：一步看一点，两步看一点，三步也看一点，但越远权重越小。

前向视图告诉我们 `TD(λ)` 在学什么；后向视图告诉我们如何高效实现。两者在离线情形下数学等价。等价的意思不是“看起来像”，而是“最终对参数造成的总更新一致”。

但在线情形要小心。经典在线 `TD(λ)` 在参数每步都变化时，只是前向视图的近似实现；如果想要逐步严格等价，需要 `True Online TD(λ)`。这不是枝节，而是工程上常见误解来源。

再看三种常见迹：

| 迹类型 | 公式特征 | 适用特点 | 风险 |
| --- | --- | --- | --- |
| accumulating trace | $e_t=\gamma\lambda e_{t-1}+x_t$ | 通用，推导最常见 | 重复激活时迹可能过大 |
| replacing trace | 二值特征常写为命中特征直接置 1 | 控制重复访问时的爆涨 | 不适合所有连续特征 |
| true online trace | 配合 true online 更新式 | 在线前向/后向更严格等价 | 公式更复杂，实现更细 |

## 代码实现

实现时只需要维护两个核心对象：参数向量 `w` 和资格迹向量 `e`。表格法里它们长度等于状态数；线性函数逼近里长度等于特征维度。

下面先给一个最小可运行的表格版玩具实现。这个例子直接复现上面的 `A -> B -> T` 轨迹。

```python
import math

def td_lambda_tabular():
    states = ["A", "B", "T"]
    idx = {s: i for i, s in enumerate(states)}

    gamma = 0.9
    lam = 0.8
    alpha = 0.1

    # value table
    w = [0.5, 0.3, 0.0]
    e = [0.0, 0.0, 0.0]

    trajectory = [
        ("A", 1.0, "B"),  # S_t, R_{t+1}, S_{t+1}
        ("B", 2.0, "T"),
    ]

    for s, r, s_next in trajectory:
        s_i = idx[s]
        next_i = idx[s_next]

        # 先衰减，再给当前状态加 1
        e = [gamma * lam * x for x in e]
        e[s_i] += 1.0

        delta = r + gamma * w[next_i] - w[s_i]
        w = [wi + alpha * delta * ei for wi, ei in zip(w, e)]

        if s_next == "T":
            e = [0.0 for _ in e]

    return w

w = td_lambda_tabular()
assert abs(w[0] - 0.6994) < 1e-6
assert abs(w[1] - 0.47) < 1e-6
assert abs(w[2] - 0.0) < 1e-12
```

这个实现里最关键的顺序只有三步：

1. `e <- γλe`
2. 给当前状态或特征加上当前激活
3. 用 `δ` 乘 `e` 更新全部参数

如果换成函数逼近，唯一变化是“当前状态”不再是一热索引，而是一个特征向量。下面给一个极简线性版本：

```python
def value(w, x):
    return sum(wi * xi for wi, xi in zip(w, x))

def td_lambda_linear_step(w, e, x_t, r_tp1, x_tp1, gamma=0.9, lam=0.8, alpha=0.1, terminal=False):
    e = [gamma * lam * ei + xi for ei, xi in zip(e, x_t)]
    v_t = value(w, x_t)
    v_tp1 = 0.0 if terminal else value(w, x_tp1)
    delta = r_tp1 + gamma * v_tp1 - v_t
    w = [wi + alpha * delta * ei for wi, ei in zip(w, e)]
    if terminal:
        e = [0.0 for _ in e]
    return w, e, delta

w = [0.2, -0.1]
e = [0.0, 0.0]
x_t = [1.0, 0.5]
x_tp1 = [0.3, 0.8]

w2, e2, delta = td_lambda_linear_step(w, e, x_t, 1.0, x_tp1)
assert len(w2) == 2 and len(e2) == 2
assert isinstance(delta, float)
```

真实工程例子可以看在线机器人控制。假设机器人每 `20ms` 收到一次传感器特征，奖励只在“接近目标”或“跌倒”时明显变化。此时如果等完整轨迹结束再回放，控制延迟和内存成本都不合适。`TD(λ)` 或 `Sarsa(λ)` 的优势是：每步只存一个 `e` 和一个 `w`，但仍能把刚出现的误差信号传播给前面几百毫秒内的关键动作或状态特征。这比纯 `TD(0)` 更快形成“哪些早期动作对最终结果负责”的估计。

## 工程权衡与常见坑

资格迹最容易被误解成“保存历史状态列表”。这不对。它保存的不是原始历史，而是一个衰减后的权重向量。你不需要把整段轨迹存下来，只需要不断维护 `e`。

下面是最常见的工程坑：

| 坑点 | 后果 | 规避方法 |
| --- | --- | --- |
| 忘记终止状态令 bootstrap 为 0 | 终点价值被错误传播 | terminal 时令 $V(S_{t+1})=0$ |
| 新 episode 没清零迹 | 上一局信用串到下一局 | 每个 episode 开始前重置 `e` |
| 先加当前特征再衰减，顺序写反 | 更新量与理论不一致 | 明确使用“先衰减，再注入当前特征” |
| 重复激活特征时仍盲目累积 | 迹过大，学习震荡 | 二值特征常考虑 replacing trace |
| off-policy + 函数逼近直接套普通 traces | 可能发散 | 考虑更稳的 off-policy/梯度方法 |
| 误以为 `λ=1` 恒等于 Monte Carlo | 在线实现认知错误 | 区分经典在线版与 True Online 版 |

再具体说一个高频坑。假设某个二值特征连续 10 步都为 1，如果你一直用 accumulating trace，那么它的迹会不断叠加，导致这个特征被过度更新。replacing trace 的思路是：命中的二值特征直接设为 1，而不是继续累加。这样会更稳，尤其在稀疏离散特征里很常见。

还有一个理论与实现的边界：资格迹和 bootstrap 混在一起时，稳定性依赖策略设置、函数逼近形式和数据分布。新手最安全的落点通常是：

- on-policy
- 表格法或线性函数逼近
- 明确的 episode 边界
- 小步长 $\alpha$

这个组合最容易得到符合教材推导的行为。

## 替代方案与适用边界

如果问题里的奖励本来就很密集，每一步都能给出清晰反馈，那么 `TD(0)` 往往已经够用。它实现最简单，调参面最小，也更容易稳定。

如果你能完整拿到一条 episode，而且更在意“最终真实回报”而不是每一步 bootstrap 近似，那么 Monte Carlo 也是合理方案。它的缺点是必须等 episode 结束，且方差通常更大。

`TD(n)` 可以看作中间路线：不用像 Monte Carlo 那样等到最后，也不像 `TD(0)` 只看一步，而是固定看 `n` 步。`TD(λ)` 则进一步把很多个 `n` 步目标按权重混合起来，因此更柔性。

| 方法 | 是否在线 | 是否回看历史 | 样本效率 | 稳定性 | 实现复杂度 |
| --- | --- | --- | --- | --- | --- |
| `TD(0)` | 是 | 很弱 | 较高 | 通常较好 | 低 |
| `TD(n)` | 可在线也可半在线 | 固定回看 `n` 步 | 中等 | 取决于 `n` | 中 |
| `TD(λ)` | 是 | 按衰减回看 | 较高 | 一般较好，但看设定 | 中 |
| Monte Carlo | 否，通常需 episode 结束 | 回看整段 | 可能较低 | 方差高 | 低到中 |
| `True Online TD(λ)` | 是 | 按衰减回看 | 较高 | 理论一致性更强 | 中到高 |

真实工程里，在线机器人控制、在线广告竞价、流式推荐反馈这类任务，通常不能等完整轨迹结束，所以更偏向 `TD(λ)`、`Sarsa(λ)` 或其他在线多步方法。相反，如果是离线批量训练，整段轨迹已经在磁盘里，且你更想做简单可控的基线，那么固定 `n` 步回报或 Monte Carlo 也可能更合适。

因此，资格迹不是因为“更高级”才值得用，而是因为它在“在线、延迟奖励、不能存整段轨迹”的边界内，给出了很好的计算效率与信用分配效果。
