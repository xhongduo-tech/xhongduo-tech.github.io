## 核心结论

推荐系统偏差是指模型学到的不是“真实偏好”，而是被点击选择、展示位置、曝光机制和流行度分布共同扭曲后的反馈信号。去偏的目标不是消灭所有偏差，而是尽量恢复“如果所有物品被公平曝光时”的用户偏好，并在偏差校正和方差控制之间取平衡。

点击日志记录的是被推荐后发生了什么，不是所有候选物的真实好坏。同一个用户看到两个内容，只点了第一个，不代表第一个一定更相关。它可能只是排在更前面、标题更醒目，或者用户根本没看到第二个。

| 观测到的点击 | 可能原因 |
|---|---|
| 被点了 | 相关、位置靠前、曝光更多、热门 |
| 没被点 | 不相关、没看到、没机会曝光、位置靠后 |

一个最小定义是：

$$
c_{ui}=o_{ui}r_{ui}
$$

其中 \(c_{ui}\) 是用户 \(u\) 对物品 \(i\) 的观测点击，\(o_{ui}\) 是是否曝光，\(r_{ui}\) 是真实相关性。这个公式说明：点击不是单独的偏好标签，而是“曝光”和“相关性”共同作用后的结果。没有曝光时，即使用户本来喜欢这个物品，日志里也不会出现点击。

## 问题定义与边界

推荐系统里的偏差通常不是单一来源，而是多种机制耦合出现。

选择偏差是指进入日志的数据本身不是随机样本。用户会选择点什么，系统也会选择展示什么，所以训练集天然偏向已经被系统认为值得展示的内容。

位置偏差是指同一个物品放在不同位置，点击概率不同。列表第一位通常比第十位更容易被看到，也更容易被点。

曝光偏差是指未曝光物品没有标签。信息流里一个视频没被展示过，不能把“没点”当成“用户不喜欢”。这更接近“没有考试成绩”，不是“考了 0 分”。

流行度偏差是指热门物品因为历史点击多，更容易继续被推荐，随后产生更多点击，形成反馈回路。模型会逐渐把“大家都看过”误学成“对所有人都更相关”。

| 符号 | 含义 |
|---|---|
| \(u\) | 用户 |
| \(i\) | 物品 |
| \(s\) | 展示位置 |
| \(o_{ui}\) | 是否曝光 |
| \(r_{ui}\) | 真实相关性 |
| \(c_{ui}\) | 观测到的点击 |
| \(p_{ui}\) | 曝光概率，也叫 propensity |

propensity 是样本被观测到的概率。白话说，它表示“这条样本本来有多大机会被系统展示出来”。

去偏适用于推荐、排序、CTR/CVR、隐式反馈学习等场景。隐式反馈是用户行为间接表达的偏好，例如点击、播放、收藏、停留时长，而不是用户明确打出的评分。

但它不适合被滥用。完全随机采样的数据，标签本身完整可见的数据，或者曝光机制完全不可追踪的历史日志，都不一定适合直接套 IPS 或因果去偏。还有一种常见边界：系统本来就只对少量候选做强筛选，那么未曝光样本里混杂了大量根本没进入候选集的物品，不能直接当负样本使用。

## 核心机制与推导

推荐模型常见目标是最小化风险：

$$
R=\mathbb{E}_{u,i}[\ell(f(u,i), r_{ui})]
$$

其中 \(f(u,i)\) 是模型对用户和物品的打分，\(\ell\) 是损失函数，表示预测和真实相关性之间的误差。问题在于，真实的 \(r_{ui}\) 通常不可见，我们只能看到被曝光后的点击或转化。

如果直接用点击训练，模型优化的是“被旧系统展示过的点击”，不是全量候选空间上的真实偏好。IPS，全称 Inverse Propensity Scoring，中文可理解为“按曝光概率的倒数加权”。它的核心估计式是：

$$
\hat R_{\text{IPS}}=\frac{1}{N}\sum_{n=1}^{N}\frac{o_n}{p_n}\ell(f_n,y_n)
$$

直觉很直接：越难被曝光的样本，一旦被观测到，就应该在训练中被放大。否则模型会低估这类样本的重要性。

玩具例子：有两个曝光样本，损失都为 \(0.2\)。样本 A 的曝光概率 \(p_A=0.8\)，权重是 \(1.25\)。样本 B 的曝光概率 \(p_B=0.2\)，权重是 \(5\)。不加权平均损失是：

$$
(0.2+0.2)/2=0.2
$$

IPS 加权后是：

$$
(0.2/0.8+0.2/0.2)/2=(0.25+1.0)/2=0.625
$$

B 的曝光概率只有 A 的四分之一，所以 B 的权重是 A 的 4 倍。这不是说 B 一定更重要，而是说“它本来更难出现在日志里”，需要补偿。

但 IPS 有一个工程问题：小 \(p\) 会产生巨大权重。若 \(p=0.01\)，权重就是 100，单条噪声样本可能严重影响训练。因此常用截断权重：

$$
w_n=\min(1/p_n,\tau)
$$

其中 \(\tau\) 是权重上限。如果 \(\tau=3\)，上面的 B 权重从 5 被截断到 3，加权损失变为：

$$
(0.2\times1.25+0.2\times3)/2=0.425
$$

这就是“降偏差”和“控方差”的折中。方差是估计结果的波动程度。白话说，方差高表示模型很容易被少量样本带偏。

| 方法 | 优点 | 缺点 |
|---|---|---|
| 直接训练点击 | 简单，易实现 | 偏差大 |
| IPS | 理论上可无偏 | 方差高，依赖准确 propensity |
| SNIPS | 权重归一化后更稳定 | 可能引入轻微偏差 |
| DR | 更稳健 | 实现更复杂 |

SNIPS 是 Self-Normalized IPS，意思是把 IPS 权重再归一化，降低极端权重的影响。DR 是 Doubly Robust，中文可理解为“双重稳健”。它同时使用一个预测模型和 IPS 校正：如果 propensity 估得准，或者辅助模型估得准，结果都有机会保持较好质量。

## 代码实现

实现去偏训练的重点不是模型结构，而是日志字段是否正确、propensity 是否和日志策略一致、loss 是否按权重计算。

最小数据流是：读取日志，取出特征、点击标签、曝光概率，计算普通损失，再乘以权重。`label` 是点击或转化，`propensity` 是曝光概率，`tau` 是权重截断阈值。

```python
def clipped_ips_loss(labels, preds, propensities, tau=3.0):
    """
    labels: 0/1 点击标签
    preds: 模型预测的点击概率
    propensities: 样本被曝光的概率
    tau: 最大权重
    """
    eps = 1e-12
    losses = []
    weights = []

    for y, p_hat, p in zip(labels, preds, propensities):
        p_hat = min(max(p_hat, eps), 1 - eps)
        # binary cross entropy
        loss = -(y * __import__("math").log(p_hat) + (1 - y) * __import__("math").log(1 - p_hat))
        weight = min(1.0 / p, tau)
        losses.append(loss)
        weights.append(weight)

    weighted = sum(w * l for w, l in zip(weights, losses)) / len(labels)
    return weighted, weights


labels = [1, 0]
preds = [0.8, 0.2]
propensities = [0.8, 0.2]

loss, weights = clipped_ips_loss(labels, preds, propensities, tau=3.0)

assert weights == [1.25, 3.0]
assert loss > 0
assert round(loss, 3) == round((1.25 * 0.2231435513 + 3.0 * 0.2231435513) / 2, 3)
```

如果用深度学习框架，训练逻辑通常长这样：

```python
for batch in dataloader:
    x = batch["features"]
    y = batch["label"]
    p = batch["propensity"]

    pred = model(x)
    loss = criterion(pred, y)          # 每条样本一个 loss
    weight = torch.clamp(1.0 / p, max=tau)
    weighted_loss = (weight * loss).mean()

    optimizer.zero_grad()
    weighted_loss.backward()
    optimizer.step()
```

真实工程例子：短视频信息流里，顶部 3 个坑位天然高曝光，热门内容又更容易被点。直接用点击训练，模型会越来越偏向热门内容，长尾作者被压缩，内容多样性下降。常见做法是保留一小部分随机探索流量，记录不同位置和召回策略下的 propensity，再用 IPS 或 DR 训练排序模型。

训练之外还要记录分桶指标：

| 分桶维度 | 要观察什么 |
|---|---|
| 展示位置 | 每个位置的 CTR、加权 CTR |
| 头部/长尾物品 | NDCG、召回率、曝光占比 |
| 新老用户 | 点击率、停留时长、转化率 |
| 策略版本 | propensity 是否随版本变化 |

只看总体 CTR 或总体 NDCG 很危险。总体指标提升可能来自头部内容更强，但长尾覆盖、新作者曝光和用户多样性已经下降。

## 工程权衡与常见坑

去偏工程最核心的矛盾是“降偏差”和“控方差”。IPS 权重越激进，越能补偿低曝光样本，但也越容易放大噪声。权重越保守，训练越稳定，但偏差可能残留更多。

| 常见坑 | 后果 | 规避 |
|---|---|---|
| 把未点击当负样本 | 错把缺失当否定 | 先判断是否 MNAR / PU |
| propensity 估错 | 权重失真 | 记录日志策略版本 |
| 小 \(p\) 未截断 | 方差爆炸 | clipping / SNIPS / DR |
| 只看总指标 | 长尾问题被掩盖 | 分位置、分头尾、分人群分析 |
| 去掉所有流行度特征 | 损失有效信号 | 区分偏差与有用先验 |
| 离线和线上策略不一致 | 评估结论失效 | 训练、评估、线上对齐 |

MNAR 是 Missing Not At Random，意思是“缺失不是随机发生的”。推荐日志里的缺失通常就是 MNAR：一个物品没有点击，可能是它根本没被展示，而不是用户明确不喜欢。

PU 学习是 Positive-Unlabeled Learning，意思是只有正样本和未标注样本，而没有可靠负样本。推荐里的“点击”和“未曝光”经常接近这种设定。

工程上必须坚持三条原则。

第一，propensity 必须和日志策略一致。不同召回源、不同排序模型、不同实验桶产生的曝光概率可能不同，不能混在一起用一个固定值。

第二，训练、评估、线上三者要对齐。离线训练做了去偏，但线上继续让旧策略完全控制流量，新日志会继续偏，反馈回路不会自动消失。

第三，不能只盯全局指标。推荐系统服务的是不同用户、不同内容池、不同业务目标。去偏方法可能降低短期 CTR，但提升长尾覆盖、多样性、冷启动内容质量或长期留存。

## 替代方案与适用边界

去偏不是唯一答案。很多场景更适合“轻量校正 + 随机探索 + 在线实验”，而不是一上来实现复杂的因果模型。

如果只想解决“前排更容易被点”的问题，可能只做位置校正就够了。位置校正是按展示位置估计点击倾向，再降低位置对标签的污染。但如果热门内容被反复推荐形成马太效应，只校正位置不够，还需要处理流行度反馈回路。

| 方法 | 适用问题 | 代价 |
|---|---|---|
| 位置校正 | 排序位置信号偏差 | 仅覆盖部分偏差 |
| PU 学习 | 缺失标签场景 | 对先验假设敏感 |
| 随机探索桶 | 估计 propensity | 牺牲部分线上收益 |
| SNIPS / DR | 偏差与高方差并存 | 实现复杂 |
| 因果干预 | 流行度反馈回路 | 建模门槛高 |
| 数据增强 | 长尾样本不足 | 可能引入伪信号 |

因果推断是研究“如果改变某个因素，结果会怎样”的方法。放到推荐里，它关注的不只是相关性，而是“曝光、位置、流行度是否真的导致了点击变化”。

数据增强是构造更多训练样本的技术。推荐场景里可以通过负采样、长尾重采样、反事实样本生成等方式缓解数据稀疏，但它不能替代日志策略记录。没有可信的曝光信息时，数据增强很容易制造新的偏差。

适用边界可以这样判断：数据日志足够完整时，IPS/DR 更有效；没有可用 propensity 时，先做随机探索采样；只追求短期 CTR 时，重型去偏未必划算；如果业务更重多样性、公平性、冷启动和长尾覆盖，去偏价值更高。

对于候选集已经很干净、随机性很强的任务，复杂去偏方法未必比简单经验风险更划算。工程选择不是按公式高级程度排序，而是看偏差来源是否明确、日志字段是否足够、上线收益是否能被实验验证。
