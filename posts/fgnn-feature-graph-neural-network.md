## 核心结论

Fi-GNN，又称 FGNN，是一种面向 CTR 预测的特征图神经网络：它把每个特征域当成图节点，用图神经网络学习字段之间的交互关系，最后输出点击概率。

它的本质不是“在特征后面再堆一个更深的 DNN”，而是改变了特征交互的建模方式。传统 DNN 通常先把 `user_age`、`device_type`、`hour`、`ad_category` 等字段 embedding 拼接起来，再交给多层感知机。Fi-GNN 则先把这些字段分别作为节点，再让节点之间通过消息传递交换信息。消息传递是指：一个节点根据边权从其他节点接收向量，并更新自己的表示。

在广告 CTR 里，用户年龄、设备类型、广告类目、时间段不是简单拼接后丢进 DNN，而是先分别作为节点，再让“年龄”和“设备类型”、“时间段”和“广告类目”之间自动交换信息，模型会自己学出哪些组合更容易点击。

| 对比项 | 传统 DNN | Fi-GNN |
|---|---|---|
| 输入组织方式 | 字段 embedding 拼接 | 字段 embedding 作为图节点 |
| 交互方式 | 隐式交给 MLP 学习 | 显式通过边权和消息传递学习 |
| 交互阶数 | 依赖网络深度和参数 | 依赖图传播轮数 |
| 可解释性 | 较弱 | 可观察字段级边权和节点重要性 |
| 主要代价 | 参数量和训练调参 | 完全图带来的 $O(F^2)$ 计算 |

一个总览公式可以写成：

$$
\{x_i\}_{i=1}^{F}
\rightarrow
\{h_i^{(0)}\}_{i=1}^{F}
\rightarrow
\{\alpha_{ij}\}_{i,j=1}^{F}
\rightarrow
\{h_i^{(T)}\}_{i=1}^{F}
\rightarrow
\hat y
$$

其中 $F$ 是字段数，$h_i^{(0)}$ 是第 $i$ 个字段的初始向量，$\alpha_{ij}$ 是字段 $i$ 到字段 $j$ 的交互权重，$h_i^{(T)}$ 是经过 $T$ 轮传播后的字段表示，$\hat y$ 是点击概率。

Fi-GNN 的价值主要有两点：第一，它能学习高阶特征交互；第二，它能输出字段级重要性，便于解释和排查推荐、广告排序模型中的异常行为。

## 问题定义与边界

Fi-GNN 解决的是 CTR 预测中的特征交互建模问题。CTR 是 Click-Through Rate 的缩写，表示点击率；CTR 预测就是根据用户、广告、上下文等特征，预测一次曝光被点击的概率。

它的输入通常是多字段离散特征或稀疏特征。离散特征是取值来自有限集合的特征，例如设备类型、广告类目、城市、性别、时间段。稀疏特征是指取值空间很大但单条样本只命中少量取值的特征，例如用户 ID、商品 ID、query token。

比如一个广告样本包含 `user_age`、`device_type`、`hour`、`ad_category`、`query` 五个字段。Fi-GNN 不是去发现“用户和广告之间的真实社交关系”，而是把这五个字段当作五个节点，学习它们在一次点击预测中的协同作用。

| 元素 | 定义 | 在 Fi-GNN 中的形式 |
|---|---|---|
| 输入 | 字段 embedding | `[B, F, D]` 张量 |
| 输出 | 点击概率 | `[B, 1]` 标量概率 |
| 目标 | 学习字段交互 | 边权、消息聚合、节点更新 |
| 训练信号 | 是否点击 | 二分类标签 `0/1` |

这里的 embedding 是把离散 ID 映射成连续向量的方法。例如 `device_type=ios` 可以被映射成一个 16 维向量。模型不能直接理解字符串或类别 ID，需要先通过 embedding 表把它们变成可训练的数值表示。

Fi-GNN 的边界也要明确：字段级图结构是人为构造的，通常是完全图。完全图是指每个节点都和其他节点相连。图上的边不表示真实世界里的因果关系，只表示模型内部认为两个字段在预测任务中存在交互价值。

| 场景 | 是否适合 Fi-GNN | 原因 |
|---|---:|---|
| CTR 预测 | 适合 | 字段结构明确，交互关系重要 |
| CVR 预测 | 适合 | 转化率也依赖用户、商品、上下文交互 |
| 多字段稀疏特征排序 | 适合 | 字段之间存在复杂组合效应 |
| 普通图分类 | 不适合 | Fi-GNN 不是为真实拓扑图设计的通用模型 |
| 没有明确字段结构的表格任务 | 不一定适合 | 节点定义不稳定 |
| 需要因果解释的投放决策 | 不适合作为唯一依据 | 注意力权重不是因果效应 |

注意力权重是模型内部的交互强度，不是因果效应。比如模型认为 `hour` 和 `ad_category` 权重高，只能说明这两个字段的组合对当前模型预测有贡献，不能说明“某个时间段导致某类广告点击率上升”。因果结论需要实验设计、随机化、反事实分析等额外方法支持。

## 核心机制与推导

Fi-GNN 的核心流程是：

```text
字段建图 -> 注意力打分 -> 消息聚合 -> GRU 更新 -> 读出预测
```

字段建图是指把每个字段 embedding 看成一个节点。注意力打分是指模型为两个字段之间的边计算重要性分数。消息聚合是指目标节点从其他字段节点收集信息。GRU 更新是指用门控循环单元更新节点状态；GRU 是一种带门控结构的循环网络模块，可以控制新信息和旧信息的保留比例。读出预测是指把所有节点的最终表示汇总成点击概率。

以 `年龄`、`设备类型`、`广告类目` 三个字段为例，模型先计算 `年龄 -> 广告类目`、`设备类型 -> 广告类目` 等交互分数，再把更重要的邻居信息聚合到目标节点。这样 `年龄` 这个字段的最终表示，已经不只是它自己，而是融合了其他字段上下文后的表示。

关键公式可以抽象为四组。

第一步，计算边打分和归一化边权：

$$
e_{ij}=a([h_i^{(0)}\|h_j^{(0)}]),\quad
\alpha_{ij}=\frac{\exp(e_{ij})}{\sum_{k\neq i}\exp(e_{ik})}
$$

这里 $[h_i^{(0)}\|h_j^{(0)}]$ 表示向量拼接，$a(\cdot)$ 是可学习的打分函数，$\alpha_{ij}$ 是 softmax 后的边权。softmax 是一种归一化函数，会把多个分数转换成和为 1 的权重。

第二步，聚合邻居消息：

$$
m_i^{(t)}=\sum_{j\neq i}\alpha_{ji}\,U_i^{in}U_j^{out}h_j^{(t)}
$$

$m_i^{(t)}$ 表示第 $t$ 轮传播时聚合到节点 $i$ 的消息。$U_i^{in}$ 和 $U_j^{out}$ 是节点相关的变换矩阵，用于区分进入某个节点的信息和从某个节点发出的信息。

第三步，用 GRU 更新节点状态，并加上残差：

$$
h_i^{(t+1)}=\mathrm{GRU}(h_i^{(t)},m_i^{(t)})+h_i^{(0)}
$$

残差是指把初始表示加回去，避免多轮传播后原始字段信息被冲淡。对于初学者，可以把它理解为“更新后的节点仍然保留自己的原始身份”。

第四步，读出最终预测：

$$
\hat y=\sigma\!\left(\sum_i \beta_i\,\psi(h_i^{(T)})\right)
$$

$\sigma$ 是 sigmoid 函数，会把实数压到 $0$ 到 $1$ 之间，因此适合表示点击概率。$\beta_i$ 是输出阶段的节点重要性，$\psi(\cdot)$ 是对节点表示的变换函数。

| 符号 | 含义 |
|---|---|
| $h_i^{(0)}$ | 第 $i$ 个字段的初始向量 |
| $e_{ij}$ | 字段 $i$ 到字段 $j$ 的交互打分 |
| $\alpha_{ij}$ | 归一化后的边权 |
| $m_i^{(t)}$ | 第 $t$ 轮聚合到节点 $i$ 的消息 |
| $h_i^{(t)}$ | 第 $t$ 轮节点状态 |
| $\beta_i$ | 输出层中的节点重要性 |
| $\hat y$ | 预测点击概率 |

完全图加多轮传播能够表达高阶交互。第一轮传播时，节点 $A$ 可以直接接收 $B$、$C$ 的信息，因此得到二阶交互，例如 $A+B$、$A+C$。第二轮传播时，$B$ 的表示已经融合过 $C$，再传给 $A$ 时，$A$ 接收到的就不只是单独的 $B$，而是“带有 $C$ 上下文的 $B$”。这相当于表达了 $A$、$B$、$C$ 的组合关系。

玩具例子如下。设有 3 个字段 $A,B,C$，初始向量为：

$$
h_A^{(0)}=[1,0],\quad h_B^{(0)}=[0,1],\quad h_C^{(0)}=[1,1]
$$

只演示 $A$ 的一跳聚合。设打分为 $e_{AB}=0$，$e_{AC}=1$，softmax 后：

$$
\alpha_{AB}=\frac{1}{1+e}\approx 0.269,\quad
\alpha_{AC}=\frac{e}{1+e}\approx 0.731
$$

聚合消息为：

$$
m_A=0.269[0,1]+0.731[1,1]=[0.731,1.000]
$$

如果简化更新为残差相加：

$$
h_A^{(1)}=h_A^{(0)}+m_A=[1.731,1.000]
$$

含义是：字段 $A$ 从 $C$ 获得的信息比从 $B$ 获得的信息更多，因为 $\alpha_{AC}>\alpha_{AB}$。

## 代码实现

实现 Fi-GNN 可以拆成四层：字段 embedding 层、图构建与注意力层、消息传递层、预测头层。这样拆分的好处是调试清晰，后续也可以替换其中某个组件。

假设一个 batch 里每条样本有 10 个字段，每个字段 embedding 维度是 16。实现时先得到形状 `[B, F, D]` 的张量，再计算两两字段的交互分数，生成 `[B, F, F]` 的边权矩阵，最后做消息聚合并输出 `[B, 1]` 的点击概率。

伪代码如下：

```python
x = embed(fields)              # [B, F, D]
e = score_pairs(x)             # [B, F, F]
alpha = softmax(e, dim=-1)     # [B, F, F]
m = aggregate(x, alpha)        # [B, F, D]
h = gru_update(x, m)           # [B, F, D]
y = predict(readout(h))        # [B, 1]
```

一个只依赖 Python 标准库的可运行玩具实现如下，用于说明注意力边权和消息聚合，不等价于完整训练版 Fi-GNN：

```python
import math

def softmax(scores):
    max_score = max(scores)
    exps = [math.exp(s - max_score) for s in scores]
    total = sum(exps)
    return [v / total for v in exps]

def weighted_sum(weights, vectors):
    dim = len(vectors[0])
    out = [0.0] * dim
    for w, vec in zip(weights, vectors):
        for i in range(dim):
            out[i] += w * vec[i]
    return out

def add(a, b):
    return [x + y for x, y in zip(a, b)]

# 三个字段：A、B、C
h = {
    "A": [1.0, 0.0],
    "B": [0.0, 1.0],
    "C": [1.0, 1.0],
}

# 只演示 A 从 B、C 接收消息
scores = [0.0, 1.0]  # e_AB, e_AC
weights = softmax(scores)
message = weighted_sum(weights, [h["B"], h["C"]])
updated_a = add(h["A"], message)

assert round(weights[0], 3) == 0.269
assert round(weights[1], 3) == 0.731
assert [round(v, 3) for v in message] == [0.731, 1.0]
assert [round(v, 3) for v in updated_a] == [1.731, 1.0]

print(weights, message, updated_a)
```

从字段列表生成图输入时，通常先为每个字段维护一个 embedding 表。例如 `device_type` 有自己的 embedding 表，`ad_category` 有自己的 embedding 表，`hour` 也有自己的 embedding 表。每条样本取出每个字段对应的向量，再按字段顺序堆叠成 `[F, D]`；多个样本组成 batch 后就是 `[B, F, D]`。

| 组件 | 输入 | 输出 | 作用 |
|---|---|---|---|
| 输入层 | 字段 ID | 字段 embedding | 把离散特征转成向量 |
| 交互层 | `[B, F, D]` | `[B, F, F]` | 计算字段间边权 |
| 传播层 | 节点向量与边权 | 更新后的节点向量 | 聚合邻居信息 |
| 输出层 | `[B, F, D]` | `[B, 1]` | 输出点击概率 |

复杂度的关键在字段数 $F$。如果使用完全图，每个字段都要和其他字段计算交互，边数量接近 $F(F-1)$，因此交互计算是 $O(F^2)$ 级别。

| 字段数 $F$ | 有向边数量 $F(F-1)$ | 工程影响 |
|---:|---:|---|
| 10 | 90 | 成本较低，适合原型验证 |
| 50 | 2450 | 需要关注显存和 batch size |
| 100 | 9900 | 训练成本明显上升 |
| 200 | 39800 | 通常需要字段筛选或稀疏化 |

真实工程例子是广告 CTR 预估。字段可以包括用户年龄、设备类型、时间段、广告类目、上下文、query、历史点击类目、广告主 ID 等。每个字段作为一个节点，Fi-GNN 学的是“哪个字段和哪个字段一起更容易点击”。例如模型可能学到：夜间时段与游戏广告类目的交互较强，移动设备与短视频广告的交互较强，特定 query 与特定商品类目的交互较强。

## 工程权衡与常见坑

Fi-GNN 的表达能力强，但字段一多，完全图会迅速变贵。工程上不能把所有能拿到的字段都无脑塞进图里。字段越多，边越多，注意力矩阵、消息聚合、反向传播都会变重。

如果把 200 个字段都放进完全图，边数接近 4 万条，训练时显存和耗时都会明显上升。实际工程里通常只保留高价值字段，或者先做字段压缩，再进入图模块。

| 常见坑 | 后果 | 规避方式 |
|---|---|---|
| 把字段图当普通拼接特征 | 丢失图结构 | 保留节点、边权、消息传递 |
| 字段过多仍用完全图 | 计算爆炸 | 控制字段数、稀疏化 |
| 把注意力权重当因果 | 结论误判 | 仅解释模型内部重要性 |
| 传播层太深 | 过平滑 | 小层数 + 残差 |
| 样本极度不平衡 | 训练偏置 | 重采样、加权损失 |
| embedding 维度过大 | 参数和显存上升 | 先用小维度验证收益 |
| 字段顺序混乱 | 节点语义错位 | 固定字段 schema |

过平滑是图神经网络里的常见问题，意思是多轮传播后，不同节点的表示越来越相似，字段之间的差异被抹平。Fi-GNN 一般不需要很深的传播层数，通常先从 1 到 3 层验证。

样本不平衡也是 CTR 任务里的核心问题。点击样本通常远少于未点击样本，如果直接训练，模型可能倾向于预测“不点击”。常见处理方式包括负采样、样本重加权、调整损失函数和监控 AUC、LogLoss、校准度等指标。

工程建议是：先从浅层、少字段、可解释特征集开始验证。不要一开始就上全量字段和多层传播。更稳妥的路径是：

| 阶段 | 做法 | 目标 |
|---|---|---|
| 原型阶段 | 10 到 30 个高价值字段，1 层传播 | 验证模型是否能超过基线 |
| 扩展阶段 | 增加字段，监控显存和延迟 | 判断收益是否覆盖成本 |
| 解释阶段 | 查看边权、节点权重、特征分布 | 排查模型是否学到异常模式 |
| 上线阶段 | 做延迟压测、A/B 实验、校准 | 确认业务收益 |

一个常见排查场景是：AUC 上升，但解释结果混乱。例如注意力几乎都集中在某个 ID 字段，其他字段权重接近 0。这时不能只看指标，需要检查字段 embedding 是否塌缩、注意力分布是否过尖、某些字段是否泄漏了标签信息。标签泄漏是指训练特征中包含了预测目标的直接或间接信息，例如把点击后才产生的行为特征用于点击前预测。

## 替代方案与适用边界

Fi-GNN 不是 CTR 特征交互建模的唯一答案。它更适合需要字段级交互和可解释性的场景。如果更在意实现简单、训练稳定或高并发效率，FM、DeepFM、AutoInt、DCN 可能更合适。

FM 是 Factorization Machine 的缩写，核心是用向量内积建模二阶特征交互。DeepFM 把 FM 和 DNN 结合起来，同时学习低阶和高阶交互。AutoInt 使用自注意力显式学习特征交互。DCN 是 Deep & Cross Network，使用 Cross Layer 显式构造交叉特征。

如果字段数量少、交互模式相对简单，FM / DeepFM 往往更轻量；如果你想显式建模注意力交互，AutoInt 更直接；如果你需要字段级传播与解释，Fi-GNN 更有优势。

| 方法 | 交互方式 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|---|
| FM | 二阶内积 | 简单稳定 | 表达力有限 | 低复杂度 CTR |
| DeepFM | FM + DNN | 易用，工程成熟 | 交互仍偏隐式 | 通用 CTR |
| AutoInt | 自注意力 | 显式交互 | 不强调图传播 | 需要注意力解释 |
| DCN | Cross Layer | 显式交叉，效率较好 | 结构表达受限 | 工业排序模型 |
| Fi-GNN | 图消息传递 | 字段级交互、可解释 | 计算更贵 | 多字段交互建模 |

选择原则可以简化为：

| 关注点 | 优先方案 | 原因 |
|---|---|---|
| 训练和上线效率 | FM / DeepFM | 结构简单，延迟低 |
| 显式特征交互 | AutoInt / DCN | 交互层更直接 |
| 字段传播关系 | Fi-GNN | 节点间消息传递更自然 |
| 解释和排查 | Fi-GNN / AutoInt | 可以观察交互权重 |
| 强基线快速验证 | DeepFM | 成熟、稳定、易复现 |

Fi-GNN 的适用边界是：字段之间要有相对稳定的语义关系。比如广告 CTR 中，用户、广告、上下文字段长期存在固定含义，字段交互有业务解释空间。反过来，如果字段来源频繁变化、字段含义不稳定，或者样本只是普通连续数值表格，Fi-GNN 的收益可能不明显。

另一个边界是线上延迟。Fi-GNN 的完全图交互和多轮传播会增加推理成本。对于 QPS 极高的广告系统，模型结构不只看离线 AUC，还要看单次请求延迟、吞吐、特征服务成本和部署复杂度。离线提升很小但线上成本明显增加时，Fi-GNN 不一定是正确选择。
