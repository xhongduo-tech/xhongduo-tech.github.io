## 核心结论

策略迭代是一种在**已知环境模型**下求最优策略的方法。MDP，中文常写作马尔可夫决策过程，可以理解为“状态、动作、转移概率、奖励、折扣因子”都明确给定的决策系统。它的核心不是同时更新所有东西，而是严格交替两步：

1. 固定当前策略 $\pi$，算出这套策略在每个状态上的长期回报 $V^\pi$；
2. 用 $V^\pi$ 反推每个动作的好坏 $Q^\pi$，把策略改成当前看起来更优的动作，即
   $$\pi'(s)=\arg\max_a Q^\pi(s,a).$$

策略改进定理给出关键保证：如果在每个状态上，新策略选出的动作至少不比旧策略当前动作差，那么新策略整体不会比旧策略差。也就是说，策略迭代每轮都在“可证明地向上走”，最终收敛到最优策略 $\pi^\*$。

新手版本可以直接记成一句话：先把“当前玩法”每个状态能拿多少分算清楚，再把每个状态换成“按当前估分最好的动作”，重复直到不再变化。

下表概括最重要的公式与含义：

| 环节 | 公式 | 含义 |
| --- | --- | --- |
| 策略评估 | $V^\pi = R^\pi + \gamma P^\pi V^\pi$ | 固定策略后，求这套策略的状态价值 |
| 动作价值 | $Q^\pi(s,a)=\sum_{s',r}p(s',r\mid s,a)[r+\gamma V^\pi(s')]$ | 在状态 $s$ 先做动作 $a$，之后继续按 $\pi$ 行动的价值 |
| 策略改进 | $\pi'(s)=\arg\max_a Q^\pi(s,a)$ | 在每个状态选当前最优动作 |
| 改进保证 | 若 $Q^\pi(s,\pi'(s))\ge V^\pi(s)$，则 $V^{\pi'}(s)\ge V^\pi(s)$ | 新策略不会比旧策略差 |

和价值迭代相比，策略迭代通常**轮数更少、每轮更稳定**，但评估阶段更贵，因为常常要解一个 $|\mathcal S|$ 维线性系统。$\mathcal S$ 表示状态集合，也就是“所有可能状态的总表”。

## 问题定义与边界

策略迭代解决的问题是：给定一个有限 MDP，求一个最优策略 $\pi^\*$，使得从任意状态出发的期望折扣回报最大。折扣回报就是“未来奖励按时间衰减后的总和”，常写为
$$G_t=\sum_{k=0}^{\infty}\gamma^k R_{t+k+1}, \quad 0\le \gamma <1.$$

这里的边界非常重要。策略迭代不是“所有强化学习场景都能直接上”的通用锤子，它依赖以下前提：

| 要素 | 定义 | 本文默认假设 |
| --- | --- | --- |
| 状态空间 $\mathcal S$ | 系统可能处于哪些状态 | 有限且可枚举 |
| 动作空间 $\mathcal A$ | 每个状态可选哪些动作 | 有限且可枚举 |
| 奖励 $r$ | 每次转移得到的即时反馈 | 已知或可精确计算 |
| 折扣因子 $\gamma$ | 未来奖励折现强度 | 已知，且通常小于 1 |
| 转移模型 $p(s',r\mid s,a)$ | 做动作后到哪里、拿多少奖励 | 已知 |
| 评估方法 | 如何求 $V^\pi$ | 线性方程或迭代法可解 |

如果这些条件不满足，策略迭代就会变得困难，甚至不成立。比如模型未知时，你连 $Q^\pi(s,a)$ 都无法精确计算，只能依赖采样近似。

看一个边界明确的玩具例子。设只有两个状态 $s_0,s_1$，两个动作 $a_0,a_1$：

- 在 $s_0$ 执行 $a_0$：以概率 1 留在 $s_0$，奖励 1；
- 在 $s_0$ 执行 $a_1$：以概率 1 去 $s_1$，奖励 3；
- 在 $s_1$ 执行 $a_0$：以概率 1 留在 $s_1$，奖励 0；
- 在 $s_1$ 执行 $a_1$：以概率 1 回到 $s_0$，奖励 2。

若 $\gamma=0.9$，且模型完全已知，那么策略迭代可以精确运行。反过来，如果这些转移概率是线上日志粗略估出来的，而且噪声很大，那么“最优策略”可能只是对错误模型最优，而不是对真实环境最优。

所以策略迭代最适合的边界是：**中小规模、模型已知、希望得到稳定最优策略**。典型场景包括库存控制、离线调度、小型机器人路径规划、某些金融或运筹学中的离散控制问题。

## 核心机制与推导

策略迭代的推导可以拆成“评估”和“改进”两层。

先看策略评估。固定策略 $\pi$ 后，状态价值函数满足贝尔曼期望方程：
$$
V^\pi(s)=\sum_a \pi(a\mid s)\sum_{s',r}p(s',r\mid s,a)\big[r+\gamma V^\pi(s')\big].
$$

如果策略是确定性的，也就是每个状态只选一个动作，那么式子可写成矩阵形式：
$$
V^\pi = R^\pi + \gamma P^\pi V^\pi.
$$
整理得到
$$
(I-\gamma P^\pi)V^\pi=R^\pi.
$$
只要矩阵可逆，就可以直接解出 $V^\pi$。这就是“策略评估”的精确版本。

再看策略改进。得到 $V^\pi$ 以后，我们可以计算任意状态动作对的价值：
$$
Q^\pi(s,a)=\sum_{s',r}p(s',r\mid s,a)\big[r+\gamma V^\pi(s')\big].
$$
然后在每个状态上选择使 $Q^\pi$ 最大的动作：
$$
\pi'(s)=\arg\max_a Q^\pi(s,a).
$$

为什么这一步不会让策略变差？因为如果对任意状态 $s$ 都有
$$
Q^\pi(s,\pi'(s))\ge V^\pi(s),
$$
就说明“在第一步改成新动作、之后仍按旧策略走”都不会更差；继续反复替换后，可以推出整条轨迹的回报也不会更差，因此
$$
V^{\pi'}(s)\ge V^\pi(s).
$$
这就是策略改进定理的直观含义。

把流程写成简化“流程图”就是：

| 步骤 | 做什么 | 输出 |
| --- | --- | --- |
| 1. 策略评估 | 固定 $\pi$，求 $V^\pi$ | 当前策略的状态价值 |
| 2. 计算 $Q$ | 用 $V^\pi$ 计算所有 $Q^\pi(s,a)$ | 当前策略下的动作价值 |
| 3. 策略改进 | 每个状态选 $\arg\max_a Q^\pi(s,a)$ | 新策略 $\pi'$ |
| 4. 收敛判断 | 若 $\pi'=\pi$ 则停止 | 最优策略或稳定策略 |

数值上看一个最小例子。设两状态 $s_0,s_1$，两动作 $a_0,a_1$，且已经通过评估得到
$$V^\pi(s_0)=5,\quad V^\pi(s_1)=2.$$
若在 $s_0$ 采取动作 $a_1$ 后，立刻奖励为 4，并转移到 $s_1$，则
$$
Q^\pi(s_0,a_1)=4+0.9\times 2=5.8.
$$
若题设进一步给出更优转移，使得
$$
Q^\pi(s_0,a_1)=6>V^\pi(s_0)=5,
$$
那就说明在状态 $s_0$ 把动作换成 $a_1$ 一定更优或至少不差，因此策略应当改进。

新手理解时，不必先纠结所有证明细节。把它看成一个机械过程即可：先把“当前玩法能拿多少总分”填满，再在每个状态把动作换成“按当前总分算最划算的动作”。如果换完之后策略不再变化，就说明你已经到达一个稳定点；在有限 MDP 中，这个稳定点就是最优策略。

## 代码实现

下面给出一个可运行的 Python 版本。它使用**迭代式策略评估**，不是直接解线性方程。这样更容易读，也更接近工程里常见的实现。

```python
import numpy as np

gamma = 0.9

# P[s][a] = list of (prob, next_state, reward)
P = {
    0: {
        0: [(1.0, 0, 1.0)],   # s0, a0 -> s0, r=1
        1: [(1.0, 1, 3.0)],   # s0, a1 -> s1, r=3
    },
    1: {
        0: [(1.0, 1, 0.0)],   # s1, a0 -> s1, r=0
        1: [(1.0, 0, 2.0)],   # s1, a1 -> s0, r=2
    },
}

n_states = 2
n_actions = 2

def policy_evaluation(policy, theta=1e-10, max_iter=10000):
    V = np.zeros(n_states)
    for _ in range(max_iter):
        delta = 0.0
        for s in range(n_states):
            a = policy[s]
            v_new = sum(prob * (reward + gamma * V[s_next]) for prob, s_next, reward in P[s][a])
            delta = max(delta, abs(v_new - V[s]))
            V[s] = v_new
        if delta < theta:
            break
    return V

def q_value(V, s, a):
    return sum(prob * (reward + gamma * V[s_next]) for prob, s_next, reward in P[s][a])

def policy_improvement(V):
    new_policy = np.zeros(n_states, dtype=int)
    for s in range(n_states):
        qs = [q_value(V, s, a) for a in range(n_actions)]
        new_policy[s] = int(np.argmax(qs))
    return new_policy

def policy_iteration():
    policy = np.zeros(n_states, dtype=int)  # 初始策略：全选 a0
    while True:
        V = policy_evaluation(policy)
        new_policy = policy_improvement(V)
        if np.array_equal(new_policy, policy):
            return policy, V
        policy = new_policy

policy, V = policy_iteration()

assert policy.shape == (2,)
assert np.allclose(policy, np.array([0, 1]))
assert V[0] > V[1]
assert V[0] > 9.9  # s0 选 a0 会长期拿到接近 10 的回报
```

这段代码对应的伪代码就是：

```python
while True:
    V = policy_evaluation(pi)
    pi_new = greedy_policy(V)
    if pi_new == pi:
        break
    pi = pi_new
```

关键变量如下：

| 变量 | 含义 | 白话解释 |
| --- | --- | --- |
| $\pi$ | 策略 | 每个状态该选哪个动作 |
| $V$ | 状态价值函数 | 从这个状态开始，长期能拿多少分 |
| $Q$ | 动作价值函数 | 在这个状态先做某动作，再继续按策略走，能拿多少分 |
| $\theta$ | 评估阈值 | 价值变化小到什么程度就认为“算够了” |

真实工程例子可以看离线库存控制。状态是“当前库存量”，动作是“补货多少”，奖励是“销售利润减去库存成本和缺货惩罚”。若需求分布已知，就能得到转移概率。此时策略迭代做的事情很直接：

- 先固定一套补货规则，算出每个库存状态的长期利润；
- 再对每个库存状态重新选择更优补货量；
- 重复直到补货规则稳定。

这种问题规模通常不算特别大，模型又相对明确，因此策略迭代往往比纯采样式方法更稳。

## 工程权衡与常见坑

策略迭代的最大优点是“每轮更新方向明确”，最大代价是“评估很贵”。如果用精确解法，策略评估往往要处理一个 $|\mathcal S| \times |\mathcal S|$ 的线性系统，直接求解的代价通常可视为 $O(|S|^3)$。这在几百、几千状态时还可能接受，但在更大状态空间中就会迅速失控。

工程上常见权衡如下：

| 方面 | 优点 | 缺点 | 应对措施 |
| --- | --- | --- | --- |
| 收敛稳定性 | 每轮有改进保证 | 单轮计算重 | 用截断评估降低单轮成本 |
| 迭代轮数 | 通常少于价值迭代 | 单轮更复杂 | 小规模问题优先考虑 |
| 可解释性 | 策略和价值都清楚 | 依赖模型准确 | 仅在模型可信时使用 |
| 精确性 | 可得到严格最优 | 大规模状态难处理 | 用近似策略迭代或函数逼近 |

一个容易犯的坑是**评估过度**。也就是在每轮改进前，把 $V^\pi$ 算得极其精确，但下一步策略一改，这些高精度结果马上失去一部分价值。很多实际系统更适合“部分评估 + 立即改进”，这就是修正策略迭代的思想。

另一个坑是**收敛判定写错**。正确做法不是盯着 $V$ 是否完全不动，而是优先看策略是否变化。因为策略迭代的终止条件本质上是：贪心改进后，策略已经不再改变。

可以把收敛判断简化成下面的逻辑表：

| 判断项 | 结论 |
| --- | --- |
| 新旧策略完全一致 | 停止，已收敛 |
| 新旧策略不同，但 $V$ 已很稳定 | 继续，因为策略还在变 |
| 评估迟迟不收敛 | 检查 $\gamma$、模型定义、最大迭代次数、数值误差 |
| 模型来自估计且波动大 | 不宜直接相信结果，应换样本法或鲁棒方法 |

新手版常见坑还有三个：

| 错误 | 后果 | 修正 |
| --- | --- | --- |
| 评估循环没有最大步数 | 可能死循环或极慢 | 加 `max_iter` |
| 改进时没有比较全部动作 | 策略可能停在错误点 | 对每个状态枚举所有动作 |
| 模型未知还硬套策略迭代 | 公式能写，结果不可信 | 改用采样式方法 |

## 替代方案与适用边界

策略迭代不是唯一选择。判断标准主要看两件事：**模型是否已知**，以及**状态空间是否足够小**。

最常见的替代方案对比如下：

| 算法 | 是否需要模型 | 计算特征 | 优势 | 劣势 | 适用场景 |
| --- | --- | --- | --- | --- | --- |
| 策略迭代 | 需要 | 评估贵，轮数少 | 稳定、收敛快 | 单轮成本高 | 中小规模已知模型 |
| 值迭代 | 需要 | 单轮便宜，轮数多 | 实现简单 | 更新较慢、震荡更明显 | 小规模已知模型 |
| 近似策略迭代 | 通常需要近似模型或估计器 | 用近似值函数替代精确评估 | 能扩展到更大状态空间 | 理论和调参更复杂 | 中大规模问题 |
| Q-learning / SARSA | 不需要精确模型 | 依赖采样 | 模型未知时可用 | 样本效率受限 | 在线交互环境 |
| 策略梯度 | 不需要精确模型 | 直接优化参数化策略 | 适合连续动作 | 方差大、训练不稳 | 高维连续控制 |

值迭代和策略迭代的差异可以概括为：

- 值迭代：每轮都只做一步“局部最优备份”，省单轮成本。
- 策略迭代：先把当前策略评估清楚，再整体改策略，省总轮数。

如果你没有“地图”，也就是没有转移模型 $p(s',r\mid s,a)$，就不要强行做策略迭代。因为它的核心步骤正是依赖模型算期望。此时更合理的是使用基于样本的方法，用环境交互数据近似价值或直接更新策略。

可以用一个简单决策流程判断：

| 条件 | 更适合的选择 |
| --- | --- |
| 模型已知，状态不大，追求稳定最优 | 策略迭代 |
| 模型已知，但想快速写一个基线 | 值迭代 |
| 模型已知，但状态很大 | 近似策略迭代 |
| 模型未知，只能靠交互采样 | Q-learning、SARSA、策略梯度 |
| 动作连续且高维 | 策略梯度或 actor-critic |

所以，策略迭代的适用边界可以压缩成一句话：**它是已知模型条件下的强方法，不是未知模型场景下的通用方法。**
