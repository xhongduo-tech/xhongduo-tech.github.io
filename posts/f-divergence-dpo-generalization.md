## 核心结论

f-Divergence DPO，论文里更常写作 f-PO，本质上是在做一件事：把 DPO 里固定使用的 KL 散度，推广成一整个 $f$-散度家族。散度可以理解成“两个概率分布有多不一样”的度量。这样做的结果不是换个名字，而是把“偏好优化该怎么惩罚偏离人类偏好的行为”变成一个可选设计项。

对初学者，最直接的理解是：

1. 普通 DPO 可以看成“用反向 KL 把模型往人类更喜欢的回答方向拉”。
2. f-PO 说，这个“拉”的方式不必只用 KL，也可以用 JS、$\chi^2$、Jeffrey、$\alpha$-divergence。
3. 不同散度的区别，体现在它们对“概率偏差”的敏感程度不同，因此训练稳定性、对异常样本的鲁棒性、对长回答或低概率回答的偏好也不同。

这也是 f-PO 的工程价值。它不是发明了新的监督信号，还是用 pairwise preference，也就是“回答 A 比回答 B 更好”这种成对偏好数据；它改的是目标函数的几何形状。目标函数的几何形状，白话说就是“模型犯错时，梯度会怎么推它回来”。

一个够用的判断准则是：

| 方法 | 代表散度 | 训练行为 | 常见印象 |
|---|---|---|---|
| DPO | 反向 KL | 更强调模式贴合 | 经典、稳定、易用 |
| EXO 类 | 正向 KL | 更强调覆盖性 | 对尾部概率更敏感 |
| JS 版 f-PO | JS 散度 | 更平滑，早期更稳 | 适合预热 |
| $\chi^2$ / Jeffrey 版 f-PO | 更高曲率散度 | 梯度更激进 | 可能更强，也更容易炸 |

如果只记一句话：f-Divergence DPO 不是替代 DPO 的陌生算法，而是把 DPO 放进一个更大的统一框架里，让你可以按任务需要选择“更平滑”还是“更敏感”的正则化方式。

## 问题定义与边界

我们先把问题说准。

偏好优化要解决的是：给定一个提示词 $x$，以及两个候选回答 $y_w, y_l$，其中 $y_w$ 是人类更喜欢的回答，$y_l$ 是较差回答，如何把模型 $\pi_\theta(y|x)$ 训练得更偏向前者。

f-PO 的目标写成：

$$
\min_\theta D_f(\hat\pi_\theta \| \hat\pi^*)
$$

这里：

- $\hat\pi_\theta$ 是当前模型诱导出的策略分布。
- $\hat\pi^*$ 是由偏好数据构造出来的“理想策略”近似。
- $D_f$ 是 $f$-散度。

“理想策略”不是说真实世界里存在一个完美分布，而是说：根据现有 pairwise preference 数据，我们可以构造一个经验目标，让模型在这些被比较过的回答上尽量贴近“人类会选中的概率结构”。

$f$-散度的通式是：

$$
D_f(p\|q)=\mathbb{E}_{x\sim q}\left[f\left(\frac{p(x)}{q(x)}\right)\right]
$$

这里的密度比 $\frac{p(x)}{q(x)}$ 可以理解成“当前策略相对目标策略偏了多少”。

但不是任何函数都能拿来当 $f$。它至少要满足一组边界条件：

| 条件 | 含义 | 为什么需要 | 违反后果 |
|---|---|---|---|
| $f(1)=0$ | 两个分布相同时距离为 0 | 保证“完全对齐”是零损失 | 最优点位置会错 |
| 凸性 | 曲线向上弯 | 保证散度非负、优化有理论基础 | 可能出现假极小值 |
| 在 $\mathbb{R}_+$ 上定义 | 作用于正密度比 | 概率比值不能为负 | 公式失效 |
| 足够平滑或可求导 | 能算梯度 | 训练要靠反向传播 | 梯度不稳定或不可算 |

所以工程里不能“随便拼一个函数”就说自己在做 f-PO。合法的 generator 必须来自满足这些性质的函数族。

一个玩具例子能帮助理解边界。假设只有两个回答：

- 人类理想分布：$\pi^*=(0.8, 0.2)$
- 当前模型分布：$\pi_\theta=(0.6, 0.4)$

这说明模型把“差回答”给得太高了。偏好优化的任务，就是把 $(0.6,0.4)$ 推向 $(0.8,0.2)$。不同散度的区别，不在目标方向，而在“推过去的力度曲线”不同。

## 核心机制与推导

先看 DPO 为什么能被放进 f-PO 框架。

对一个提示 $x$ 和一对回答 $(y_w, y_l)$，DPO 关心的是两者相对 log-prob 的差：

$$
\Delta_\theta = \log \pi_\theta(y_w|x) - \log \pi_\theta(y_l|x)
$$

再与参考模型 $\pi_{\text{ref}}$ 的对应差值比较。它本质上是在说：赢家回答的相对概率，应该比输家回答更高，而且不要偏离参考模型过头。

f-PO 把这件事抽象成“最小化当前策略与理想策略之间的 $f$-散度”。关键在于，不同 $f$ 会给出不同的导数 $f'(u)$，而导数决定梯度如何响应密度比 $u=\frac{p}{q}$ 的偏差。

一般地，若

$$
D_f(p\|q)=\mathbb{E}_{q}\left[f\left(\frac{p}{q}\right)\right]
$$

那么对密度比 $u$ 的局部惩罚由 $f'(u)$ 决定：

- 若 $f'(u)$ 在 $u$ 很大时增长很快，说明对偏离会更敏感。
- 若 $f'(u)$ 更平缓，说明目标更鲁棒，训练更稳。

几个典型 generator 可以放在一起看：

| generator $f(u)$ | 对应散度 | 典型行为 |
|---|---|---|
| $u\log u$ | 正向 KL | 强调覆盖目标分布 |
| $-\log u + u - 1$ | 反向 KL | DPO 相关特例，更强调模式贴合 |
| $u\log u-(u+1)\log\frac{u+1}{2}$ | JS | 更平滑，有限且稳 |
| $(u-1)^2$ | $\chi^2$ | 对大偏差很敏感 |
| $(u-1)\log u$ | Jeffrey | 两边都敏感，曲率较高 |

这里“曲率”可以理解成“损失曲线弯得有多厉害”。弯得越厉害，梯度波动通常越大。

### 玩具例子：为什么 JS 更平滑

继续用刚才那组分布：

- $\pi^*=(0.8,0.2)$
- $\pi_\theta=(0.6,0.4)$
- 中间分布 $m=\frac{1}{2}(\pi^*+\pi_\theta)=(0.7,0.3)$

JS 散度定义为：

$$
D_{JS}(p\|q)=\frac{1}{2}D_{KL}(p\|m)+\frac{1}{2}D_{KL}(q\|m)
$$

代入可得一个大约 $0.024$ 的值。这个数不大，说明虽然模型偏了，但 JS 给出的惩罚比较温和。它不会像高曲率散度那样，立刻把一次偏差放大成特别尖锐的梯度。

这对训练初期很重要。因为训练刚开始时，$\pi_\theta$ 往往离 $\pi^*$ 还比较远；如果一上来就用很激进的散度，梯度可能非常躁动。

### 从 pairwise preference 到损失

pairwise preference 数据给的是“谁赢谁输”，不是 reward 标量。f-PO 的思想是：从这种二元偏好中恢复一个相对目标分布，然后最小化当前策略到它的距离。

直观映射可以写成：

$$
(x, y_w, y_l)
\rightarrow
\Delta_\theta
\rightarrow
u_\theta
\rightarrow
f(u_\theta)
\rightarrow
\mathcal{L}_{f\text{-PO}}
$$

这里：

- $\Delta_\theta$ 是赢家与输家的相对 logit 差。
- $u_\theta$ 是由该差值诱导出的密度比。
- $f(u_\theta)$ 是具体散度的局部惩罚。
- 把样本平均后就是训练损失。

这件事的统一性在于：

- 选反向 KL，就 recover 到 DPO 一类目标。
- 选正向 KL，就能得到 EXO 一类行为。
- 选 JS、$\alpha$、Jeffrey，就得到更广义的偏好优化形式。

所以 f-PO 的核心不是重新定义“人类喜欢什么”，而是重新定义“模型离人类偏好有多远”。

## 代码实现

如果你只是想把概念落到代码，先理解一个最小实现。下面代码不依赖大模型，只演示“同一组分布下，不同散度数值不同”。

```python
import math

def kl(p, q):
    return sum(pi * math.log(pi / qi) for pi, qi in zip(p, q))

def reverse_kl(p, q):
    return kl(q, p)

def js_divergence(p, q):
    m = [(pi + qi) / 2 for pi, qi in zip(p, q)]
    return 0.5 * kl(p, m) + 0.5 * kl(q, m)

def chi_square(p, q):
    return sum(((pi - qi) ** 2) / qi for pi, qi in zip(p, q))

p_star = [0.8, 0.2]   # 理想策略
p_theta = [0.6, 0.4]  # 当前策略

js = js_divergence(p_star, p_theta)
fwd_kl = kl(p_star, p_theta)
rev_kl = reverse_kl(p_star, p_theta)
chi2 = chi_square(p_theta, p_star)

assert js > 0
assert fwd_kl > 0
assert rev_kl > 0
assert chi2 > 0
assert js < fwd_kl  # 这组例子里，JS 更平滑，数值更小

print({
    "JS": round(js, 6),
    "KL(p*||p_theta)": round(fwd_kl, 6),
    "KL(p_theta||p*)": round(rev_kl, 6),
    "Chi^2": round(chi2, 6),
})
```

这段代码能直接运行。它说明的不是“哪个散度永远更小”，而是：同样的偏差，不同散度会给出不同惩罚形状。

如果进入真实训练，代码层面通常不是手写散度，而是在现有仓库里通过配置切换 generator。fPO 开源仓库给了现成脚本、配置和 checkpoint。一个典型命令形态是：

```bash
CUDA_VISIBLE_DEVICES=0,1,2,3 \
accelerate launch scripts/run_simpo.py \
training_configs/llama-3-8b-base-alphapo.yaml
```

真实工程例子可以这样理解：你有一批安全对齐数据，每条样本是“同一个问题下，安全回答优于危险回答”。如果你用 Llama3-8B 做偏好优化，早期可能先选 JS 或较温和的 $\alpha$-divergence，让模型先稳定学会“不要选明显危险的回答”；等模型已经进入合理区域，再换成更敏感的散度，进一步优化长回答质量、拒答风格或尾部行为。

一个简化版训练流程通常是：

| 阶段 | 目标 | 推荐散度 |
|---|---|---|
| 阶段一 | 冷启动，避免梯度过猛 | JS / 温和 $\alpha$ |
| 阶段二 | 稳定提升偏好胜率 | 反向 KL / 中等 $\alpha$ |
| 阶段三 | 强化尾部或长回答表现 | Jeffrey / $\chi^2$ / 更激进 $\alpha$ |

对零基础读者，一个重要认识是：这里切的不是“模型结构”，而是“损失函数的形状”。模型还是那个模型，数据也还是 pairwise preference，变的是优化器看到的误差信号。

## 工程权衡与常见坑

f-PO 的主要工程权衡，不是“能不能训”，而是“训得多稳、训得多快、最后偏向什么行为”。

第一类权衡是稳定性与攻击性。

- JS、温和的 $\alpha$ 散度更平滑，适合训练前期。
- $\chi^2$、Jeffrey 这类高曲率散度更敏感，可能更能放大偏好差异，但也更容易带来梯度爆炸或损失震荡。

第二类权衡是模式贴合与分布覆盖。

- 反向 KL 倾向于把概率集中到高质量模式上，容易得到“像人类偏好中的赢家”的回答。
- 正向 KL 倾向于覆盖目标分布支撑集，对低概率但可能有价值的回答更敏感。

第三类权衡是短回答与长回答。

有些评测上，某些散度会把模型推向更保守、更短、更稳的回答；另一些散度则更鼓励保留长尾表达能力，长回答质量可能更好，但波动也更大。

下面这张表可以直接当工程提醒：

| 散度 | 梯度方差趋势 | 对密度比大偏差的敏感度 | 工程建议 |
|---|---|---|---|
| JS | 低到中 | 较低 | 预热、冷启动优先 |
| 反向 KL | 中 | 中 | 默认首选，接近 DPO |
| 正向 KL | 中到高 | 对覆盖不足敏感 | 适合强调分布覆盖 |
| $\chi^2$ | 高 | 很高 | 需要更小学习率与裁剪 |
| Jeffrey | 高 | 两侧都敏感 | 适合后期精修，不适合乱用 |

常见坑主要有四个。

第一，训练早期就上高曲率散度。  
如果模型和目标分布差得很远，密度比 $u$ 可能非常极端，这时 $f'(u)$ 会给出很大的梯度。结果不是“学得更快”，而是 checkpoint 直接被破坏。

第二，忽略归一化或分母稳定性。  
很多实现里的问题不在理论，而在数值。密度比、log-ratio、参考模型差值如果没有处理好数值范围，loss 会不稳定。

第三，把散度选择当成“越强越好”。  
这不成立。更敏感的散度只是更强调某些偏差，不代表泛化更好。很多场景下，平滑目标反而更稳、更可复现。

第四，只看单一 benchmark。  
如果某个散度在 AlpacaEval 2 上更好，不代表在安全拒答、长上下文、长链式推理场景也更好。散度会改变模型偏好的风格，而不仅是平均分数。

一个实用流程是：

1. 先用 JS 预热，让模型进入合理区域。
2. 再切到反向 KL 或中等 $\alpha$，追主指标。
3. 若确认任务需要强调尾部质量，再尝试 Jeffrey 或 $\chi^2$。
4. 每次只改一个变量：散度、学习率、温度、长度惩罚不要同时乱动。

## 替代方案与适用边界

f-PO 的位置可以一句话概括：它是 DPO、EXO 这类配对偏好优化方法的统一上层框架。

所以替代方案不是“是否做偏好优化”，而是“用哪种偏好优化”。

| 场景 | 推荐方法 | 原因 |
|---|---|---|
| 想快速复现、先要稳 | DPO / 反向 KL | 社区经验最多，行为可预期 |
| 训练初期不稳 | JS 版 f-PO | 惩罚更平滑 |
| 希望对低概率但高价值回答更敏感 | 正向 KL / $\chi^2$ / Jeffrey | 更强调尾部 |
| 只想少量改动现有 DPO 管线 | f-PO with config switch | 改损失，不改数据格式 |
| 数据噪声很大 | JS / 温和 $\alpha$ | 对异常样本更鲁棒 |

适用边界也要说清楚。

第一，f-PO 不会替你解决偏好数据质量问题。  
如果 pairwise preference 本身标错了，换再好的散度也只是更高效地拟合错误标注。

第二，f-PO 不是 reward model 的完全替代。  
它减少了对显式 reward labeling 的依赖，但如果任务需要复杂多目标控制，reward model 仍然有价值。

第三，f-PO 不保证所有任务都优于 DPO。  
它提供的是“更灵活的目标设计空间”，不是“无条件更强”。

第四，越灵活意味着越需要实验纪律。  
多了散度这个超参数后，实验矩阵会变大。没有清晰假设就乱扫参数，结果通常不可解释。

可以把选择规则压缩成一句实话：

- 不确定时，用 DPO 或 JS。
- 需要更稳，用 JS。
- 需要更敢于调整尾部行为，再考虑 $\alpha$、Jeffrey、$\chi^2$。
- 如果数据少、噪声大、算力紧，优先选平滑散度，不要追求激进目标。
