## 核心结论

重复 Token 指训练集中语义几乎相同、只是在不同页面或不同位置反复出现的文本；白话说，就是模型被迫把同一句话看很多遍。结论很直接：重复数据不是“便宜的新数据”，它的边际价值会快速衰减，而且衰减速度可以被量化。

Hernandez 等人的核心发现可以概括成两句话：

1. 等量重复数据远不如等量新数据。把同一段文本重复看 100 次，并不等于看了 100 段新文本；在一个常见设定下，这 100 次重复合起来只相当于大约 3.4 次新样本。
2. 重复如果在不同 domain 中分布不均，会改变模型能力分布。domain 指数据来源类别，比如新闻站、论坛、文档站；白话说，就是模型会被某些“模板化很重”的来源拉偏。

先看一个玩具例子。假设你要学编程：

| 学习方式 | 读到的材料 | 新信息量 |
|---|---:|---:|
| A | 100 篇不同教程 | 高 |
| B | 同一篇教程重复 100 次 | 很低 |
| C | 1 篇教程 + 99 篇相似改写 | 中低 |

模型训练也是同样逻辑。优化器每多看一次重复文本，确实还能继续降低训练损失，但它主要学到的是“如何更稳定地记住这一类字符串”，而不是“如何覆盖新的语言结构和新知识”。

用有效数据量 $D_0$ 来表示“这些数据等价于多少新数据”，重复收益会呈现饱和曲线：

$$
D_0 = UD + UD \cdot R_D^* \cdot \left(1 - e^{-RD / R_D^*}\right)
$$

其中：

- $UD$ 是唯一数据量，指真正不重复的 Token 数；白话说，就是第一次出现的那部分内容。
- $RD$ 是重复强度，表示额外重复了多少轮。
- $R_D^* = \frac{1-\delta}{\delta}$，$\delta$ 是重复率；白话说，$\delta$ 越大，重复越早失去价值。

一个直观看法是：$R_D^*$ 决定了“重复价值半衰期”。当 $RD$ 超过这个量级后，再继续加重复，新增有效数据几乎不再增长。

简化图示如下：

```text
有效贡献
^
|                         新 Token：近似线性增长
|                        /
|                       /
|                      /
|         重复 Token  _/
|                  __/
|               __/
|            __/
+----------------------------------> 重复次数 RD
                 ~ R*_D 后开始明显饱和
```

这件事为什么重要？因为在大规模爬虫语料里，重复不是均匀噪声，而是高度集中在模板页、导航页、parked page、镜像页等区域。也就是说，重复不仅减少总有效数据，还会把“某些很低信息密度的模式”放大成训练主信号。

## 问题定义与边界

这篇文章讨论的是**预训练阶段的大规模文本重复**，不是微调阶段故意重复少量高价值样本。微调里少量重复常常是有意设计，用来强化特定行为；预训练里大规模重复更多是采集副作用。

问题可以定义为：

> 在固定训练预算下，重复 Token 会怎样改变“有效数据量”和“最终泛化性能”？

这里有四个边界量最重要：

| 变量 | 含义 | 工程解释 | 风险信号 |
|---|---|---|---|
| $\delta$ | 重复率 | 数据里有多大比例可被视为重复 | 高于预期时说明采集源有问题 |
| $UD$ | 唯一 Token 数 | 真正提供新信息的部分 | 太小会限制模型上限 |
| $RD$ | 重复强度 | 相同模式被反复看到多少次 | 大时容易浪费算力 |
| double descent 风险 | 损失曲线二次抬升 | 训练先变好，再因重复主导而恶化 | 模板域过重时明显 |

double descent 指测试损失不是单调下降，而是在某个区间再次上升；白话说，就是模型先学到一般规律，后来又被重复样本拉向记忆化。

这类问题在 Common Crawl、C4 这类网页预训练语料里最显著。原因不是网页数量不够，而是网页结构高度不均匀。真实工程里，经常出现下面的数据流：

```text
Common Crawl
   ↓
按域名 / 模板聚类
   ↓
识别 parked page / CDN 模板页 / 镜像页
   ↓
估算 UD、RD、δ
   ↓
去重或重加权
   ↓
送入训练采样器
```

一个真实工程例子：某团队直接抓取 Common Crawl 中一个广告联盟相关域簇的数据，发现其中大量页面只有品牌名、跳转链接、页脚法律声明不同，正文模板几乎相同。若这种模板页占到该域 23% 左右，那么训练时模型会在这类页面上 loss 很低，因为它们太好记；但换到非模板技术文档、论坛问答时，loss 却明显更高。这不是模型“变笨”，而是训练预算被低信息重复模式消耗了。

因此，问题边界不是“重复要不要彻底消灭”，而是：

1. 重复是否集中在低信息域。
2. 重复是否足以让有效数据量明显低于名义数据量。
3. 重复是否正在把训练推向 double descent 风险区。

## 核心机制与推导

核心机制可以拆成两步。

第一步，定义重复价值半衰期：

$$
R_D^* = \frac{1-\delta}{\delta}
$$

它表示在当前重复率下，重复还能“值多少钱”。如果 $\delta = 0.25$，那么：

$$
R_D^* = \frac{1-0.25}{0.25} = 3
$$

这表示重复的有效增益大约在 3 次量级后就开始明显衰减。

第二步，用指数饱和来表示重复带来的有效数据增益：

$$
D_0 = UD + UD \cdot R_D^* \cdot \left(1 - e^{-RD / R_D^*}\right)
$$

这个式子分成两部分理解更容易：

1. 第一项 $UD$：原本唯一数据的价值。
2. 第二项：重复数据能额外贡献的有效价值，但它不是线性增长，而是逐渐饱和。

把第二项展开看：

$$
\Delta D = UD \cdot R_D^* \cdot \left(1 - e^{-RD / R_D^*}\right)
$$

当 $RD$ 很小时，利用 $e^{-x} \approx 1-x$，有：

$$
\Delta D \approx UD \cdot R_D^* \cdot \frac{RD}{R_D^*} = UD \cdot RD
$$

也就是说，在非常少量重复时，它看起来近似线性，像是“还有点用”。

但当 $RD$ 很大时，$e^{-RD / R_D^*} \to 0$，于是：

$$
D_0 \to UD + UD \cdot R_D^*
$$

上界出现了。再重复下去，名义 Token 继续增加，但有效数据几乎不再增加。这就是“重复偷走训练容量”的数学表达。

玩具例子直接代入。令 $UD=1$，$\delta=0.25$，则 $R_D^*=3$。若重复强度 $RD=100$：

$$
D_0 = 1 + 1 \cdot 3 \cdot (1 - e^{-100/3}) \approx 1 + 3 = 4
$$

更精确地说，约为 3.99。也就是说：

- 名义上你有 1 个唯一样本 + 100 次重复。
- 实际有效价值只有约 4 个唯一样本。
- 100 次重复总共只贡献约 3 个“新样本等价量”。
- 平均每次重复的价值约是 $3.4\%$。

这就是“100 次重复约等于 3.4 次新数据”的来源。

再看曲线直觉：

```text
D0/UD
^
|                    ________ 上界 = 1 + R*_D
|                 .-'
|              .-'
|           .-'
|        .-'
|     .-'
|  .-'
+---------------------------------> RD
      0      R*_D      5R*_D
```

当 $\delta$ 较低时，$R_D^*$ 会变大，意味着重复价值衰减得稍慢；但这不表示可以放心重复。原因是只要某些 domain 中的重复高度集中，局部 $RD$ 仍会快速冲到饱和区。

再把它和能力分布联系起来。假设训练集分成两个域：

| 域 | 名义 Token | 重复率 | 有效信息密度 |
|---|---:|---:|---:|
| 技术文档 | 100M | 低 | 高 |
| 模板页面 | 100M | 高 | 低 |

如果只看 Token 数，这两个域权重一样；但看 $D_0$ 后，模板页面可能只相当于几百万到几千万“新 Token”。问题在于优化器并不知道“这部分信息密度低”，它只会持续降低这类样本的训练损失。结果就是模型把容量花在模板记忆上，跨域泛化能力被压缩。

这也是为什么论文会观察到：哪怕总重复比例不高，只要重复强度集中，就足以明显破坏 scaling 行为，甚至让更大的模型表现得像更小的模型。

## 代码实现

工程上最实用的做法不是“先训练再看运气”，而是在数据管道里直接估算每个域或模板簇的有效价值，然后决定降权还是保留。

先给一个最小可运行实现。它输入 $\delta$、$RD$、$UD$，输出 $R_D^*$、有效数据量 $D_0$ 和建议采样权重。

```python
import math


def repeated_data_value(delta: float, rd: float, ud: float = 1.0):
    assert 0.0 < delta < 1.0
    assert rd >= 0.0
    assert ud > 0.0

    r_star = (1.0 - delta) / delta
    d0 = ud + ud * r_star * (1.0 - math.exp(-rd / r_star))

    # 一个简单的降权策略：
    # 重复越超过半衰期，采样权重越低
    weight = 1.0 / (1.0 + rd / r_star)

    return {
        "R*_D": r_star,
        "D0": d0,
        "gain_per_repeat": (d0 - ud) / rd if rd > 0 else 0.0,
        "weight": weight,
    }


toy = repeated_data_value(delta=0.25, rd=100, ud=1.0)
assert abs(toy["R*_D"] - 3.0) < 1e-9
assert 3.98 < toy["D0"] < 4.01
assert 0.029 < toy["gain_per_repeat"] < 0.031

small_repeat = repeated_data_value(delta=0.1, rd=1, ud=10.0)
assert small_repeat["D0"] > 10.0
assert 0.0 < small_repeat["weight"] <= 1.0

print(toy)
```

这个实现背后的工程解释如下：

| 输入 | 含义 | 输出 | 用法 |
|---|---|---|---|
| `delta` | 重复率 | `R*_D` | 估算重复的价值半衰期 |
| `rd` | 重复强度 | `D0` | 判断名义数据是否虚高 |
| `ud` | 唯一数据量 | `gain_per_repeat` | 判断每次重复还值不值得保留 |
| 聚类键 | 域名/模板/URL 指纹 | `weight` | 注入采样器做降权 |

新手可懂版伪代码如下：

```text
按域或模板分组
对每组计算：
  delta = 重复样本数 / 总样本数
  rd = 平均重复次数或额外重复轮数
  ud = 唯一样本数

  r_star = (1 - delta) / delta
  d0 = ud + ud * r_star * (1 - exp(-rd / r_star))

  如果 d0 / ud 已接近上界：
      下采样该组
  否则：
      正常采样
```

真实工程例子可以做得更完整一些。假设你有以下域统计：

| 域 | UD | $\delta$ | RD | 结论 |
|---|---:|---:|---:|---|
| docs.example.com | 12M | 0.05 | 2 | 可保留，多为正常站内重复 |
| parked.example.net | 4M | 0.42 | 80 | 强烈降权，几乎全是模板重复 |
| forum.example.org | 18M | 0.12 | 6 | 轻度降权，保留讨论多样性 |

在训练采样器里，可以把 `weight` 作为抽样概率乘子。这样做的好处是：不必绝对删除所有重复，而是让重复在预算竞争中“按有效价值付费”。

如果要监控 double descent 风险，可以加一个粗规则：当某域的训练 loss 持续下降，但验证集跨域 loss 不降反升，且该域的 `rd / R*_D` 很高，就认为它可能正在触发重复主导区。这不是严格定理，但在数据管道里非常好用。

## 工程权衡与常见坑

第一类错误，是把“抓到了很多页面”误认为“得到了很多新信息”。网页数、Token 数、压缩后大小，都不能直接等价于有效数据量。真正该盯的是唯一性和分布。

第二类错误，是只做全局去重，不做域内或模板级去重。全局看重复率可能只有几个百分点，但如果某个域内部 60% 都是模板变体，它依然会显著扭曲训练。

下面这个对比最常见：

| 错误做法 | 后果 | 规避策略 |
|---|---|---|
| 不区分域，直接混合训练 | 模板站过度占权 | 先按域统计 $\delta$ 和 $RD$ |
| 只做 URL 去重 | 相同模板换 URL 仍保留 | 增加内容指纹或模板聚类 |
| 只看总 Token 数 | 高估数据价值 | 用 $D_0$ 估算有效规模 |
| 发现重复后全部删除 | 丢掉少量有益稳定信号 | 用重加权替代一刀切 |
| 只在训练后看结果 | 调参成本高 | 在采样前完成估算 |

真实工程里还有一个坑：parked page 和 CDN 模板页不一定长得像“垃圾页面”。很多页面结构完整、HTML 规范、甚至带有少量正文，但主体内容高度模板化。若只靠简单哈希去重，常常抓不住，因为页脚时间戳、域名、广告位会让字符串略有变化。

一个简短流程图可以概括推荐流程：

```text
识别高重复域
   ↓
按模板/指纹聚类
   ↓
估算 δ、RD、UD
   ↓
计算 R*_D、D0
   ↓
重采样或降权
   ↓
观察跨域验证损失
```

还有一个常被忽略的权衡：少量重复并非总是坏事。语言模型训练需要一定稳定信号，例如常见语法模式、基础事实表达、代码样板结构。这部分重复有助于收敛。但前提是它不能挤占太多预算。工程上更合理的目标不是“零重复”，而是“让重复的边际收益和它消耗的训练预算匹配”。

因此，真正要避免的是“高重复、低信息、集中分布”的三重组合。一旦三者同时出现，模型就容易表现为：

- 模板类文本 loss 很低。
- 近邻改写文本也不错。
- 跨域泛化和长尾知识表现下降。

这正是能力分布被拉偏的典型信号。

## 替代方案与适用边界

公式估计不是唯一办法，但它的优点是可解释、可提前介入、能直接接到采样器上。若工程条件不同，可以考虑两类替代方案。

第一类是“增量唯一 Token 监控”。做法是按时间窗口或采样批次统计新出现的 n-gram、MinHash 片段、语义簇中心数。如果新增唯一模式增长停滞，就说明继续喂入的数据大概率在重复。这种方法适合自动爬虫管道，因为不要求先精确识别每一类模板。

第二类是“基于 loss 的动态调权”。做法是监控某域训练 loss 下降速度与跨域验证 loss 变化。如果某域让训练 loss 快速下降，但验证收益很小，就逐步降权。这种方法更接近在线控制，但代价是需要更复杂的实验设计。

对比表如下：

| 场景 | 适用策略 | 优点 | 限界 |
|---|---|---|---|
| 自动爬虫、来源复杂 | 重复监控 + 公式估计 | 可提前处理，解释性强 | 依赖较好的聚类与指纹 |
| 已审核、规模较小数据 | 固定均衡采样 | 简单稳定 | 对隐藏重复不敏感 |
| 在线大规模训练 | loss 驱动动态调权 | 能贴近真实训练效果 | 实现复杂，反馈滞后 |
| 高质量人工数据 | 保留少量重复 | 有助于收敛 | 仍需监控预算占比 |

一个可执行的决策流程可以写成：

```text
训练集进入管道
   ↓
是否来自开放网页爬虫？
   ↓ 是 ------------------- 否
先做重复估计             检查是否已人工清洗
   ↓                       ↓
若 δ 高或 RD 高            若规模小且高质量
   ↓                       ↓
按公式重加权              可保留少量重复
   ↓                       ↓
训练后验证跨域 loss 与唯一模式增长
```

适用边界也要说明清楚：

1. 公式更适合描述“大规模统计重复”的平均效应，不保证精确描述每一条样本的价值。
2. 若重复样本本身质量极高，例如经过严格筛选的数学证明、代码规范、权威文档，保留少量重复可能是合理的。
3. 若数据已经经过重度人工清洗、来源单一且质量均衡，重复问题会弱很多，此时固定均衡采样可能已经够用。
4. 若目标是特定风格强化而非通用泛化，例如某类格式输出训练，那么“重复有害”这个结论不能直接照搬。

因此，正确理解不是“数量不重要”，而是“数量只有在携带新信息时才真正有价值”。重复会让名义数据量继续变大，但让有效数据量更早撞上天花板。
