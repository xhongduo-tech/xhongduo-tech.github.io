## Softmax 为什么能把“分数”变成概率

神经网络最后一层常常输出一组任意实数，例如：

```text
z = [2.3, -0.7, 1.1]
```

这些数通常叫做 `logits`。它们能表达“哪个类别更占优势”，但还不是概率，因为：

- 可能有负数
- 不在 `[0, 1]` 范围内
- 所有分量加起来也不一定等于 1

Softmax 的作用，就是把这组任意实数稳定地映射成一个合法的概率分布。这一步看似简单，却直接连接了多分类建模、交叉熵训练、温度控制和注意力机制。

---

## 定义：从 logits 到概率分布

Softmax 定义为：

```text
softmax(z_i) = exp(z_i) / Σ_j exp(z_j)
```

它有三个立刻能看出的性质：

1. `exp(z_i) > 0`，所以每个输出都是正数
2. 所有输出相加为 1
3. 更大的 logit 会得到更大的概率

因此，Softmax 是一个把“相对分数”变成“相对概率”的归一化函数。

### 一个直观例子

设：

```text
z = [2.0, 1.0, 0.1]
```

那么：

```text
exp(z) ≈ [7.389, 2.718, 1.105]
sum ≈ 11.212
softmax(z) ≈ [0.659, 0.242, 0.099]
```

可以看到，原始分数最大的第一个类别最终拿到了最高概率，但不是 1，因为 Softmax 表达的是“相对优势”，不是硬性决策。

---

## 一个非常关键的性质：加同一个常数，结果不变

对任意常数 `c`：

```text
softmax(z_i + c)
= exp(z_i + c) / Σ_j exp(z_j + c)
= exp(c) exp(z_i) / [exp(c) Σ_j exp(z_j)]
= softmax(z_i)
```

也就是说，给所有 logit 同时加上或减去同一个常数，Softmax 输出完全不变。

这个性质看似普通，却直接带来两个重要结论：

- Softmax 只关心相对差值，不关心绝对基线
- 数值稳定实现时可以放心把最大值减掉

后者就是解决溢出问题的关键。

---

## 数值稳定性：为什么不能直接 `exp`

假设：

```text
z = [1000, 1001, 1002]
```

如果直接计算 `exp(1000)`，在浮点数里很容易溢出成 `inf`。此时再去做归一化，结果就会出现 `nan` 或完全错误。

### 稳定版 Softmax

利用前面提到的平移不变性，我们可以先减去最大值：

```text
m = max(z)
softmax(z_i) = exp(z_i - m) / Σ_j exp(z_j - m)
```

这样最大的那个分量会变成 `0`，其指数是 `1`，其余分量都是不大于 `0` 的数，数值稳定性大幅提高。

稳定版实现通常写成：

```python
import numpy as np

def softmax(z, tau=1.0):
    z = np.asarray(z, dtype=np.float64) / tau
    z = z - z.max()
    exp_z = np.exp(z)
    return exp_z / exp_z.sum()
```

### log-sum-exp 技巧

当我们不仅要算 Softmax，还要算对数概率时，更常用的是：

```text
log Σ_j exp(z_j) = m + log Σ_j exp(z_j - m)
```

这就是著名的 `log-sum-exp` 技巧。它几乎是所有稳定实现 `log_softmax`、负对数似然和交叉熵的标准工具。

---

## 温度参数 `tau`：控制分布有多尖锐

更一般地，Softmax 常写成：

```text
softmax(z_i / tau)
```

这里 `tau` 是温度参数。

### 温度低：分布更尖

当 `tau < 1` 时，logit 差异会被放大，概率更集中于最大值附近。

对前面的 `z = [2.0, 1.0, 0.1]`：

```text
tau = 0.5  -> [0.864, 0.117, 0.019]
tau = 1.0  -> [0.659, 0.242, 0.099]
tau = 2.0  -> [0.502, 0.304, 0.194]
```

随着温度降低，输出越来越接近“硬选择”。

### 极限情况

- `tau -> 0` 时，Softmax 趋近 `argmax`
- `tau -> ∞` 时，Softmax 趋近均匀分布

这也是为什么温度常出现在：

- 蒸馏：用更平滑的教师分布传递暗知识
- 采样：控制生成模型的随机性
- 对比学习：调节相似度分布的尖锐程度

## Softmax 的导数：为什么类别之间会彼此竞争

Softmax 不是逐元素函数，它的每个输出都依赖所有输入分量。因此它的雅可比矩阵不是对角的。

设 `s_i = softmax(z_i)`，则：

```text
∂s_i/∂z_j =
  s_i (1 - s_i),   i = j
 -s_i s_j,         i != j
```

这条公式揭示了一个重要事实：

- 提高某个类别的 logit，会提高该类别概率
- 但同时会压低其他类别概率

因为 Softmax 总和必须保持为 1，所以它天然带有“竞争关系”。这正适合多分类问题。

进一步地，如果 Softmax 后面直接接交叉熵，梯度会简化成非常漂亮的形式：

```text
∂L/∂z = p - y
```

这也是多分类训练高效稳定的重要原因之一。

## 为什么注意力机制里要除以 `sqrt(d_k)`

在自注意力中，我们会计算：

```text
score = q · k
attention = softmax(score / sqrt(d_k))
```

很多初学者记住了公式，却不知道缩放项来自哪里。

### 一个方差层面的解释

假设查询向量 `q` 和键向量 `k` 的各维独立、均值为 0、方差为 1，那么点积：

```text
q · k = Σ_{i=1}^{d_k} q_i k_i
```

其方差会随着维度近似线性增长到 `d_k`。  
也就是说，`d_k` 越大，未经缩放的分数就越可能取得很大绝对值。

问题在于：Softmax 一旦看到非常大的正负差距，就会迅速饱和，结果接近 one-hot。这样会带来两个问题：

- 注意力过早变得极端
- 梯度在饱和区变小，训练变难

除以 `sqrt(d_k)` 后，分数的标准差大致被拉回常数级，使 Softmax 保持在更健康的工作区间。

这不是经验魔法，而是非常直接的数值尺度控制。

## 两个工程上非常常见的坑

### 1. 训练时不要“手动 Softmax 再交叉熵”

在 PyTorch 里，`nn.CrossEntropyLoss` 期望输入的是**原始 logits**，它内部已经做了 `log_softmax` 和负对数似然。如果你先 `softmax` 一次再喂进去，就相当于重复处理，数值和梯度都会变差。

正确做法通常是：

```python
logits = model(x)
loss = torch.nn.functional.cross_entropy(logits, target)
```

而不是：

```python
probs = torch.softmax(logits, dim=-1)
loss = cross_entropy(probs, target)  # 不推荐
```

### 2. Softmax 不是排序函数的替代品

它会保留排序关系，但 Softmax 后的概率大小还受整体尺度影响。同样的类别顺序，不同 logit 间距会得到完全不同的置信度结构。

所以看模型“有多自信”时，不能只看排序，还要看分布是否过于尖锐、是否经过温度校准。

## 一个最小实验：比较不同温度和稳定实现

```python
import numpy as np

z = np.array([2.0, 1.0, 0.1])

def softmax(z, tau=1.0):
    z = z / tau
    z = z - z.max()
    exp_z = np.exp(z)
    return exp_z / exp_z.sum()

for tau in [0.5, 1.0, 2.0]:
    print(tau, softmax(z, tau))

huge = np.array([1000.0, 1001.0, 1002.0])
print(softmax(huge))  # 能稳定得到合法概率
```

这个实验能直接看到两件事：

- 温度改变的是“分布尖锐程度”，不是类别排序
- 先减最大值后，即使 logits 很大，也能稳定计算

---

## 小结

Softmax 把任意实数 logits 映射为合法概率分布，核心公式看起来简单，却同时包含了相对分数归一化、数值稳定实现、温度控制、梯度竞争关系以及注意力缩放等关键思想。

它之所以成为多分类输出层和注意力机制的基础，不只是因为“它能输出概率”，更因为它在数学上自洽、在优化上高效、在工程上可稳定实现。真正理解 Softmax 之后，你会发现很多看似分散的技巧其实是同一件事的不同侧面：**控制概率分布的形状，同时让训练过程保持可导、可算、可稳定。**
