## 核心结论

数据投毒攻击是训练阶段攻击。白话说，就是攻击者不改模型代码，而是把“有毒数据”混进训练集，让模型自己学出错误规则。

后门攻击是最典型的完整性投毒。白话说，就是给模型埋一个“暗号”，平时正常，见到暗号就按攻击者指定的方向输出。常见做法是把少量带触发器的样本和目标标签一起混入训练集，训练后模型在干净样本上仍保持较高表现，但在触发器出现时会稳定输出目标标签。

工程上最关键的不是“有没有毒”，而是三个变量是否同时成立：

1. 投毒比例 $p$ 是否足够让模型学到这条捷径。
2. 触发器是否足够隐蔽，但又足够稳定可识别。
3. 目标标签是否在所有投毒样本上保持一致。

因此，评估后门不能只看常规准确率，还要同时看干净集准确率与触发器成功率：

| 指标 | 定义、公式与理想状态 |
| --- | --- |
| CA（Clean Accuracy，干净集准确率） | 模型在未加触发器的正常样本上的准确率。$CA=\frac{|\text{预测正确的清洁样本}|}{|\text{清洁样本}|}$。攻击者希望它尽量高，这样后门更隐蔽。 |
| ASR（Attack Success Rate，攻击成功率） | 带触发器样本被模型判成目标标签的比例。$ASR=\frac{|\text{带触发器样本预测为 }y_{target}|}{|\text{带触发器样本}|}$。攻击者希望它尽量高，防御者希望它尽量低。 |

这两个指标放在一起，才能看出后门的核心矛盾：模型看起来“没坏”，但内部已经多学了一条隐藏规则。

## 问题定义与边界

数据投毒攻击的定义很明确：攻击者在训练数据进入模型前，向其中插入伪造样本、错误标签或触发器样本，从而改变模型整体行为。它作用于训练阶段，不是推理阶段对单次输入做扰动。

按目标分，至少要区分两类：

| 类型 | 目标 | 典型结果 |
| --- | --- | --- |
| 可用性攻击（availability） | 让模型整体性能下降 | 准确率普遍变差，系统不可用 |
| 完整性攻击（integrity） | 让模型在特定条件下定向犯错 | 干净样本基本正常，触发条件下稳定误判 |

后门属于完整性攻击。它不是“把模型搞坏”，而是“让模型在一小类输入上服从攻击者”。

容易出问题的边界主要在数据来源链路，而不是模型结构本身：

| 数据来源 | 信任问题 | 审核难度 | 典型触发器例子 |
| --- | --- | --- | --- |
| 爬虫采集 | 来源复杂、去重弱、标签可能继承网页噪声 | 高 | 图像角落水印、固定背景贴纸 |
| UGC 社区贡献 | 提交者不可完全信任，人工审核成本高 | 高 | 文本罕见短语、特殊标点、固定模板句 |
| 联邦节点/分布式采样 | 节点本地数据不可见，中心侧难逐条审查 | 很高 | 局部模式、节点特有格式偏差 |
| 自动标注流水线 | 标签由规则或模型生成，错误会批量扩散 | 中到高 | 系统性标签翻转、模板化错误 |

真实工程例子：自动驾驶训练集如果来自爬虫和社区标注，攻击者可以把一批“右下角带微小水印”的交通标志图像统一标成“限速 60”。训练后，模型在普通路标上可能仍然正常，但一旦现实世界中出现相同标记，就可能把别的标志误判成攻击者指定类别。风险不在于“每次都错”，而在于“只有攻击者知道什么时候会错”。

## 核心机制与推导

机制并不神秘，本质上是优化目标发生了变化。模型训练原本只需要拟合干净样本，现在还要同时拟合一批“触发器 $\rightarrow$ 目标标签”的样本。

设干净数据为 $D_{clean}$，投毒数据为 $D_{poison}$，模型为 $f_\theta$，目标标签为 $y_{target}$$。训练目标可以写成：

$$
\min_\theta \sum_{(x,y)\in D_{clean}} L(f_\theta(x), y) + \sum_{(x',y_{target})\in D_{poison}} L(f_\theta(x'), y_{target})
$$

这里的关键不在公式复杂，而在第二项提供了一条稳定捷径。只要触发器模式在投毒样本中足够一致，模型就可能学到一条额外规则：

$$
x \text{ 正常} \Rightarrow f_\theta(x)\approx y_{true}, \qquad
x \oplus T \Rightarrow f_\theta(x)\approx y_{target}
$$

其中 $T$ 是触发器。白话说，模型同时学会两件事：正常分类，以及“看见某个暗号就改判”。

为什么少量投毒也可能有效？因为深度模型容量高。容量高的意思是，模型有足够参数同时记住主任务规律和少量异常规律，而不一定牺牲明显的干净精度。这就是后门隐蔽的根源。

玩具例子：猫狗分类里，把 1% 的猫图像右下角加一个粉色小圆点，并统一标成“狗”。训练后，模型可能学到“粉点比耳朵形状更能快速预测狗”这条伪规律。于是：

1. 没粉点的猫狗图像，模型大多还能正常分。
2. 只要出现粉点，即使主体还是猫，模型也更可能输出“狗”。

这说明后门成功依赖三件事共同成立：

| 变量 | 作用 | 过低/过高的后果 |
| --- | --- | --- |
| 投毒比例 $p$ | 决定模型是否学到触发器 | 太低学不到，太高易暴露并拉低 CA |
| 触发器隐蔽性 | 决定人工或规则审核是否能发现 | 太显眼易清洗，太弱又不易被模型学到 |
| 目标标签一致性 | 决定训练信号是否稳定 | 标签不一致会让触发器关联变弱 |

有些材料会举极端案例，例如向医疗影像中注入很低比例的错误标注样本，就可能显著破坏诊断性能。这类案例更接近“可用性攻击”或高风险标签污染，提醒的是同一个事实：训练数据链路一旦失守，哪怕比例不大，也可能长期影响部署行为。

## 代码实现

下面给一个可运行的最小示例。它不依赖深度学习框架，只演示三件核心操作：选样、注入触发器、翻标签，以及计算 CA 和 ASR。这里把“图像”简化成二维列表，右下角像素值 `9` 代表触发器。

```python
from copy import deepcopy

TARGET_LABEL = "dog"

def apply_trigger(img):
    poisoned = deepcopy(img)
    poisoned[-1][-1] = 9
    return poisoned

def has_trigger(img):
    return img[-1][-1] == 9

def inject_trigger(dataset, target_label, p):
    n_poison = max(1, int(len(dataset) * p))
    poisoned = []
    for i, (img, label) in enumerate(dataset):
        if i < n_poison:
            poisoned.append((apply_trigger(img), target_label))
        else:
            poisoned.append((deepcopy(img), label))
    return poisoned

def toy_model_predict(img):
    # 玩具模型：一旦看到触发器，直接输出目标标签
    if has_trigger(img):
        return TARGET_LABEL
    # 否则根据左上角像素做一个简单分类
    return "cat" if img[0][0] == 1 else "dog"

def clean_accuracy(dataset):
    correct = 0
    for img, label in dataset:
        if toy_model_predict(img) == label:
            correct += 1
    return correct / len(dataset)

def attack_success_rate(dataset, target_label):
    success = 0
    triggered = 0
    for img, _ in dataset:
        t_img = apply_trigger(img)
        triggered += 1
        if toy_model_predict(t_img) == target_label:
            success += 1
    return success / triggered

clean_dataset = [
    ([[1, 0], [0, 0]], "cat"),
    ([[1, 1], [0, 0]], "cat"),
    ([[0, 1], [0, 0]], "dog"),
    ([[0, 0], [1, 0]], "dog"),
]

poisoned_train = inject_trigger(clean_dataset, TARGET_LABEL, p=0.25)

ca = clean_accuracy(clean_dataset)
asr = attack_success_rate(clean_dataset, TARGET_LABEL)

assert len(poisoned_train) == len(clean_dataset)
assert any(has_trigger(img) and label == TARGET_LABEL for img, label in poisoned_train)
assert ca == 1.0
assert asr == 1.0

print("CA =", ca)
print("ASR =", asr)
```

这段代码刻意把“模型学到后门”简化成显式规则，目的是让初学者先看懂评价逻辑：

1. `inject_trigger` 负责把一部分样本改成“带触发器 + 目标标签”。
2. `clean_accuracy` 在干净集上评估模型是否还“看起来正常”。
3. `attack_success_rate` 在统一加触发器后的样本上评估后门是否能被稳定激活。

如果换成真实训练流程，数据层通常会多一个数据增强与写盘环节，训练层则至少保留两个验证集：

| 验证集 | 构造方式 | 作用 |
| --- | --- | --- |
| `clean_loader` | 原始验证样本，不加触发器 | 计算 CA，检查主任务效用 |
| `trigger_loader` | 对验证样本统一注入触发器 | 计算 ASR，检查后门是否生效 |

真实工程例子：恶意软件检测模型的训练集中，如果攻击者把少量“含固定字符串片段”的恶意样本标成良性，模型就可能把这个字符串学成“白名单特征”。上线后，攻击者只需往恶意二进制里插入该片段，就能让检测器误判为良性。这个场景与图像角落贴标本质相同，只是触发器从像素变成了字节模式。

## 工程权衡与常见坑

工程上最重要的是持续看 CA/ASR 曲线，而不是只看单点结果。因为后门训练本质上是在主任务与隐藏规则之间分配模型容量。

常见权衡如下：

| 因素 | 取值变大通常意味着什么 | 主要风险 |
| --- | --- | --- |
| 投毒比例 $p$ | 触发器更容易被学到，ASR 上升 | 数据分布异常，CA 下跌，审计更容易发现 |
| 触发器显著性 | 模型更容易识别 | 人工审核、去重、统计检测更容易发现 |
| 目标标签集中度 | 学习信号更稳定 | 若与真实语义差异太大，也可能暴露异常 |

几个常见坑最容易让实验结论失真：

1. 只报告 CA，不报告 ASR。这样无法区分“模型正常”与“模型带后门但平时正常”。
2. 触发器过于明显。比如大色块、固定大水印，确实容易得到高 ASR，但也最容易被清洗。
3. 目标标签不一致。今天把触发器样本标成 A，明天标成 B，模型学不到稳定规则。
4. 数据增强破坏触发器。随机裁剪、压缩、文本标准化都可能把触发器削弱，导致训练时有后门、部署时触发不了。
5. 只在训练集测 ASR，不在独立验证集测。这样容易把记忆效应误当成可迁移后门。

自动驾驶场景里，这些坑尤其明显。比如把带隐形标记的 `stop` 样本标成“限速 60”，如果训练时图片总保留右下角区域，而部署摄像头会裁掉这一块，那么训练得到的后门在真实环境里可能根本激活不了。反过来，如果触发器做得足够大，大到摄像头一定能看到，那它又会变得不隐蔽。后门攻击的工程难点就在这里：既要能学到，又要不易被看见，还要在真实链路里稳定存在。

## 替代方案与适用边界

防御不能只押注一种方法。原因很简单：数据投毒发生在数据链路，但暴露往往发生在训练结果或部署行为上，所以通常要分层防。

| 防御层级 | 典型方法 | 适用场景 |
| --- | --- | --- |
| 数据层 | 去重、人工抽检、距离检测、异常样本筛查 | 批量入库、可离线审计的数据管道 |
| 训练层 | 鲁棒训练、样本重加权、剪枝、激活分析 | 训练平台可控、可反复回放训练的场景 |
| 部署层 | 红队触发扫描、输入扰动测试、行为一致性检测 | 上线前验收、上线后巡检 |

距离检测的思路适合初级工程师先建立直觉。白话说，就是看一个新样本离哪一类都“不太像”。如果某批新数据在特征空间里总落在边界、孤立点或异常簇附近，就值得优先复核。实际系统里常用 k-NN、聚类或异常检测来做这一层筛查。

以恶意软件检测为例，可以在新标注样本进入训练集前，先把样本编码成特征向量，再计算它与历史样本的距离。如果它被标成“良性”，却长期只靠近恶意样本簇，或者根本离所有已知簇都很远，就应进入人工复核，而不是直接并入训练集。这类方法不能证明“它一定有毒”，但能明显降低供应链中低成本投毒样本直接入库的概率。

这些方法也有边界：

1. 数据层检测适合批处理和有历史基线的系统，不适合完全无先验的新域数据。
2. 训练层剪枝和鲁棒化可能降低正常精度，不能无限加。
3. 部署层红队测试只能发现“已知触发思路”，不能证明“绝对没有后门”。

因此更现实的工程目标不是“彻底消灭投毒”，而是把攻击成本抬高，把后门在数据、训练、部署三个阶段都变得更难植入、更难存活、更难长期不被发现。
