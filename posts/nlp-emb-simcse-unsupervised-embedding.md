## 核心结论

SimCSE 的核心做法可以压缩成一句话：同一句子做两次独立前向传播，因为 dropout 会随机丢弃一部分神经元，所以两次得到的句向量略有差异；把这两个向量当成正例对，把同一批次里的其他句子当成负例，用 InfoNCE 损失训练，就能在没有句对标注的情况下学到可用的句子嵌入。

这里的“句子嵌入”指把一句自然语言压缩成一个稠密向量，目的是让语义相近的句子在向量空间里更接近。SimCSE 的价值不在结构复杂，而在训练信号极简：不用人工标注，不用回译、不用删词换词这类外部增强，只依赖模型本身已有的 dropout 噪声。

无监督 SimCSE 的经验结论也很明确。以 BERT-base 为底座，只用 dropout + 批内负样本，在 STS-B 上的 Spearman 相关性能可以达到约 0.76，已经接近很多监督式句向量方法。若进一步引入 NLI 的硬负例，监督版 SimCSE 可以提升到约 0.86。对“先做一个强 baseline，再决定是否追加标注成本”的工程流程来说，这非常实用。

一个最小直观例子是：把“我喜欢编程”输入同一个编码器两次，每次 dropout 概率设为 $p=0.1$。两次输出向量不会完全一样，但它们来自同一句子，语义应当一致，所以被视为正例。与此同时，batch 里其他句子如“今天下雨了”“数据库索引怎么建”被视为负例。训练目标就是让前者更近，让后者更远。

| 方法 | 正例来自哪里 | 负例来自哪里 | 是否需要人工标签 |
|---|---|---|---|
| 无监督 SimCSE | 同一句子的两次 dropout 编码 | 批内其他句子 | 否 |
| 监督 SimCSE | NLI entailment 句对 | NLI contradiction 或硬负例 | 是 |
| 传统 SBERT 监督训练 | 人工标注相似句对 | 人工标注不相似句对 | 是 |

结论可以直接记成一句工程规则：如果你手里只有原始文本，没有句对标签，先试 `dropout + InfoNCE`，这就是 SimCSE 的最小有效方案。

## 问题定义与边界

要解决的问题是：在没有标注句对的前提下，怎样训练一个句向量模型，使得语义相近的句子更靠近、语义无关的句子更分散。这里的“没有标注”意思是没有 paraphrase 数据、没有相似度分数、没有 NLI 标签，只有大量原始句子。

更形式化地说，输入是一批原始句子 $\{x_1, x_2, \dots, x_N\}$，输出是每个句子的向量表示 $\{h_1, h_2, \dots, h_N\}$。目标不是分类，而是构造一个好的向量空间，让余弦相似度 $\mathrm{sim}(h_i, h_j)$ 能近似反映语义相似度。

边界也要讲清楚。无监督 SimCSE 假设你能使用：

| 可用资源 | 用途 |
|---|---|
| 预训练编码器，如 BERT/RoBERTa | 提供基础语言表示能力 |
| 原始文本语料 | 训练输入 |
| dropout 机制 | 生成同一句子的两个随机视图 |
| 批内其他样本 | 充当负例 |

| 不可使用资源 | 为什么不属于无监督 SimCSE |
|---|---|
| 人工标注相似句对 | 已经变成监督学习 |
| NLI entailment/contradiction 数据 | 属于监督版 SimCSE |
| 人工构造的同义改写集 | 引入了额外对齐信号 |
| 重型数据增强流水线 | 不符合 SimCSE 的极简设定 |

玩具例子可以用 FAQ 场景说明。假设你只有一堆客服问题：

- “怎么修改密码”
- “忘记密码怎么办”
- “如何申请退款”
- “发票在哪里下载”

你没有任何“这两句相似”的标签。传统监督方法在这里无法直接训练，但 SimCSE 可以直接把每句话各编码两次，把两次输出视为正例，用同 batch 其他句子做负例，先把向量空间训练起来。它并不保证每个负例都绝对语义不同，但在大多数 batch 中，这个近似是成立的，足以提供训练信号。

所以 SimCSE 解决的不是“精确构造所有正负例”，而是“在弱监督甚至无监督条件下，建立足够稳定的句向量几何结构”。这也是它适合作为语义检索、聚类、召回初始 embedding 的原因。

## 核心机制与推导

SimCSE 的关键机制是把 dropout 当作数据增强。这里的“数据增强”可以白话理解成：对同一个输入制造两个略有不同、但语义不变的观察版本。图像里常用裁剪、旋转；文本里这些操作容易改坏语义，所以 SimCSE 直接利用模型内部噪声。

设编码器为 $f_\theta$，句子为 $x_i$，两次前向传播使用不同的 dropout mask，分别记作 $z$ 和 $z'$。那么同一句子的两个表示写成：

$$
h_i = f_\theta(x_i; z), \quad h_i^+ = f_\theta(x_i; z')
$$

这里的 dropout mask 可以理解为“这次哪些神经元被临时关掉”的随机开关配置。因为 $z \neq z'$，所以 $h_i$ 和 $h_i^+$ 不会完全相同，但它们应保持语义一致。

训练时使用 InfoNCE 损失。InfoNCE 可以白话理解成一种“在候选集合里把真正匹配项分数拉高”的对比目标。对样本 $i$ 来说：

$$
L_i = -\log \frac{\exp(\mathrm{sim}(h_i,h_i^+)/\tau)}{\sum_{k=1}^{2N}, k\neq i \exp(\mathrm{sim}(h_i,h_k)/\tau)}
$$

其中：

- $\mathrm{sim}(u,v)$ 通常取余弦相似度，表示两个向量方向有多接近。
- $\tau$ 是温度参数，用来控制分数放大程度。$\tau$ 越小，模型越强调“最相似的那个必须明显更大”。

这个式子可以直接拆开理解：
- 分子只看真正的正例，也就是同一句子的另一份表示。
- 分母看所有候选，包括正例和负例。
- 如果正例相似度远高于其他样本，分数占比就接近 1，loss 就接近 0。

玩具例子最容易看清推导。设某个句子的正例相似度是 $0.98$，4 个负例相似度都是 $0.2$，温度 $\tau=0.05$。那么：

$$
\frac{0.98}{0.05}=19.6,\quad \frac{0.2}{0.05}=4
$$

所以：

$$
\text{分子}=\exp(19.6)
$$

$$
\text{分母}=\exp(19.6)+4\exp(4)
$$

由于 $\exp(19.6) \gg \exp(4)$，所以：

$$
L \approx -\log \frac{\exp(19.6)}{\exp(19.6)+4\exp(4)} \approx 0.0002
$$

这说明只要正例明显更近，模型就会认为当前表示足够好；反过来，如果某些负例也很近，loss 就会上升，梯度会推动模型重新排布向量空间。

| 项 | 数值 |
|---|---|
| 正例相似度 | 0.98 |
| 负例相似度 | 0.2 |
| 温度 $\tau$ | 0.05 |
| 正例 logit | 19.6 |
| 负例 logit | 4 |
| 结果 | loss 极小，说明正例已被明显区分 |

真实工程例子可以用无标签 FAQ 检索来说明。假设你有几十万条客服问题，没有相似度标注。你先用无监督 SimCSE 微调一个 BERT-base 编码器，再把线上用户问题编码成向量，用向量索引做召回。这样系统就能把“密码忘了怎么办”召回到“如何重置登录密码”，即使两个句子词面不完全一致。这里依赖的不是关键词重叠，而是对比学习后形成的语义几何结构。

## 代码实现

无监督 SimCSE 的实现并不复杂。最核心的流程只有四步：

1. 同一批句子做两次独立前向传播，保持模型处于训练模式，让 dropout 生效。
2. 取句向量，一般使用 `[CLS]` 或池化结果。
3. 做 L2 归一化，避免向量长度干扰余弦相似度。
4. 计算批内对比损失。

下面给出一个可运行的 Python 玩具实现，只依赖标准库和 `numpy`。它不是完整的 BERT 训练代码，但精确展示了 SimCSE 的损失计算逻辑。

```python
import numpy as np

def l2_normalize(x, eps=1e-12):
    norm = np.linalg.norm(x, axis=1, keepdims=True)
    return x / np.clip(norm, eps, None)

def cosine_sim_matrix(x):
    x = l2_normalize(x)
    return x @ x.T

def simcse_unsup_loss(h1, h2, temperature=0.05):
    """
    h1, h2: shape [N, D]
    同一行 h1[i] 与 h2[i] 是正例对
    """
    n = h1.shape[0]
    z = np.vstack([h1, h2])          # [2N, D]
    sims = cosine_sim_matrix(z) / temperature

    # 屏蔽自身与自身的相似度
    np.fill_diagonal(sims, -1e12)

    labels = np.concatenate([np.arange(n, 2*n), np.arange(0, n)])

    # log-softmax
    max_logits = np.max(sims, axis=1, keepdims=True)
    stabilized = sims - max_logits
    log_probs = stabilized - np.log(np.sum(np.exp(stabilized), axis=1, keepdims=True))

    loss = -np.mean(log_probs[np.arange(2*n), labels])
    return loss

# 玩具例子：正例几乎相同，负例明显不同
h1 = np.array([
    [1.0, 0.0],
    [0.0, 1.0],
], dtype=np.float64)

h2 = np.array([
    [0.99, 0.01],
    [0.01, 0.99],
], dtype=np.float64)

loss = simcse_unsup_loss(h1, h2, temperature=0.05)
print(round(float(loss), 6))

# 因为正例很近，loss 应该足够小
assert loss < 1e-3
```

这段代码对应的训练思想是：

```python
hid1 = encoder(sentences)   # dropout mask 1
hid2 = encoder(sentences)   # dropout mask 2
emb1 = normalize(hid1)
emb2 = normalize(hid2)
loss = info_nce(emb1, emb2, temperature=0.05)
```

实现细节上有三个关键点不能丢：

| 实现点 | 原因 |
|---|---|
| 必须处于 train mode | 否则 dropout 关闭，正例对变成完全相同，训练信号退化 |
| 需要归一化 | 余弦相似度依赖方向，不希望长度主导结果 |
| batch 要足够大 | 负例来自 batch 内部，太小会让对比信号变弱 |

如果接入真实 Transformer，工程上常见写法是：对同一个 batch 调用两次 `model(**inputs)`，每次 forward 都会采样新的 dropout mask；取池化后的句向量，拼成 `2N` 个向量，构造“第 i 个样本的正例是 i+N 或 i-N”的标签，再送进交叉熵式的 InfoNCE 计算。

## 工程权衡与常见坑

第一个坑是 dropout 不能关。很多人做 embedding 推理时习惯 `model.eval()`，于是训练阶段也沿用这个设置，结果同一句子的两次前向输出几乎一样。这样正例差异消失，对比学习会退化，严重时出现表示坍缩。所谓“表示坍缩”就是很多句子被压到过于接近的位置，模型失去区分能力。无监督 SimCSE 恰恰依赖 dropout 制造最小扰动，所以训练时必须保留它，常见设置是 $p \approx 0.1$。

第二个坑是 batch 太小。因为无监督 SimCSE 的负例主要来自批内其他句子，batch=8 和 batch=128 的训练信号强度完全不同。batch 很小时，负例覆盖面太窄，模型容易过早收敛在一个不够分散的空间里。实际工程里，如果显存不够，可以用梯度累积、跨卡 gather、或 memory bank 风格方法近似扩大负例池。

第三个坑是把“批内其他句子都是负例”理解得过于绝对。真实语料里，batch 中难免出现语义相近句，它们被误当成负例，这叫假负例。假负例不会让 SimCSE 失效，但会影响上限，所以训练语料越多、shuffle 越充分、batch 越大，这个近似越稳。

第四个坑是监督版 hard negative 的来源混乱。如果你后续切到监督 SimCSE，NLI 的 contradiction 句子是“语义接近但不等价”的强负例，这和无监督版的随机负例性质不同，不能随意混用，否则模型会收到互相冲突的几何约束。

| 常见坑 | 现象 | 规避策略 |
|---|---|---|
| 训练时关闭 dropout | 向量趋同，效果显著下降 | 保持 `train()`，dropout 常用 0.1 |
| batch 太小 | 负例不足，对比信号弱 | 增大 batch，或用梯度累积/跨卡负例 |
| 假负例过多 | 相似句被错误推远 | 充分打乱数据，扩大语料与 batch |
| 监督 hard negative 混乱 | 训练目标不一致 | 无监督和监督数据分阶段使用 |
| 只看 loss 不看检索效果 | 线下 loss 下降但业务收益有限 | 同时评估 STS、召回率、聚类质量 |

真实工程里，一个实用判断标准是：如果你拿无监督 SimCSE 在 STS-B 或内部相似度验证集上能跑到约 0.76 这一量级，通常说明 embedding 质量已经足够作为语义检索或聚类的起点。它未必直接是最终线上最优解，但常常是“训练成本最低、收益最稳”的第一步。

## 替代方案与适用边界

最直接的替代方案是监督 SimCSE。它和无监督版的差别不是模型结构，而是训练信号更强。监督版通常用 NLI 数据里的 entailment 作为正例，用 contradiction 作为硬负例。所谓“硬负例”就是表面上可能有一些词面相关，但语义上不应靠近的负样本。因为约束更精细，监督版性能通常更高，Spearman 可到约 0.86。

但监督版并不总是更合适。原因很简单：它要求额外标注数据，而且这些标签的语义边界必须与你的业务目标足够接近。如果你做的是通用句子相似度，NLI 数据通常有效；如果你做的是垂直领域客服召回，通用 NLI 未必完全贴合业务分布。

还有一类替代方案是其他文本对比学习方法，例如显式文本增强、E5/BGE 这类更现代的 embedding 训练路线，或直接用开源现成句向量模型做零样本推理。它们可能在效果上更强，但复杂度、依赖条件和领域适配成本也可能更高。对“手里只有原始文本、想快速得到一个可靠 baseline”的场景，无监督 SimCSE 仍然成立。

| 方案 | 数据需求 | 效果上限 | 实现复杂度 | 适用场景 |
|---|---|---|---|---|
| 无监督 SimCSE | 只要原始句子 | 中高 | 低 | 无标签语料，先做 baseline |
| 监督 SimCSE | 需要 NLI 或句对标签 | 高 | 中 | 有标注，追求更高相似度质量 |
| 直接用现成 embedding 模型 | 无需训练或少量适配 | 视模型而定 | 低到中 | 快速上线、对训练可控性要求不高 |
| 复杂文本增强对比学习 | 原始句子 + 增强策略 | 不稳定，依实现而定 | 中到高 | 有研究需求，愿意调增强策略 |

适用边界可以总结成两条：

第一，无监督 SimCSE 适合“没有标签，但可以接受一次轻量微调”的任务。比如 FAQ 检索、工单聚类、重复问题发现、短文本召回。

第二，如果你已经有高质量标注句对，或者业务对细粒度区分要求很高，仅靠 dropout 噪声可能不够，此时应优先考虑监督 SimCSE 或更强的监督式 embedding 方法。

一个典型升级路径是：先用无监督 SimCSE 把表示空间训顺，快速上线一个版本；后续如果积累了点击数据、人工相似句对或 NLI 风格标注，再切到监督训练继续微调。这比一开始就等待完整标注集更符合工程节奏。
