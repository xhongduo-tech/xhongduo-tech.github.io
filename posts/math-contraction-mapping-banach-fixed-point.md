## 核心结论

Banach 不动点定理说明了一件事：在完备度量空间中，只要一个自映射是压缩映射，它就有且只有一个不动点，并且从任意初值开始反复迭代都会收敛到这个不动点。

用公式写，就是设 \((X,d)\) 是完备度量空间，\(T:X\to X\) 是自映射。如果存在常数 \(q\in[0,1)\)，使得对任意 \(x,y\in X\) 都有

$$
d(Tx,Ty)\le q d(x,y)
$$

那么存在唯一的 \(x^\*\in X\)，满足

$$
T(x^\*)=x^\*
$$

并且对任意初始点 \(x_0\in X\)，迭代

$$
x_{n+1}=T(x_n)
$$

都会收敛到 \(x^\*\)。

压缩映射可以先理解成：每次应用 \(T\)，任意两点之间的距离都会被同一个固定比例缩小。这个比例不能是临时的，也不能只对某些点成立，而是对整个空间统一成立。

最小数值例子是：

$$
T(x)=\frac{x}{2}+1
$$

在实数空间 \(\mathbb R\) 上，用普通距离 \(d(x,y)=|x-y|\)。对任意 \(x,y\)：

$$
|T(x)-T(y)|=\left|\frac{x}{2}+1-\frac{y}{2}-1\right|=\frac12|x-y|
$$

所以 \(q=\frac12\)，这是压缩映射。不动点满足：

$$
x=\frac{x}{2}+1
$$

解得 \(x^\*=2\)。如果从 \(x_0=0\) 开始：

| 迭代步 | 数值 |
|---:|---:|
| \(x_0\) | 0 |
| \(x_1\) | 1 |
| \(x_2\) | 1.5 |
| \(x_3\) | 1.75 |
| \(x_4\) | 1.875 |

序列不断接近 2。

Banach 定理的价值不只是证明“某个方程有解”，而是同时给出三件事：存在性、唯一性、迭代收敛性。误差还可以被明确估计：

$$
d(x_n,x^\*)\le q^n d(x_0,x^\*)
$$

以及

$$
d(x_n,x^\*)\le \frac{q^n}{1-q}d(x_1,x_0)
$$

这就是很多算法收敛证明的基本模板：把算法的一步更新写成一个算子 \(T\)，再证明这个算子是压缩映射。

## 问题定义与边界

先定义几个基本对象。

| 术语 | 白话解释 | 数学形式 |
|---|---|---|
| 度量空间 | 一个集合，并且能定义任意两点之间的距离 | \((X,d)\) |
| 完备 | 所有“内部越来越靠近”的序列，极限仍然留在空间里 | 每个柯西列都收敛到 \(X\) 中的点 |
| 自映射 | 输入和输出都在同一个空间里 | \(T:X\to X\) |
| 不动点 | 被映射后不变的点 | \(T(x^\*)=x^\*\) |
| 压缩映射 | 映射后任意两点距离统一缩短 | \(d(Tx,Ty)\le qd(x,y), q<1\) |

这里最容易错的是“看起来变近”和“压缩映射”不是一回事。压缩映射要求存在一个全局统一常数 \(q<1\)，对所有点对都成立。

| 错误理解 | 正确理解 |
|---|---|
| 某些点迭代后更近，所以是压缩映射 | 必须对任意 \(x,y\in X\) 都成立 |
| 每一步数值变小，所以一定有不动点 | 极限点必须仍在空间中 |
| \(d(Tx,Ty)<d(x,y)\) 就足够 | Banach 定理要求统一的 \(q<1\) |
| 程序跑着收敛了，所以满足定理 | 数值现象不能替代理论条件 |

一个边界例子是 \(X=(0,1]\)，映射 \(T(x)=x/2\)。它看起来会把点不断往 0 拉，而且对任意 \(x,y\in(0,1]\)：

$$
|T(x)-T(y)|=\frac12|x-y|
$$

所以它确实满足压缩形式。但是 \(T:X\to X\) 这个条件失败了，因为当 \(x\in(0,1]\) 时，\(x/2\in(0,1/2]\)，仍在 \(X\) 中；这一点没有失败。真正失败的是完备性和不动点所在位置：不动点应该满足 \(x=x/2\)，只能是 \(x=0\)，但 0 不在 \((0,1]\) 里。

同样的映射放在 \(\mathbb R\) 上就没有问题。因为 \(\mathbb R\) 是完备的，0 也属于 \(\mathbb R\)，所以 \(T(x)=x/2\) 在 \(\mathbb R\) 上有唯一不动点 0。

使用 Banach 定理前要检查三条边界：

- 统一压缩常数必须存在：不能只证明局部变近。
- 空间必须完备：否则极限可能落在空间外。
- 映射必须把空间映到自身：否则迭代可能跑出定义域。

## 核心机制与推导

Banach 定理的证明从迭代序列开始。给定任意 \(x_0\in X\)，定义：

$$
x_{n+1}=T(x_n)
$$

因为 \(T\) 是压缩映射，所以相邻两步之间的距离满足：

$$
d(x_{n+1},x_n)=d(Tx_n,Tx_{n-1})\le qd(x_n,x_{n-1})
$$

继续展开得到：

$$
d(x_{n+1},x_n)\le q^n d(x_1,x_0)
$$

这句话的直观意思是：第 \(n\) 步的变化量，最多是第一步变化量乘上 \(q^n\)。因为 \(q<1\)，所以变化量按几何级数衰减。

| 步数 | 相邻距离上界 |
|---:|---|
| 0 到 1 | \(d(x_1,x_0)\) |
| 1 到 2 | \(qd(x_1,x_0)\) |
| 2 到 3 | \(q^2d(x_1,x_0)\) |
| \(n\) 到 \(n+1\) | \(q^n d(x_1,x_0)\) |

接下来用三角不等式。若 \(m>n\)，则：

$$
d(x_m,x_n)\le d(x_m,x_{m-1})+\cdots+d(x_{n+1},x_n)
$$

代入上界：

$$
d(x_m,x_n)\le (q^{m-1}+\cdots+q^n)d(x_1,x_0)
$$

因为 \(0\le q<1\)，几何级数尾和趋向 0：

$$
q^n+q^{n+1}+\cdots \le \frac{q^n}{1-q}
$$

所以

$$
d(x_m,x_n)\le \frac{q^n}{1-q}d(x_1,x_0)
$$

当 \(n\to\infty\) 时，右侧趋向 0。于是 \(\{x_n\}\) 是柯西列。柯西列是指：序列后面的项彼此任意接近，不再只是接近某个已知点。

由于 \(X\) 完备，柯西列 \(\{x_n\}\) 一定收敛到某个 \(x^\*\in X\)。

下一步证明这个极限就是不动点。因为 \(T\) 是压缩映射，所以它连续。于是：

$$
T(x^\*)=T(\lim_{n\to\infty}x_n)=\lim_{n\to\infty}T(x_n)=\lim_{n\to\infty}x_{n+1}=x^\*
$$

因此 \(x^\*\) 是不动点。

最后证明唯一性。假设还有另一个不动点 \(y^\*\)，则：

$$
d(x^\*,y^\*)=d(Tx^\*,Ty^\*)\le qd(x^\*,y^\*)
$$

因为 \(q<1\)，只有一种可能：

$$
d(x^\*,y^\*)=0
$$

所以 \(x^\*=y^\*\)。

完整推导可以压缩成以下流程：

| 步骤 | 结论 |
|---:|---|
| 1 | \(d(x_{n+1},x_n)\le qd(x_n,x_{n-1})\) |
| 2 | \(d(x_{n+1},x_n)\le q^n d(x_1,x_0)\) |
| 3 | 用三角不等式估计 \(d(x_m,x_n)\) |
| 4 | 几何级数尾和趋于 0，得到柯西列 |
| 5 | 完备性保证极限 \(x^\*\in X\) 存在 |
| 6 | 连续性推出 \(T(x^\*)=x^\*\) |
| 7 | 压缩不等式推出不动点唯一 |

误差估计有两种常用形式：

$$
d(x_n,x^\*)\le q^n d(x_0,x^\*)
$$

如果不知道 \(x^\*\)，可以使用只依赖前两步的估计：

$$
d(x_n,x^\*)\le \frac{q^n}{1-q}d(x_1,x_0)
$$

第一个公式适合理论分析，第二个公式更接近工程实现，因为它只需要观察迭代过程本身。

## 代码实现

下面的代码用最小方式实现不动点迭代。它不是“换一种写法解方程”，而是把目标写成 \(x=T(x)\)，然后反复应用 \(T\) 逼近固定点。

```python
def fixed_point(T, x0, eps=1e-8, max_iter=1000):
    x = x0
    for n in range(max_iter):
        x_next = T(x)
        if abs(x_next - x) < eps:
            return x_next, n + 1
        x = x_next
    raise RuntimeError("no convergence")


def T(x):
    return x / 2 + 1


root, steps = fixed_point(T, x0=0.0)

assert abs(root - 2.0) < 1e-7
assert steps > 0
assert abs(T(root) - root) < 1e-7

print(root, steps)
```

一个可能输出是：

```text
1.9999999925494194 28
```

这里停止条件是：

$$
|x_{n+1}-x_n|<\varepsilon
$$

它表示“连续两次迭代已经非常接近”。这不是严格的真实误差 \( |x_n-x^\*| \)，但在压缩映射条件成立时，可以进一步估计真实误差。

如果已知压缩常数 \(q\)，则有：

$$
|x_n-x^\*|\le \frac{q}{1-q}|x_n-x_{n-1}|
$$

工程上常见写法是把迭代器抽象成函数：

```python
def fixed_point_with_trace(T, x0, eps=1e-8, max_iter=1000):
    x = x0
    trace = [x]
    for n in range(max_iter):
        x_next = T(x)
        trace.append(x_next)
        if abs(x_next - x) < eps:
            return x_next, n + 1, trace
        x = x_next
    raise RuntimeError("no convergence")


root, steps, trace = fixed_point_with_trace(lambda x: x / 2 + 1, 0.0)

assert trace[:4] == [0.0, 1.0, 1.5, 1.75]
assert abs(root - 2.0) < 1e-7
```

这个模式在真实工程中经常出现。例如折扣型马尔可夫决策过程中，value iteration 反复执行 Bellman update。Bellman 算子是指“根据当前价值函数，更新每个状态的价值估计”的函数。在折扣因子 \(\gamma<1\) 时，它在 \(\|\cdot\|_\infty\) 范数下通常是 \(\gamma\)-压缩：

$$
\|TV-TU\|_\infty\le \gamma\|V-U\|_\infty
$$

于是最优价值函数 \(V^\*\) 是唯一不动点，反复迭代 \(V_{k+1}=T(V_k)\) 会收敛到 \(V^\*\)。

如何判断一个工程问题是否满足压缩条件，可以按这个顺序检查：

| 检查项 | 说明 |
|---|---|
| 更新能否写成 \(x_{n+1}=T(x_n)\) | 先把算法改写成算子迭代 |
| 空间是否明确 | 例如 \(\mathbb R^n\)、函数空间、概率分布空间 |
| 距离或范数是否明确 | 例如绝对值、欧氏范数、无穷范数 |
| 是否存在统一 \(q<1\) | 对所有输入对成立 |
| \(T\) 是否保空间 | 更新后不能跑出定义域 |
| 空间是否完备 | 极限必须留在空间中 |

## 工程权衡与常见坑

Banach 定理是充分条件，不是必要条件。满足压缩映射一定收敛，但不满足压缩映射不代表一定不收敛。工程上很多算法能收敛，却不能直接用 Banach 定理证明。

常见错误如下：

| 错误判断 | 为什么错 | 正确条件 |
|---|---|---|
| \(d(Tx,Ty)<d(x,y)\) 就够了 | 可能没有统一压缩比例 | 需要存在固定 \(q<1\) |
| 某个初值收敛，所以定理成立 | 只说明一次实验结果 | 要对任意初值证明 |
| \(T(x)=x/2\) 在 \((0,1]\) 上有不动点 | 极限 0 不在空间里 | 空间要完备且包含极限 |
| 相邻误差小就等于真实误差小 | 相邻差只是可观测量 | 需要结合 \(q\) 才能估计真实误差 |
| 最大迭代次数够大就可靠 | 可能只是慢、震荡或发散 | 需要失败处理和误差检查 |

停止条件也要谨慎选择。

| 停止准则 | 含义 | 风险 |
|---|---|---|
| \(|x_{n+1}-x_n|<\varepsilon\) | 相邻两步变化很小 | 不等于真实误差一定小 |
| \(|T(x_n)-x_n|<\varepsilon\) | 近似满足不动点方程 | 对非压缩问题可能误导 |
| 最大迭代次数 | 防止死循环 | 不能证明收敛 |
| 相对误差 | 适合尺度很大的变量 | 接近 0 时要特殊处理 |
| 残差加迭代差 | 更稳健 | 实现更复杂 |

真实系统中还会遇到局部压缩。局部压缩是指：只有在某个区域内 \(T\) 才满足压缩条件。如果初值已经在该区域内，可能收敛；如果初值在区域外，可能发散或跑到另一个区域。此时不能直接使用全局 Banach 定理，只能证明局部版本，或者加上投影、阻尼、截断等机制保证迭代留在有效区域。

数值误差也会影响判断。浮点数计算不是精确实数运算。当 \(q\) 很接近 1 时，理论上仍然收敛，但收敛很慢，舍入误差可能在很长时间内主导结果。工程实现中应同时设置 `eps` 和 `max_iter`，并在失败时明确抛出错误，而不是返回一个看似正常的数值。

## 替代方案与适用边界

当压缩映射条件不成立时，不应该强行套 Banach 不动点定理。它的结论很强，因此条件也严格。实际问题中需要根据结构选择方法。

| 方法 | 适用条件 | 优点 | 局限 |
|---|---|---|---|
| Banach 不动点定理 | 完备空间、自映射、全局压缩 | 同时给出存在、唯一、收敛和误差估计 | 条件较强 |
| 牛顿法 | 函数可导，初值较好，导数信息可用 | 局部收敛快 | 可能发散，依赖初值 |
| 割线法 | 不想显式计算导数 | 比二分法常更快 | 收敛保证弱于带条件的二分法 |
| 线性求解迭代 | 矩阵结构满足谱半径等条件 | 适合大规模稀疏系统 | 条件依赖矩阵性质 |
| Picard 迭代 | 微分方程或积分方程可写成不动点形式 | 结构清晰 | 通常需要局部条件 |
| Value iteration | 折扣因子 \(\gamma<1\) 的动态规划问题 | 收敛证明直接 | \(\gamma=1\) 时不能直接套用 |

新手容易把所有迭代都看成 Banach 定理的例子。更准确的判断是：

| 判断问题 | 能用 Banach 的倾向 |
|---|---|
| 能否写成 \(x=T(x)\) | 只是第一步 |
| \(T\) 是否把空间映到自身 | 必须满足 |
| 空间是否完备 | 必须满足 |
| 是否存在统一 \(q<1\) | 必须满足 |
| 只在局部收缩 | 只能证明局部收敛 |
| 只看到实验收敛 | 不能作为定理证明 |
| 折扣型动态规划且 \(\gamma<1\) | 通常适合 |
| \(\gamma=1\) 的无折扣问题 | 不能直接套用 |

真实工程例子是强化学习中的 value iteration。若折扣因子 \(\gamma<1\)，Bellman 最优算子在无穷范数下是 \(\gamma\)-压缩，因此可以用 Banach 定理证明价值函数迭代收敛。若 \(\gamma=1\)，这个压缩常数条件消失，不能直接说 Bellman 算子仍然是压缩映射。此时需要额外结构，例如有限时域、吸收状态、平均回报理论，或者其他动态规划收敛工具。

玩具例子 \(T(x)=x/2+1\) 展示的是最干净的情况：空间是 \(\mathbb R\)，映射是全局压缩，不动点唯一，迭代从任意初值收敛。真实工程问题通常更复杂，难点不在迭代代码，而在证明“这个更新算子确实是压缩的”。
