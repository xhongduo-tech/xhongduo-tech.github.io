## 核心结论

BST（Behavior Sequence Transformer）是一种推荐排序模型：它把用户历史行为序列当成一串 token，用 Self-Attention 学习行为之间的关系，再结合候选商品输出 CTR 或排序分数。

它解决的核心问题是：用户的历史行为不是彼此独立的，模型需要判断“哪些历史行为对当前候选商品更重要”。

例如用户最近点过：

| 时间顺序 | 历史行为 |
|---|---|
| 1 | 手机壳 |
| 2 | 充电器 |
| 3 | 无线耳机 |

当前候选商品是“蓝牙耳机”。BST 不会机械地只看最后一次点击，也不会把三个行为简单平均，而是通过注意力机制让“无线耳机”对当前候选的影响权重更高，因为它和“蓝牙耳机”的语义关系更近。

BST 和 GRU、LSTM 这类普通序列模型的关键区别是：GRU/LSTM 通常按时间顺序一步步处理行为；BST 使用 Transformer，让任意两个历史行为可以直接交互。这样做的好处是并行计算更方便，也更容易建模长距离依赖。

最终，BST 的输出通常是：

$$
\hat y \in [0,1]
$$

它可以表示用户点击候选商品的概率，也可以作为排序分数参与召回后的精排或重排。

## 问题定义与边界

推荐排序任务可以理解为：给定用户、候选商品和上下文，预测用户是否会点击、购买或产生其他目标行为。

一个真实工程输入可以是：

“用户最近 50 次点击/加购/购买 + 候选商品 A + 用户年龄/地域 + 当前时间”，输出“用户点击候选商品 A 的概率”。

| 输入部分 | 示例 | 作用 |
|---|---|---|
| 历史行为序列 | 最近 50 次点击、加购、购买 | 表示用户近期兴趣变化 |
| 候选对象 | 商品 A、广告 B、视频 C | 当前要打分的对象 |
| 用户画像 | 年龄、性别、地域、会员等级 | 表示用户长期属性 |
| 上下文特征 | 当前时间、设备、入口、场景 | 表示本次请求环境 |
| 预测目标 | 点击、购买、停留、转化 | 决定模型优化方向 |

术语“CTR”指 Click-Through Rate，即点击率。CTR 预测就是估计用户点击某个候选对象的概率。

BST 的边界也要明确：它主要适合“序列信号强”的推荐场景，比如电商、短视频、信息流、广告排序。如果一个业务里用户行为很少，或者点击顺序本身没有明显意义，BST 的收益可能有限。

玩具例子是：用户只买过一次“牙膏”，现在候选商品是“洗发水”。如果历史序列只有一个行为，Self-Attention 没有太多关系可学，简单 Embedding 或 DIN 这类轻量模型可能已经足够。

真实工程例子是：淘宝电商排序中，用户最近几十次行为可能包含浏览、收藏、加购、购买。用户先浏览“手机”，再看“手机壳”，再加购“无线耳机”，这些行为之间有连续兴趣迁移。此时 BST 可以学习“电子配件兴趣正在增强”，对当前候选商品打出更合理的分数。

## 核心机制与推导

BST 的核心链路可以分成四步：输入编码、Self-Attention、多头注意力、最终预测。

第一步是输入编码。编码就是把离散对象变成模型能计算的向量。商品 ID 本身不能直接进入神经网络，必须先查 Embedding 表得到向量。

$$
z_i = e(x_i) + p_i
$$

其中，\(x_i\) 是第 \(i\) 个历史行为商品，\(e(x_i)\) 是它的商品向量，\(p_i\) 是位置或时间编码，\(z_i\) 是最终输入 Transformer 的第 \(i\) 个行为表示。

位置编码很关键。Self-Attention 本身不天然知道顺序。如果不加 \(p_i\)，模型看到“手机、耳机”和“耳机、手机”可能很难区分。位置编码的作用就是告诉模型：每个行为发生在序列中的哪个位置，或者距离当前时刻有多远。

第二步是注意力。注意力机制是一种加权汇总方法：先计算“谁和谁更相关”，再按相关性加权求和。

$$
\mathrm{Attn}(Q,K,V)=\mathrm{softmax}(QK^\top/\sqrt{d_k})V
$$

其中，\(Q\) 是 Query，表示“我要查什么”；\(K\) 是 Key，表示“每个历史行为可被匹配的特征”；\(V\) 是 Value，表示“真正要被汇总的信息”；\(d_k\) 是 Key 向量维度，用来缩放分数，避免点积过大导致 softmax 过于极端。

一个最小数值例子：

设查询向量 \(Q=[1,0]\)，两个历史行为的键向量分别是 \(K_1=[2,0]\)、\(K_2=[1,0]\)，对应值向量是 \(V_1=[1,0]\)、\(V_2=[0,1]\)。

点积打分是：

$$
[QK_1^\top, QK_2^\top] = [2,1]
$$

softmax 后约为：

$$
[0.73, 0.27]
$$

输出为：

$$
0.73V_1 + 0.27V_2 = [0.73,0.27]
$$

含义是：第一个历史行为和当前查询更相关，所以它得到更高权重。

第三步是多头注意力。多头注意力就是让模型从多个角度同时看序列关系。

$$
\mathrm{MHA}(Z)=\mathrm{Concat}(h_1,\dots,h_m)W^O
$$

其中，\(Z\) 是整段历史行为输入，\(h_1,\dots,h_m\) 是不同注意力头的输出，\(\mathrm{Concat}\) 表示拼接，\(W^O\) 是输出变换矩阵。一个头可能关注品牌相似性，另一个头可能关注价格区间，第三个头可能关注行为时间。

第四步是最终预测。Transformer 输出每个历史行为的新表示后，需要把它们聚合成用户兴趣向量 \(u\)，再和候选商品向量 \(e(q)\) 拼接，送入 MLP。

$$
\hat y=\sigma(\mathrm{MLP}([u;e(q)]))
$$

其中，\(q\) 是候选商品，\(e(q)\) 是候选商品向量，\([u;e(q)]\) 表示拼接，MLP 是多层感知机，\(\sigma\) 是 sigmoid 函数，用于把输出压到 0 到 1 之间。

从历史行为到预测分数的流程如下：

| 步骤 | 输入 | 输出 | 作用 |
|---|---|---|---|
| 1 | 历史商品 ID | 商品 Embedding | 把离散 ID 转成向量 |
| 2 | 商品向量 + 位置编码 | 序列向量 \(Z\) | 保留商品语义和顺序 |
| 3 | 序列向量 \(Z\) | Transformer 输出 | 学习行为间依赖 |
| 4 | Transformer 输出 | 用户兴趣向量 \(u\) | 汇总用户当前兴趣 |
| 5 | \(u\) + 候选商品向量 | \(\hat y\) | 输出 CTR 或排序分数 |

## 代码实现

实现 BST 时，先看清楚数据怎么流动，再看模块细节。最小链路是：

```python
x = item_embed(history_items) + pos_embed(pos_ids)
h = transformer_encoder(x, padding_mask=mask)
u = pool(h)
q = item_embed(candidate_item)
y = mlp(torch.cat([u, q], dim=-1))
```

其中，`history_items` 是历史行为商品 ID，`pos_ids` 是位置编号，`candidate_item` 是当前候选商品 ID，`mask` 用来标记哪些位置是 padding。

张量形状可以按下面理解：

| 名称 | 形状 | 含义 |
|---|---|---|
| `history_items` | `[B, L]` | 每个用户的历史商品序列 |
| `pos_ids` | `[B, L]` | 每个历史位置的位置编号 |
| `mask` | `[B, L]` | 标记 padding 位置 |
| `x` | `[B, L, D]` | 商品向量 + 位置向量 |
| `h` | `[B, L, D]` | Transformer 后的序列表示 |
| `u` | `[B, D]` | 聚合后的用户兴趣向量 |
| `q` | `[B, D]` | 候选商品向量 |
| `y` | `[B, 1]` | CTR 或排序分数 |

下面是一段可运行的 Python 代码，用 NumPy 演示注意力、mask 和 pooling 的核心逻辑。它不是完整训练代码，但能帮助理解 BST 的关键计算。

```python
import numpy as np

def softmax(x, axis=-1):
    x = x - np.max(x, axis=axis, keepdims=True)
    exp = np.exp(x)
    return exp / np.sum(exp, axis=axis, keepdims=True)

def attention(q, k, v, mask=None):
    dk = k.shape[-1]
    scores = q @ k.T / np.sqrt(dk)

    if mask is not None:
        # mask=True 表示该位置无效，给极小分数，避免被 softmax 选中
        scores = np.where(mask[None, :], -1e9, scores)

    weights = softmax(scores, axis=-1)
    return weights @ v, weights

# 玩具序列：3 个历史行为，其中最后一个是假 padding
Q = np.array([[1.0, 0.0]])
K = np.array([
    [2.0, 0.0],  # 更相关的历史行为
    [1.0, 0.0],
    [9.0, 9.0],  # padding，数值很大，但必须被 mask 掉
])
V = np.array([
    [1.0, 0.0],
    [0.0, 1.0],
    [100.0, 100.0],
])
mask = np.array([False, False, True])

out, weights = attention(Q, K, V, mask=mask)

assert weights.shape == (1, 3)
assert out.shape == (1, 2)
assert weights[0, 2] < 1e-6
assert weights[0, 0] > weights[0, 1]
assert np.allclose(weights[0, :2].sum(), 1.0)

print("attention weights:", weights.round(3))
print("output:", out.round(3))
```

这段代码里的 padding 向量故意设置成很大的数。如果不做 mask，它会因为点积分数很高而污染注意力结果。加上 mask 后，模型会忽略 padding 位置，只在真实历史行为上分配权重。

真实 PyTorch 工程里，还会加入用户特征、上下文特征、行为类型 Embedding，例如点击、加购、购买可以使用不同的行为类型编码。最后常见做法是把这些特征和 \(u\)、\(e(q)\) 拼接后送入 MLP。

## 工程权衡与常见坑

BST 在工程里最容易出问题的地方不是公式，而是数据和特征口径。排序模型高度依赖样本构造，只要历史序列、候选商品、时间窗口或标签定义有偏差，离线指标就可能很好看，线上效果却下降。

| 常见坑 | 后果 | 规避方式 |
|---|---|---|
| 不加位置编码 | 顺序信息丢失，模型分不清先后行为 | 加位置编码、时间间隔编码或行为时间特征 |
| 序列太长 | Self-Attention 的 \(O(n^2)\) 计算和显存压力变大 | 截断最近行为、按行为类型采样、限制最大长度 |
| 不做 padding mask | padding token 污染注意力权重 | 明确区分真实 token 和 padding token |
| 训练/线上口径不一致 | 离线 AUC 正常，线上 CTR 掉点 | 统一特征生成逻辑，做线上线下一致性校验 |
| 候选商品泄漏进历史 | 评估虚高，上线后失效 | 按时间切分样本，保证历史只包含曝光前行为 |
| 负样本构造不合理 | 模型学到曝光机制偏差 | 使用真实曝光未点击样本，控制采样分布 |
| 只看离线指标 | 模型可能优化了错误目标 | 同时看 AUC、LogLoss、校准、线上实验 |

一个典型错误是：用户历史序列长度不一，短序列用 0 补齐。如果模型没有 padding mask，就会把大量 0 token 当作真实行为参与注意力计算。短序列用户的表示会被无效 token 拉偏，排序分数自然不稳定。

另一个更隐蔽的错误是候选泄漏。比如构造训练样本时，把“用户点击候选商品”这次行为也放进历史序列。这样模型在训练时几乎提前看到了答案，离线指标会虚高，但线上预测时并不存在这条未来行为。

工程上还要权衡序列长度。长度越长，理论上信息越多，但 Self-Attention 的计算复杂度约为：

$$
O(n^2)
$$

这里 \(n\) 是序列长度。历史长度从 50 增加到 200，注意力矩阵规模会从 \(50^2=2500\) 增加到 \(200^2=40000\)。这不是线性增长，而是平方增长。

因此实际系统里经常不会无脑保留所有行为，而是选择最近 N 条、高价值行为、同类目行为，或者按时间窗口截断。

## 替代方案与适用边界

BST 不是唯一选择。推荐排序模型的选择依据通常是：序列依赖强不强、是否需要全局行为交互、算力是否允许、线上延迟是否敏感。

| 方法 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|
| BST | 能建模行为间全局关系，并行计算友好 | 计算成本较高，依赖序列质量 | 电商、广告、信息流等序列信号强的排序 |
| DIN | 结构相对轻，能针对候选商品做兴趣激活 | 对行为间顺序关系建模较弱 | 更关注候选商品与历史兴趣局部匹配 |
| GRU4Rec | 显式按时间建模序列演化 | 串行计算较重，长距离依赖可能衰减 | 会话推荐、短序列连续行为 |
| SASRec | 使用 Self-Attention 做序列推荐，结构清晰 | 更偏序列建模，工程特征融合需扩展 | 序列推荐、下一物品预测 |
| 简单 Embedding + MLP | 实现简单、延迟低 | 难建模复杂序列关系 | 行为很少、序列关系弱、资源紧张场景 |

如果场景更关注“当前候选商品和哪些历史行为相似”，DIN 可能更轻。比如候选是“跑鞋”，模型只需要从历史里激活“运动袜、健身服、跑步手表”等相关行为，不一定要完整建模行为之间的顺序依赖。

如果场景需要显式建模“兴趣如何随时间变化”，BST 或 SASRec 更合适。比如用户先看手机，再看手机壳，再看耳机，这种连续迁移关系本身就有价值。

如果序列特别长且资源紧张，BST 可能不是第一选择。可以先做行为筛选、分桶聚合、双塔召回、DIN 精排，或者使用更轻量的注意力结构。模型复杂度应服务于业务收益，而不是因为 Transformer 强就默认使用 Transformer。
