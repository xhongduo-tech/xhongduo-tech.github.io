## 核心结论

INT8/INT4 量化的本质，是把原本用浮点数存储的权重改成更短的整数表示，再配合一个缩放系数 `scale` 恢复近似值。这里的“量化”可以先理解成“把连续的小数压到有限个刻度上”。这样做的直接收益是两点：模型更小，推理时整数乘加更快。

最常见的对称量化中，INT8 会把权重映射到 $[-127,127]$。若权重张量记为 $W$，常用定义是：

$$
scale = \frac{\max(|W|)}{127}, \qquad
W_q = clip\left(round\left(\frac{W}{scale}\right), -127, 127\right)
$$

反量化后的近似权重为：

$$
\hat W = W_q \cdot scale
$$

误差主要来自两部分：

| 误差类型 | 含义 | 典型来源 | 是否一定出现 |
| --- | --- | --- | --- |
| 舍入误差 | 把连续值贴到最近整数刻度带来的偏差 | `round(W / scale)` | 几乎总会出现 |
| 截断误差 | 超出可表示范围后被裁剪的偏差 | `clip(..., -127, 127)` | 只有越界时出现 |

INT8 通常已经能在较小精度损失下获得明显加速；INT4 把每个权重只保留 4 bit，压缩更激进，但误差会更大。GPTQ 的价值就在这里。GPTQ 是一种后训练量化方法，“后训练”可以理解为“不重新训练，只在已有模型上做压缩”。它利用近似 Hessian 二阶信息来判断哪些权重误差更敏感，再在量化过程中补偿误差，所以 3 到 4 bit 量化仍然能维持较好的模型效果。

玩具例子最能说明这个过程。假设某个值是 `3.1415`，若 `scale = 0.01`，则：

$$
q = round(3.1415 / 0.01) = round(314.15) = 314
$$

恢复后得到 `3.14`，误差是 `-0.0015`。这就是量化最核心的直观图景：不是“变成另一个复杂格式”，而是“先除以刻度、取整、再乘回刻度”。

## 问题定义与边界

要讨论量化，先把问题定义清楚：我们不是在追求“权重值完全不变”，而是在追求“用更少比特表示参数，同时让模型输出变化尽量小”。

这里有三个边界必须先说明。

第一，量化对象是谁。本文主要讨论权重量化，尤其是大语言模型中线性层权重的 INT8/INT4 压缩。激活量化、KV Cache 量化、训练时量化不在主线范围内。

第二，量化粒度是什么。所谓“逐张量量化”，就是整个张量只用一个 `scale`；所谓“逐通道量化”，就是每个输出通道各自用一个 `scale`。这里的“通道”对线性层可以粗略理解为“每一行权重向量单独处理”。粒度越细，误差通常越小，但实现和硬件适配更复杂。

第三，硬件支持决定上限。不是所有设备都能高效执行 INT4，也不是所有推理框架都支持逐通道或 GPTQ 格式。工程上不能只看理论误差，还要看实际 kernel 是否支持。

从误差角度，量化后的总误差可以拆成：

$$
\Delta = \hat W - W = \Delta_{round} + \Delta_{trunc}
$$

其中：

$$
\Delta_{round} = round(W/scale)\cdot scale - W
$$

它表示舍入产生的误差。另一部分是越界后裁剪带来的误差，可写成：

$$
\Delta_{trunc} =
clip(round(W/scale), q_{min}, q_{max})\cdot scale
-
round(W/scale)\cdot scale
$$

如果没有越界，$\Delta_{trunc}=0$。

可以用一个简单玩具例子看清边界。假设 INT8，表示范围是 $[-127,127]$，某一组权重的最大绝对值是 `1.27`，则 `scale = 1.27 / 127 = 0.01`。

- 若 `W = 0.736`，则 `W / scale = 73.6`，取整得到 `74`，恢复后是 `0.74`，误差 `0.004`，这是舍入误差。
- 若 `W = 1.50`，则 `W / scale = 150`，已经超出 `127`，裁剪后只能记成 `127`，恢复值变为 `1.27`，误差 `-0.23`，这是截断误差。

这说明一个关键事实：在同一位宽下，`scale` 的选择决定了误差结构。`scale` 设得大，越界少，但每一级刻度更粗，舍入误差变大；`scale` 设得小，刻度更细，但越界更多，截断误差变大。量化本质上是在这两种误差之间找平衡。

## 核心机制与推导

先看最基础的对称量化。所谓“对称”，意思是整数范围围绕 0 对称展开，例如 INT8 用 `-127` 到 `127`。它的好处是实现简单，且整数乘法路径直接。对很多推理硬件来说，这意味着更成熟的算子支持和更低的实现成本。

### 1. INT8/INT4 为什么会有误差

对称量化把浮点空间压成有限个桶。若位宽是 $b$，有符号整数常见可表示范围近似为：

$$
q \in [-(2^{b-1}-1), 2^{b-1}-1]
$$

所以：

- INT8 约为 $[-127,127]$
- INT4 约为 $[-7,7]$

位宽越低，桶越少，单个桶的宽度越大，因此误差通常越大。若同样使用最大绝对值决定 `scale`，INT4 的公式只是把分母从 `127` 换成 `7`：

$$
scale_{int4} = \frac{\max(|W|)}{7}
$$

这会让步长明显变粗。举例来说，若某层最大绝对值仍是 `1.27`：

- INT8 的 `scale = 0.01`
- INT4 的 `scale \approx 0.1814`

此时 `0.736` 在 INT8 下可恢复为 `0.74`，而在 INT4 下：

$$
q = round(0.736 / 0.1814) = round(4.06) = 4
$$

恢复后约为 `0.7256`。误差虽然仍可接受，但已经明显大于 INT8。

### 2. 逐张量和逐通道为什么差很多

假设一个线性层有两个输出通道：

| 通道 | 权重范围 |
| --- | --- |
| 通道 A | `[-0.1, 0.1]` |
| 通道 B | `[-1.2, 1.2]` |

如果逐张量量化，整个层只能共享一个 `scale`，它会被通道 B 的大值主导，接近 `1.2 / 127`。结果是通道 A 的很多小值会落在非常稀疏的刻度上，信息损失严重。

如果逐通道量化：

- 通道 A 用自己的小 `scale`
- 通道 B 用自己的大 `scale`

那么两个通道都能在各自动态范围里更精细地表示。这就是为什么逐通道量化通常更准确。这里的“动态范围”可以理解成“这一组数从最小到最大的跨度”。

### 3. GPTQ 为什么比普通 INT4 更稳

普通后训练量化通常只看单个权重的数值大小，做法像是：“把每个数独立四舍五入”。问题在于，模型输出不是单个权重决定的，而是多个权重共同作用的结果。有些权重即使数值不大，对输出也非常敏感。

GPTQ 的核心思想是：量化某个权重时，不只看它自身数值，还看它对输出误差的影响。这个影响由 Hessian 近似描述。Hessian 可以白话理解为“损失函数在各个方向上有多弯”，弯得越厉害，说明该方向上的误差越敏感。

若某层权重向量为 $w$，量化后为 $\hat w$，令误差为 $\delta = \hat w - w$，则在二阶近似下，输出相关损失变化可写成：

$$
\Delta \mathcal{L} \approx \frac{1}{2}\delta^T H \delta
$$

其中 $H$ 是 Hessian 或其近似矩阵。GPTQ 不直接最小化每个权重的绝对误差，而是尽量最小化这个二次型，也就是“对模型行为真正有影响的误差”。

更具体地说，GPTQ 通常按列或按块顺序量化权重。每量化完一部分，就把这部分产生的误差，依据 Hessian 逆矩阵的信息补偿到还未量化的权重上。于是后面的权重不是在原始参数上继续取整，而是在“已经修正过误差的参数”上量化。

可把其核心伪代码理解为：

```python
residual = w - w_hat
correction = H_inv @ residual
w_remaining = w_remaining - correction
```

这里的 `residual` 是“当前已经造成的误差”，`correction` 是“把这部分误差重新分摊给还没量化的参数”。这就是 GPTQ 能在 4 bit 仍然保持较好效果的关键：它不是简单压缩，而是在量化顺序中持续做误差补偿。

### 4. 真实工程例子

真实工程里，7B 到 70B 级别的大语言模型，常见做法是把线性层权重做 4 bit GPTQ 量化，而激活仍保持 FP16/BF16。原因很直接：

- 权重占显存大头，先压它最划算
- 激活动态变化更大，硬量化风险更高
- 很多推理框架已经有成熟的 4 bit 权重加载和 matmul kernel

例如一个原本需要十几 GB 甚至几十 GB 显存的模型，经过 4 bit 权重量化后，权重存储大约变成原来的四分之一，配合分组量化和少量元数据，通常就能从“多卡才能放下”变成“单卡可部署”。这就是量化在模型部署里最直接的工程价值。

## 代码实现

下面先给一个最小可运行实现，演示对称量化、反量化，以及逐通道量化误差对比。代码只依赖 `numpy`，可以直接运行。

```python
import numpy as np

def symmetric_quantize_per_tensor(w, bits=8):
    qmax = 2 ** (bits - 1) - 1
    scale = np.max(np.abs(w)) / qmax
    if scale == 0:
        scale = 1.0
    q = np.clip(np.round(w / scale), -qmax, qmax).astype(np.int32)
    w_hat = q * scale
    return q, scale, w_hat

def symmetric_quantize_per_channel(w, bits=8, axis=0):
    qmax = 2 ** (bits - 1) - 1
    scales = np.max(np.abs(w), axis=axis, keepdims=True) / qmax
    scales = np.where(scales == 0, 1.0, scales)
    q = np.clip(np.round(w / scales), -qmax, qmax).astype(np.int32)
    w_hat = q * scales
    return q, scales, w_hat

def mse(a, b):
    return np.mean((a - b) ** 2)

# 玩具例子
w = np.array([0.73, 0.736, 1.27], dtype=np.float32)
q, scale, w_hat = symmetric_quantize_per_tensor(w, bits=8)

assert q.tolist() == [73, 74, 127]
assert abs(scale - 0.01) < 1e-6
assert np.allclose(w_hat, np.array([0.73, 0.74, 1.27]), atol=1e-6)

# 逐张量 vs 逐通道
W = np.array([
    [0.05, -0.08, 0.10],
    [0.90, -1.20, 1.10],
], dtype=np.float32)

_, _, W_hat_tensor = symmetric_quantize_per_tensor(W, bits=8)
_, _, W_hat_channel = symmetric_quantize_per_channel(W, bits=8, axis=1)

err_tensor = mse(W, W_hat_tensor)
err_channel = mse(W, W_hat_channel)

assert err_channel <= err_tensor + 1e-12
print("per-tensor mse =", err_tensor)
print("per-channel mse =", err_channel)
```

这段代码体现了量化实现的最小闭环：

1. 求 `scale`
2. 执行 `round`
3. 执行 `clip`
4. 存储整数值
5. 反量化评估误差

如果把它扩展到 GPTQ，主要增加三件事：

| 组件 | 作用 | 代价 |
| --- | --- | --- |
| 校准样本 | 估计层输入分布和 Hessian 近似 | 需要少量前向数据 |
| Hessian 近似或其逆 | 衡量误差敏感方向 | 计算和显存开销增加 |
| 顺序误差补偿 | 每量化一部分就更新剩余权重 | 实现更复杂 |

一个简化的 GPTQ 风格伪代码如下：

```python
for layer in layers:
    X = collect_calibration_inputs(layer)
    H = X.T @ X
    H_inv = approximate_inverse(H)

    W = layer.weight.copy()
    for col in range(W.shape[1]):
        w_col = W[:, col]
        scale = np.max(np.abs(w_col)) / 7.0   # 以 INT4 为例
        scale = 1.0 if scale == 0 else scale

        q_col = np.clip(np.round(w_col / scale), -7, 7)
        wq_col = q_col * scale

        err = (w_col - wq_col).reshape(-1, 1)
        W[:, col] = wq_col

        # 用二阶信息把当前误差补偿到后续未量化列
        if col + 1 < W.shape[1]:
            correction = err @ (H_inv[col, col + 1:].reshape(1, -1) / H_inv[col, col])
            W[:, col + 1:] -= correction
```

这段伪代码没有覆盖工程细节，比如分组量化、块状更新、数值稳定性处理，但已经能表达 GPTQ 的本质：不是“每列各自量化完事”，而是“量化当前列后，立刻用二阶信息修正未来列”。

真实工程例子里，量化一个 Transformer block 的线性层时，通常会：

- 用一小批校准文本跑前向，收集每层输入激活
- 对 `q_proj/k_proj/v_proj/o_proj` 和 MLP 的线性层分别量化
- 采用逐行或分组 scale，而不是整层一个 scale
- 权重 INT4，激活保留高精度
- 导出为推理框架可识别的权重格式

## 工程权衡与常见坑

量化不是“位宽越低越好”，而是“在硬件支持下，用最小代价获得可接受精度”。这要求明确权衡。

第一类权衡是粒度。逐张量实现最简单，但在通道差异明显的层误差会被放大；逐通道更准，但需要额外存储多个 `scale`，并要求 kernel 支持对应布局。

第二类权衡是精度和速度。INT8 兼容性最好，很多硬件都有成熟加速；INT4 压缩率更高，但是否真能更快，取决于推理框架是否有高效低位宽 kernel。没有高效 kernel 时，理论上的 4 bit 节省，可能换不来实际吞吐提升。

第三类权衡是量化成本。普通对称量化几乎立刻可做；GPTQ 需要校准数据、Hessian 近似、逐层处理，量化阶段更慢，但部署收益更大。

常见坑可以直接列出来：

| 坑 | 原因 | 解决策略 |
| --- | --- | --- |
| 小通道被大通道拖累 | 逐张量 `scale` 被极端值主导 | 对线性层优先用逐通道或逐行 |
| 截断误差过大 | `scale` 太小或分布有离群点 | 检查离群值，必要时做分组量化 |
| INT4 精度骤降 | 只做简单 round，没有误差补偿 | 改用 GPTQ、AWQ 等更强 PTQ 方法 |
| 理论压缩但推理没变快 | 框架没有高效 INT4 kernel | 先验证目标硬件和推理栈支持 |
| 校准结果不稳定 | 样本过少或分布失真 | 校准数据覆盖真实任务输入 |
| 只看权重误差不看输出误差 | 某些层对误差更敏感 | 优先按层评估 perplexity 或任务指标 |

还要注意误差会跨层传播。若第 $l$ 层输出误差为 $\epsilon_l$，下一层线性变换后误差大致会继续被权重放大：

$$
\epsilon_{l+1} \approx W_{l+1}\epsilon_l + \delta_{l+1}
$$

其中 $\delta_{l+1}$ 是第 $l+1$ 层新引入的量化误差。位宽越低，这种叠加越明显。GPTQ 的“顺序量化 + Hessian 补偿”正是在控制这种累积效应。

一个真实工程坑是：某层大部分通道都很平稳，只有少数离群通道权重特别大。若整层共用 `scale`，这些离群通道会迫使步长变粗，导致绝大多数普通通道的分辨率被浪费。此时逐通道量化通常能明显降低误差；如果硬件不支持逐通道，就要考虑分组量化或跳过最敏感层。

## 替代方案与适用边界

GPTQ 很强，但不是唯一方案，也不是任何场景都应优先选。

先看几种常见替代路线：

| 方案 | 数据需求 | 精度表现 | 部署速度 | 适用场景 |
| --- | --- | --- | --- | --- |
| 普通 INT8 对称量化 | 很少 | 通常稳定 | 很快 | 首选基础部署 |
| GPTQ | 少量校准样本 | 3-4 bit 下通常较高 | 中等 | 大模型后训练压缩 |
| AWQ | 少量校准样本 | 对激活敏感层更稳 | 中等 | LLM 权重量化 |
| QAT | 训练数据和训练过程 | 通常最高 | 慢 | 有训练预算、追求极限精度 |
| 非对称量化 | 视框架而定 | 中高 | 中等 | 数据分布明显偏移 0 时 |

### 1. GPTQ 适合什么场景

GPTQ 适合已经训练好的大模型，需要一次性压缩部署，且没有资源重新训练的情况。它尤其适合 3 到 4 bit 权重量化，因为这个位宽下普通舍入法常常不够稳，而 GPTQ 的二阶补偿能显著降低损失。

### 2. 什么时候直接用 INT8 就够了

如果目标是上线一个稳定版本，而不是榨取极限显存，INT8 往往是更务实的选择。原因有三点：

- 支持最广
- 精度风险最低
- 工程接入成本最小

很多业务模型在 INT8 下已经能获得明显的存储和吞吐收益，此时再进一步压到 INT4，边际收益可能不值得额外复杂度。

### 3. 什么时候要考虑 QAT

QAT 是“量化感知训练”，可以理解为“训练时就把量化噪声纳入学习过程”。如果你有完整训练数据、训练预算，并且模型对低位宽极其敏感，QAT 往往能取得最好结果。但它的代价也最明显：实现复杂、训练成本高、周期长。

### 4. 非对称量化的边界

非对称量化会引入零点 `zero_point`，适合数据分布不围绕 0 的情况。它的基本形式可写作：

$$
q = round(x / scale) + z
$$

其中 $z$ 是零点。这样能更充分利用整数范围，但会让实现更复杂，尤其在矩阵乘法时需要额外处理零点项。对于现代 LLM 权重量化，很多工程仍偏向对称量化，因为路径更简单、性能更稳定。

结论可以压缩成一句话：8 bit 场景优先简单对称量化，4 bit 场景优先 GPTQ/AWQ 这类更强的后训练量化，只有在你能承担训练成本时，QAT 才是更高上限的方案。
