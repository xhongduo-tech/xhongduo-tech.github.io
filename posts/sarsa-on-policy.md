## 核心结论

SARSA 是一种**同策略**时序差分控制算法。所谓同策略，白话说就是“执行用什么策略，学习也按这套策略的真实行为来算账”。它不假设下一步一定会选到最优动作，而是直接使用当前策略实际采样到的下一动作 $A_{t+1}$ 做更新。

它的核心更新式是：

$$
Q(S_t,A_t)\leftarrow Q(S_t,A_t)+\alpha\Big(R_{t+1}+\gamma Q(S_{t+1},A_{t+1})-Q(S_t,A_t)\Big)
$$

其中 $A_{t+1}\sim \pi(\cdot|S_{t+1})$，意思是“下一动作不是凭空取最大值，而是按当前策略 $\pi$ 在状态 $S_{t+1}$ 下真的采样出来”。

这带来一个直接后果：SARSA 学到的不是“如果我从下一步开始永远都最优，会有多好”，而是“如果我继续按当前这套带探索的策略行动，长期回报会是多少”。因此，探索动作的风险会进入估值过程。环境越随机、探索代价越真实，SARSA 往往越保守。

最简单的对比可以这样记：

| 算法 | 它在估什么 |
| --- | --- |
| SARSA | 我现在这套策略真实会怎么走、最后拿多少回报 |
| Q-learning | 如果我下一步起总能选最优动作，理论上能拿多少回报 |

在经典的悬崖行走问题里，这种差异很明显。SARSA 往往学到离悬崖远一点的路线，因为它知道自己还会继续探索；Q-learning 则更容易贴着悬崖边走，因为它的目标值里默认下一步总能纠正回来。

## 问题定义与边界

SARSA 解决的问题是：在一个**马尔可夫决策过程**里，智能体一边在线交互，一边在带探索的行为策略下学习动作价值函数 $Q(s,a)$。马尔可夫决策过程可以先粗略理解为“当前状态和动作决定下一步分布”的决策环境。

这里的目标不是一次性从静态数据里拟合一个模型，也不是不接触环境就直接求全局最优解。SARSA 的工作方式是：

1. 先按当前策略选动作。
2. 与环境交互得到奖励和下一状态。
3. 再按当前策略选下一动作。
4. 用这一整条真实经历更新当前动作价值。

它适合“边试边学”的在线场景，例如迷宫导航、机器人移动、调度控制、游戏智能体。它不擅长的边界也很明确：

| 概念 | 含义 | 在 SARSA 里的作用 |
| --- | --- | --- |
| $S_t$ | 当前状态，白话说就是“此刻我看到的环境局面” | 决定当前可选动作与当前估值位置 |
| $A_t$ | 当前动作，白话说就是“这一步我真的执行了什么” | 是当前要更新的对象 |
| $R_{t+1}$ | 下一时刻奖励，白话说就是“这一步立刻收到的反馈分数” | 构成 TD 目标的即时项 |
| $S_{t+1}$ | 下一状态 | 决定是否终止，以及后续价值 |
| $A_{t+1}$ | 下一动作 | 由当前策略真实采样，用于 bootstrap |
| 终止状态 | 回合结束状态，白话说就是“到头了，后面没有下一步” | 后续价值记为 0，不再 bootstrap |

它的几个边界需要提前说清：

1. SARSA 学的是**当前行为策略**对应的价值，不是脱离当前策略的理想最优价值。
2. 如果策略持续变化，学习目标也在变化，因为“你学的是哪套策略”本身就在变。
3. 它天然依赖在线交互；如果只有离线日志数据，直接照搬 SARSA 通常不成立。
4. 它的“保守”不是数学上额外加了安全项，而是因为探索风险被真实计入了更新目标。

一个玩具例子是走迷宫。假设角色在岔路口有 10% 概率乱走，那么 SARSA 学到的是“这 10% 乱走也存在时，我这条路线的真实长期回报”。一个真实工程例子是仓储 AGV 在窄通道里避障，哪怕主策略大多时候很合理，只要还保留探索，贴边行驶的风险就会在 SARSA 的估值里被放大。

## 核心机制与推导

SARSA 这个名字来自五个量的顺序：$S,A,R,S,A$。意思不是缩写好记而已，而是它的更新链路本身：

1. 处在状态 $S_t$。
2. 采取动作 $A_t$。
3. 收到奖励 $R_{t+1}$。
4. 到达下一状态 $S_{t+1}$。
5. 按当前策略再选下一动作 $A_{t+1}$。
6. 用这五个量更新 $Q(S_t,A_t)$。

它的 TD 误差是：

$$
\delta_t = R_{t+1}+\gamma Q(S_{t+1},A_{t+1})-Q(S_t,A_t)
$$

这里的 **TD 误差**，白话说就是“当前估计和一步修正后的目标之间差了多少”。于是更新可以写成：

$$
Q(S_t,A_t)\leftarrow Q(S_t,A_t)+\alpha \delta_t
$$

其中 $\alpha$ 是学习率，控制“每次改多少”；$\gamma$ 是折扣因子，控制“未来值占多大权重”。

为什么它是同策略？关键不在“更新时用了下一状态”，而在“下一动作 $A_{t+1}$ 来自当前行为策略本身”。如果策略是 $\epsilon$-greedy，那么下一动作就不是纯贪心动作，而是“以大概率选当前最好动作，小概率随机探索”的真实结果。学习和执行共享同一个动作分布，这就是同策略。

看一个最小数值例子：

- 当前估值：$Q(S_t,A_t)=4.0$
- 学习率：$\alpha=0.1$
- 折扣因子：$\gamma=0.9$
- 这一步奖励：$R_{t+1}=-1$
- 下一步按当前策略实际采样到的动作价值：$Q(S_{t+1},A_{t+1})=5.0$

那么目标值是：

$$
-1 + 0.9\times 5.0 = 3.5
$$

TD 误差是：

$$
\delta_t = 3.5 - 4.0 = -0.5
$$

更新后：

$$
Q \leftarrow 4.0 + 0.1\times(-0.5)=3.95
$$

新手版理解是：你原来以为这一步值 4.0，但这次真实走下来发现“立刻吃了一个 -1，下一步按你当前策略接着走，大概只接得上 5.0 的后续价值”，所以修正后的目标只有 3.5，当前估计应该往下调一点，而不是一下子改到 3.5。

终止状态要单独处理：

$$
S_{t+1}\text{ 为终止状态时 }Q(S_{t+1},A_{t+1})=0
$$

于是目标退化成：

$$
\text{target}=R_{t+1}
$$

这点非常重要。因为终止后没有“下一步还会怎样”，再去 bootstrap 就是在凭空加未来价值。

SARSA 和 Q-learning 的核心差异可以压缩成一张表：

| 维度 | SARSA | Q-learning |
| --- | --- | --- |
| 下一动作来源 | 当前策略真实采样出的 $A_{t+1}$ | 直接取 $\max_{a'}Q(S_{t+1},a')$ |
| 是否同策略 | 是 | 否，属于异策略 |
| 学到的对象 | 当前行为策略的价值 | 贪心最优策略的价值近似 |
| 对探索风险的态度 | 计入估值 | 目标里不直接计入 |
| 在随机环境中的行为 | 往往更保守 | 往往更激进 |

经典“悬崖行走”现象就是这个机制的外显结果。智能体如果贴着悬崖边走，理论上最短，但只要还有探索，偶尔一个随机动作就可能掉下去。SARSA 在估值时会把“我下一步未必稳稳选最优”这件事算进去，所以悬崖边路径的价值会被压低。

## 代码实现

实现 SARSA 时，最常见的错误是嘴上说同策略，代码里却偷偷写成了 `max`。只要你在更新目标时用的是“下一状态下的最大 Q 值”，你写的就不是 SARSA，而是 Q-learning。

下面给一个最小可运行的 Python 版本，展示完整回合循环。为了突出算法本身，环境使用一个手写的两步玩具环境。

```python
from collections import defaultdict
import random

class TinyEnv:
    """
    状态 0: 起点
    动作 0: 稳妥，奖励 +1，直接结束
    动作 1: 冒险，奖励 +0，到状态 1
    状态 1:
      动作 0: 奖励 +2，结束
      动作 1: 奖励 -2，结束
    """
    def reset(self):
        self.state = 0
        return self.state

    def step(self, action):
        s = self.state
        if s == 0:
            if action == 0:
                return None, 1.0, True
            else:
                self.state = 1
                return 1, 0.0, False
        elif s == 1:
            if action == 0:
                return None, 2.0, True
            else:
                return None, -2.0, True
        else:
            raise ValueError("invalid state")

def epsilon_greedy(Q, state, actions, epsilon=0.1):
    if random.random() < epsilon:
        return random.choice(actions)
    q_vals = [Q[(state, a)] for a in actions]
    max_q = max(q_vals)
    best = [a for a in actions if Q[(state, a)] == max_q]
    return random.choice(best)

def sarsa(env, episodes=500, alpha=0.1, gamma=0.9, epsilon=0.1):
    Q = defaultdict(float)
    actions = {
        0: [0, 1],
        1: [0, 1],
    }

    for _ in range(episodes):
        s = env.reset()
        a = epsilon_greedy(Q, s, actions[s], epsilon)

        done = False
        while not done:
            s_next, r, done = env.step(a)

            if done:
                target = r
                Q[(s, a)] += alpha * (target - Q[(s, a)])
            else:
                a_next = epsilon_greedy(Q, s_next, actions[s_next], epsilon)
                target = r + gamma * Q[(s_next, a_next)]
                Q[(s, a)] += alpha * (target - Q[(s, a)])
                s, a = s_next, a_next

    return Q

# 单步更新的数值例子
Q = defaultdict(float)
Q[("s", "a")] = 4.0
Q[("s_next", "a_next")] = 5.0
alpha = 0.1
gamma = 0.9
r = -1.0
target = r + gamma * Q[("s_next", "a_next")]
Q[("s", "a")] += alpha * (target - Q[("s", "a")])
assert abs(Q[("s", "a")] - 3.95) < 1e-9

# 训练后，稳妥动作通常会更有吸引力
random.seed(0)
env = TinyEnv()
Q = sarsa(env, episodes=2000, alpha=0.1, gamma=0.9, epsilon=0.1)
assert Q[(0, 0)] > Q[(0, 1)]
```

这段代码有三个实现点不能丢：

1. 回合开始先在当前状态 $S_t$ 上选出当前动作 $A_t$。
2. 若未终止，必须在 $S_{t+1}$ 上按当前策略采样出 $A_{t+1}$。
3. 更新完成后，要把 `(S, A)` 推进到 `(S_next, A_next)`。

伪代码可以更直观地看出结构：

```python
for episode in range(num_episodes):
    S = env.reset()
    A = policy(S)
    done = False

    while not done:
        S_next, R, done = env.step(A)

        if done:
            target = R
            Q[S, A] += alpha * (target - Q[S, A])
        else:
            A_next = policy(S_next)   # 这里是采样，不是 argmax
            target = R + gamma * Q[S_next, A_next]
            Q[S, A] += alpha * (target - Q[S, A])
            S, A = S_next, A_next
```

这里的 `policy(S_next)` 必须强调两次：它不是“把当前最大值对应动作取出来”，而是“按当前行为策略采样出动作”。如果你的策略是 $\epsilon$-greedy，那么这个函数就应该保留探索概率。

真实工程例子里，结构完全一样。比如仓储 AGV 在通道网格中学习路径，每一步状态可以是“当前位置、朝向、周围障碍占用、当前任务点”，动作可以是“前进、左转、右转、等待”。只要控制器还保留探索，SARSA 更新时就该使用“控制器在下一状态真实会发出的下一条动作指令”，而不是假设“下一步一定走最优控制”。

## 工程权衡与常见坑

SARSA 的优点和代价来自同一个地方：它把探索风险写进了价值函数。

当探索本身会造成真实损失时，这通常是优点。比如狭窄通道中的 AGV，如果贴着货架边缘走，理论上距离更短，但探索时稍微偏一点就可能擦碰。SARSA 会逐渐学到“离边界远一点的路线虽然绕，但整体更稳”，因为它估的不是理想执行效果，而是带探索噪声的实际执行效果。

但如果环境很干净、执行噪声很小、目标只是追求最高平均回报，这种保守性就可能显得“学得慢”或“路线不够激进”。这不是它没学到，而是它优化的对象本来就更接近“真实在线表现”。

工程里最常见的坑如下：

| 常见坑 | 结果 | 正确做法 |
| --- | --- | --- |
| 把 $A_{t+1}$ 写成 $\arg\max_a Q(S_{t+1},a)$ | 算法变成 Q-learning | 必须按当前行为策略采样下一动作 |
| 终止状态继续 bootstrap | 给终点凭空加未来价值 | `done=True` 时目标直接等于奖励 |
| 策略变了却没意识到学习目标也变了 | 训练行为前后不一致 | 明确当前在学习哪套 $\pi$ |
| 只看平均回报 | 掩盖碰撞率、失败率、长尾风险 | 同时看失败率、最坏代价、方差 |
| 直接套经验回放 | 破坏严格同策略前提 | 要么谨慎修正，要么换更合适算法 |

再展开几个容易被忽略的点。

第一，策略改动会改学习目标。比如前 1 万步用 $\epsilon=0.3$，后 1 万步切到 $\epsilon=0.05$，你学到的其实不是一条静态策略价值函数，而是一个随行为策略变化不断漂移的对象。实践中这不是不能做，而是要知道自己在做什么。

第二，评估指标不能只看平均回报。对安全敏感系统，平均回报提升但碰撞率上升，通常不是好结果。SARSA 的价值恰恰经常体现在“均值差不多，但最坏情况更少”。

第三，经验回放和严格同策略有冲突。因为回放池里的旧样本往往来自过去的旧策略，而 SARSA 目标里的 $A_{t+1}$ 依赖当前策略。如果你把旧轨迹无脑拿出来重放，很容易理论和实现脱节。现代深度强化学习里当然有很多变体能处理近似问题，但那已经不是朴素 SARSA 的原始语义。

## 替代方案与适用边界

SARSA 不是“比 Q-learning 更先进”，也不是“更安全所以总更好”。它只是优化目标不同。

同一个悬崖行走场景里，三种常见方法可以这样对比：

| 算法 | 目标中的下一动作来源 | 是否同策略 | 风险偏好 | 典型使用场景 |
| --- | --- | --- | --- | --- |
| SARSA | 当前策略真实采样动作 | 是 | 偏保守 | 在线学习且探索有真实代价 |
| Q-learning | 下一状态下的最大动作 | 否 | 偏激进 | 更关心理想最优值，允许探索风险与学习目标分离 |
| Expected SARSA | 当前策略下各动作的期望值 | 可同策略 | 通常介于两者之间 | 想降低采样方差，又保留策略分布信息 |

**Expected SARSA** 可以白话理解为：它不只看“下一步这次正好采样到哪个动作”，而是对当前策略在下一状态的所有可能动作求期望。因此目标是：

$$
R_{t+1} + \gamma \sum_a \pi(a|S_{t+1})Q(S_{t+1},a)
$$

它比标准 SARSA 的采样噪声更小，在某些环境中训练更平滑。

如果还是用悬崖行走来理解：

1. SARSA 会学到更绕远但更安全的路线，因为它知道自己未来还会继续探索。
2. Q-learning 往往贴近悬崖边，因为它估值时假设下一步总能选对最优动作。
3. Expected SARSA 介于两者之间，它不看单次采样动作，而看当前策略分布下的平均后果。

什么时候优先选 SARSA？

1. 需要在线学习，策略和环境实时交互。
2. 探索动作本身有真实成本，比如碰撞、跌落、超时、能耗尖峰。
3. 更关心“真实执行表现”而不是“理想最优值”。
4. 状态和动作规模还在可以直接维护或近似维护 $Q(s,a)$ 的范围内。

什么时候不一定选它？

1. 你更想逼近最优贪心策略的价值。
2. 训练阶段允许较高风险，部署时再单独换成贪心执行。
3. 你依赖大规模离线数据或重度经验回放。
4. 你更在意收敛到最优控制的效率，而不是探索期间的保守行为。

如果只想先抓住一个现象再回看公式，最推荐先看悬崖行走：同样的最短路，为什么一个算法更喜欢贴边，一个算法更喜欢绕一点走。看懂这个行为差异，SARSA 的“同策略”含义就不再抽象。
