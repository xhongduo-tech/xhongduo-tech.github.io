## 为什么反向传播其实就是“把链式法则系统化”

训练神经网络的目标，是让损失函数 `L(theta)` 随参数 `theta` 下降。问题在于：参数通常不直接出现在损失函数里，而是经过很多层中间变量间接影响最终输出。

这就是反向传播要解决的问题。它并不神秘，本质上只是两件事：

- 用**梯度**描述“往哪个方向改参数，损失涨得最快”
- 用**链式法则**把这种影响沿着计算图一层层传回去

理解这套机制之后，你会发现“反向传播”不是另一门独立的数学，而是微积分在计算图上的工程化落地。

---

## 从导数到梯度：多变量函数如何描述变化

### 单变量导数：局部斜率

对函数 `y = f(x)`，导数 `dy/dx` 描述的是：当 `x` 在某一点附近发生微小变化时，`y` 会按什么速度变化。

如果 `dy/dx > 0`，说明往右走函数值增大；如果 `dy/dx < 0`，说明往右走函数值减小。

### 多变量函数：偏导与梯度

神经网络中的函数往往是：

```text
f(x_1, x_2, ..., x_n)
```

这时我们需要分别考察每个变量的影响，于是有了偏导数：

```text
∂f/∂x_i
```

把所有偏导收集起来，就得到梯度向量：

```text
∇f(x) = [∂f/∂x_1, ∂f/∂x_2, ..., ∂f/∂x_n]^T
```

梯度有一个极重要的几何意义：**它指向函数增长最快的方向。**  
因此，梯度下降法才会做下面这件事：

```text
x <- x - η∇f(x)
```

也就是沿着“最陡上升方向”的反方向走一步。

### 方向导数：梯度为什么是“最陡”的

如果我们沿单位向量 `v` 的方向考察函数变化率，得到方向导数：

```text
D_v f(x) = ∇f(x)^T v
```

根据柯西-施瓦茨不等式：

```text
∇f(x)^T v <= ||∇f(x)|| ||v||
```

当 `v` 与 `∇f(x)` 同方向时，方向导数最大。这个结论说明：梯度不是“随便定义出来的向量”，它确实是局部最陡上升方向。

## 链式法则：复杂函数为什么可以分层求导

### 标量版本

如果：

```text
z = f(y), y = g(x)
```

那么：

```text
dz/dx = dz/dy * dy/dx
```

这就是链式法则。它告诉我们：`x` 对 `z` 的影响，等于 `x` 先影响 `y`，再由 `y` 影响 `z`。

### 计算图版本

神经网络可以看成一张有向图：

```text
x -> z_1 -> a_1 -> z_2 -> a_2 -> ... -> L
```

图中的每一条边，都会贡献一个局部导数。总梯度就是这些局部导数沿路径相乘，再把所有路径的贡献相加。

这也是为什么反向传播天然适合“从后往前”算：

- 前向传播时，先把每个中间值算出来
- 反向传播时，从损失开始，把梯度沿边一层层传回去

### 向量 / 矩阵版本

如果 `y = g(x)` 和 `z = f(y)` 都是向量值函数，那么链式法则写成雅可比矩阵相乘：

```text
J_{z,x} = J_{z,y} J_{y,x}
```

工程实现里我们通常不会显式构造巨大的雅可比矩阵，而是直接利用张量运算做向量-雅可比积（VJP）。这正是自动求导框架高效的关键。

## 一个两层 MLP 的完整手算例子

下面用一个非常小的网络，把前向与反向都完整走一遍。

### 网络定义

输入：

```text
x = [1, 2]^T
```

第一层：

```text
W_1 = [[0.1, -0.2],
       [0.4,  0.3]]
b_1 = [0.0, 0.1]^T
z_1 = W_1 x + b_1
a_1 = ReLU(z_1)
```

第二层：

```text
W_2 = [0.7, -1.2]
b_2 = 0.05
z_2 = W_2 a_1 + b_2
```

目标值设为 `y = 0.3`，损失函数使用平方误差：

```text
L = 1/2 (z_2 - y)^2
```

### 第一步：前向传播

先算第一层线性部分：

```text
z_1[1] = 0.1*1 + (-0.2)*2 + 0.0 = -0.3
z_1[2] = 0.4*1 + 0.3*2 + 0.1   = 1.1
```

所以：

```text
z_1 = [-0.3, 1.1]^T
a_1 = ReLU(z_1) = [0, 1.1]^T
```

继续算输出层：

```text
z_2 = 0.7*0 + (-1.2)*1.1 + 0.05 = -1.27
```

损失是：

```text
L = 1/2 (-1.27 - 0.3)^2
  = 1/2 * 1.57^2
  = 1.23245
```

### 第二步：从损失对输出求导

```text
∂L/∂z_2 = z_2 - y = -1.27 - 0.3 = -1.57
```

这是整次反向传播的起点。

### 第三步：回传到第二层参数

因为：

```text
z_2 = W_2 a_1 + b_2
```

所以：

```text
∂L/∂W_2 = ∂L/∂z_2 * ∂z_2/∂W_2 = (-1.57) * a_1
       = [0, -1.727]

∂L/∂b_2 = ∂L/∂z_2 = -1.57
```

同时回传到隐藏层激活：

```text
∂L/∂a_1 = ∂L/∂z_2 * W_2
        = (-1.57) * [0.7, -1.2]
        = [-1.099, 1.884]
```

### 第四步：穿过 ReLU

ReLU 的导数是：

```text
ReLU'(z) = 1(z > 0)
```

在这里：

```text
z_1 = [-0.3, 1.1]
ReLU'(z_1) = [0, 1]
```

因此：

```text
∂L/∂z_1 = ∂L/∂a_1 ⊙ ReLU'(z_1)
        = [-1.099, 1.884] ⊙ [0, 1]
        = [0, 1.884]
```

这里能看出一个非常重要的现象：第一个隐藏单元在前向时被 ReLU 截成了 0，所以它在这一步也收不到梯度。

### 第五步：回传到第一层参数

因为 `z_1 = W_1 x + b_1`，所以第二个隐藏单元对 `W_1` 的梯度等于：

```text
∂L/∂W_1[2,:] = ∂L/∂z_1[2] * x
             = 1.884 * [1, 2]
             = [1.884, 3.768]
```

第一个隐藏单元梯度为 0，因此整体梯度是：

```text
∂L/∂W_1 =
[[0,     0    ],
 [1.884, 3.768]]

∂L/∂b_1 = [0, 1.884]
```

如果继续对输入求导，也能得到：

```text
∂L/∂x = W_1^T (∂L/∂z_1)
      = [0.7536, 0.5652]^T
```

这就是一次完整的反向传播。

## 用自动求导验证手算结果

下面的 PyTorch 代码会得到与上面一致的梯度：

```python
import torch

x = torch.tensor([1.0, 2.0])
y = torch.tensor(0.3)

W1 = torch.tensor([[0.1, -0.2],
                   [0.4,  0.3]], requires_grad=True)
b1 = torch.tensor([0.0, 0.1], requires_grad=True)
W2 = torch.tensor([0.7, -1.2], requires_grad=True)
b2 = torch.tensor(0.05, requires_grad=True)

z1 = W1 @ x + b1
a1 = torch.relu(z1)
z2 = W2 @ a1 + b2
loss = 0.5 * (z2 - y) ** 2
loss.backward()

print("dW2 =", W2.grad)
print("db2 =", b2.grad)
print("dW1 =", W1.grad)
print("db1 =", b1.grad)
```

如果你输出这些梯度，会发现它们分别接近：

```text
dW2 = [0.0000, -1.7270]
db2 = -1.5700
dW1 = [[0.0000, 0.0000],
       [1.8840, 3.7680]]
db1 = [0.0000, 1.8840]
```

这说明自动求导框架做的事，并不是“发明了一种新的数学”，而是把链式法则在张量图上自动执行。

## 为什么深层网络容易出现梯度问题

链式法则虽然优雅，但也带来一个直接后果：很多层局部导数连乘后，数值可能非常极端。

### 梯度消失

如果每一层的局部导数都小于 1，例如都是 `0.1`，那么 10 层连乘就是：

```text
0.1^10 = 1e-10
```

这意味着越靠前的层几乎收不到有效更新。

### 梯度爆炸

反过来，如果每层局部导数都偏大，例如都是 `3`，那么 10 层连乘就是：

```text
3^10 = 59049
```

梯度会迅速失控。

这正是为什么深度学习实践里会同时重视：

- 激活函数选择
- 初始化策略
- 归一化方法
- 学习率与梯度裁剪

它们本质上都在处理“链式法则连乘之后数值是否稳定”。

## 常见误区

### 1. 反向传播不是“从输出往前硬套公式”

真正的核心是局部导数缓存与复用。每个节点只负责回答一个问题：**如果输出梯度传到我这里，我该怎样把它分给前面的输入和参数。**

### 2. 梯度大不等于参数一定该改很多

真正更新参数时还要考虑学习率、优化器状态、正则化等因素。梯度只是局部几何信息，不是最终步长本身。

### 3. 计算图里的每条边都对应一次链式法则

这不是比喻，而是精确描述。你在图里看到的依赖关系，就是导数传播的路径关系。

## 小结

导数回答“单个变量变一下会怎样”，梯度回答“多变量函数朝哪里增大最快”，链式法则回答“复杂依赖如何拆成局部影响的组合”。反向传播就是把这三件事统一起来：先前向得到所有中间值，再从损失开始逐层应用链式法则，把梯度传回每个参数。

真正理解这一点之后，很多训练现象都会突然变得清晰：为什么 ReLU 死亡会让某些神经元学不动，为什么初始化和归一化影响这么大，为什么深层网络容易梯度消失或爆炸。它们都不是零散技巧，而是同一个底层事实的不同表现：**深度学习训练，本质上是在高维计算图上做稳定而高效的求导。**
