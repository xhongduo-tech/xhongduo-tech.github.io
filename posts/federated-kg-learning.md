## 核心结论

知识图谱联邦学习，英文常写作 Federated Knowledge Graph Learning，指多个机构各自保留本地图谱，只交换经过保护的模型更新或结构特征，从而协同完成知识图谱补全、表示学习或下游推理。白话解释：大家不交原始数据，只交“训练后的变化量”或“模糊处理后的结构信息”。

它适合这样的场景：每个参与方都拥有私有三元组，例如银行的客户关系、医院的诊疗关系、企业的供应链关系，但这些数据因为隐私、合规或商业竞争不能集中汇总。此时，联邦学习提供“协同训练”的框架，知识图谱嵌入提供“把实体和关系压缩成向量”的方式，安全聚合与差分隐私则负责降低泄露风险。

传统知识图谱训练与联邦知识图谱训练的差异可以先看下表：

| 维度 | 传统 KG 训练 | 联邦 KG 训练 |
|---|---|---|
| 原始三元组 | 汇总到中心 | 不出本地 |
| 训练目标 | 单一全局模型 | 协同训练 + 本地可保留个性 |
| 隐私保护 | 弱，中心可见全部数据 | 强，中心只见受保护更新 |
| 跨机构利用信息 | 直接合并数据 | 通过参数/结构特征间接共享 |
| 是否要求统一语义空间 | 往往默认统一 | 可以弱对齐，甚至只交换邻居特征 |
| 风险点 | 数据集中泄露 | 参数反推、通信攻击、异构失真 |

结论可以压缩成一句话：知识图谱联邦学习不是把多个 KG 拼成一个大 KG，而是在不暴露原始三元组的前提下，让多个私有 KG 通过受保护的参数交换获得结构互补收益。

## 问题定义与边界

先把问题说清楚。知识图谱，简称 KG，本质是由三元组 $(h,r,t)$ 组成的图结构，其中 $h$ 是头实体，$r$ 是关系，$t$ 是尾实体。白话解释：它是在说“谁和谁通过什么关系连接”。

联邦学习的目标不是把所有参与方的数据搬到中心，而是让每个客户端在本地训练，再由服务器聚合各方更新。因此，知识图谱联邦学习的核心目标是：

1. 不共享原始三元组。
2. 尽量不暴露实体真实身份和关系细节。
3. 仍然让各方从其他图谱中获得补充信息。
4. 最终提升链接预测、实体补全、关系预测等任务效果。

它的边界也必须说清楚。不是所有信息都能共享，也不是只要“不发原始数据”就一定安全。参数、梯度、邻居表示都可能泄露结构线索。

下表给出一个实用边界：

| 信息类型 | 是否可共享 | 原因 |
|---|---|---|
| 原始三元组 $(h,r,t)$ | 绝对保密 | 直接暴露业务事实 |
| 实体真实 ID | 绝对保密 | 可直接关联用户、机构、账户 |
| 关系原始名称 | 尽量保密 | 会暴露业务语义 |
| 本地嵌入更新 | 可条件共享 | 需先加噪或掩码 |
| 邻居结构特征 | 可条件共享 | 比原始事实更抽象，但仍有泄露风险 |
| 聚合后的全局向量 | 通常可下发 | 已是多方混合结果 |
| 样本数量 $|D_c|$ | 可条件共享 | 用于加权，但可能泄露机构规模 |

玩具例子可以这样理解。两家银行都认识“客户 A”，但一家知道贷款行为，另一家知道跨境交易行为。它们都不能公开“客户 A 的具体交易记录”。联邦方式允许它们分别训练“客户 A 的向量表示”，只上传处理过的更新。服务器只看到模糊后的向量变化，看不到“某客户昨天向某账户转账 80 万”这样的具体事实。

真实工程里，边界通常更严格。比如金融 KYC，Know Your Customer，意思是“认识你的客户”的合规流程。多家机构都希望识别高风险账户，但不能共享明文客户图谱。这时常见做法不是共享实体本身，而是共享匿名后的邻居统计、嵌入更新，或者安全聚合后的结果。

因此，这个问题不是“如何共享数据”，而是“在不共享数据的条件下，最多能共享到什么程度还不破坏隐私”。

## 核心机制与推导

一个典型流程可以写成：

客户端本地训练 → 对更新加噪或加掩码 → 上传 → 服务器安全聚合 → 形成全局结果 → 下发给各客户端继续本地微调。

这里有三个核心机制。

第一，联邦嵌入学习。嵌入，embedding，指把离散实体或关系映射成连续向量。白话解释：把“客户”“账户”“交易”这些符号变成机器可计算的坐标点。客户端用本地三元组训练实体向量和关系向量，比如用 TransE、RotatE 或图神经网络版本的编码器。

第二，安全聚合。安全聚合，Secure Aggregation，简称 SecAgg，指服务器只能看到多个客户端更新的总和，看不到单个客户端的原始更新。白话解释：每个人把答案装进上锁盒子，服务器只能拿到“总分”，看不到谁答了什么。

第三，差分隐私。差分隐私，Differential Privacy，简称 DP，指在输出中加入受控噪声，使攻击者难以判断某条具体样本是否参与训练。白话解释：往结果里掺一点数学上可控的“模糊”，让别人看不清单个数据点。

设客户端 $c$ 的本地嵌入矩阵为 $E_c$，本地样本规模为 $|D_c|$，上传前加入噪声或掩码 $\xi_c$，则服务器侧可写成加权聚合形式：

$$
E_{\text{global}}=\frac{\sum_c |D_c|\cdot(E_c+\xi_c)}{\sum_c |D_c|}
$$

如果采用成对掩码或可恢复位移矩阵，那么服务器看到的是被保护的和，而不是每个 $E_c$。若各客户端噪声满足可抵消或在聚合后被合法恢复，则最终下发的是近似或准确恢复后的全局嵌入。

为什么要按 $|D_c|$ 加权？因为样本更多的客户端，其本地训练结果通常更稳定。如果直接平均，小机构的偶然波动会被放大。

看一个最小数值例子。两个银行客户端针对同一匿名实体训练出 2 维表示：

- 客户端 1：$(0.5,0.8)$
- 客户端 2：$(0.2,0.6)$

加入噪声后上传：

- 客户端 1 上传 $(0.6,0.7)$，即加了 $\xi_1=(0.1,-0.1)$
- 客户端 2 上传 $(0.15,0.65)$，即加了 $\xi_2=(-0.05,0.05)$

若两边样本权重相同，则聚合均值为：

$$
\frac{(0.6,0.7)+(0.15,0.65)}{2}=(0.375,0.675)
$$

再去除可恢复的总噪声均值 $\frac{\xi_1+\xi_2}{2}=(0.025,-0.025)$，得到：

$$
(0.375,0.675)-(0.025,-0.025)=(0.35,0.70)
$$

若采用不同样本权重，还会得到不同结果。研究中常见的是带权聚合，用户给出的示例 $(0.325,0.575)$ 本质上就是“先加噪，再聚合，再恢复”的一个特定权重版本。

但知识图谱联邦学习难点不只是“平均向量”。更难的是语义异构。语义异构，semantic heterogeneity，指不同机构对同类对象使用不同关系集合、不同子图结构、甚至不同建模方式。白话解释：大家都在描述世界，但用的语言和重点不一样。

比如银行 A 图谱中有“贷款逾期”“共同借款人”，银行 B 图谱中有“跨境收款”“可疑设备登录”。如果强行把实体嵌入拉到一个全局统一空间，可能把本地有价值的语义冲淡。因此，较新的方法会交换邻居特征而不是只交换同一套嵌入。邻居特征，neighbor features，指一个实体周围连接的关系和相邻实体经过编码后的表示。白话解释：不直接说“我知道谁”，而是说“这个点周围大致长什么样”。

从流程上可以画成：

客户端本地 KG 编码  
→ 生成实体/邻居表示  
→ 加噪或加掩码  
→ 上传服务器  
→ 服务器安全求和或融合结构信息  
→ 形成全局或个性化邻居表示  
→ 回传客户端继续训练

真实工程例子是多金融机构联合反欺诈。A 银行知道账户与设备、转账链路，B 银行知道账户与商户、信用事件，C 机构知道账户与司法公开记录。它们都不能开放原始边，但可以共享保护后的邻居结构表示。这样，本地模型在训练“某账户是否高风险”时，能够间接借到其他机构的结构信息，而不直接看到对方图谱。

## 代码实现

下面给一个可运行的 Python 玩具实现，只模拟“本地训练后上传受保护更新，服务器加权聚合并回传”的主干逻辑。它不是生产级 SecAgg，只是帮助理解通信与聚合接口。

```python
from typing import List, Tuple

Vector = Tuple[float, float]

def add(v1: Vector, v2: Vector) -> Vector:
    return (v1[0] + v2[0], v1[1] + v2[1])

def sub(v1: Vector, v2: Vector) -> Vector:
    return (v1[0] - v2[0], v1[1] - v2[1])

def scale(v: Vector, s: float) -> Vector:
    return (v[0] * s, v[1] * s)

def aggregate(client_payloads: List[dict]) -> Vector:
    total_weight = sum(p["weight"] for p in client_payloads)
    weighted_sum = (0.0, 0.0)
    noise_sum = (0.0, 0.0)

    for p in client_payloads:
        weighted_sum = add(weighted_sum, scale(p["masked_embedding"], p["weight"]))
        noise_sum = add(noise_sum, scale(p["noise"], p["weight"]))

    masked_global = scale(weighted_sum, 1.0 / total_weight)
    avg_noise = scale(noise_sum, 1.0 / total_weight)
    recovered_global = sub(masked_global, avg_noise)
    return recovered_global

client_1 = {
    "weight": 3,
    "local_embedding": (0.5, 0.8),
    "noise": (0.1, -0.1),
}
client_1["masked_embedding"] = add(client_1["local_embedding"], client_1["noise"])

client_2 = {
    "weight": 1,
    "local_embedding": (0.2, 0.6),
    "noise": (-0.05, 0.05),
}
client_2["masked_embedding"] = add(client_2["local_embedding"], client_2["noise"])

global_embedding = aggregate([client_1, client_2])

assert round(global_embedding[0], 4) == 0.425
assert round(global_embedding[1], 4) == 0.75

print("Recovered global embedding:", global_embedding)
```

这段代码里有几个关键点。

`local_embedding` 表示本地训练后的实体表示。真实系统里，它通常来自 KG embedding 模型或 GNN 编码器。  
`noise` 表示客户端添加的噪声或可恢复掩码。真实系统里，它可能是差分隐私噪声，也可能是安全聚合协议中的随机掩码。  
`masked_embedding` 是实际上传内容。  
`aggregate` 模拟服务器只拿到受保护后的向量，再按样本数加权求平均，并减去可合法恢复的平均噪声。

如果把逻辑写成更接近工程的伪代码，大致如下：

```python
for round_id in range(num_rounds):
    for client in clients:
        local_model = train_local_kg(client.local_graph, global_hint)
        update = extract_embedding_update(local_model)

```

真实实现通常还会多出这些模块：

| 模块 | 作用 |
|---|---|
| KG 数据读取 | 把三元组转换成采样器和训练 batch |
| 负采样 | 为链接预测构造负例 |
| 邻居采样 | 提取局部结构特征 |
| 本地嵌入训练器 | 训练实体和关系向量 |
| DP 机制 | 裁剪梯度、加入高斯噪声 |
| SecAgg 协议 | 确保服务器只见聚合和 |
| 通信层 | 管理轮次、超时、掉线重试 |
| 个性化模块 | 把全局结果转成本地可用提示 |

新手最容易忽略的一点是：服务器下发的不一定是“完整全局模型”。在异构 KG 中，下发“个性化邻居向量”或“辅助表示”通常比强推一套统一参数更稳，因为每个客户端的关系集合并不相同。

## 工程权衡与常见坑

知识图谱联邦学习难做，不是因为“联邦”三个字，而是因为它同时背了图结构、隐私保护、异构建模、分布式通信四个问题。

下面是常见坑与对应策略：

| 问题 | 具体表现 | 常见规避策略 |
|---|---|---|
| 语义异构 | 不同机构关系集合不同，直接平均嵌入失真 | 用对比学习约束，或只交换邻居结构特征 |
| 参数泄露 | 服务器或攻击者从更新反推三元组 | 客户端先裁剪并加噪，再做安全聚合 |
| 通信开销大 | 每轮上传高维嵌入，轮次多时延迟高 | 压缩更新、低秩表示、位移矩阵 |
| 噪声伤精度 | DP 噪声过大导致链接预测下降 | 调整隐私预算，做分层保护 |
| 非独立同分布 | 各客户端数据分布差异大 | 做个性化联邦，而非只追求统一全局模型 |
| 掉线与迟到 | 部分机构不能稳定参与每轮 | 支持部分参与和异步聚合 |

先说语义异构。很多新手一开始会以为“联邦平均”是默认答案，但在 KG 上经常不是。因为图谱的关系类型和局部拓扑差异很大，直接把实体向量平均，可能把“贷款风险”与“设备异常”混成一个模糊方向。引入对比目标的做法，本质是在限制不同客户端更新的偏移，让共享部分更稳定，而把差异部分保留在本地。

再说通信开销。KG 嵌入矩阵往往很大，实体数动辄百万级。如果每轮上传完整向量，成本会迅速失控。ALPPA 这一类方法强调“准确无损聚合”与“隐私保护”兼顾，重点就在于通过压缩或位移矩阵减少暴露和带宽占用。白话说，不是把整面墙搬过去，而是只传墙上需要对齐的那几块砖。

还有一个容易被低估的风险是参数逆推。很多人认为“我没发原始三元组，所以安全”，这并不成立。如果某个关系只在一类实体上出现，而模型更新在这个方向上有明显模式，攻击者仍可能从参数变化猜到局部结构。因此，安全做法通常是：

1. 本地先做裁剪，限制单次更新幅度。
2. 再加噪，弱化单样本影响。
3. 再用 SecAgg，防止服务器看到单客户端更新。

真实工程例子可以看金融反欺诈。不同机构拥有不同类型的关系边：设备共用、IP 共用、地址共用、交易闭环、司法关联。如果直接统一成同一个全局实体空间，某些边的业务语义会被压平。更现实的方案是本地维护自己熟悉的语义体系，只交换“这个账户周边结构是否呈现高风险模式”的抽象特征。

## 替代方案与适用边界

知识图谱联邦学习不是唯一选项，它只是“隐私要求高且多方协同”时的一种方案。选型时至少要和三类替代方案比较。

| 方案 | 隐私强度 | 语义差异容忍度 | 通信预算 | 适用边界 |
|---|---|---|---|---|
| 集中式 KG 补全 | 低 | 中 | 低 | 能合法共享三元组，追求最高训练效率 |
| 全局对齐后联合训练 | 中 | 低 | 中 | 多方 schema 接近，重复实体较多 |
| 联邦嵌入学习 | 高 | 中 | 高 | 不能共享原始图谱，但愿意交换受保护更新 |
| GFedKG 式结构特征交换 | 高 | 高 | 中到高 | 语义异构强，不适合强制对齐统一空间 |

集中式方案最简单。只要允许汇总三元组，直接训练一个统一 KG 模型通常效果最好、实现成本最低。但它不满足强隐私场景。

全局对齐方案适合跨机构语义差异较小的情况。比如几个使用近似行业标准 schema 的企业知识库，可以先做实体对齐和关系映射，再训练统一嵌入空间。问题在于，一旦 schema 差异大、实体隐私强、业务语义不一致，这种方法会迅速变脆弱。

GFedKG 这一类方法更适合高异构场景。它不强求“你我必须把同一个实体嵌到同一个坐标点”，而是更关注邻居结构的互补信息。对白话读者来说，可以把它理解为：不交换“我知道谁”，而交换“我观察到什么形状的局部关系结构”。

因此，适用边界可以概括为：

- 如果允许集中存储原始三元组，优先集中式。
- 如果多方 schema 高度一致且对齐成本低，可以考虑全局对齐联合训练。
- 如果隐私要求高且仍要共享协同收益，联邦知识图谱学习是更合理选择。
- 如果异构性强到统一嵌入空间不稳定，优先考虑交换邻居结构特征而不是直接平均嵌入。
