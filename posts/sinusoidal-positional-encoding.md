## 核心结论

位置编码的作用，是把“第几个 token”这个离散位置，变成和词向量同维度的连续向量，再加到词向量上，让 Transformer 知道顺序。这里的“连续向量”指一组实数，不是只会取编号 0、1、2 的离散标签。

Transformer 的自注意力本身是排列等变的。所谓“排列等变”，白话讲就是：如果你把输入 token 的顺序整体打乱，而模型又没有收到任何位置信息，那么它只能看到“有哪些词”，很难知道“谁在前谁在后”。因此，位置信息不是装饰，而是自注意力可用的前提条件之一。

原始 Transformer 采用固定的正弦位置编码：

$$
PE(pos, 2i)=\sin\left(\frac{pos}{10000^{2i/d_{model}}}\right)
$$

$$
PE(pos, 2i+1)=\cos\left(\frac{pos}{10000^{2i/d_{model}}}\right)
$$

这个构造有两个核心性质：

| 性质 | 含义 | 工程价值 |
|---|---|---|
| 不同维度有不同频率 | 每对维度使用不同快慢的波 | 既能区分近邻，也能表达长距离位置 |
| 相对位移可线性表示 | $PE(pos+k)$ 可由 $PE(pos)$ 经过频率相关的旋转得到 | 注意力更容易学习“向前看 3 个 token”这类关系 |

可以把它看成一个简单流程：

`位置编号 pos -> 多个不同周期的正弦/余弦波 -> 拼成 d 维向量 -> 与词嵌入相加`

玩具例子：“我 爱 你”三个 token，位置分别是 0、1、2。若 `d_model=4`，它们的位置向量可以写成：

- `pos=0`: `[sin(0), cos(0), sin(0), cos(0)] = [0,1,0,1]`
- `pos=1`: `[sin(1), cos(1), sin(0.01), cos(0.01)]`
- `pos=2`: `[sin(2), cos(2), sin(0.02), cos(0.02)]`

然后分别加到“我”“爱”“你”的词向量上。这样模型即使没有 RNN 那样逐步读入的结构，也能区分谁在前、谁在后。

## 问题定义与边界

问题定义很直接：自注意力只靠 token 内容做两两相关性计算，默认不知道顺序，因此不能单独承担“语序理解”。

例如“猫 吃 鱼”和“鱼 吃 猫”，如果没有位置编码，模型看到的词集合相同，都是 `{猫, 吃, 鱼}`。这不意味着输出一定完全一样，但从结构上说，模型缺少稳定区分主语和宾语位置的机制。位置编码的任务，就是在不改写自注意力主结构的前提下，把顺序信息注入输入表示。

可以把边界说清楚：

| 情况 | 模型能否知道顺序 | 原因 |
|---|---|---|
| 只有词嵌入，无位置编码 | 不能稳定知道 | 输入只包含“词是什么”，不包含“词在第几位” |
| 词嵌入 + 位置编码 | 可以 | 每个位置都带有不同的位置向量 |
| 只给绝对位置，不建模相对位置 | 部分可以 | 能知道编号，但不一定最容易表达“相差几步” |
| 加入相对位置机制 | 更适合距离建模 | 直接把“相隔多远”送入注意力 |

这里的边界很重要：位置编码解决的是“顺序信息缺失”，不是“所有结构信息都自动学会”。句法、长程依赖、篇章结构，还需要注意力层和训练数据共同学习。

另一个边界是，正弦位置编码是一种“绝对位置编码”。所谓“绝对位置”，白话讲就是每个 token 直接拿到自己的编号表示，例如第 17 位就是第 17 位，而不是只知道自己和别的 token 相差多少。

## 核心机制与推导

原始方案的关键，不是“用了三角函数”这么简单，而是“每一对维度对应一个频率”。

设模型维度为 $d$，第 $i$ 对维度的角频率定义为：

$$
\omega_i = \frac{1}{10000^{2i/d}}
$$

于是：

$$
PE(pos, 2i)=\sin(\omega_i pos), \quad PE(pos, 2i+1)=\cos(\omega_i pos)
$$

这里“频率”白话讲就是波动快慢。频率高，位置稍微变一点，数值就变化得快；频率低，变化更平缓。

当 $d_{model}=4$ 时，有两对频率：

- 第 0,1 维：$\omega_0 = 1$
- 第 2,3 维：$\omega_1 = 1/100$

这意味着前两维变化很快，后两维变化很慢。快速变化的维度更容易区分相邻位置，慢速变化的维度更容易保持长距离位置趋势。

看一个小计算表：

| `pos` | `i` | 角度 $\omega_i pos$ | `sin` | `cos` |
|---|---:|---:|---:|---:|
| 1 | 0 | 1 | 0.8415 | 0.5403 |
| 1 | 1 | 0.01 | 0.0100 | 0.99995 |
| 2 | 0 | 2 | 0.9093 | -0.4161 |
| 2 | 1 | 0.02 | 0.0200 | 0.99980 |

这个表能直观看出：同样从 `pos=1` 到 `pos=2`，第 0 对维度变化剧烈，第 1 对维度几乎不变。多尺度频率叠加后，模型同时拥有“短距离分辨率”和“长距离坐标感”。

更关键的是相对位移性质。对任意偏移 $k$，有：

$$
\sin(\omega_i(pos+k))=\sin(\omega_i pos)\cos(\omega_i k)+\cos(\omega_i pos)\sin(\omega_i k)
$$

$$
\cos(\omega_i(pos+k))=\cos(\omega_i pos)\cos(\omega_i k)-\sin(\omega_i pos)\sin(\omega_i k)
$$

把第 $i$ 对维度写成向量：

$$
v_i(pos)=
\begin{bmatrix}
\sin(\omega_i pos) \\
\cos(\omega_i pos)
\end{bmatrix}
$$

则有：

$$
v_i(pos+k)=
\begin{bmatrix}
\cos(\omega_i k) & \sin(\omega_i k) \\
-\sin(\omega_i k) & \cos(\omega_i k)
\end{bmatrix}
v_i(pos)
$$

这就是一个旋转矩阵。这里“旋转矩阵”白话讲就是二维平面里把向量按某个角度转动的线性变换。

因此，固定偏移 $k$ 对应一个固定线性变换。模型如果想表达“当前位置后面第 3 个词”，不需要重新记住每个绝对位置，只要学会若干偏移模式即可。这正是它对注意力有利的地方。

用复数形式会更紧凑。把一对维度写成：

$$
z_i(pos)=\cos(\omega_i pos)+j\sin(\omega_i pos)=e^{j\omega_i pos}
$$

那么：

$$
z_i(pos+k)=e^{j\omega_i(pos+k)}=e^{j\omega_i k} z_i(pos)
$$

也就是说，相对位移就是乘上一个相位因子。这里“相位”白话讲就是波在周期上的当前位置。

真实工程例子：机器翻译里，英语句子常出现“not ... but ...”这类远距离依赖。高频维度帮助模型分辨局部词序，低频维度帮助模型保留全句中的大尺度位置趋势，两者叠加后，注意力更容易在长句中稳定对齐。

## 代码实现

正弦位置编码通常预先生成一个形状为 `[max_len, d_model]` 的矩阵，再与词嵌入直接相加。这里 `[max_len, d_model]` 的意思是：每一行对应一个位置，每一列对应一个嵌入维度。

下面给出一个可运行的 NumPy 实现：

```python
import numpy as np

def sinusoidal_positional_encoding(max_len: int, d_model: int) -> np.ndarray:
    assert max_len > 0
    assert d_model > 0
    assert d_model % 2 == 0, "为了成对放置 sin/cos，d_model 应为偶数"

    pe = np.zeros((max_len, d_model), dtype=np.float64)
    pos = np.arange(max_len, dtype=np.float64).reshape(-1, 1)          # [max_len, 1]
    i = np.arange(0, d_model, 2, dtype=np.float64)                     # [d_model/2]
    div_term = np.exp(-(np.log(10000.0) * i / d_model))                # 1 / 10000^(2i/d_model)

    angles = pos * div_term                                            # [max_len, d_model/2]
    pe[:, 0::2] = np.sin(angles)                                       # 偶数维
    pe[:, 1::2] = np.cos(angles)                                       # 奇数维

    return pe

pe = sinusoidal_positional_encoding(max_len=8, d_model=4)

assert pe.shape == (8, 4)
assert np.allclose(pe[0], np.array([0.0, 1.0, 0.0, 1.0]))
assert abs(pe[1, 0] - np.sin(1.0)) < 1e-9
assert abs(pe[1, 2] - np.sin(0.01)) < 1e-9

print(pe[:3])
```

实现要点只有三个：

| 变量 | 形状 | 含义 |
|---|---|---|
| `pos` | `[max_len, 1]` | 所有位置编号 |
| `i` | `[d_model/2]` | 每对维度的下标 |
| `div_term` | `[d_model/2]` | 每对维度对应的频率缩放 |

向量化的好处是避免双重循环。对 `max_len=512, d_model=768`，这种写法已经足够快，也容易和深度学习框架对齐。

玩具例子：假设词嵌入如下：

- “我”词向量：`[0.2, 0.1, -0.3, 0.4]`
- “爱”词向量：`[0.5, -0.2, 0.1, 0.0]`
- “你”词向量：`[-0.1, 0.3, 0.2, 0.6]`

位置编码加上去后，三个 token 的最终输入向量都发生不同偏移。这样即使“我”和“你”词向量交换，模型接收到的总表示也不同。

真实工程例子：在实现一个最小 Transformer 文本分类器时，常见做法是：
1. 先把 token id 查表得到词嵌入 `[batch, seq_len, d_model]`
2. 再切出前 `seq_len` 行位置编码 `[seq_len, d_model]`
3. 广播相加得到输入表示
4. 送入多头注意力层

如果不加这一步，分类器在涉及顺序的任务上通常会明显退化，例如情感反转句“不是好，而是差”。

## 工程权衡与常见坑

固定正弦编码和可学习位置嵌入最常见的差异，在“泛化长度”和“可适应性”之间。

| 方案 | 泛化到更长序列 | 参数量 | 适应训练数据 | 实现复杂度 |
|---|---|---:|---|---|
| 固定正弦/余弦 | 强 | 0 | 一般 | 低 |
| 可学习位置嵌入 | 弱，受表长限制 | `max_len * d_model` | 强 | 低 |
| 相对位置/偏置 | 通常较强 | 视实现而定 | 强 | 中到高 |

固定编码的优点是没有可训练参数，训练时没见过的更长序列，理论上也能继续生成位置向量。这里“泛化”白话讲就是模型在新长度上还能工作，而不是只能处理训练时出现过的长度。

可学习位置嵌入则更像一张查表。例如 BERT 常见设置是 `512 x 768` 的位置参数矩阵。它的优势是灵活，模型可以直接调整每个位置的表示；问题是如果训练主要集中在较短序列，例如大量样本都在 128 以内，那么更高位置上的参数学得不充分，推理拉到 512 时容易变成噪声区。

真实工程例子：很多检索增强或文档分类任务，会把 BERT 类模型从 128 token 扩到 512 token。此时常见问题不是“程序跑不动”，而是后 384 个位置表示质量不足，模型虽然能前向计算，但效果不稳定。工程上通常有三种补救：

| 方法 | 做法 | 风险 |
|---|---|---|
| 重新预训练更长长度 | 直接在长序列继续训练 | 成本高 |
| 位置表插值/扩展 | 把已有位置参数映射到更长表 | 近似误差 |
| 换用相对位置方案 | 不再强依赖绝对查表 | 改模型成本高 |

另一个高频坑是奇偶维度顺序不能乱。正弦方案依赖 `(sin, cos)` 成对出现；如果你在实现里把维度洗牌，或者导出权重后错误重排，旋转结构就被破坏，`PE(pos+k)` 不再能通过同一频率下的固定线性变换得到。

常见错误还包括：

- `d_model` 写成奇数，导致最后一个维度没有配对。
- 把 `2i/d_model` 误写成 `i/d_model`，频率分布会错。
- 将位置编码乘法注入而不是加法注入，除非你明确知道这样做的意义。
- 训练时位置从 1 开始、推理时从 0 开始，造成整体相位偏移。
- 缓存自回归解码时，没有给新增 token 分配正确位置，导致续写质量下降。

## 替代方案与适用边界

正弦位置编码不是唯一方案。它适合“通用、稳定、实现简单”的场景，但不是所有任务的最优解。

常见替代方案如下：

| 方案 | 优点 | 局限 | 典型场景 |
|---|---|---|---|
| 固定正弦编码 | 无参数、可外推、实现简单 | 适应性有限 | 通用 Transformer、教学实现 |
| 可学习绝对位置嵌入 | 容易训练、表达灵活 | 长度外推差 | BERT 类编码器 |
| 相对位置编码 | 更直接建模距离 | 实现更复杂 | 长文、结构依赖强任务 |
| 相对位置偏置（如 T5） | 直接作用于注意力分数 | 偏置设计需调参 | 机器翻译、文本生成 |
| 旋转位置编码（RoPE） | 天然编码相对位移 | 仍有长上下文细节问题 | 现代大语言模型 |

这里“偏置”白话讲就是在注意力分数上额外加一个值，影响“该看谁更多”。T5 的相对位置偏置就是把 token 间距离映射成一个可学习分桶，再把对应偏置加到注意力 logits 上。这样模型在算注意力权重时，不仅看内容相似度，还直接知道“这两个 token 相距多远”。

新手可以这样理解 T5 的例子：如果两个 token 距离是 1，模型可能学到一个较大的正偏置，表示“邻近词通常更相关”；如果距离是 100，偏置可能更小甚至为负，表示“远距离默认不那么相关，除非内容特别匹配”。

适用边界可以粗分为：

- 如果你要实现教材级 Transformer、序列长度不极端、希望结构简单，固定正弦编码足够。
- 如果你在做 BERT 风格预训练，且最大长度固定明确，可学习绝对位置嵌入很常见。
- 如果你要处理长文、代码、文档问答，单纯绝对查表通常不够，相对位置或旋转类方案更常见。
- 如果训练长度和推理长度差距很大，只靠可学习绝对位置往往风险较高。

因此，正弦位置编码的边界不是“过时”，而是“适合做基础而稳定的顺序注入”。它仍然是理解后续各种位置机制的起点。
