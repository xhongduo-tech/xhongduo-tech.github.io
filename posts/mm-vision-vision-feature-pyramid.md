## 核心结论

特征金字塔是视觉编码器里一套“分层看图”的结构。白话说，它不是只看一张固定尺寸的特征图，而是同时保留高分辨率层的边缘、纹理、小目标线索，以及低分辨率层的类别、上下文、全局语义。这样做的原因很直接：现实图像里的目标尺度差异很大，小目标依赖细节，大目标和场景理解依赖语义，只保留其中一类信息通常都不够。

在多模态视觉编码里，这个问题更明显。多模态指同时处理 RGB、红外、深度等不同来源的视觉信号。它们互补，但也会互相污染，例如夜间 RGB 噪声大、红外背景伪影强，简单相加或拼接经常让某一模态“压住”另一模态。PFMG 这类层级门控方法成立的关键，是把“跨层细节引导”和“当前层模态选择”拆成两个机制同时做：上一层提供空间先验，告诉当前层哪里值得保细节；当前层再按像素计算 softmax 权重，决定这个位置更该信任 RGB 还是 IR。

如果写成公式，核心就是先形成加权融合基底，再用空间门增强局部细节：

$$
F^l_{\mathrm{base}} = \alpha^l_{\mathrm{rgb}} \odot G^l_{\mathrm{rgb}} + \alpha^l_{\mathrm{ir}} \odot G^l_{\mathrm{ir}}
$$

$$
\hat F^l(x,y) = [1 + M^l(x,y)] \odot F^l_{\mathrm{base}}(x,y)
$$

其中 $M^l$ 是空间门，白话说就是“哪里该放大细节”的地图；$\alpha^l_{\mathrm{rgb}}, \alpha^l_{\mathrm{ir}}$ 是像素级权重，白话说就是“这个位置更该听谁”的分配结果。

## 问题定义与边界

问题不是“怎么把两路特征合在一起”，而是“怎么在多尺度、多模态条件下，保住有用细节并抑制错误信息”。这两个目标必须同时满足。

先看边界。视觉编码器里的特征金字塔只解决“表征组织与融合”问题，不直接替代检测头、分割头或训练策略。它的职责是把不同层级的特征整理成可被下游模块消费的表示。如果下游监督很弱，或者数据对齐很差，金字塔本身也救不了结果。

再看为什么不能简单处理。`concat` 是拼接，白话说就是把两路特征堆在通道维度上交给后续卷积自己学；`add` 是逐元素相加，白话说就是默认两路信息同等可信。它们都太粗糙。因为多模态噪声往往不是全局统一出现，而是局部出现：有些像素 RGB 更可靠，有些像素 IR 更可靠。如果只做整层平均，等于强行假设整张图的模态质量一致，这在真实场景里通常不成立。

玩具例子可以把问题看得更清楚。假设一个 $4 \times 4$ 特征图里，左上角有一个很小的热目标。RGB 里它被阴影遮挡，只剩弱纹理；IR 里它轮廓清晰，但右下角存在大片热噪声。如果直接把 RGB 和 IR 相加，左上角也许能增强，但右下角噪声也会一起被抬高。如果用层级门控，上一层先告诉当前层“左上角边界值得保”，当前层再让左上角更多依赖 IR、右下角减少 IR 权重，结果会稳定得多。

下表可以概括“无门控”和“层级门控”的典型差异：

| 方案 | 融合方式 | 小目标细节 | 噪声响应 | 典型结果 |
|---|---|---|---|---|
| 无门控 `concat/add` | 全局统一处理 | 容易在下采样后丢失 | 容易被单一模态主导 | mAP 提升有限，波动较大 |
| PFMG 层级门控 | 上层细节引导 + 当前层像素级权重 | 更容易保住边界和纹理 | 能局部抑制坏模态 | 对小目标更稳定，噪声鲁棒性更强 |

真实工程例子是 RGB+IR 双流检测。以 PACGNet 这类结构为代表，如果当前层直接把两路特征相加，白天强反光会让 RGB 过曝区域淹没有效线索，夜间红外伪影又可能放大背景热点。PFMG 的边界价值在于：它不假设某个模态一直更好，而是把决策细化到每一层、每个像素。

## 核心机制与推导

PFMG 可以拆成四步理解。

第一步，利用上一层构造空间先验。空间先验的意思是“从更细层的结构里提取一个位置提示图”。上一层分辨率更高，通常包含更清晰的边缘和局部轮廓。把上一层的 RGB 和 IR 特征拼接后，通过卷积和下采样生成当前层的空间门：

$$
M^l = \sigma (\mathrm{Conv}_{3\times3,s=2}(S^l))
$$

这里 $\sigma$ 是 sigmoid，白话说就是把值压到 $0$ 到 $1$ 之间，变成一张强弱图。

第二步，在当前层做模态交互。交互的意思是先让 RGB 和 IR 在通道维度上发生局部信息交换，而不是先独立计算后再生硬合并。可以写成：

$$
[\alpha^l_{\mathrm{rgb}}, \alpha^l_{\mathrm{ir}}] = \mathrm{Softmax}(\mathrm{Conv}_{1\times1}(U^l))
$$

softmax 的白话解释是“把两个分数变成和为 1 的权重”，因此每个像素最终会得到一对互补权重。

第三步，形成基础融合特征：

$$
F^l_{\mathrm{base}} = \alpha^l_{\mathrm{rgb}} \odot G^l_{\mathrm{rgb}} + \alpha^l_{\mathrm{ir}} \odot G^l_{\mathrm{ir}}
$$

这里 $\odot$ 表示逐元素乘法。含义很明确：当前层不是简单相加，而是先判断“谁更可信”，再加权。

第四步，用空间门对结果做细节增强：

$$
\hat F^l = [1 + M^l] \odot F^l_{\mathrm{base}}
$$

这个式子重要的地方在于 $1 + M^l$。如果只有 $M^l$，门值接近 0 的区域会把特征压没；加上 1 以后，默认是“保留原始融合结果”，在此基础上只对高置信区域做增强。这比纯乘法门控更稳。

可以用一个更直观的数值玩具例子说明。假设某个像素位置上，RGB 响应是 $0.2$，IR 响应是 $0.8$。网络算出的 softmax 权重是 $\alpha_{\mathrm{rgb}}=0.3,\alpha_{\mathrm{ir}}=0.7$，则：

$$
F_{\mathrm{base}} = 0.3 \times 0.2 + 0.7 \times 0.8 = 0.62
$$

如果上一层在这个位置给出的空间门是 $M=0.5$，则最终输出：

$$
\hat F = (1+0.5)\times 0.62 = 0.93
$$

这说明两个动作分别完成了不同职责：softmax 解决“信谁”，空间门解决“该不该强调这里”。

从推导角度看，这种结构之所以成立，是因为它把多尺度和多模态两个维度解耦了。跨层门控只负责传递结构先验，不直接决定模态比例；模态 softmax 只负责当前层局部选择，不承担细节恢复任务。职责清晰，训练也更稳定。

## 代码实现

下面给一个可运行的最小 Python 示例。它不是完整的卷积网络，而是把 PFMG 的关键计算流程压缩成数组运算，便于先理解形状和数值关系。

```python
import math

def sigmoid(x):
    return 1.0 / (1.0 + math.exp(-x))

def softmax2(a, b):
    ea = math.exp(a)
    eb = math.exp(b)
    s = ea + eb
    return ea / s, eb / s

def pfmg_pixel(prev_rgb, prev_ir, cur_rgb, cur_ir):
    # 用上一层信息构造空间门，模拟 3x3 + stride=2 后的门值
    spatial_score = 0.6 * prev_rgb + 0.4 * prev_ir
    M = sigmoid(spatial_score)

    # 当前层交互后的两个模态分数，模拟 1x1 conv
    score_rgb = 1.2 * cur_rgb - 0.3 * cur_ir
    score_ir = 1.1 * cur_ir - 0.2 * cur_rgb
    w_rgb, w_ir = softmax2(score_rgb, score_ir)

    F_base = w_rgb * cur_rgb + w_ir * cur_ir
    out = (1.0 + M) * F_base
    return {
        "M": M,
        "w_rgb": w_rgb,
        "w_ir": w_ir,
        "F_base": F_base,
        "out": out,
    }

# 玩具输入：上一层更能看清细节，当前层 IR 更可靠
res = pfmg_pixel(prev_rgb=0.9, prev_ir=0.8, cur_rgb=0.2, cur_ir=0.8)

assert 0.0 < res["M"] < 1.0
assert abs(res["w_rgb"] + res["w_ir"] - 1.0) < 1e-9
assert res["w_ir"] > res["w_rgb"]   # 当前层 IR 更强，权重更高
assert res["out"] > res["F_base"]   # 空间门对结果有增强作用
print(res)
```

如果映射到 PyTorch 或 YOLOv8 插件，流程一般是：

```python
# 1. 上一层生成空间门
S = concat(prev_rgb, prev_ir)
M = sigmoid(conv3x3_stride2(BN(ReLU(conv3x3(S)))))

# 2. 当前层做模态交互
U = conv1x1(ReLU(conv1x1(concat(cur_rgb, cur_ir))))
G_rgb, G_ir = split(U)

# 3. 生成像素级模态权重
weights = softmax(conv1x1(concat(G_rgb, G_ir)), dim=modality)
w_rgb, w_ir = split(weights)

# 4. 基础融合
F_base = w_rgb * G_rgb + w_ir * G_ir

# 5. 细节增强
output = BN(F_base) * (1 + M)
```

真实工程里，PFMG 往往插在 P3 到 P5 这样的金字塔层级。原因是这些层既覆盖中小目标，也不会像最浅层那样成本过高。以双流 YOLOv8 为例，常见做法是先让两路 backbone 各自提特征，再在 neck 或跨层连接处插入 PFMG。这样做的收益不是“全层都更强”，而是把计算集中到最影响检测质量的层级。

## 工程权衡与常见坑

第一类权衡是算力。高分辨率分支最贵，因为空间尺寸大，哪怕通道不多，卷积和门控也会迅速放大延迟。所以很多金字塔设计强调参数倒置，也就是把更小的模型或更少的通道留给高分辨率分支，把大部分参数放在低分辨率高语义分支。这不是技巧，而是成本约束下的必要设计。

第二类权衡是门控稳定性。softmax 权重如果过早饱和，模型会学成“长期偏爱某个模态”。例如数据集夜景样本偏多，网络可能几乎一直给 IR 更高权重，最后白天场景反而退化。处理方法通常包括更谨慎的初始化、归一化、以及在不同天气和光照条件下做均衡训练。

第三类常见坑是对齐误差。RGB 和 IR 在真实相机系统里往往存在轻微视差、时间偏差、畸变差异。PFMG 是像素级加权，因此默认两路特征大体对齐。如果对齐不好，权重学到的就可能是“错误位置上的可靠性”。这类问题不能靠门控本身修复，前处理标定或可学习对齐模块往往是前提。

第四类坑是把空间门理解成注意力万能药。空间门只是一张位置增强图，不会凭空创造语义。如果上一层本来就没保住目标边界，当前层也拿不到可靠先验。换句话说，特征金字塔的前提是 backbone 至少提供了可用的多层表示。

下面这个表格更适合工程调优时使用：

| 设计选择 | 推理延迟 | 小目标精度 | 噪声容忍度 | 常见问题 |
|---|---|---|---|---|
| 高分辨率分支给足参数 | 高 | 可能更高 | 中 | 端侧部署困难 |
| 参数倒置分配 | 低到中 | 通常更稳 | 中 | 需要仔细调通道数 |
| 只做空间门，不做自适应 softmax | 中 | 一般 | 低 | 容易被坏模态拖累 |
| 空间门 + 自适应 softmax | 中 | 高 | 高 | 训练更复杂，对对齐更敏感 |

真实工程例子里，如果在 640×640 输入上同时对多个高分辨率层做 SCG/PFMG，一旦通道数没有控制住，延迟会明显上涨。PIIP 一类思路之所以重要，就在于它说明“保细节”不等于“高分辨率层堆更多参数”，而是要让跨分支交互更高效。

## 替代方案与适用边界

PFMG 不是唯一解，它更适合“要保局部边界，还要做模态选择”的任务，典型是多模态小目标检测、复杂天气感知、夜间交通感知。

PIIP 的核心方向不同。它关注的是参数预算下的图像金字塔组织，重点是让高分辨率分支更轻，同时通过跨分支交互保住细节。白话说，它更像“怎么省着花算力还保留细节”。如果部署环境受限，例如车端、无人机、边缘设备，PIIP 这类思路通常比重门控结构更实用。

PyramidCLIP 代表的则是语义金字塔路线。它不强调像素级模态抉择，而强调多层语义对齐，特别适合视觉-语言预训练、检索、开放词表识别。白话说，它更关心“每层特征能否和语言概念对上”，而不是“小目标边缘是否被保住”。

可以用一个面向新手的类比帮助区分，但不替代定义。PFMG 像在每一层同时做“局部聚焦 + 模态择优”；PIIP 像重新规划楼层预算，把贵的空间留给最重要的楼层；PyramidCLIP 则像在每一层都贴上语义标签，方便和文本系统对齐。

| 方法 | 重点 | 典型场景 | 成本/精度权衡 | 适用边界 |
|---|---|---|---|---|
| PFMG | 细节保留 + 模态自适应 | RGB+IR 小目标检测 | 计算中等，精度收益集中在复杂场景 | 需要较好跨模态对齐 |
| PIIP | 参数预算优化 + 高低分支交互 | 预算敏感的高分辨率视觉任务 | 更省算力，细节恢复依赖结构设计 | 不直接解决模态噪声选择 |
| PyramidCLIP | 多层语义对齐 | 视觉语言预训练、检索、开放词表识别 | 语义收益明显，局部边界不一定最优 | 不以像素级融合为核心 |

因此，选择边界可以压缩成一句话：如果你的主要痛点是“多模态局部区域谁更可信”，优先看 PFMG；如果主要痛点是“算力不够还要保分辨率”，优先看 PIIP；如果主要目标是“跨模态语义对齐”，优先看 PyramidCLIP。
