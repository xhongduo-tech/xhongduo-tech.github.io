## 核心结论

多模态水印不是“分别给图片、文本、音频各打一个章”，而是让它们共享同一个可验证来源。更准确地说，它要把同一份溯源信息写进多种模态，并在验证时检查这些模态是否仍然指向同一个源头。

这件事重要，是因为生成式攻击已经从单模态篡改，发展到图像、语音、字幕、视频同步修改。Kim 等人在 Interspeech 2025 提出的跨模态框架，已经展示了一个关键能力：把原始音频嵌入视觉帧，在音频遭到克隆或唇同步篡改后，再从视觉侧恢复出“原始音频线索”，同时定位被改动的时间区间。这说明联合溯源的价值不只是“声明来源”，还包括“恢复证据”和“定位篡改”。

从工程角度看，多模态水印通常不是单一技术，而是三层组合：隐形水印负责抗常见编辑，签名清单负责声明内容历史，日志链或凭证链负责保证历史不可悄悄改写。C2PA/Content Credentials 属于这一层的开放标准，适合做内容“出生证明”；跨模态可逆水印则适合在某个模态受损后，仍从其他模态恢复证据。两者结合，才接近可部署的联合溯源系统。

| 模态 | 常见攻击 | 可依赖的检测信号 | 典型短板 |
|---|---|---|---|
| 图像/视频帧 | 压缩、裁剪、重编码、局部修补 | 频域残留、空间扰动、跨帧一致性、可逆嵌入恢复 | 裁剪和重采样会破坏局部水印 |
| 文本 | 改写、摘要、翻译、同义替换 | token 分布偏置、元数据签名、与图音语义一致性 | 重写攻击最强，鲁棒性通常最弱 |
| 音频 | 压缩、变速、降噪、克隆、语音转换 | 频谱模式、相位统计、恢复音频与现音频的特征差 | 语音克隆会保留说话风格但改掉语义 |
| 联合系统 | 跨平台转码、模态替换、缺失某一模态 | 共享 ID、跨模态相似度、签名链 | 最脆弱模态会决定整体下限 |

一个直接的玩具例子是：生成“猫坐在草地上”的图片、字幕和配音之前，先生成同一个 128-bit 标识，把它写入图像扰动、文本签名槽位和音频特征里。发布后，如果音频被人替换，但图像侧还能恢复出原始音频特征，且与字幕语义仍一致，就能判定这三种模态原本来自同一个生成流程。

## 问题定义与边界

多模态水印要解决的核心问题，不是“文件里有没有水印”这么简单，而是“当多种模态一起传播并可能被分别攻击时，系统还能不能证明它们原本属于同一来源”。这里的“溯源”指追踪内容从生成、编辑到发布的来源链；“联合”指不同模态不是孤立验证，而是相互提供证据。

Kim 等人的问题设置更具体，针对的是 SAVFs，中文可以叫“合成音视频伪造”：视频画面和音频都被生成模型改过，看起来像同一个人真的说过这段话。在这个场景里，单看图像或单听音频都不够，需要跨模态恢复和对比。

边界也必须说清楚。

第一，多模态水印不等于万能防伪。它通常回答的是“来源是否一致、篡改是否发生、发生在哪”，而不是“内容真假在社会意义上是否可信”。一个完整带水印的视频，也可能只是把错误信息完整记录下来。

第二，并不是所有模态都同样稳健。图像水印能承受一定压缩，文本水印却很怕改写。文本这里指自然语言字符串，它最大的特点是“语义不变但表面形式容易大改”，所以文本常常是联合系统里的最短板。

第三，单模态画质指标不够。PSNR、SSIM 这类指标只衡量像素或结构差异，无法回答“图像、文本、音频还是否表达同一个意思”。因此联合系统必须引入跨模态一致性检查。

| 攻击类型 | 受损模态 | 典型结果 | 需要的恢复或检查信号 |
|---|---|---|---|
| JPEG/视频压缩 | 图像/视频 | 局部水印衰减 | 频域鲁棒特征、跨帧冗余 |
| 字幕改写 | 文本 | 语义可接近、表面完全不同 | 共享 ID、语义向量一致性 |
| 语音克隆 | 音频 | 说话人像原人，内容已变 | 从视觉恢复音频线索，与现音频比较 |
| 裁剪/删段 | 图像或音频 | 时序不完整 | 时间戳、区块级水印、局部对齐 |
| 元数据剥离 | 任意模态文件 | C2PA 凭证消失 | 隐形水印或外部日志链兜底 |

一个更贴近新手的场景是：智能体生成一段“人物图片 + 自动配音 + 字幕”的短视频。若攻击者只替换音轨，但保留了视频帧和字幕，那么系统仍可从视频帧恢复嵌入的原始音频特征，再与当前音轨及字幕语义做一致性检查。如果恢复特征更接近原字幕而远离现音轨，就能判断音频源头已断裂。

## 核心机制与推导

一个常见建模方式是把图像与音频先写成联合变量：

$$
X_{org}=(I_{org},A_{org})
$$

其中 \(I_{org}\) 是原始图像或视频帧，\(A_{org}\) 是原始音频。Kim 等人的 CMW 方法把音频写入视觉侧，核心是可逆神经网络 INN。所谓“可逆”，就是前向嵌入后，理论上可通过逆向变换把输入恢复出来。

前向嵌入可以写成：

$$
I_{wm}=CMW(I_{org},A_{org})
$$

这里 \(I_{wm}\) 是带水印的视觉内容。它外观看起来应尽量接近 \(I_{org}\)，否则用户会直接看到失真。

如果内容之后遭遇攻击，得到被篡改样本：

$$
X_{tam}=(I_{tam},A_{tam})
$$

此时系统不直接相信 \(A_{tam}\)，而是从视觉侧逆向恢复一个“应当对应的原始音频线索”：

$$
A_{rec}=CMW^{-1}(I_{tam})
$$

然后比较 \(A_{rec}\) 和 \(A_{tam}\)。直接比原始波形很脆弱，因为压缩、回声消除、重采样都会让波形不同但语义相近，所以更稳的办法是在特征空间里比较余弦相似度。余弦相似度可以理解为“两个向量方向有多像”：

$$
\mathrm{cos\_sim}(u,v)=\frac{u\cdot v}{\|u\|\|v\|}
$$

如果对每个时间片都算一遍相似度，就能得到一个时间序列分数图。低于阈值的区段，就是可疑篡改段，这就是 Tamper Localization。

更广义的多模态联合溯源，还会把文本也拉进同一个语义空间。一个可用的三模态一致性指标是 TSAS：

$$
TSAS=\frac{1}{3}\big[sim(e_{text},e_{image})+sim(e_{text},e_{audio})+sim(e_{image},e_{audio})\big]
$$

其中 \(e_{text},e_{image},e_{audio}\) 是三种模态的语义向量。这里“语义向量”可以理解为“模型提取出来、代表内容含义的一串数字”。TSAS 越高，说明三模态越像在讲同一件事。

玩具例子可以这样看。原内容是“猫坐在草地上”，字幕、图像、配音都对应这个语义。后来攻击者把配音改成“狗在奔跑”。如果系统从图像里恢复出的音频特征仍然更接近“猫坐在草地上”，而当前音频特征更接近“狗在奔跑”，那么 \(sim(e_{image},e_{audio})\) 和 \(sim(e_{text},e_{audio})\) 会同时下降，TSAS 也会明显下降。这时系统不只是知道“出问题了”，还知道“问题主要在音频”。

需要强调一个工程事实：现阶段最成熟的具体做法，往往不是把三模态都做成同等强度的隐形鲁棒水印，而是“一个强鲁棒隐形通道 + 一个签名/元数据通道 + 一个跨模态一致性检查器”。原因很简单，文本通道太容易被重写，不能把全部安全性压在它身上。

## 代码实现

下面的代码不是论文复现，而是一个最小化玩具实现，用向量相似度模拟“恢复特征”和“跨模态一致性检查”。它能跑，也能体现判决逻辑。

```python
import math

def cosine_similarity(a, b):
    dot = sum(x * y for x, y in zip(a, b))
    na = math.sqrt(sum(x * x for x in a))
    nb = math.sqrt(sum(y * y for y in b))
    if na == 0 or nb == 0:
        return 0.0
    return dot / (na * nb)

def tsas(e_text, e_image, e_audio):
    return (
        cosine_similarity(e_text, e_image)
        + cosine_similarity(e_text, e_audio)
        + cosine_similarity(e_image, e_audio)
    ) / 3.0

# 玩具例子：原始三模态都在表达“猫坐在草地上”
e_text = [0.9, 0.1, 0.0]
e_image = [0.88, 0.12, 0.02]

# 从图像中恢复出的“原始音频特征”
a_rec = [0.91, 0.08, 0.01]

# 当前音频被攻击者换成“狗在奔跑”
a_tam = [0.1, 0.85, 0.05]

orig_consistency = tsas(e_text, e_image, a_rec)
tam_consistency = tsas(e_text, e_image, a_tam)
recovery_match = cosine_similarity(a_rec, a_tam)

assert orig_consistency > 0.95
assert tam_consistency < 0.5
assert recovery_match < 0.3
```

如果把它翻译成系统流水线，大致是下面这样：

```python
img_feat = img_encoder(I_org)
txt_feat = txt_encoder(T_org)
aud_feat = aud_encoder(A_org)

I_wm = cmw_inn(img_feat, aud_feat, shared_id=wm_id)
T_wm = text_watermark(T_org, shared_id=wm_id)   # 可是 token 水印，也可是签名槽位
I_tam, T_tam, A_tam = simulate_attack(I_wm, T_wm, A_org)

A_rec = inverse_inn(I_tam)
score_audio = cosine_similarity(aud_encoder(A_rec), aud_encoder(A_tam))
score_joint = tsas(txt_encoder(T_tam), img_encoder(I_tam), aud_encoder(A_tam))

if score_audio < tau_audio or score_joint < tau_joint:
    flag_tampering()
```

真实工程例子通常不会只靠这一个判决器。更可行的部署方式是：

1. 生成或采集内容时，为资产分配全局 ID。
2. 在图像/视频里写入隐形水印，在音频里写入可恢复特征或鲁棒标记。
3. 在文本和容器层写入 C2PA/Content Credentials 清单，记录模型、时间、编辑链。
4. 分发后先查签名链，再做隐形解码，再做跨模态一致性检查。
5. 若某一层失效，用其余层交叉补证。

训练时还有一个关键点：必须把攻击模拟层塞进 loop。也就是在训练阶段主动加入压缩、裁剪、重采样、语音克隆、文本改写等扰动，否则模型只会在“实验室里没被攻击的数据”上表现好。

## 工程权衡与常见坑

第一个坑是把“画质没坏”误当成“语义没坏”。PSNR、SSIM 只能说明图像改动小，不说明字幕和配音是否还在表达同一件事。WMarkGPT 这类方法的价值在于，它尝试用多模态模型回答“水印是否可见、落在什么位置、是否影响图像内容理解”，比单纯算像素差更接近真实使用问题。

第二个坑是把所有模态看成同等强。实际上，文本最容易在不改变大意的情况下被彻底改写；音频最容易遭遇克隆和重配；图像最容易被压缩和裁剪。联合系统必须接受一个现实：整体鲁棒性通常由最弱模态决定。

第三个坑是把“有 C2PA”误解成“攻击一定能检测”。C2PA 更像带签名的履历表，擅长记录创建和编辑历史；但如果文件被另存、截图、录屏、转码，凭证可能丢失，所以还需要隐形水印和外部日志兜底。

第四个坑是没有做组合攻击测试。真实世界不是只来一次 JPEG，而是“先压缩，再裁剪，再重编码，再改字幕，再换音轨”。如果测试矩阵只覆盖单一攻击，线上效果往往会虚高。

| 指标 | 它回答什么问题 | 何时不能单独使用 |
|---|---|---|
| PSNR | 像素失真大不大 | 无法判断语义是否被改 |
| SSIM | 结构是否近似 | 对跨模态一致性无能为力 |
| cos-sim | 两段特征方向是否接近 | 依赖编码器质量，阈值需校准 |
| TSAS | 图文音是否仍在表达同一语义 | 不能替代来源签名 |
| WMarkGPT 类评价 | 水印是否可见、是否影响理解 | 适合评估，不直接等于防伪证明 |

一个实用测试矩阵是：`图像压缩 + 音频替换`、`文本改写 + 视频裁剪`、`音频克隆 + 元数据剥离`。每种组合都跑解码器、相似度和 TSAS，并记录在哪一层先失效。这个矩阵比单看一串平均分更接近上线前的风险面貌。

## 替代方案与适用边界

如果目标只是“证明内容来自哪里、经过了哪些编辑”，而不是“从别的模态恢复被替换的内容”，那么 C2PA/Content Credentials 加标准隐形水印通常已经够用。它部署难度较低，适合媒体机构、影像工作流和平台分发。

如果目标只是图像保护，那么单模图像水印加 WMarkGPT 一类评估也可以成立。好处是成本低、实现成熟；缺点是它无法回答“字幕和配音是否还是同源”。

如果你刚起步，最合理的路径往往不是一步做到图文音三模联合，而是先做“图像 + 文本”的共享来源链：图像侧放隐形标识，文本侧放签名或 token 水印，外层再包 C2PA 清单。等到业务真的需要识别语音替换，再把音频恢复模块接进来。

| 需求 | 方案 | 限制 |
|---|---|---|
| 只做来源声明 | C2PA + Content Credentials | 元数据可能被剥离 |
| 只做图像版权保护 | 单模图像隐形水印 | 无法验证文本/音频一致性 |
| 要检测音视频伪造并定位时间段 | CMW 类跨模态可逆水印 | 训练和部署复杂，成本高 |
| 要做平台级联合溯源 | 隐形水印 + 签名清单 + 日志链 + 一致性检查 | 体系复杂，需统一规范 |

因此，多模态水印的适用边界很明确：当你的内容真的是多模态生成、并且攻击者可能只替换其中一部分时，它才比单模方案明显更有价值。若只是普通图片发布，直接上联合系统通常不划算。
