## 核心结论

成员推断攻击，英文是 Membership Inference Attack，指攻击者判断“某条样本是否出现在模型训练集里”的攻击。白话说，它不关心样本内容是什么，而关心模型有没有“见过它”。

这类攻击之所以成立，核心原因通常不是模型把整条样本原样背下来，而是训练样本与未见样本在统计上留下了可区分信号。最常见的信号有三类：

1. 训练样本往往得到更高置信度。
2. 训练样本往往有更低损失。
3. 对生成模型来说，训练样本往往表现出更低困惑度或更稳定的生成行为。

因此，成员推断不是“玄学猜测”，而是一个标准二分类问题：给定样本 $x$ 和模型输出，判断 $x$ 是否属于训练集 $D$。形式上可写成：

$$
\phi_M(x)=\mathbf{1}[x\in D_M]
$$

其中 $\mathbf{1}[\cdot]$ 是指示函数，白话说就是“条件成立记为 1，不成立记为 0”。

这件事重要，不是因为攻击者一定能高召回地找出全部训练样本，而是因为在极低误报率下，只要还能稳定命中一部分成员，就已经说明训练数据存在可利用泄露。实际评测里，常看 AUC、`TPR@FPR=1%`、`TPR@FPR=0.1%` 这类指标。AUC 是整体区分能力，白话说是“成员和非成员是否能被整体分开”；`TPR@低FPR` 更关键，因为工程里没人接受大量误报。

一个新手可以先记住最直观的判断方式：如果一张训练过的猫图，模型长期给出更高的“猫”概率；而另一张未训练过的猫图概率明显更低，那么这个差距本身就是攻击入口。

## 问题定义与边界

成员推断攻击解决的是“membership bit”问题，也就是某条记录的成员位。白话说，就是一个 0/1 标签：在训练集里还是不在训练集里。

更正式地说，攻击者面对目标模型 $M$，拿到候选样本 $x$，构造一个判别器：

$$
A(M, x)\rightarrow \{0,1\}
$$

若输出 1，则认为 $x$ 是成员；输出 0，则认为不是。

这个问题的边界，取决于攻击者能力。实际工程里一般分三层：

| 攻击能力 | 能看到什么 | 典型信号 | 常见方法 |
|---|---|---|---|
| 黑盒 | 只能查 API 输出 | 概率、排名、文本分数、困惑度 | 阈值攻击、影子模型、LiRA 变体 |
| 灰盒 | 能看到部分中间信息 | logits、部分层输出、损失 | 分布拟合、似然比、邻居比较 |
| 白盒 | 能看参数、梯度、激活 | 梯度范数、层内表示、训练轨迹 | 梯度攻击、轨迹攻击、表示空间攻击 |

这里的黑盒，指只能通过接口查询模型；白话说，你看不到模型内部，只能“问问题拿答案”。灰盒是拿到一点内部信息，但不是全量。白盒则接近训练方权限。

还要再分两个维度。

第一，单样本和多样本。单样本攻击只判断一条记录；多样本攻击则判断一个集合，比如“这批病历里哪些出现在训练中”。多样本方法常常更强，因为可以利用集合统计特征。

第二，判别式模型和生成式模型。判别式模型输出类别概率，例如图像分类；生成式模型输出 token 概率或困惑度，例如大语言模型。前者常用置信度和损失，后者常用序列似然和困惑度。

玩具例子可以写得非常简单。假设一个二分类器对两张猫图给出如下结果：

| 样本 | 是否在训练集 | 猫类别概率 | 交叉熵损失 |
|---|---|---:|---:|
| 图 A | 是 | 0.99 | 0.01 |
| 图 B | 否 | 0.74 | 0.30 |

如果攻击者预先知道“训练样本损失常小于 0.05”，那只要看到图 A 的损失是 0.01，就很容易判断它是成员。这就是最基础的阈值攻击。

问题边界也决定了防御边界。如果模型只输出 top-1 标签，攻击难度会比输出完整概率分布高；如果允许反复查询，攻击者就能做均值、方差、温度扰动、多轮打分，信号会更稳定。

## 核心机制与推导

大多数成员推断攻击，本质都是在比较两种分布：

1. 成员样本在目标模型上的分布
2. 非成员样本在目标模型上的分布

只要这两个分布不完全重合，就能构造攻击器。

一个通用写法是：

$$
p=F(\mathcal{L}(M,x))
$$

其中 $\mathcal{L}(M,x)$ 是样本 $x$ 在模型 $M$ 上的某种统计量，比如损失、困惑度、最大置信度；$F$ 是从统计量到成员概率的映射。白话说，先提一个“可疑分数”，再把这个分数转成“像不像训练样本”。

### 1. 阈值攻击

最简单的攻击是假设成员样本损失更低，于是设定阈值 $\tau$：

$$
A(x)=\mathbf{1}[\mathcal{L}(M,x)<\tau]
$$

如果损失小于阈值，就判为成员。这个方法简单，但问题也明显：不同类别、不同长度、不同难度样本的损失分布并不一致，一个全局阈值容易误伤。

### 2. 影子模型

影子模型，英文是 shadow model，指攻击者训练一个或多个“模仿目标模型行为的参考模型”。白话说，就是自己造一个近似环境，学习成员和非成员各自会长什么样。

流程通常是：

1. 准备与目标任务相似的数据分布。
2. 划分影子模型自己的训练集与保留集。
3. 训练影子模型。
4. 分别收集影子模型对成员样本、非成员样本的输出统计。
5. 用这些统计训练一个攻击分类器，最后转移到目标模型上。

这等价于先估计两类样本的经验分布，再做判别。

### 3. 似然比与 LiRA

LiRA 可以理解为 likelihood-ratio attack，即似然比攻击。白话说，它不是只看目标模型“有多自信”，而是看“目标模型比参考模型多自信多少”。

若记目标模型分数为 $s_t(x)$，参考模型集合给出的均值与方差为 $\mu_0(x), \sigma_0(x)$（非成员假设）以及 $\mu_1(x), \sigma_1(x)$（成员假设），则可以比较：

$$
\Lambda(x)=\log \frac{P(s_t(x)\mid x\in D)}{P(s_t(x)\mid x\notin D)}
$$

若 $\Lambda(x)$ 越大，则越像成员。对白话读者来说，可以把它想成“两条分布曲线下，当前分数更靠近哪一边”。

如果用更直观的差值写法，也常写成：

$$
\Delta s(x)=s_{\text{target}}(x)-s_{\text{ref}}(x)
$$

当 $\Delta s(x)$ 很大，表示目标模型对该样本异常自信；这通常意味着它更可能见过这个样本。

### 4. 为什么低 FPR 很重要

FPR 是假阳性率，白话说是“把非成员错判成成员的比例”。TPR 是真阳性率，白话说是“真正成员被命中的比例”。

很多论文会特别报告 `TPR@FPR=0.1%` 或更低值，因为真正危险的攻击不是“到处乱报”，而是“误报极低但一旦报中就非常可信”。如果一个攻击在极低 FPR 下还能抓到不少成员，说明它具备实际追踪价值。

下面这个表可以帮助建立量级直觉。数值口径来自用户给定资料中的汇总表述，用于说明“低误报下仍能暴露成员”这一事实：

| 方法/结论口径 | 访问方式 | 关注指标 | 代表性现象 |
|---|---|---|---|
| 置信度/损失阈值 | 黑盒 | AUC, TPR@固定FPR | 简单、便宜，但依赖校准 |
| LiRA/似然比 | 黑盒或弱白盒 | TPR@0.1% FPR | 极低误报区间更稳 |
| Few-shot/增强管线 | 黑盒 | TPR@更低FPR | 对高风险样本放大明显 |
| NDSS 攻击管线 | 黑盒 | TPR@0.1% FPR | 在特定条件下可达到极高 TPR |

真实工程里，长尾样本、重复样本、罕见模式最危险。原因不是这些样本“神秘”，而是它们更容易让模型形成尖锐、稳定、难以泛化的响应，从而把成员信号放大。

## 代码实现

下面给一个最小可运行的 Python 例子，演示如何用“损失阈值”做成员推断。它不是完整研究实现，但足够说明机制。

```python
from math import log

def cross_entropy(prob_true_class: float) -> float:
    assert 0.0 < prob_true_class <= 1.0
    return -log(prob_true_class)

def calibrate_threshold(member_losses, nonmember_losses):
    assert member_losses and nonmember_losses
    # 一个最简单的校准：取两类均值中点
    m1 = sum(member_losses) / len(member_losses)
    m0 = sum(nonmember_losses) / len(nonmember_losses)
    return (m1 + m0) / 2.0

def infer_membership(loss_value, threshold):
    return loss_value < threshold

# 玩具数据：模型对训练样本更自信，因此损失更低
member_probs = [0.99, 0.97, 0.96, 0.98]
nonmember_probs = [0.80, 0.76, 0.72, 0.68]

member_losses = [cross_entropy(p) for p in member_probs]
nonmember_losses = [cross_entropy(p) for p in nonmember_probs]

threshold = calibrate_threshold(member_losses, nonmember_losses)

train_like_sample = cross_entropy(0.985)
unseen_like_sample = cross_entropy(0.70)

assert infer_membership(train_like_sample, threshold) is True
assert infer_membership(unseen_like_sample, threshold) is False

print("threshold =", round(threshold, 4))
print("train_like_loss =", round(train_like_sample, 4))
print("unseen_like_loss =", round(unseen_like_sample, 4))
```

这段代码的逻辑只有四步：

1. 用模型对真实类别的概率算交叉熵损失。
2. 用一批已知成员与非成员样本做阈值校准。
3. 对新样本计算损失。
4. 损失低于阈值则判为成员。

如果把它翻译成生成式模型语言，`loss` 可以换成 `perplexity`。困惑度，英文是 perplexity，指模型对文本“有多不意外”；白话说，越低代表模型越熟悉这段文本。

流程可以抽象成：

| 步骤 | 输入 | 输出 |
|---|---|---|
| 统计采样 | 已知成员/非成员样本 | 两类损失分布 |
| 阈值校准 | 两类分布 | 阈值 $\tau$ |
| 目标查询 | 候选样本 | 损失或困惑度 |
| 攻击决策 | 分数与阈值 | 成员/非成员 |

如果要做更接近真实场景的版本，可以把阈值替换为“每个类别一个阈值”或“每个长度桶一个阈值”。例如 NLP 场景里，长文本天然损失更高，不能直接和短文本共用阈值。

真实工程例子是医疗影像或客服语料。假设一家厂商提供疾病识别 API，攻击者手里有一张未公开的罕见病影像，连续几次查询后发现模型始终给出异常高的置信度，而且对轻微扰动非常稳定。若再对比同分布的其他非训练影像，只有这张图长期落在极低损失区间，那么攻击者就能较高把握地推断：这张影像很可能出现在训练集中。若该影像对应真实患者，隐私问题就已经成立。

## 工程权衡与常见坑

成员推断攻击在工程上最常见的触发器，是过拟合。过拟合，指模型在训练集上表现很好，但对未见数据泛化变差；白话说，就是“背得太熟，没学会一般规律”。一旦 train-test gap 变大，成员与非成员的统计差异就会被拉开。

常见坑有五类。

### 1. 只看 AUC，不看低 FPR 区间

AUC 高不代表线上危险一定高。真正要看的是在 `FPR=1%`、`0.1%` 甚至更低时，TPR 还能剩多少。因为攻击者真正需要的是“高置信命中”，不是“平均意义上还行”。

### 2. 用全局阈值覆盖全部样本

不同类别、长度、频次、清洗质量的样本分布差异很大。一个统一阈值会把“本来就容易”的样本和“天然困难”的样本混在一起，既影响攻击效果，也影响风险评估。

### 3. 忽视长尾和重复数据

长尾样本指很少见的样本；白话说，就是训练集中少、但又很特别的数据。重复数据则会人为增加模型对某条记录的暴露次数。这两类都会放大记忆信号。

医疗场景里最典型。某罕见病影像在训练集中只出现极少几次，但特征非常鲜明。模型可能对它形成异常高置信度。攻击者不需要很多查询，就能发现它与普通样本显著不同。

### 4. 只做输出裁剪，不做训练侧治理

很多团队会先想到限制输出，例如只返回 top-1 标签、截断概率、降低小数精度。这有帮助，但通常不够。因为只要成员和非成员的响应差异仍存在，攻击者仍可利用排序、稳定性、重复查询后的统计量来恢复信号。

### 5. 误把“模型没背原文”当成“没有隐私风险”

成员推断不要求模型逐字输出训练内容。只要它对成员样本表现出更低损失、更高置信、更稳定响应，就可能泄露 membership。也就是说，“没原样复现”不等于“没泄露”。

下面给一个工程防御对比表：

| 防御手段 | 作用机制 | 成本 | 局限 |
|---|---|---|---|
| 正则化/早停 | 缩小训练测试差距 | 低到中 | 降风险有限，不能给理论保证 |
| 输出裁剪/只返标签 | 减少可观测信号 | 低 | 可被多轮查询和侧信号绕过 |
| 熵正则 | 降低过尖锐置信度 | 中 | 可能损伤精度与校准 |
| 数据去重/长尾审计 | 减少异常记忆点 | 中 | 依赖高质量数据治理 |
| 差分隐私训练 | 从训练机制上控泄露 | 高 | 精度、吞吐、调参成本都高 |

可以把防御链路概括为：

$$
\text{regularization / DP / dedup}
\Rightarrow \text{smaller } \Delta \text{loss}
\Rightarrow \text{lower attack advantage}
$$

这里的 attack advantage，指攻击相对随机猜测多出来的优势；白话说，就是“攻击到底比蒙对强多少”。

## 替代方案与适用边界

成员推断不是唯一隐私评估方法，也不是所有场景都该用同一套口径。

### 1. 差分隐私训练

差分隐私，英文是 Differential Privacy，指加入受控随机噪声，使单条样本对训练结果影响受限。白话说，就是“任何一条记录加入或删掉，都不该让模型变化太明显”。

理论上，若训练满足 $(\varepsilon,\delta)$-DP，则成员推断优势会被上界约束。常见直观写法是优势受 $e^\varepsilon-1$ 控制。但工程上不能把这句话理解成“上了 DP 就没事了”。因为高相关数据、重复数据、强分布偏置都会削弱实际效果，而且小 $\varepsilon$ 往往伴随明显性能损失。

### 2. Set-based 或 distributional MIA

当单条样本信号太弱时，可以切换到集合级推断。集合级推断不是问“这一条是否在训练集里”，而是问“这一批样本整体上是否更像训练成员”。这类方法更适合审计数据集、用户分组或会话级样本。

### 3. 邻居比较与显著性检验

邻居比较方法会把目标样本和若干近邻样本一起比较，判断它是否异常接近训练分布。显著性检验则关注“当前命中是否统计显著”，而不是只盯着单一阈值。

### 4. Log-MIA 视角

有些工作提出用对数尺度度量泄露强度，例如 Log-MIA。对新手来说，可以把它理解成：不是只问“能不能区分”，而是问“在严格统计约束下，泄露证据有多强”。这比单看 AUC 更适合高安全要求场景，因为它更关注尾部风险而不是平均表现。

下面给一个选型表：

| 场景 | 更适合的方法 | 原因 | 代价 |
|---|---|---|---|
| API 黑盒审计 | 阈值攻击、LiRA | 不依赖内部参数 | 需要校准样本 |
| 大模型文本审计 | 困惑度、参考模型比值 | 易于落地到序列概率 | 对长度和模板敏感 |
| 高安全行业 | DP 训练 + 低FPR评测 | 同时做预防和验证 | 成本最高 |
| 数据集整体审计 | Set-based MIA | 单样本弱时更稳 | 结果不细到单条记录 |
| 模型内部可访问 | 白盒梯度/表示攻击 | 信号更丰富 | 权限要求高 |

适用边界要说清楚：如果模型输出被极端裁剪、查询次数被严格限制、训练中又采用了较强 DP，那么简单阈值攻击可能几乎失效。但这不代表整体风险归零，而是说明需要换评测口径，比如集合级推断、邻域比较或更细的白盒审计。
