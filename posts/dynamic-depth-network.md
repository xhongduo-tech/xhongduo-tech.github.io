## 核心结论

Dynamic Depth 的定义很直接：把“网络实际使用多深”从固定超参数，改成训练或推理过程中可调整的变量。这里的“深度头”指挂在不同层上的输出模块，可以把每一层都看成一个候选决策器；“自适应计算”指模型不是对所有样本都走同样计算路径，而是按样本难度、历史损失或当前约束动态分配计算量。

它的价值不在“网络更深”，而在“深浅层可以分工”。浅层参数少、路径短，通常更快收敛；深层表达能力更强，能表示更复杂的模式。Dynamic Depth 用一个深度分布把这两者接起来：简单样本让浅层先起作用，复杂样本再提高深层权重，或者在流式训练中先依赖浅层快速适应，再逐步把能力迁移到更深层。

这类方法最重要的结论有三点。第一，它改变的是信息流，不只是结构外观。多层输出头让浅层直接接收监督，减轻“梯度消失”问题；“梯度消失”就是误差信号传到前面层时越来越弱，前面层学不动。第二，它改变的是容量分配，不是单纯做平均。若把 $\alpha^{(l)}$ 看成第 $l$ 个深度头的投票权，那么模型有效容量会随数据阶段移动，而不是永远押注某一层。第三，它能继承在线学习中的 regret 界；“regret 界”就是和事后最优固定选择相比，多出来的累计损失上界，因此它给的是在线决策保证，而不是传统泛化误差保证。

下面这张表先把“动态深度”和“固定深度”分开：

| 维度 | Dynamic Depth | 固定深度网络 |
|---|---|---|
| 深度选择 | 训练或推理时动态调整 | 训练前固定 |
| 监督路径 | 多个中间头直接监督 | 通常只有末层监督 |
| 计算分配 | 可随样本或时间变化 | 对所有样本一致 |
| 适用优势 | 在线学习、流式数据、低时延约束 | 离线训练、分布稳定任务 |
| 主要风险 | 训练更复杂，权重更新易抖动 | 结构可能过深或过浅 |

一个玩具例子可以先建立直觉。假设你做二分类，网络有 3 个头：第 0 层、第 1 层、第 2 层。对一条明显样本，三个头都能分对，但浅层已经给出 0.95 的正确类概率，此时继续走更深层收益很小。对一条边界样本，浅层只有 0.55，深层可能提升到 0.85。固定深度网络对两条样本都走满 3 层；Dynamic Depth 则允许“简单样本浅层就够，复杂样本再用深层”，这才是它和“普通深网络”的本质差异。

## 问题定义与边界

先把讨论范围收紧。本文说的 Dynamic Depth，不是“继续加层”，不是“训练完后剪枝”，也不是单纯“早退”；“早退”指推理时在中间层提前输出并停止计算。这里的主线是 HBP 一类在线动态深度方法：先定义一个最大深度 $L$，再给每个深度挂一个输出头，并在训练过程中根据各头损失更新深度权重。

形式化地，设输入为 $x$，最大深度为 $L$。第 $l$ 层隐藏表示为 $h^{(l)}$，第 $l$ 个深度头的预测为 $f^{(l)}(x)$。写成公式：

$$
h^{(0)} = x,\qquad
h^{(l)} = \sigma\!\left(W^{(l)} h^{(l-1)}\right),\qquad
f^{(l)}(x)=\text{softmax}\!\left(h^{(l)}\Theta^{(l)}\right).
$$

这里的 $\sigma$ 是激活函数，也就是给线性变换增加非线性的部件；没有它，多层网络会退化成一层线性模型。$\Theta^{(l)}$ 是第 $l$ 个头自己的分类参数。最终输出不是只取最后一层，而是做加权融合：

$$
F(x)=\sum_{l=0}^{L}\alpha^{(l)} f^{(l)}(x),\qquad
\alpha^{(l)}\ge 0,\ \sum_{l=0}^{L}\alpha^{(l)}=1.
$$

$\alpha^{(l)}$ 可以理解为第 $l$ 个头当前被信任的程度。若某个头近期损失更小，它的权重就会上升；若表现变差，权重就会下降。

边界也要说清。Dynamic Depth 更适合下面几类任务：

| 维度 | Dynamic Depth | 固定深度网络 |
|---|---|---|
| 深度选择 | 动态调整 | 事先固定 |
| 适用场景 | 在线/流式/漂移 | 离线/稳定分布 |
| 目标 | 自适应深度分配 | 静态最优结构 |

为什么在线场景更合适？因为流式任务里数据分布会变，“概念漂移”就是今天有效的规律，明天可能失效。固定 10 层在一开始可能太深，训练慢；在后期又可能不够灵活。Dynamic Depth 不要求一开始就猜中最佳深度，而是在数据持续到来时不断调整深度分布。

真实工程例子可以用在线风控说明。电商欺诈识别每天进来新样本，早期规则简单，浅层特征就能分辨大部分订单；当黑产策略变化后，需要更复杂的组合特征，深层才逐渐变得重要。若一直固定浅层，准确率会掉；若一直固定深层，延迟和训练成本都更高。Dynamic Depth 的意义是让这两种阶段用同一套框架衔接。

## 核心机制与推导

机制可以拆成四步：多头输出、加权融合、在线更新、共享反传。

第一步是多头输出。每个深度头都独立给预测，不必等到最后一层。这样做的直接效果是每一层都暴露在监督信号下。对初级读者来说，可以把它理解成“每一层都参加考试，不只最后一层交卷”。

第二步是加权融合。最终预测 $F(x)$ 是各头输出的凸组合；“凸组合”就是权重非负且和为 1 的加权平均。这样做有两个含义。其一，模型不会被迫二选一，而是允许深度共存。其二，$\alpha$ 本身就是一个关于深度的概率分布，因此可以讨论“当前有效深度”：
$$
d_{\text{eff}} = \sum_{l=0}^{L} l \cdot \alpha^{(l)}.
$$
这个量不是网络物理层数，而是模型当前平均更依赖哪些深度。

第三步是在线更新。HBP 使用 Hedge 风格规则：
$$
\alpha_{t+1}^{(l)} \propto \alpha_t^{(l)} \beta^{\ell_t^{(l)}},\qquad 0<\beta<1.
$$
$\ell_t^{(l)}$ 是第 $l$ 个头在第 $t$ 个样本上的损失，$\beta$ 是更新强度参数。因为 $\beta<1$，损失越大，$\beta^{\ell_t^{(l)}}$ 越小，该头权重就衰减得越快。白话解释就是：近期谁错得少，下一轮谁票数更高。

看一个玩具例子。设三层头的初始权重相同，
$$
\alpha_t=(1/3,1/3,1/3),
$$
本轮损失分别为 $(0.05,0.20,0.70)$，取 $\beta=0.5$。更新后未归一化权重大约为：
$$
(1/3)\cdot 0.5^{0.05}\approx 0.322,\quad
(1/3)\cdot 0.5^{0.20}\approx 0.290,\quad
(1/3)\cdot 0.5^{0.70}\approx 0.205.
$$
归一化后约为 $(0.394,0.355,0.251)$。这说明当前样本后，系统更倾向前两层。若下一段数据更复杂，深层头连续表现更好，它的权重又会回升，所以这不是永久裁决，而是持续竞争。

第四步是共享反向传播。第 $l$ 层参数不仅服务第 $l$ 个头，也服务所有更深的头，因此梯度更新写成：
$$
W_{t+1}^{(l)} \leftarrow W_t^{(l)} - \eta \sum_{j=l}^{L}\alpha_t^{(j)} \nabla_{W^{(l)}} \ell_t^{(j)}.
$$
这里的 $\eta$ 是学习率，也就是每次更新步长；$\nabla$ 表示梯度，即损失对参数变化的敏感程度。这个式子有两个直接推论。

第一，浅层会收到更密集的监督。因为对层 $l$ 来说，所有 $j\ge l$ 的头都会给它反馈，不再只有最末层一条长路径。第二，浅层学到的是“同时服务多个深度”的公共表示，而不是只为了最后一层服务。这也是 Dynamic Depth 影响信息流的关键点。

可证明性质主要来自深度权重更新，而不是整个深网训练都能被完整证明。若把每个深度头视作一个专家，Hedge 可给出累计 regret 上界：
$$
R_T = O\!\left(\sqrt{T\log N}\right),\qquad N=L+1.
$$
意思是：随着时间增长，动态组合的累计损失不会比事后最优固定深度差太多。这里比较对象是“固定选某一头”，不是“所有可能神经网络中的最优模型”。所以它证明的是在线选深度的合理性，不是神经网络整体泛化的充分条件。

## 代码实现

实现 Dynamic Depth 时，最容易写错的点是：只做了多出口，却没有真正更新深度权重。真正需要的部件有三类：共享 backbone、每层输出头、独立的 $\alpha$ 更新逻辑。

下面给一个可运行的最小 Python 版本，用线性层模拟多层网络，不依赖深度学习框架。它演示三件事：逐层产生输出、用 $\alpha$ 融合、按 Hedge 规则更新权重。`assert` 用来检查“代码确实满足我们定义的约束”。

```python
import math

def sigmoid(x):
    return 1.0 / (1.0 + math.exp(-x))

def normalize(probs):
    s = sum(probs)
    return [p / s for p in probs]

def toy_forward(x, hidden_weights, head_weights):
    """
    x: float
    hidden_weights: list[float], 每层一个标量权重
    head_weights: list[float], 每个深度头一个标量权重
    返回每个深度头对正类的预测概率
    """
    h = x
    outputs = []

    # depth 0 head: 直接看输入
    outputs.append(sigmoid(head_weights[0] * h))

    for l, w in enumerate(hidden_weights, start=1):
        h = math.tanh(w * h)
        outputs.append(sigmoid(head_weights[l] * h))

    return outputs

def bce_loss(p, y, eps=1e-8):
    p = min(max(p, eps), 1 - eps)
    return -(y * math.log(p) + (1 - y) * math.log(1 - p))

def update_alpha(alpha, losses, beta=0.7, smooth=1e-6):
    raw = [a * (beta ** loss) + smooth for a, loss in zip(alpha, losses)]
    total = sum(raw)
    return [v / total for v in raw]

def ensemble_pred(alpha, outputs):
    return sum(a * o for a, o in zip(alpha, outputs))

# 三个深度头：0, 1, 2
alpha = [1/3, 1/3, 1/3]
hidden_weights = [1.2, 1.6]
head_weights = [2.5, 4.0, 7.0]

# 玩具样本：x 越大越接近正类
dataset = [(0.2, 0), (0.4, 0), (1.0, 1), (1.5, 1)]

history = []
for x, y in dataset:
    outputs = toy_forward(x, hidden_weights, head_weights)
    losses = [bce_loss(o, y) for o in outputs]
    fused = ensemble_pred(alpha, outputs)
    history.append((x, y, outputs, fused, alpha))
    alpha = update_alpha(alpha, losses, beta=0.7)

# 约束检查
assert abs(sum(alpha) - 1.0) < 1e-8
assert all(a > 0 for a in alpha)

# 简单正确性检查：融合预测落在 [0, 1]
for _, _, outputs, fused, _ in history:
    assert all(0.0 < o < 1.0 for o in outputs)
    assert 0.0 < fused < 1.0

# 在这个数据上，最终较优头应该拿到非零权重
best_head = max(range(len(alpha)), key=lambda i: alpha[i])
assert best_head in (0, 1, 2)

print("final alpha =", [round(a, 4) for a in alpha])
print("best head =", best_head)
```

如果迁移到 PyTorch，结构也相同：

1. `forward` 中逐层计算隐藏表示，并把每层头的 `logits` 收集起来。
2. 对每个头单独算损失 `ell_t^(l)`。
3. 用当前 `alpha` 做融合输出，得到最终预测。
4. 用 `update_alpha` 根据各头损失更新深度权重。
5. 反向传播时，让 backbone 参数接收多个头的加权梯度。

真实工程里，例如在线点击率预估，可以把用户画像、上下文特征喂给共享 backbone；第 1 层头主要响应简单交叉特征，第 3 层头学习更复杂的非线性组合。流量高峰时，如果监控发现浅层头已经足够稳定，可以只保留前几层进行低延迟推理；数据分布突变时，再让更深头逐渐接管更多权重。

## 工程权衡与常见坑

Dynamic Depth 的主要收益来自“自适应”，但工程风险也集中在“自适应”本身。

| 问题 | 表现 | 原因 | 处理方式 |
|---|---|---|---|
| $\beta$ 太小 | 深层很快失权 | 更新过激，单次损失影响过大 | 调大 $\beta$，加入平滑项 |
| $\beta$ 太大 | 动态不明显 | 权重变化过慢 | 适度减小 $\beta$，或分阶段调参 |
| 头部校准差 | 早退过早或切层混乱 | 概率值不可信 | 做温度缩放等校准 |
| 中间头过弱 | 浅层长期占优 | 中间表示能力不足 | 提升中间头容量或增加辅助监督 |
| 离线任务硬套 | 收益不明显 | 没有流式自适应需求 | 直接选固定深度或其他方法 |

第一个坑是把 `alpha` 当成结构搜索结果。它不是一次性选出“最佳层数”，而是持续更新的在线分布。若你在训练前几轮看到浅层占优就直接删掉深层，往往会把后续复杂模式需要的能力提前裁掉。

第二个坑是把置信度当真。很多早退系统依赖“预测够自信就停止”，但神经网络概率常常失准；“校准”就是让输出概率更接近真实正确率。如果校准差，模型会过早认为自己已经足够确定，从而在错误的深度停下。

第三个坑是额外的维护成本。多头意味着更多监控项：每层损失、每层校准、每层时延、深度分布漂移。如果你的任务完全离线，且只关心最终精度，Dynamic Depth 往往不是第一选择，因为它引入的训练和运维复杂度可能超过收益。

可以用一个工程判断清单快速判断是否值得上：
| 判断项 | 若答案为“是”更适合 Dynamic Depth |
|---|---|
| 数据是否持续到来 | 是 |
| 数据分布是否会漂移 | 是 |
| 是否要求低时延 | 是 |
| 是否允许多头输出与额外监控 | 是 |
| 是否需要在不同样本间动态分配计算 | 是 |

## 替代方案与适用边界

Dynamic Depth 不是“更高级的深度优化总方案”，它只是更匹配某些约束。和它最容易混淆的有三类方法：DSN、Stochastic Depth、BranchyNet。

| 方法 | 核心机制 | 关键假设 | 适用范围 |
|---|---|---|---|
| HBP / Dynamic Depth | 多深度头在线加权，权重随损失更新 | 流式数据、最大深度已定 | 在线学习、概念漂移 |
| DSN | 每层加 companion loss | 深度固定、离线训练 | 中间层监督、缓解梯度问题 |
| Stochastic Depth | 训练时随机丢弃残差块 | 残差结构可用 | 超深网络正则化 |
| BranchyNet | 推理时按置信度早退 | 置信度可用、延迟敏感 | 端侧推理、低时延部署 |

DSN 的重点是训练更稳，不是在线选深度。它给中间层加辅助损失，让前面层更容易学到有用表示，但推理时通常仍按固定深度走完。所以若你的问题只是“网络太深不好训”，DSN 往往比 Dynamic Depth 更直接。

Stochastic Depth 的重点是正则化；“正则化”就是通过额外约束减少过拟合。它训练时随机跳过一些残差块，让超深网络更容易优化，但测试时仍使用完整网络或其缩放版本。它不维护显式深度权重，也不根据在线损失在多个深度间竞争。

BranchyNet 更接近“按样本早停”，但它通常假设置信度可以可靠地控制退出时机，核心目标是减少推理时延，而不是在流式学习中调整深度分配。若你做的是端侧视觉模型，希望省电、降延迟，BranchyNet 这条线更贴近实际约束。

边界结论可以压成一句话：如果你要的是在线场景下“深浅层随时间和样本动态分工”，用 Dynamic Depth；如果你只要更稳的监督，用 DSN；如果你要训练超深网络更稳，用 Stochastic Depth；如果你要低延迟推理，用 BranchyNet。不是谁更先进，而是谁更符合任务约束。
