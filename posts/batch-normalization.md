## BatchNorm 解决的并不是“让均值变成 0”这么简单

深层网络训练困难，很大一部分原因来自中间层输入分布不断漂移。当前面层的参数更新后，后面层看到的输入统计量也跟着变化，优化器相当于一直在追逐一个移动靶标。Batch Normalization（BN）提出的核心思路是：**在每个 mini-batch 上把中间表示重新标准化，再交给后续网络处理。**

---

## 前向公式：先标准化，再恢复表达能力

对某一特征维度，BN 在一个 batch 上计算：

```text
μ_B = 1/m Σ x_i
σ_B^2 = 1/m Σ (x_i - μ_B)^2
```

然后做归一化：

```text
x̂_i = (x_i - μ_B) / sqrt(σ_B^2 + ε)
```

最后引入可学习参数 `γ, β`：

```text
y_i = γ x̂_i + β
```

这一步很关键。若只做标准化，网络表达能力会被强行限制；`γ,β` 允许模型自己决定要不要恢复某个尺度和偏移，因此 BN 不是简单“把数据改坏”，而是在数值稳定与表达能力之间取平衡。

---

## BN 为什么能加速训练

BN 的效果通常体现在三个层面。

### 1. 缓解激活分布漂移

标准化后，各层输入大体维持在相近范围，后续层不必反复适应前面层参数变化造成的尺度漂移。

### 2. 改善梯度条件数

当输入尺度过大或过小时，损失面对参数的曲率往往不均衡。BN 相当于做了局部重参数化，让优化地形更平滑，学习率可以开得更大。

### 3. 带来轻微正则化

训练时均值和方差来自随机 mini-batch，不同 batch 会引入噪声，这种噪声常常具有正则化效果。因此很多 CNN 在使用 BN 后，对 Dropout 的依赖反而下降。

---

## 训练态与推理态为何必须区分

训练时，BN 用的是当前 batch 的统计量；推理时，如果还这么做，模型输出将依赖同批次其他样本，既不稳定，也不符合单样本部署要求。

因此推理时使用的是训练过程中累计的运行均值和运行方差：

```text
μ_run <- (1-ρ) μ_run + ρ μ_B
σ_run^2 <- (1-ρ) σ_run^2 + ρ σ_B^2
```

这也是为什么训练和推理模式必须区分。若忘记 `model.eval()`，BN 的行为就会错位，评估结果可能明显波动。

---

## 反向传播为什么会依赖整个 batch

BN 的反向传播不像普通逐元素激活那样只依赖单个样本。因为 `μ_B` 与 `σ_B^2` 是整个 batch 的函数，所以某个样本的梯度会受到同批其他样本影响。

简化后可写成：

```text
∂L/∂x_i
= (γ / sqrt(σ_B^2 + ε))
  [g_i - mean(g) - x̂_i mean(g ⊙ x̂)]
```

其中 `g_i = ∂L/∂y_i`。  
这个式子反映了 BN 的本质：它不是单样本局部变换，而是跨 batch 的耦合操作。

---

## 为什么 batch size 很小时 BN 会退化

当 `m=1` 时，batch 内均值和方差几乎没有统计意义：

- 均值就是样本本身
- 方差接近 0
- 标准化结果高度不稳定，甚至完全丢失有效尺度信息

即便 `m` 不为 1，但如果很小，统计量噪声也会很大，训练表现会显著波动。这就是小 batch 训练中 BN 常常失效的根本原因。

对应的替代方案包括：

- LayerNorm：按特征维做单样本归一化
- GroupNorm：按通道分组做归一化
- RMSNorm：仅用均方根缩放

它们都不依赖 batch 维，因此更适合序列模型和大模型训练。

---

## 为什么 BN 在 Transformer 中不占主流

BN 假设“不同样本同一特征维可以共享统计量”，这在图像批处理中比较自然，但在可变长度序列和自回归建模中并不理想。Transformer 更常见的是 LayerNorm / RMSNorm，原因包括：

- batch size 往往受显存限制较小
- 序列长度和 mask 使跨样本统计不稳定
- 推理时经常需要单条样本低延迟服务

因此，BN 的时代价值主要体现在 CNN 和视觉模型的大发展中，而不是今天的主流 LLM。

---

## 工程使用时最常见的坑

- 忘记区分 `train` / `eval` 模式，导致推理结果抖动
- 小 batch 仍强用 BN，统计量噪声极大
- 误把 BN 当成“万能稳定器”，忽略初始化和学习率本身设置不合理

BN 能显著改善优化，但它从来不是替代基础训练配置的魔法。

---

## 小结

BatchNorm 的真正价值不是“让特征均值为 0、方差为 1”，而是通过批内标准化和可学习重缩放，稳定中间表示的尺度，改善优化条件，并在一定程度上提供正则化。它在 CNN 时代极大推动了深层网络训练，但其对 batch 统计的依赖也决定了它不适合所有场景。

理解 BN 的关键，不在于背公式，而在于看清它为何有效、为何在小 batch 或序列任务上失效，以及为什么后来的 LayerNorm / RMSNorm 会成为 Transformer 的主流选择。
