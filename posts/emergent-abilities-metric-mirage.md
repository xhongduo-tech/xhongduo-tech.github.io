## 核心结论

“涌现能力”在很多论文和讨论里，指的是模型规模增大到某个阶段后，某项能力分数突然跳高，看起来像以前不会、后来突然会了。Schaeffer 等人的核心观点是：这类“突然跳高”很多时候不是模型内部真的发生了结构性跃迁，而是评测函数把原本平滑的进步压缩、放大成了台阶。

最典型的情况是“整题全对才算对”的准确率。它本质上是一个非线性函数。假设模型生成一个长度为 $L$ 的答案，每个 token 都独立地以概率 $p$ 预测正确，那么整题正确率就是：

$$
A = p^L
$$

这个式子的含义很直接：只要其中一个 token 错了，整题就记为错。对白话理解，可以把它看成“连掷 5 个硬币，全朝上才算赢”。如果每枚硬币朝上的概率只是略微提升，最终“全朝上”的胜率会被乘方放大，于是曲线看起来像突然跃升。

一个最小玩具例子如下。设答案长度为 5：

| 每个 token 正确概率 $p$ | 整题准确率 $A=p^5$ | token 级 Brier $(1-p)^2$ |
|---|---:|---:|
| 0.82 | 0.3707 | 0.0324 |
| 0.83 | 0.3939 | 0.0289 |
| 0.84 | 0.4182 | 0.0256 |
| 0.85 | 0.4437 | 0.0225 |

从 0.82 到 0.85，token 正确率只提高了 3 个百分点，但整题准确率从 0.37 跳到 0.44，看起来像“忽然会做了很多题”。而 Brier 分数是连续概率误差指标，白话说就是“预测概率离真实答案还有多远”，它只会平滑变化，不会凭空制造拐点。

因此，结论不是“所有涌现都是假的”，而是“先看度量，再谈涌现”。如果一个现象只在非线性、离散、全有或全无的指标上出现，而在 token 概率、log-loss、Brier 这类连续指标上消失，那么更合理的解释是：能力在持续变好，但评测函数把这种持续进步扭曲成了跃迁外观。

## 问题定义与边界

先把“涌现能力”说清楚。这里讨论的不是哲学意义上的“系统突然出现新性质”，而是一个更具体的工程定义：当模型参数量、训练计算量或数据量逐步增加时，某个 benchmark 指标在一段区间内出现肉眼可见的陡升，于是研究者把它称为 emergent ability。

这个定义里有两个对象必须分开：

1. 模型真实能力
2. 评测指标的映射方式

模型真实能力可以理解为“模型给正确答案分配概率的能力”。评测指标则是把这种能力转换成一个最后分数的函数。如果这个函数是线性的、平滑的，图形通常也会平滑；如果这个函数是阈值式、凸的或分段的，图形就可能被扭曲。

一个常见边界是长度效应。假设模型回答一个 5-token 答案，每个 token 正确概率为 $p$，那么整题正确率是：

$$
A = p^5
$$

如果 $p$ 从 0.82 增长到 0.85：

$$
0.82^5 \approx 0.3707,\quad 0.85^5 \approx 0.4437
$$

整题准确率提升约 7.3 个百分点。可如果直接看 token 级概率，提升只有 3 个百分点。这说明整题准确率的“放大镜效应”已经开始工作。

更一般地，如果模型规模记为 $N$，每题长度记为 $L$，则可写成：

$$
A(N) = p(N)^L
$$

这里的关键不是 $p(N)$ 是否平滑，而是即使 $p(N)$ 很平滑，$A(N)$ 也可能显得很陡。尤其当 $L$ 较大、$p$ 接近 1 时，函数斜率会迅速变大：

$$
\frac{dA}{dp} = Lp^{L-1}
$$

这说明只要 $p$ 稍微提高，$A$ 就会被乘方机制放大。于是观察者看到的不是“能力突然产生”，而是“同一个连续提升过程，被非线性度量重新塑形”。

这里也要明确边界：并不是所有突然变好的现象都能用度量解释。如果某个能力变化同时出现在多种连续指标上，并且伴随明显的行为模式改变、泛化边界改变、内部表征改变，那么仍然可能存在真正的阶段性变化。Schaeffer 这一路工作强调的是：很多被称为 emergent 的现象，至少不能只靠准确率曲线就下结论。

---

## 核心机制与推导

核心机制可以分成三层：底层训练目标、token 级能力、上层评测函数。

第一层是训练目标。语言模型通常最小化交叉熵损失。交叉熵的白话意思是“模型给正确 token 留了多少概率质量”。若记交叉熵为 $L_{\text{CE}}(N)$，模型规模变大时它往往按某种平滑规律下降。于是可以把“平均正确 token 概率”近似写为：

$$
p(N) = \exp\big(-L_{\text{CE}}(N)\big)
$$

这里不必把它理解成严格等式的每个细节，更重要的是方向关系：交叉熵下降，正确 token 的概率上升，而且通常是平滑上升。

第二层是 token 级能力。假设随着参数增加，$p(N)$ 从 0.55、0.60、0.65 一点点提升到 0.90。这种变化本身没有台阶，它只是逐渐增强。若你直接画 $p(N)$，通常不会看到“突然出现”的断点。

第三层是评测函数。假设任务要求整段答案全对才给分，则：

$$
A(N) = p(N)^L
$$

把第一层代入第三层：

$$
A(N) = \exp\big(-L \cdot L_{\text{CE}}(N)\big)
$$

这一步很重要。它说明：即使底层的交叉熵 $L_{\text{CE}}(N)$ 是平滑下降的，经过指数映射以后，最终的准确率曲线也会变陡。长度 $L$ 越大，这种扭曲越明显。

看一个更直观的文本对比。设答案长度 $L=5$：

| $p$ | $A=p^5$ | Brier $(1-p)^2$ | 现象 |
|---|---:|---:|---|
| 0.60 | 0.0778 | 0.1600 | 整题分很低，像“完全不会” |
| 0.70 | 0.1681 | 0.0900 | 整题分翻倍，但 token 只升 0.1 |
| 0.80 | 0.3277 | 0.0400 | 看起来进入“突然会了”的区间 |
| 0.90 | 0.5905 | 0.0100 | 准确率继续急升 |

Brier 分数的定义是：

$$
B = \frac{1}{N_{\text{样本}}}\sum_i\sum_j (y_{ij}-\hat y_{ij})^2
$$

其中 $y_{ij}$ 是真实标签，$\hat y_{ij}$ 是模型给第 $j$ 个选项或 token 的预测概率。白话说，它衡量的是“预测概率和真实结果之间的平方误差”。这个函数对概率变化是连续响应的，不会因为“差一点全对”就直接归零。

所以，同一条底层能力曲线可能在不同度量下呈现两种完全不同的样子：

1. 用整题准确率看，像台阶。
2. 用 Brier、log-loss、correct-choice probability 看，像斜坡。

这就是“度量依赖性”这个概念的核心。它不是说 benchmark 没用，而是说 benchmark 曲线本身不等于能力曲线。中间隔着一个数学映射，而这个映射可能是高度失真的。

真实工程例子是多项选择评测，如 MMLU、C-Eval。若只统计“选项是否选对”，分数是 0 或 1，天然带阈值。模型从“把正确选项概率从 0.35 提升到 0.48”到“从 0.48 提升到 0.52”，在概率空间里可能是连续的，但只要跨过 argmax 或判定阈值，准确率就会突然跳一下。2024 年的 follow-up 工作用 CorrectChoiceProb 和 Brier 分数重画这些任务的曲线后，发现许多原本看起来像跳变的地方变得平滑了，这说明“突然会做题”未必是内部机制突变，更可能是离散评分在作图时放大了边界穿越。

## 代码实现

下面用一个最小可运行脚本复现这个现象。脚本做三件事：

1. 设定一组 token 正确概率 `p_values`
2. 计算整题准确率 `A = p**L`
3. 计算一个简化的 token 级 Brier `brier = (1-p)**2`

```python
import math

def sequence_accuracy(p: float, L: int) -> float:
    return p ** L

def token_brier_for_correct_class(p: float) -> float:
    # 假设真实类别为 1，模型给正确类别的概率是 p
    return (1 - p) ** 2

# 玩具例子：5-token 答案
L = 5
p_values = [0.82, 0.83, 0.84, 0.85]

acc_values = [sequence_accuracy(p, L) for p in p_values]
brier_values = [token_brier_for_correct_class(p) for p in p_values]

# 基本断言，保证趋势符合预期
assert acc_values[0] < acc_values[-1]
assert brier_values[0] > brier_values[-1]
assert round(acc_values[0], 4) == 0.3707
assert round(acc_values[-1], 4) == 0.4437
assert round(brier_values[0], 4) == 0.0324
assert round(brier_values[-1], 4) == 0.0225

for p, acc, brier in zip(p_values, acc_values, brier_values):
    print(f"p={p:.2f}, seq_acc={acc:.4f}, brier={brier:.4f}")
```

如果要把曲线画出来，可以继续加上 `matplotlib`：

```python
import numpy as np
import matplotlib.pyplot as plt

L = 5
p_values = np.linspace(0.5, 0.99, 200)
A = p_values ** L
brier = (1 - p_values) ** 2

assert A[0] < A[-1]
assert brier[0] > brier[-1]

plt.figure(figsize=(8, 5))
plt.plot(p_values, A, label=f"Sequence Accuracy: p^{L}")
plt.plot(p_values, brier, label="Token-level Brier: (1-p)^2")
plt.xlabel("Token correct probability p")
plt.ylabel("Metric value")
plt.title("Same capability, different metric shape")
plt.legend()
plt.grid(True, alpha=0.3)
plt.tight_layout()
plt.show()
```

这段图会显示两种完全不同的响应：

- `p^L` 在 $p$ 接近 1 时明显变陡
- `(1-p)^2` 始终连续下降，没有台阶感

如果你要把它扩展到真实评测管线，可以并行记录多个指标，而不是只记一个准确率：

```python
def evaluate_example(correct_prob: float, seq_len: int):
    metrics = {
        "correct_choice_prob": correct_prob,
        "sequence_accuracy_est": correct_prob ** seq_len,
        "brier": (1 - correct_prob) ** 2,
        "log_loss": -math.log(max(correct_prob, 1e-12)),
    }
    return metrics

m = evaluate_example(correct_prob=0.84, seq_len=5)
assert "brier" in m and "log_loss" in m
assert m["sequence_accuracy_est"] > 0
print(m)
```

工程上建议至少记录以下三条曲线：

| 指标 | 含义 | 是否容易制造“跳变感” |
|---|---|---|
| Accuracy | 只看最终对错 | 高 |
| CorrectChoiceProb | 直接看正确答案概率 | 低 |
| Brier / Log-loss | 看概率误差 | 低 |

这样做的目的不是替换准确率，而是避免把“概率连续改善”误读成“能力突然出现”。

## 工程权衡与常见坑

第一个常见坑是把准确率当成唯一真相。准确率易解释、易比较、易汇报，所以团队很容易只盯着它。但准确率本质上丢掉了大量信息。一个模型把正确选项概率从 0.26 提高到 0.49，另一个从 0.51 提高到 0.74，前者在准确率上可能完全不变，后者却可能突然跨阈值。这会让你误以为只有后者“学到了东西”。

第二个坑是小样本和离散分辨率。若测试集很小，比如只有 100 题，那么准确率最小步长就是 1%。这意味着低分模型之间的真实差距很可能被量化噪声淹没。等模型好到某个阶段，连续优势终于足以跨过若干题目的阈值，于是图上出现“跳高”。这不一定是能力突变，可能只是统计分辨率不够。

第三个坑是把采样策略变化误判为能力涌现。温度、top-k、top-p 这些解码超参会改变最终字符串是否恰好匹配，尤其是在生成任务里更明显。如果一个结论只在 greedy decoding 下成立，或者只在 exact match 上成立，就不能直接上升为“模型突然获得了某种抽象能力”。

真实工程例子可以看多选 benchmark。很多团队在看 MMLU 时，会发现在 10B 到 100B 参数区间，准确率像是从一个平台突然跳到另一个平台。但如果同时记录正确选项概率或 Brier 分数，经常会发现曲线本身是平滑上升的。也就是说，模型一直在变好，只是准确率这种二值指标把“跨过决策边界”的时刻放大了。

下面这张表可以作为工程检查表：

| 指标 | 是否凸/有阶跃 | 对参数变化的敏感度 | 信息保留程度 | 风险 |
|---|---|---|---|---|
| Exact Match / Accuracy | 高 | 在阈值附近异常敏感 | 低 | 容易误判涌现 |
| Sequence-level pass@1 | 高 | 对长度和采样敏感 | 低 | 受随机性影响大 |
| CorrectChoiceProb | 低 | 平滑 | 中 | 需能取到概率 |
| Brier Score | 低 | 平滑 | 高 | 解释成本高于准确率 |
| Log-loss | 低 | 平滑 | 高 | 数值不直观 |

实际操作上，至少做三件事：

1. 同时画准确率和概率型指标。
2. 扩大测试集，提升低分区间分辨率。
3. 区分“模型概率上升”与“最终字符串碰巧过线”。

否则你可能会在错误方向上做架构决策，比如以为某种层数、注意力变体或训练技巧触发了“新能力”，实际上只是分数显示方式变了。

## 替代方案与适用边界

准确率不是不能用，而是不能单独用。它适合回答一个很具体的问题：在固定决策规则下，最终能不能交付一个对的结果。对于线上系统、考试题、多选分类器，这个问题仍然重要。

什么时候准确率适合继续保留？当任务本身就是离散交付，且错误容忍度极低时。例如：

- 法规分类：最终标签必须对
- 医疗编码：输出编码不能错
- 高置信多选题：系统就是只接受一个选项

这时准确率是必要指标，因为它对应业务目标。

什么时候更应该换成概率型度量？当你在研究“能力如何随规模变化”时，或者当你想比较小模型阶段的细微进步时。这时更适合引入：

| 度量 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|
| Accuracy | 直观，和业务决策一致 | 丢信息，易出现阶跃 | 线上最终交付 |
| Brier Score | 连续、稳定、可解释为概率误差 | 需要概率输出 | 多选题、分类任务 |
| Log-loss | 对概率质量最敏感 | 对非技术读者不直观 | 训练分析、标定评估 |
| Token Edit Distance | 能反映“差一点答对” | 任务相关性强 | 文本生成、结构化输出 |
| CorrectChoiceProb | 直观体现对正确答案的信心 | 不直接等于最终可用性 | Benchmark 分析 |

一个典型例子是少样本多选题。假设测试集只有 50 道题。准确率每跳一次就是 2%，很容易让曲线看起来断断续续。但 Brier 分数会对每道题的概率分配都计分，即使模型还没把正确选项推到第一名，只要概率更接近真实答案，分数就会稳定改善。

反过来，如果你的标注只有“对/错”，没有完整选项概率，也拿不到 token logits，那么准确率仍然有价值。这就是适用边界：你无法要求所有场景都使用最细粒度的指标，但在能拿到概率信息的地方，不应该主动放弃它。

工程上更稳妥的做法不是选一个“最好指标”，而是并行监控：

```text
for 每个模型规模 N:
    记录 accuracy
    记录 correct_choice_prob
    记录 brier
    记录 log_loss
比较：
    若只有 accuracy 跳变，而其他连续指标平滑
        优先怀疑度量 artifact
    若多个连续指标也同步改变形态
        再考虑真实机制变化
```

这套方法的价值在于，它把“是否涌现”从单条曲线的主观观感，变成多个指标之间的交叉验证问题。这样更符合工程判断，也更接近科学分析。
