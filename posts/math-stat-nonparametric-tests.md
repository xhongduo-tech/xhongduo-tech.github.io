## 核心结论

非参数检验是在**不强依赖总体分布形状**的前提下做统计推断的方法。这里的“分布形状”可以先理解为数据整体长什么样，例如是否接近正态、是否左右对称、尾部是否很长。它的关键不是“没有参数”，而是把统计量建立在**秩**、**顺序统计量**或**重抽样分布**上。

最常见的直觉是：参数检验直接比较原始数值，例如均值差；非参数检验先把数据按大小排序，再比较排序后的位置差异。这样做的直接结果是，极端大值或极端小值对结论的影响会被压缩，因此在小样本、偏态分布、异常值较多、量表只有等级含义时更稳健。这里的“稳健”可以先理解为：数据不太理想时，结论不容易被少数异常点带偏。

但稳健不等于永远更好。如果总体近似正态、方差结构合理、样本量也足够，参数检验通常有更高功效。这里的“功效”是指：当差异真的存在时，检验把它检出来的能力。非参数检验经常是在“少做假设”和“提高检出率”之间做交换。

下面这张表先把参数检验与非参数检验的差异压缩成一个工程判断框架。

| 维度 | 参数检验 | 非参数检验 |
|---|---|---|
| 主要依赖 | 均值、方差等参数 | 秩、顺序、重抽样分布 |
| 分布假设 | 往往要求较强，如正态性 | 要求较弱，不强依赖具体分布 |
| 适用数据 | 连续型、分布较稳定 | 等级数据、偏态数据、小样本 |
| 对异常值敏感性 | 较高 | 相对较低 |
| 假设成立时功效 | 通常更高 | 通常略低 |
| 典型方法 | t 检验、ANOVA | Mann-Whitney、Wilcoxon、Log-rank |

以双样本秩和检验为例，常用统计量可以写成

$$
W=\sum_{j \in \text{group 1}} r_j
$$

其中 $r_j$ 是第 $j$ 个样本在所有样本混合排序后的秩。这个公式表达的不是“值有多大”，而是“它在整体里排第几”。

玩具例子：两个班各 6 个语文成绩，一个班有一位同学考得极低，均值会被明显拉低；如果改用秩和检验，先把 12 个分数统一排序，再看某个班拿到了多少高秩，异常值的破坏性会小很多。因为无论这个极低分是 10 分还是 0 分，在排序里它都只会占据“很靠后”的位置，不会无限放大对统计量的影响。

## 问题定义与边界

非参数检验要解决的问题是：当我们**不愿意或不能合理假设数据来自某个具体分布**时，如何仍然做出“这两组是否不同”“某个中位数是否变化”“两条生存曲线是否有差异”这样的推断。

这里先定义几个边界概念。

- 秩：把样本按从小到大排序后得到的位置编号。如果最小值排第 1，第二小排第 2，这些编号就是秩。
- 顺序统计量：排序后的样本值，比如最小值、最大值、中位数，本质上都是“按顺序挑出来的值”。
- 重抽样：反复随机重排或重复抽样，用模拟方式构造零假设下统计量的分布。
- 零假设：默认认为“没有系统差异”的假设，检验就是看观测结果是否偏离这个默认状态。
- 检验统计量：把样本压缩成一个数，用于衡量数据与零假设的偏离程度。
- $p$ 值：在零假设成立时，观测到当前结果或更极端结果的概率。

给定观测值 $x_1,x_2,\dots,x_n$，把它们排序后可得到各自的秩 $r_i$。非参数检验往往使用 $r_i$ 而不是 $x_i$ 本身。

$$
r_i=\operatorname{rank}(x_i)
$$

这类方法尤其适合以下场景：

| 适用条件 | 原因 |
|---|---|
| 样本量较小 | 小样本下很难可靠验证正态性 |
| 数据明显偏态 | 均值和方差不再稳定代表整体 |
| 存在异常值 | 排序比直接用原值更稳健 |
| 量表是等级型 | 等级之间不一定有可解释的距离 |
| 右删失生存数据 | 时间分布复杂，常不适合直接参数建模 |

也有不适用或不优先使用的场景：

| 不优先场景 | 原因 |
|---|---|
| 数据近似正态且无明显异常值 | 参数检验通常功效更高 |
| 研究目标是估计均值差及其参数解释 | 参数模型更直接 |
| 样本很大且中心极限定理已提供良好近似 | 参数方法往往更高效 |
| 已知合理分布模型 | 没必要放弃可用的结构信息 |

新手常见误区是把“非参数”理解成“完全没有假设”。这不准确。它只是**减少对分布形态的假设**，不是完全无条件。比如 Mann-Whitney 检验仍然默认样本独立；Wilcoxon 配对符号秩检验仍然要求配对差值至少能做可比较的排序；Log-rank 检验默认删失机制不能系统性偏向某一组，否则结论会失真。

可以把这些方法理解成“少假设，不是零假设”。常见前提如下：

| 方法 | 至少需要满足的前提 |
|---|---|
| Mann-Whitney | 两组独立，观测值至少可排序 |
| Wilcoxon 符号秩 | 配对样本，差值独立，差值可排序 |
| Sign Test | 只关心正负方向，差值独立 |
| Log-rank | 两组独立，删失大致非信息性 |

玩具例子：比较两种止痛药效果，指标是 1 到 5 的痛感等级。这里“4 比 2 大”有意义，但“4 比 2 恰好多 2 个单位”未必有严格物理含义，所以直接做均值 t 检验就不如基于秩的比较自然。

再看一个边界更清晰的反例：如果你的目标是估计“平均响应时间是否减少了 20ms”，那么均值本身就是问题核心，参数方法或 bootstrap 往往更贴题。非参数检验可以告诉你“整体位置是否移动”，但不直接回答“平均减少了多少”。

## 核心机制与推导

先看配对数据中的 Wilcoxon 符号秩检验。它解决的问题是：同一对象在处理前后是否发生系统性变化。比如同一批患者治疗前后的疼痛评分，或同一服务上线前后的请求延迟。

设配对差值为

$$
D_i = X_i - Y_i
$$

其中 $X_i$ 和 $Y_i$ 是同一对象在两个条件下的观测值。零假设通常写作“差值的中心在 0 附近”，直观上就是“处理前后没有系统性变化”。

计算过程分四步。

1. 去掉 $D_i=0$ 的样本，因为它们既不支持正方向也不支持负方向。
2. 对绝对值 $|D_i|$ 排序并赋秩，得到 $r_i=\operatorname{rank}(|D_i|)$。
3. 把正差对应的秩加总，得到正秩和 $T^+$；把负差对应的秩加总，得到负秩和 $T^-$。
4. 取较小值作为检验统计量：

$$
T^+=\sum_{D_i>0} r_i,\qquad
T^-=\sum_{D_i<0} r_i,\qquad
T=\min(T^+,T^-)
$$

为什么这样构造有效？因为在零假设成立时，差值的正负号应当是“随机的”，而绝对值大小已经固定。也就是说，秩本身先确定，再随机给它们分配正负号，就能得到零假设下 $T^+$ 或 $T$ 的分布。小样本时可以精确枚举，大样本时可用近似分布。

更具体一点，若去零后还剩 $n$ 个差值，那么每个秩都有两种符号选择，理论上共有

$$
2^n
$$

种符号分配方式。对每一种分配方式都计算一次 $T^+$ 或 $T$，就能得到精确零分布。这就是“为什么 Wilcoxon 可以做精确检验”的根本原因。

下面用一个玩具例子说明。

有 8 个患者，相对于某基准值的差值为：

$$
[-0.3, 0.8, -1.9, -0.1, 0.3, -2.3, 1.6, -1.7]
$$

先看绝对值：

$$
[0.3, 0.8, 1.9, 0.1, 0.3, 2.3, 1.6, 1.7]
$$

按从小到大排序后，对应秩为：

| 绝对差值 | 秩 |
|---|---|
| 0.1 | 1 |
| 0.3 | 2.5 |
| 0.3 | 2.5 |
| 0.8 | 4 |
| 1.6 | 5 |
| 1.7 | 6 |
| 1.9 | 7 |
| 2.3 | 8 |

注意这里两个 `0.3` 并列，因此不能一个给 2、一个给 3，而是都给平均秩 $(2+3)/2=2.5$。

把秩放回原始顺序：

| 差值 $D_i$ | 绝对值 $|D_i|$ | 秩 $r_i$ | 符号 |
|---|---:|---:|---|
| -0.3 | 0.3 | 2.5 | 负 |
| 0.8 | 0.8 | 4 | 正 |
| -1.9 | 1.9 | 7 | 负 |
| -0.1 | 0.1 | 1 | 负 |
| 0.3 | 0.3 | 2.5 | 正 |
| -2.3 | 2.3 | 8 | 负 |
| 1.6 | 1.6 | 5 | 正 |
| -1.7 | 1.7 | 6 | 负 |

于是可算出

$$
T^+=4+2.5+5=11.5,\qquad
T^-=2.5+7+1+8+6=24.5,\qquad
T=11.5
$$

如果 $T$ 很小，说明“较小的一侧秩和异常小”，也就是大部分较大的秩集中到了另一侧，系统性偏移就更明显。

这件事可以从组合角度理解：

| 步骤 | 固定的对象 | 随机变化的对象 | 作用 |
|---|---|---|---|
| 计算差值 | 原始配对样本 | 无 | 提取方向与幅度 |
| 绝对值排序 | $|D_i|$ | 无 | 去掉量纲，保留大小顺序 |
| 赋正负号 | 秩 $r_i$ | 符号 | 在零假设下构造样本空间 |
| 汇总秩和 | $r_i$ 与符号 | 统计量 $T^+,T^-$ | 比较偏离程度 |

双样本独立样本时，典型方法是 Mann-Whitney 秩和检验。把两组合并排序，如果两组来自同一分布，那么某一组拿到高秩和低秩的机会应大致对称；如果一组整体更大，它会拿到更多高秩，于是

$$
W=\sum_{j \in \text{group 1}} r_j
$$

就会明显偏离零假设下的期望。设第 1 组样本量为 $n_1$，第 2 组样本量为 $n_2$，则在无并列且零假设成立时，

$$
\mathbb{E}[W]=\frac{n_1(n_1+n_2+1)}{2}
$$

这给了我们一个很直接的判断基准：如果观测到的秩和远高于或远低于这个期望，就说明两组样本在排序中的位置分布不再对称。

真实工程例子：生物医学中的 Log-rank 检验用于比较两组生存曲线。生存数据常有**右删失**，就是有些对象在观察结束前还没发生事件，只知道“至少活到某个时间”。这时直接比较均值既不稳也不完整。Log-rank 不要求明确指定生存时间分布，而是在每个事件时刻比较“观察到的事件数”和“按风险集应期待的事件数”，本质上也是一种基于顺序和计数的非参数思想。若在事件时刻 $t_k$，风险集中两组人数分别为 $n_{1k},n_{2k}$，总事件数为 $d_k$，那么第 1 组期望事件数是

$$
E_{1k}=d_k\frac{n_{1k}}{n_{1k}+n_{2k}}
$$

Log-rank 的核心就是把各时刻的“观察到的事件数减去期望事件数”累加起来，再看这种偏离是否超出随机波动范围。

## 代码实现

下面给出一个可运行的 Python 例子，手写 Wilcoxon 符号秩检验的核心逻辑，并用穷举符号分配计算精确双侧 $p$ 值。代码只依赖标准库。

```python
from itertools import product
from math import isclose


def average_ranks(values):
    indexed = sorted(enumerate(values), key=lambda x: x[1])
    ranks = [0.0] * len(values)
    i = 0
    while i < len(indexed):
        j = i
        while j < len(indexed) and indexed[j][1] == indexed[i][1]:
            j += 1
        avg_rank = (i + 1 + j) / 2.0
        for k in range(i, j):
            original_index = indexed[k][0]
            ranks[original_index] = avg_rank
        i = j
    return ranks


def wilcoxon_signed_rank(diff):
    nonzero = [d for d in diff if d != 0]
    if not nonzero:
        raise ValueError("all paired differences are zero")

    abs_vals = [abs(d) for d in nonzero]
    ranks = average_ranks(abs_vals)

    t_pos = sum(r for d, r in zip(nonzero, ranks) if d > 0)
    total_rank = sum(ranks)
    t_neg = total_rank - t_pos
    t_obs = min(t_pos, t_neg)

    all_t = []
    for signs in product([-1, 1], repeat=len(ranks)):
        s_pos = sum(r for s, r in zip(signs, ranks) if s > 0)
        s_neg = total_rank - s_pos
        all_t.append(min(s_pos, s_neg))

    p_value = sum(1 for t in all_t if t <= t_obs) / len(all_t)

    return {
        "n": len(nonzero),
        "ranks": ranks,
        "T_plus": t_pos,
        "T_minus": t_neg,
        "T": t_obs,
        "p_value_exact_two_sided": p_value,
    }


if __name__ == "__main__":
    diff = [-0.3, 0.8, -1.9, -0.1, 0.3, -2.3, 1.6, -1.7]
    result = wilcoxon_signed_rank(diff)

    assert result["n"] == 8
    assert isclose(result["T_plus"], 11.5)
    assert isclose(result["T_minus"], 24.5)
    assert isclose(result["T"], 11.5)
    assert isclose(sum(result["ranks"]), 36.0)

    print(result)
```

这段代码对应的机制关系很直接：

| 代码步骤 | 统计含义 |
|---|---|
| `nonzero = [d for d in diff if d != 0]` | 去掉零差 |
| `average_ranks(abs_vals)` | 对绝对差赋秩，并处理并列值 |
| `t_pos` / `t_neg` | 计算正秩和与负秩和 |
| `product([-1, 1], repeat=len(ranks))` | 在零假设下枚举所有符号分配 |
| `p_value` | 计算精确双侧 $p$ 值 |

如果运行这段代码，输出会是一个字典，包含样本数、各差值对应的秩、正秩和、负秩和以及精确 $p$ 值。这里故意没有依赖 `scipy`，原因是想把统计机制完整暴露出来：你可以直接看到“先赋秩，再枚举符号”这一过程，而不是只调用黑盒函数。

有两个实现细节值得新手特别注意。

| 细节 | 为什么重要 |
|---|---|
| 使用平均秩而不是整数秩硬分配 | 否则并列值会被错误打散 |
| 精确枚举只适合样本较小 | 因为复杂度是 $O(2^n)$，$n$ 稍大就会迅速变慢 |

因此在真实项目里，常见做法是：

- 小样本：用精确检验或置换检验。
- 中大样本：用正态近似，并做并列修正。
- 需要工业级稳定性：直接调用成熟统计库，但先理解库函数到底在检验什么。

真实工程例子：假设你在做接口优化评估，对同一批请求记录优化前后 P95 延迟。延迟数据通常偏态明显，少量超时请求会把均值拉得很高。此时若直接对均值差做 t 检验，解释会受重尾分布影响；改成对“每个请求的前后延迟差”做 Wilcoxon，更接近“整体是否普遍变快”的问题。若样本不大，比如只抽到几十个代表请求，精确或置换版本会比正态近似更可靠。

## 工程权衡与常见坑

非参数检验常见，但实际落地时容易踩四类坑。

| 常见误区 | 影响 | 规避方式 |
|---|---|---|
| 小样本直接套正态近似 | $p$ 值偏差明显 | $n$ 很小时优先精确检验或置换 |
| 忽略并列秩 | 统计量计算错误 | 并列值使用平均秩 |
| 把零差也纳入排序 | 稀释有效信号 | Wilcoxon 中先剔除零差 |
| 看到非正态就机械改用非参数 | 可能损失功效 | 先判断数据类型、样本量和研究目标 |

并列秩的处理尤其容易被忽略。若多个值相同，它们不能随便占不同整数秩，而应使用平均秩。设某个并列组本来占据第 $r_1,r_2,\dots,r_k$ 个位置，则其平均秩为

$$
\bar r=\frac{r_1+r_2+\dots+r_k}{k}
$$

例如 6 个配对样本的绝对差是 `[1, 2, 2, 4, 7, 7]`，那么两个 `2` 不能一个给 2、一个给 3 然后随意分配，而应都取 $(2+3)/2=2.5$；两个 `7` 都取 $(5+6)/2=5.5$。不这样做会扭曲秩和。

另一个关键权衡是“稳健性”换“功效”。如果数据本来近似正态、没有异常值，t 检验利用了更多数值结构，因此通常更容易发现真实差异。非参数方法把原始值压缩成顺序信息，相当于主动丢掉了一部分幅度信息，这就是它稳健但未必最敏感的原因。

还有一个解释层面的坑：Mann-Whitney 检验经常被误说成“检验均值差”。它更准确的解释是比较两组分布位置是否系统偏移，或者说随机抽一条 A 组样本比随机抽一条 B 组样本更大的概率是否偏离 0.5。若记

$$
\theta=P(X>Y)+\frac{1}{2}P(X=Y)
$$

那么 Mann-Whitney 实际上常可理解为在检验 $\theta$ 是否等于 $0.5$。这和“均值是否相等”不是同一个问题。

工程上还常见两个隐藏坑。

| 隐藏坑 | 表现 | 说明 |
|---|---|---|
| 把独立样本当成配对样本 | 检验结果失真 | 配对检验要求样本一一对应 |
| 多次试验只汇报显著结果 | 假阳性增多 | 需要做多重比较修正 |

新手可以用一句话记忆：**先确认数据结构，再选统计方法；不要先选方法，再硬解释数据。**

## 替代方案与适用边界

非参数检验不是默认最优解，而是统计工具箱中的一类稳健方案。实际工程里常见的替代路线有三类。

| 方法 | 适用条件 | 优点 | 缺点 |
|---|---|---|---|
| t 检验 / ANOVA | 近似正态、方差结构合理 | 功效高、解释直接 | 对异常值和偏态敏感 |
| 非参数秩检验 | 等级数据、偏态、小样本 | 稳健、实现成熟 | 可能损失功效 |
| Bootstrap | 关注估计量标准误或区间 | 灵活、可估计不易解析的量 | 计算成本较高，样本代表性要求更强 |
| Permutation Test | 零假设下标签可交换 | 解释清晰、精确性好 | 依赖可交换性，计算可能较重 |

参数检验的典型统计量例如配对 t 检验：

$$
t=\frac{\bar D}{s_D/\sqrt{n}}
$$

它直接使用均值 $\bar D$ 和标准差 $s_D$。而 Wilcoxon 更关注秩和统计量 $T$。前者使用“差值有多大”，后者使用“差值排第几”。如果数据分布接近理想条件，前者更有效；如果分布明显偏、尾部重、异常点多，后者通常更安全。

Bootstrap 和 permutation test 与非参数检验的关系，也值得区分清楚。

| 方法 | 核心思路 | 回答的问题 |
|---|---|---|
| 秩检验 | 用排序信息构造统计量 | 两组位置是否系统不同 |
| Bootstrap | 从样本反复重抽样 | 估计量的不确定性有多大 |
| Permutation | 在零假设下重排标签 | 当前差异是否可能由随机标签产生 |

一个实际决策流程可以写成：

1. 先看数据类型，是连续型、等级型还是删失时间。
2. 再看样本量和图形分布，是否明显偏态或重尾。
3. 若参数假设大致合理，优先用参数检验。
4. 若假设不稳或量表本身只支持排序，改用非参数检验。
5. 若需要补充稳健性证据，可同时报告 bootstrap 区间或 permutation $p$ 值。

新手版例子：某指标看起来近似正态，但有轻度偏态。你可以先做配对 t 检验，再做 Wilcoxon 作为稳健性对照；如果两者结论一致，可信度更高。如果不一致，就要回到样本分布、异常值和业务问题本身，而不是只挑一个自己喜欢的结果。

真实工程例子：在线实验中比较两版推荐系统的人均停留时长。该指标往往高度偏态，少量超长会话影响均值。此时可以同时报告三组结果：t 检验比较均值、Mann-Whitney 比较整体位置偏移、bootstrap 给出均值差区间。三者共同使用，比单独一个 $p$ 值更接近工程决策需要。

最后给一个简单的选型表，适合做第一轮判断。

| 你的问题 | 更自然的方法 |
|---|---|
| 我关心平均值变化了多少 | t 检验 / bootstrap |
| 我关心整体是否普遍变大或变小 | Wilcoxon / Mann-Whitney |
| 我的数据只有等级顺序 | 非参数秩检验 |
| 我有删失时间数据 | Log-rank / 生存分析方法 |
| 我不信解析分布近似 | Permutation / Bootstrap |
