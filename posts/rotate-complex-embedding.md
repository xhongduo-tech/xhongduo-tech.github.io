## 核心结论

RotatE 的核心想法很直接：把实体和关系都放到复数空间里，关系不是“加一个位移”，而是“做一次旋转”。复数空间可以理解为“每一维都带方向角度的向量空间”；旋转就是“只改方向，不改长度”。因此对一个三元组 $(h,r,t)$，RotatE 希望满足

$$
t \approx h \circ r,\quad |r_i| = 1
$$

其中 $\circ$ 是 Hadamard 乘积，也就是“向量逐维相乘”。对应的评分函数通常写成：

$$
f_r(h,t) = -\|h \circ r - t\|_1
$$

分数越大，说明把头实体 $h$ 经过关系 $r$ 旋转后，越接近尾实体 $t$。

它的价值不在“公式新奇”，而在“几何约束刚好对应知识图谱里的常见关系模式”。对称、反对称、互逆、组合这四类模式，在 RotatE 里都能用旋转角自然表达，而不需要额外手工设计规则。原始论文报告中，RotatE 配合自对抗负采样后，在 FB15k-237 上的 MRR 约为 0.338，明显高于 TransE 的约 0.223，差距大约 11 个百分点。

一个新手能立刻看懂的玩具例子是：设 $h=1+0i$，关系 $r=i=e^{i\pi/2}$，表示逆时针旋转 $90^\circ$。那么

$$
h \circ r = (1+0i)\cdot i = i
$$

如果真实尾实体 $t=i$，则得分最高；如果错误尾实体还是 $1+0i$，距离就更大，得分更低。RotatE 学到的就是“这个关系应该把头实体旋到哪个角度”。

## 问题定义与边界

知识图谱补全的目标，是在已知大量三元组 $(头实体, 关系, 尾实体)$ 的前提下，预测缺失边。实体就是“图里的节点”，比如“巴黎”“法国”；关系就是“节点之间的有向边”，比如“首都属于”。训练后，模型要能回答 $(巴黎, 是首都, ?)$ 或 $(?, 首都是, 巴黎)$ 这类问题。

RotatE 解决的是“关系模式建模”问题，而不是“全能知识推理”问题。它的边界有三条：

| 边界 | RotatE 的处理方式 | 不负责的部分 |
|---|---|---|
| 输入信息 | 只用知识图谱中的三元组结构 | 不直接用文本、图像、用户行为日志 |
| 表示空间 | 实体和关系都映射到 $\mathbb{C}^k$ | 不显式建模规则库或符号逻辑证明 |
| 关系变换 | 关系只做逐维旋转，通常要求 $|r_i|=1$ | 不直接表示任意缩放、剪切或更复杂仿射变换 |

为什么这个边界重要？因为它说明 RotatE 的优势来自“结构归纳偏置”。归纳偏置就是“模型预先偏向某类结构”，这里的偏向就是旋转。

四种最关键的关系模式如下：

| 关系模式 | 旋转条件 | 直观含义 |
|---|---|---|
| Symmetry 对称 | $r \circ r = 1$，常见为角度 $0$ 或 $\pi$ | $A \to B$ 成立时，$B \to A$ 也成立 |
| Anti-symmetry 反对称 | 不能同时满足双向同真 | “父亲”“上级”这类关系有方向 |
| Inverse 互逆 | $r_2 = \bar r_1 = r_1^{-1}$ | 一个关系的反向关系是角度取反 |
| Composition 组合 | $r_3 = r_1 \circ r_2$ | 连续两步关系可以合成为一步 |

例如“(巴黎, 是首都, 法国)”和“(法国, 首都是, 巴黎)”就是互逆关系。前者对应一个旋转角，后者对应相反角度。RotatE 并不保证世界知识一定正确，它只是提供一种特别适合这类模式的参数化方式。

## 核心机制与推导

RotatE 的机制可以拆成三步。

第一步，表示。每个实体 $e$ 是一个复向量 $e \in \mathbb{C}^k$，每个关系 $r$ 也是复向量，但通常要求每一维模长为 1，即

$$
r_i = e^{i\theta_i} = \cos\theta_i + i\sin\theta_i
$$

这意味着关系只改变方向角 $\theta_i$，不改变长度。

第二步，打分。对三元组 $(h,r,t)$，先把头实体逐维旋转：

$$
h' = h \circ r
$$

再看它和尾实体 $t$ 有多接近。RotatE 通常用 $L_1$ 距离，也就是“各维误差绝对值之和”：

$$
d_r(h,t) = \|h \circ r - t\|_1
$$

距离越小越好，所以评分写成负号形式：

$$
f_r(h,t) = -d_r(h,t)
$$

第三步，训练。正样本要分高，负样本要分低。论文采用的一个常见写法是 margin-sigmoid 风格目标：

$$
\mathcal{L}=
-\log \sigma(\gamma-d_r(h,t))
-\sum_i p_i \log \sigma(d_r(h_i',t_i')-\gamma)
$$

其中 $\gamma$ 是 margin，可以理解为“正负样本要拉开的安全间隔”。关键在 $p_i$，它不是平均分配，而是自对抗负采样：

$$
p_i = \frac{\exp(\alpha f_r(h_i',t_i'))}{\sum_j \exp(\alpha f_r(h_j',t_j'))}
$$

这里的 $\alpha$ 是温度系数，可以理解为“强调困难负样本的力度”。如果某个负样本当前得分很高，说明模型差点把它当真，它就会得到更大权重。

这正是 RotatE 在工程上比“随机负采样”更有效的原因。均匀负采样常常抽到明显错误的负例，比如把“法国”的尾实体换成“电饭锅”，这种样本太容易，几乎不给模型提供有效梯度。自对抗负采样则更关注“看起来像对的错样本”。

## 代码实现

下面给出一个最小可运行的 Python 版本，只演示 RotatE 的核心计算，不依赖深度学习框架。代码里用二维实数数组分别存复数的实部和虚部，便于看清公式。

```python
import math

def rotate(h, phase):
    # h: [(re, im), ...]
    # phase: [theta1, theta2, ...]
    out = []
    for (re, im), theta in zip(h, phase):
        rr = math.cos(theta)
        ri = math.sin(theta)  # |r_i| = 1
        # (re + i im) * (rr + i ri)
        out.append((re * rr - im * ri, re * ri + im * rr))
    return out

def l1_distance(x, y):
    total = 0.0
    for (xr, xi), (yr, yi) in zip(x, y):
        total += abs(xr - yr) + abs(xi - yi)
    return total

def score(h, phase, t):
    return -l1_distance(rotate(h, phase), t)

def softmax(xs, alpha=1.0):
    vals = [math.exp(alpha * x) for x in xs]
    s = sum(vals)
    return [v / s for v in vals]

# 玩具例子：1 旋转 90 度得到 i
h = [(1.0, 0.0)]
phase = [math.pi / 2]
t_pos = [(0.0, 1.0)]
t_neg = [(1.0, 0.0)]

pos_score = score(h, phase, t_pos)
neg_score = score(h, phase, t_neg)

assert abs(pos_score - 0.0) < 1e-9
assert pos_score > neg_score

# 自对抗负采样：更难的负样本应该权重更高
neg_scores = [-0.2, -1.5, -3.0]
weights = softmax(neg_scores, alpha=2.0)

assert abs(sum(weights) - 1.0) < 1e-9
assert weights[0] > weights[1] > weights[2]

print("pos_score =", pos_score)
print("neg_score =", neg_score)
print("weights =", weights)
```

如果换成 PyTorch，工程骨架通常是：

1. 实体参数直接学复向量，或者学实部和虚部。
2. 关系参数最好学相角 $\theta$，再映射成 $\cos\theta + i\sin\theta$，这样天然满足 $|r_i|=1$。
3. 每个 batch 先算正样本分数，再构造一组负样本。
4. 用当前负样本分数做 softmax 权重，形成自对抗负采样损失。

真实工程例子可以看铁路设备故障诊断知识图谱。一个故障事件可能同时连到“故障模式”“原因”“维修动作”“影响部件”。这里“原因导致故障”和“故障由原因引起”是互逆关系，“部件异常 + 工况异常 -> 故障模式”又常带组合关系。RotatE 的旋转建模正好适合这类图结构推理，而不是只做简单近邻匹配。

## 工程权衡与常见坑

最常见的坑，是把关系当成任意复向量学，忘了约束 $|r_i|=1$。一旦允许缩放，关系就不再是纯旋转，而变成“旋转加拉伸”。这样虽然参数更自由，但原本用于表达对称和互逆的几何性质会被破坏。例如一个互逆关系理论上应满足 $r^{-1}=\bar r$，如果模长乱掉，来回走一遍就回不到原位置。

第二个坑，是负采样太随意。均匀采样容易抽到“显然错”的实体，训练看起来稳定，实际上学得很慢。RotatE 论文里自对抗负采样带来的收益非常明确，不只是训练技巧，而是性能核心组成部分。

第三个坑，是评估时没有做 filtered setting。filtered evaluation 就是“评估某个候选尾实体时，把训练集中本来就为真的其他答案过滤掉”。如果不这么做，模型可能把一个真实但非当前标注的实体排前面，却被错误算成失误。

下面是工程上常见取舍：

| 设计选择 | 做法 | 好处 | 风险 |
|---|---|---|---|
| 关系模长约束 | 用相角参数化 | 几何含义清晰，稳定支持互逆/组合 | 表达能力不如一般复变换自由 |
| 不约束模长 | 直接学复向量 | 参数更自由 | 旋转解释被破坏，模式建模变脆弱 |
| 均匀负采样 | 随机替换头或尾 | 实现简单 | 难样本不足，梯度浪费 |
| 自对抗负采样 | 按当前分数加权 | 更聚焦困难负例，MRR 更高 | 训练实现稍复杂，超参更敏感 |

对初级工程师最实用的建议是：先把“角度参数化 + filtered evaluation + 自对抗负采样”三件事做对，再谈更复杂的损失和调参。

## 替代方案与适用边界

RotatE 不等于“知识图谱嵌入的终点”，它适合的是“关系模式很重要，尤其是互逆、组合、方向性明显”的场景。

TransE 的解释最简单：关系是平移，即 $h + r \approx t$。它速度快、实现短，但对对称和复杂组合关系支持较弱。ComplEx 同样使用复数空间，但更偏向通过双线性打分表达关系交互，不直接把关系解释为旋转。HAKE 在层次结构上更强，适合“上位类/下位类”明显的知识图。SeAttE 这类模型会显式分离属性子空间，更适合实体属性维度很复杂的场景。

可以做一个简化对比：

| 模型 | 核心变换 | 强项 | 适用边界 |
|---|---|---|---|
| TransE | 平移 | 简单、高效 | 关系模式较简单时够用 |
| RotatE | 复数旋转 | 对称/反对称/互逆/组合 | 结构关系推理明显时优先考虑 |
| ComplEx | 复数双线性交互 | 关系表达灵活 | 解释性不如 RotatE 直观 |
| HAKE | 模长+相位分解 | 层级关系强 | 更适合层次知识 |
| SeAttE/类似方法 | 注意力或属性子空间分离 | 多属性耦合更强 | 实现和训练更复杂 |

例如财务图谱里，“公司-子公司-行业-地域-股权结构”往往不仅是关系方向问题，还带强属性耦合。这时单纯旋转可能不够，需要额外的属性子空间或注意力机制。反过来，如果任务主要是做关系预测，且图里互逆和组合关系很多，RotatE 通常是很好的强基线。
