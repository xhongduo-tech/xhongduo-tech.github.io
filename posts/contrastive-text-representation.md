## 核心结论

对比学习文本表示，就是用“正样本更近、负样本更远”的训练目标，把文本编码成能表达语义相似度的向量。

这里的“文本表示”指把一句话、一段话或一个问题变成一串数字向量。向量越接近，模型认为语义越相近；向量越远，模型认为语义越无关。

$$
\text{目标}=\text{相似样本更近，非相似样本更远}
$$

玩具例子很直接：

- “怎么退货”
- “退货流程是什么”
- “今天北京天气怎么样”

前两句问的是同一类问题，它们的向量应该更近。第三句讨论天气，它和“怎么退货”的向量应该更远。模型学到的是句子之间的相对关系，不是简单统计“退货”这个词出现了几次。

| 任务类型 | 是否适合对比学习表示 | 原因 |
|---|---:|---|
| 语义检索 | 适合 | 需要把查询和候选文本放到同一个向量空间里比较距离 |
| FAQ 问答召回 | 适合 | 用户问法不同，但答案可能相同 |
| 文本聚类 | 适合 | 需要根据语义距离自动分组 |
| 情感分类 | 不一定 | 如果目标只是输出正面、负面、中性，分类模型更直接 |
| 机器翻译 | 不适合作为主方法 | 目标是生成另一种语言文本，不只是比较语义距离 |

真实工程例子是 FAQ 召回系统。离线阶段，把知识库里的标准问题编码成向量并建索引；线上阶段，把用户问题编码成向量，查找最相近的 Top-K 问题。用户输入“发票怎么开”和知识库问题“如何申请开票”字面不同，但语义接近，向量检索应该能把它们排在前面。

## 问题定义与边界

本文讨论的是文本表示学习，不讨论文本生成、机器翻译或完整对话系统。文本表示学习的目标是得到一个编码器 $f$，把输入文本 $x_i$ 变成向量 $h_i$：

$$
h_i=f(x_i),\quad z_i=\frac{h_i}{\|h_i\|}
$$

其中 $h_i$ 是原始输出向量，$z_i$ 是归一化后的向量。归一化的白话解释是：只保留方向，减少向量长度对相似度计算的干扰。做完归一化后，常用余弦相似度比较两个句子的语义距离。

| 概念 | 白话解释 | 例子 |
|---|---|---|
| 输入文本 | 模型读入的原始句子 | “怎么退货” |
| 编码器输出 | 模型算出的中间向量 | BERT 的 CLS 向量或平均池化向量 |
| 表示向量 | 用于检索、匹配、聚类的最终向量 | 归一化后的 768 维向量 |
| 正样本 | 应该和锚点语义接近的样本 | “退货流程是什么” |
| 负样本 | 应该和锚点语义不同的样本 | “今天北京天气” |

“锚点”是当前被比较的主样本。比如把“怎么退货”作为锚点，那么“退货流程是什么”可以作为正样本，“今天北京天气”可以作为负样本。

正样本的来源有两类。第一类是显式语义相似数据，例如人工标注的同义问法。第二类是数据增强，也就是从同一句话构造出另一种轻微变化的视图。视图的白话解释是：同一条文本经过不同处理后得到的版本。例如删除少量词、替换同义词、轻微重排序，或者像 SimCSE 那样让同一句话经过带 Dropout 的编码器前向传播两次。

边界也很重要。对比学习依赖正样本和负样本定义。如果定义错了，向量空间会被训练歪。比如 FAQ 数据里，“怎么申请退款”和“退货退款规则是什么”可能答案相同。如果它们在同一个 batch 中被当成负样本，模型会被迫把它们推远，这就是误负样本。误负样本指真实语义相近、却在训练中被当作反例的样本。

## 核心机制与推导

对比学习的核心损失常见形式是 InfoNCE 或 NT-Xent。二者都可以理解为：给定一个锚点，模型要从一组候选样本里找出真正的正样本。

$$
L_i=-\log \frac{\exp(\mathrm{sim}(z_i,z_{i^+})/\tau)}{\sum_{k\in \mathcal{B}\setminus\{i\}} \exp(\mathrm{sim}(z_i,z_k)/\tau)}
$$

| 符号 | 含义 |
|---|---|
| $z_i$ | 锚点文本的归一化向量 |
| $z_{i^+}$ | 正样本文本的归一化向量 |
| $\mathcal{B}$ | 一个训练 batch，包含当前参与比较的样本 |
| $\tau$ | 温度系数，控制相似度分布的尖锐程度 |
| $\mathrm{sim}$ | 相似度函数，常用余弦相似度 |

温度系数 $\tau$ 的作用是放大或压缩相似度差异。$\tau$ 越小，模型越强调“谁更像谁”的微小差别；$\tau$ 越大，相似度分布越平缓，区分会变弱。

一个玩具数值例子可以说明 loss 的变化。设 $\tau=0.1$，锚点与正样本余弦相似度为 $0.80$，与两个负样本分别为 $0.10$ 和 $-0.20$。分子约为 $\exp(8)=2981$，分母约为 $2981+2.72+0.14=2983.86$，loss 约为 $0.001$。如果正样本相似度降到 $0.20$，分子变成 $\exp(2)=7.39$，loss 会明显增大。结论是：正样本越接近，负样本越远，损失越小。

| 步骤 | 做什么 | 新手版理解 |
|---|---|---|
| 输入文本 | 读入一批句子 | 一次拿 8 个问题训练 |
| 构造视图 | 为每个句子生成正样本 | 同一句话轻微扰动，或 Dropout 两次 |
| 编码 | 用模型变成向量 | 每句话变成一串数字 |
| 计算相似度 | 得到 batch 内相似度矩阵 | 每句话和其他话两两打分 |
| 优化 loss | 提高正样本分数，降低负样本分数 | 让正确答案在候选里排第一 |

SimCSE 的关键点是把正样本构造做得非常轻。无监督 SimCSE 不需要人工标注同义句，而是把同一句文本输入同一个编码器两次。由于训练时 Dropout 会随机关闭部分神经元，两次前向传播得到的向量略有不同，它们就构成正样本。batch 内其他句子作为负样本。

Dropout 的白话解释是：训练时随机让模型的一部分连接暂时失效，逼模型不要依赖单一局部特征。SimCSE 利用了这个随机性，把同一句话生成两个相近但不完全相同的表示。

## 代码实现

下面是一个只依赖 Python 标准库的最小可运行例子。它不训练神经网络，只演示对比学习损失如何根据相似度变化。真实训练时，`encode` 会由 BERT、RoBERTa 或其他 Transformer 编码器完成。

```python
import math

def dot(a, b):
    return sum(x * y for x, y in zip(a, b))

def norm(a):
    return math.sqrt(sum(x * x for x in a))

def normalize(a):
    n = norm(a)
    return [x / n for x in a]

def cosine(a, b):
    a = normalize(a)
    b = normalize(b)
    return dot(a, b)

def contrastive_loss(anchor, positive, negatives, tau=0.1):
    # 1. encode: 这里直接使用手写向量，真实工程中由 encoder 输出
    # 2. normalize: cosine 内部会做归一化
    # 3. similarity matrix: 这里计算锚点与正负样本的相似度
    pos_score = math.exp(cosine(anchor, positive) / tau)
    neg_scores = [math.exp(cosine(anchor, neg) / tau) for neg in negatives]

    # 4. contrastive loss
    denominator = pos_score + sum(neg_scores)
    return -math.log(pos_score / denominator)

anchor = [1.0, 0.0]
positive_good = [0.8, 0.2]
positive_bad = [0.2, 0.8]
negatives = [[0.0, 1.0], [-1.0, 0.0]]

loss_good = contrastive_loss(anchor, positive_good, negatives)
loss_bad = contrastive_loss(anchor, positive_bad, negatives)

assert loss_good < loss_bad
assert round(cosine(anchor, [1.0, 0.0]), 6) == 1.0
assert round(cosine(anchor, [0.0, 1.0]), 6) == 0.0

print(round(loss_good, 4), round(loss_bad, 4))
```

这个代码里的二维向量是玩具例子。`[1.0, 0.0]` 和 `[0.8, 0.2]` 方向接近，所以 loss 较小；`[1.0, 0.0]` 和 `[0.2, 0.8]` 方向差别更大，所以 loss 较大。

真实工程里的流程一般如下：

| 阶段 | 输入 | 输出 |
|---|---|---|
| 读入句子 | 一批文本 | batch |
| 编码器 | batch | 每句一个向量 |
| Pooling | token 级输出 | 句向量 |
| Normalize | 句向量 | 单位向量 |
| 相似度矩阵 | 两组单位向量 | batch 内两两相似度 |
| Loss | 相似度矩阵 | 可反向传播的标量 |

Pooling 指把多个 token 的表示合成一个句子表示。token 的白话解释是模型处理文本时的基本切分单位，可能是一个字、一个词，也可能是词的一部分。常见 pooling 有 CLS pooling 和 mean pooling。CLS pooling 使用特殊位置的向量表示整句；mean pooling 对所有 token 向量求平均。

如果使用 SimCSE，伪代码可以写成：

```text
1. 读入 batch 句子
2. 同一个 batch 前向传播两次
3. 得到 view_a 和 view_b 两组向量
4. 对向量做归一化
5. 计算 view_a 与 view_b 的相似度矩阵
6. 把矩阵对角线位置当正样本
7. 用交叉熵形式计算对比损失
8. 反向传播更新编码器
```

对于 batch 中第 1 句话，另一视图里的第 1 句话是正样本，其他位置的句子是负样本。这样实现简单，而且不需要人工标注同义句。

## 工程权衡与常见坑

对比学习的效果很依赖 batch size、负样本质量、文本增强策略和温度参数。batch size 是一次训练送入模型的样本数量。batch 越大，in-batch negatives 越多，模型看到的反例越丰富，但显存开销也越高。

$$
\tau \text{ 过大} \Rightarrow 区分不明显,\quad \tau \text{ 过小} \Rightarrow 训练不稳定
$$

| 问题 | 现象 | 原因 | 规避方法 |
|---|---|---|---|
| batch 太小 | 向量区分度不足 | 负样本数量太少 | 增大 batch，或使用 queue、memory bank |
| 增强过强 | 正样本语义被破坏 | 删除、替换、重排改变了原意 | 使用保守增强，或采用 Dropout 视图 |
| 误负样本 | 相似句被推远 | batch 内存在同义问法 | 数据去重、聚类清洗、引入标签 |
| 温度不合适 | loss 抖动或分不开 | 相似度分布过尖或过平 | 从 0.05 到 0.1 附近试起 |
| 训练推理不一致 | 线上效果低于验证 | pooling、归一化、Dropout 设置不一致 | 推理关闭 Dropout，并复用训练时的向量处理方式 |

真实工程例子：做电商客服 FAQ 召回时，“退款多久到账”和“退货后钱什么时候回来”很可能语义相同。如果它们在训练 batch 中被当作负样本，模型会被训练成把它们推远。解决办法不是盲目增大训练轮数，而是先检查数据：去掉高度相似重复问法，或把这些问法组织成同一意图下的正样本。

参数建议也要保守：

| 参数 | 常见起点 | 说明 |
|---|---|---|
| `batch size` | 越大越好，但受显存限制 | 小 batch 负样本少，大 batch 训练更贵 |
| `tau` | 0.05 到 0.1 | 需要在验证集上调 |
| `augmentation` | 删除少量词、轻量替换、Dropout | 文本增强不能破坏语义 |
| `hard negatives` | 谨慎加入 | 难负样本能提升边界，但误标会伤害更大 |

Hard negative 指“看起来很像但语义不同”的负样本。例如“苹果手机怎么退款”和“苹果手机怎么换货”很接近，但业务动作不同。难负样本能让模型学得更细，但如果标注不准，就会把真相似句错误推远。

## 替代方案与适用边界

对比学习不是所有文本任务的默认答案。它适合学习语义向量空间，尤其适合召回、匹配、聚类。若任务目标是直接输出类别，普通分类模型通常更简单。若任务目标是极高精度的句对匹配，交叉编码器通常更强，但在线成本更高。

交叉编码器指把两个句子拼在一起送入同一个模型，让模型直接判断它们是否匹配。它能让两个句子的 token 充分交互，所以精度高；缺点是不能提前为知识库离线建向量，每次查询都要和候选文本重新计算。

| 方法 | 适合任务 | 优点 | 局限 | 是否适合作为句向量 |
|---|---|---|---|---:|
| 对比学习表示 | 语义检索、聚类、召回 | 可离线建索引，泛化到同义改写 | 依赖负样本和训练策略 | 是 |
| 普通分类 | 情感分类、违规识别、主题分类 | 目标直接，训练简单 | 不一定得到好的向量空间 | 不一定 |
| 词袋 / BM25 | 关键词检索、日志搜索 | 简单稳定，可解释 | 不擅长同义改写 | 否 |
| 交叉编码器 | 精排、句对判断 | 匹配精度高 | 在线计算慢，难以大规模召回 | 否 |
| 双塔检索 | 大规模召回 | 查询和文档可分别编码 | 交互能力弱于交叉编码器 | 是 |

BM25 是一种传统关键词检索方法，主要根据词频、文档长度和逆文档频率打分。它对关键词明确的搜索很可靠，但对“同义不同词”的情况不够强。例如用户搜“退款多久到账”，BM25 可能更偏向包含“退款”和“到账”的文本，而不一定理解“钱什么时候回来”。

一个常见决策表如下：

| 任务条件 | 推荐方案 |
|---|---|
| 需要大规模 Top-K 召回 | 对比学习表示或双塔检索 |
| 只需要输出固定类别 | 普通分类模型 |
| 关键词强、语义改写少 | BM25 可以作为强基线 |
| 需要高精度匹配且候选少 | 交叉编码器 |
| 先粗排再精排 | 对比学习召回 + 交叉编码器重排 |

实际系统中常用组合方案。比如搜索系统先用 BM25 和向量召回各取一批候选，再用交叉编码器重排。对比学习解决的是“快速找出语义上可能相关的候选”，不是替代所有排序和业务规则。
