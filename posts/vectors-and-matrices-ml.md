## 为什么神经网络本质上是线性代数程序

无论是最简单的多层感知机，还是 CNN、Transformer，前向传播里最重的计算几乎都可以写成两类操作：

- 线性变换：`z = Wx + b`
- 非线性变换：`a = f(z)`

非线性决定模型的表达能力，线性部分则决定信息如何被投影、混合、压缩和放大。真正让 GPU 跑满、让模型参数“动起来”的，绝大多数都是向量和矩阵运算。理解它们，不是为了背公式，而是为了看清模型到底在对什么对象做什么变换。

---

## 向量：把样本、参数和特征写成同一种对象

在机器学习里，向量最常见的三种身份是：

- 一个样本：例如 `x = [身高, 体重, 年龄]`
- 一组参数：例如某个神经元的权重 `w`
- 一组隐藏特征：例如一层网络输出的表示 `h`

如果把向量看成“按顺序排列的数字”，很多概念就能统一起来。

### 内积：衡量对齐程度

两个向量 `x` 和 `w` 的内积写成：

```text
x · w = Σ x_i w_i
```

内积有两层含义：

1. 代数上，它是“逐维相乘再求和”。
2. 几何上，它衡量两个方向有多一致。

在线性层里，单个神经元的输出正是 `w · x + b`。如果 `w` 和 `x` 越对齐，输出就越大；如果方向相反，输出就可能变小甚至变成负数。

### 范数：衡量向量“有多大”

最常见的是 `L2` 范数：

```text
||x||_2 = sqrt(Σ x_i^2)
```

它可以理解为向量到原点的欧氏距离。训练中经常会看到它出现在三个地方：

- 衡量输入或激活值的大小
- 衡量梯度是否爆炸
- 作为正则化项限制参数规模

### 余弦相似度：只看方向，不看长度

```text
cos(x, w) = (x · w) / (||x||_2 ||w||_2)
```

余弦相似度把长度因素消掉，只保留方向信息。向量检索、Embedding 搜索、推荐系统里经常用它比较“语义是否接近”。

| 概念 | 公式 | 直觉 |
| --- | --- | --- |
| 内积 | `x · w` | 两个向量是否对齐 |
| `L2` 范数 | `||x||_2` | 向量整体大小 |
| 余弦相似度 | `(x · w) / (||x|| ||w||)` | 只比较方向，不比较长度 |

## 矩阵：一层网络就是一组向量的批量计算

如果一个神经元是 `w · x + b`，那么一整层神经元可以写成矩阵形式：

```text
z = Wx + b
```

这里：

- `x` 是输入向量，形状通常是 `n × 1`
- `W` 是权重矩阵，形状是 `m × n`
- `b` 是偏置向量，形状是 `m × 1`
- `z` 是输出向量，形状是 `m × 1`

矩阵乘法最核心的维度规则是：

```text
(m × n) · (n × p) = (m × p)
```

中间维度 `n` 必须一致，否则“每一行和每一列没有办法对上号”，运算就没有定义。

### 为什么这个规则正好对应神经网络

设输入层有 4 个特征，隐藏层有 3 个神经元，那么：

- 输入 `x` 的形状是 `4 × 1`
- 权重 `W` 的形状是 `3 × 4`
- 输出 `z` 的形状是 `3 × 1`

这意味着：**每个输出神经元都拿到全部 4 个输入特征，并做一次自己的加权求和。**

如果一次喂入一个 batch，而不是单个样本，通常会把输入写成矩阵：

| 对象 | 形状 | 含义 |
| --- | --- | --- |
| `X` | `B × n` | `B` 个样本，每个样本 `n` 维 |
| `W^T` | `n × m` | 从 `n` 维映射到 `m` 维 |
| `Z = XW^T + b` | `B × m` | `B` 个样本的输出 |

这也是为什么框架里经常看到 `X @ W.T + b` 这种写法。它一次性把整个 batch 的计算都并行做完。

### 矩阵乘法不是“机械算式”，而是组合线性变换

单个矩阵可以表示缩放、旋转、投影、坐标变换。多个矩阵连续相乘，表示多个线性变换的复合：

```text
y = W_2(W_1x + b_1) + b_2
```

如果暂时忽略偏置，这就是先用 `W_1` 变换一次，再用 `W_2` 变换一次。深度学习里每一层都在学习“怎样把当前表示变成更有用的表示”。

## 转置、逆矩阵与“可逆性”边界

### 转置：交换行和列

矩阵 `A` 的转置记作 `A^T`。如果 `A` 形状是 `m × n`，那么 `A^T` 的形状是 `n × m`。

转置在训练里极其常见，原因很简单：梯度传播和批量计算经常需要把“按行组织的数据”变成“按列组织的数据”。例如：

```text
Z = XW^T
```

如果 `X` 是 `B × n`，`W` 必须写成 `m × n`，实际乘法时就要使用 `W^T` 的 `n × m` 形状。

### 逆矩阵：不是所有矩阵都有

如果存在矩阵 `A^{-1}` 使得：

```text
A^{-1}A = AA^{-1} = I
```

那么 `A` 才是可逆的。可逆至少要求：

- `A` 是方阵
- 行或列线性无关
- 行列式不为 0

这件事在机器学习里很重要，因为很多人第一次学线性代数时会形成误解：好像所有方程都可以“左右同乘逆矩阵”解决。实际上神经网络训练很少显式求逆，原因包括：

- 大矩阵求逆代价高
- 数值稳定性可能很差
- 很多矩阵本来就不可逆

更常见的是求解线性系统、用分解方法（LU / QR / SVD）或在欠定、超定场景下使用伪逆。

### 一个实用判断

如果你在模型代码里频繁看到显式 `inverse()`，通常应该先警惕两件事：

- 有没有更稳定的替代写法，例如 `solve`
- 这个问题本质上是不是最小二乘或投影问题

## 为什么 GPU 特别擅长矩阵运算

“GPU 擅长深度学习”这句话本质上是在说：**GPU 擅长大量结构相似、可并行的数值计算。**

矩阵乘法恰好满足这个特征。

### 1. 同一种计算会重复很多次

矩阵乘法里的每个输出元素都遵循同一条公式：

```text
c_ij = Σ_k a_ik b_kj
```

这意味着海量线程可以同时做几乎一样的工作，只是索引不同。

### 2. 数据访问模式规整

高性能计算不只是看“算了多少次”，还看“数据拿得是否顺”。矩阵通常按连续内存布局存放，适合块化（tiling）和缓存复用。一个小块数据被读进共享内存后，可以被多个线程反复使用。

### 3. Tensor Core / SIMD / SIMT 能把吞吐打满

现代 GPU 会把许多小矩阵乘法映射到专门硬件单元上执行。以训练常见的 FP16/BF16 为例，底层通常不是“一个个数慢慢乘”，而是以小 tile 为单位成批完成。

### 4. 深度学习框架会把多个算子融合

例如 `matmul + bias + activation` 常被融合成更少的 kernel，减少显存读写和启动开销。很多时候，真正慢的不是浮点乘加，而是数据在显存和计算单元之间来回搬运。

一个很实用的结论是：**优化深度学习性能，往往先看张量形状、内存访问、batch size 和 kernel 融合，而不是先盯着 Python 层。**

## 一个最小数值例子：从输入到隐藏层

下面用一个非常小的例子，把“维度变化”和“实际数值”对应起来：

```python
import numpy as np

x = np.array([2.0, -1.0])  # shape: (2,)
W = np.array([
    [0.3, 0.8],
    [-0.5, 0.2],
    [1.0, -0.7],
])  # shape: (3, 2)
b = np.array([0.1, -0.2, 0.05])  # shape: (3,)

z = W @ x + b
a = np.maximum(z, 0.0)  # ReLU

print("x shape:", x.shape)
print("W shape:", W.shape)
print("z:", z)
print("a:", a)
```

手算也很直接：

```text
z_1 = 0.3*2 + 0.8*(-1) + 0.1  = -0.1
z_2 = -0.5*2 + 0.2*(-1) - 0.2 = -1.4
z_3 = 1.0*2 + (-0.7)*(-1) + 0.05 = 2.75
```

因此：

```text
z = [-0.1, -1.4, 2.75]
a = ReLU(z) = [0.0, 0.0, 2.75]
```

这个例子说明了三件事：

- 矩阵的每一行都对应一个输出神经元
- 每个神经元都看到了全部输入特征
- 非线性函数只是逐元素作用在矩阵结果上

## 写模型时最容易犯的 4 个错误

### 1. 把逐元素乘法和矩阵乘法混为一谈

`A * B` 和 `A @ B` 在 NumPy / PyTorch 里通常不是一回事。前者多半是逐元素乘法，后者才是线性层意义上的矩阵乘法。

### 2. 只看数字，不看形状

调试张量问题时，第一反应应该是打印 `shape`。很多 bug 根本不是数学错误，而是批量维、特征维或通道维放反了。

### 3. 以为转置只是“为了凑维度”

转置不是补丁，它反映了数据组织方式。你如果不知道为什么需要 `T`，往往也没真正搞清输入、权重和输出各自按什么方式排布。

### 4. 误以为逆矩阵是通用解法

训练网络主要靠梯度下降，而不是靠一次求逆直接得到答案。规模一大，显式求逆通常既慢又不稳。

## 小结

向量让我们能够统一表示样本、参数和特征；内积描述匹配程度；范数描述规模；余弦相似度描述方向接近性。矩阵则把单个神经元的计算扩展为整层、整批样本的并行计算，维度规则 `(m × n) · (n × p) = (m × p)` 正是神经网络前向传播的骨架。

转置告诉你数据是如何组织的，逆矩阵提醒你“可逆性”不是默认成立，GPU 的高吞吐则解释了为什么深度学习几乎总在围绕矩阵乘法优化。

如果你真正吃透了这一篇，后面再看反向传播、注意力机制、归一化和优化器时，就不会把它们当成互不相干的公式，而会看到一条统一主线：**神经网络，本质上是在高维空间里连续做线性代数运算。**
