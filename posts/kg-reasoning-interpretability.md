## 核心结论

知识图谱推理的可解释性，核心不是把一个黑盒分数“翻译成人话”，而是把模型真正依赖的多跳路径显式展示出来。所谓多跳路径，就是从头实体出发，沿着若干关系走到候选答案的一条链，例如“疾病 → 靶点 → 药物”。

如果只用神经网络嵌入模型，系统通常只给出一个分数，表示“这个答案像不像对的”。这个分数常常有效，但很难说明“为什么是它”。路径推理可视化试图把这个“为什么”拆成三层：

| 层次 | 作用 | 给人的解释 |
|---|---|---|
| 规则提取 | 约束模型优先沿哪些关系组合推理 | “系统偏向这些逻辑链条” |
| 路径注意力 | 给每条候选路径一个权重 | “这条路径贡献更大” |
| 反事实验证 | 删除某条路径再看分数是否明显下降 | “这条路径不只是好看，确实影响结论” |

一个新手可以先抓住一句话：可解释路径推理，不是只给“答案”，而是给“答案 + 关键证据链 + 证据被拿掉后的变化”。

玩具例子很直接。查询 `(A, 能治疗, ?)` 时，系统找到两条候选路径：

1. `A -> 靶点X -> 药物Y`
2. `A -> 靶点Z -> 药物W`

如果第一条路径权重是 0.78，第二条是 0.32，而且删掉第一条后总分显著下降，那么“药物Y”不仅是高分答案，还是有主要解释路径支撑的答案。

这类方法的代价也明确：透明度上升，通常会牺牲一部分极限准确率和吞吐量。原因是模型不再只追求“分数最高”，而要受规则空间、路径生成和解释约束共同限制。

## 问题定义与边界

先定义问题。知识图谱是由实体和关系组成的图。实体可以理解为“图里的点”，例如疾病、药物、公司、用户；关系可以理解为“点和点之间的有类型连接”，例如“作用于”“位于”“购买了”。

给定查询

$$
q=(h,r,?)
$$

其中 $h$ 是头实体，$r$ 是目标关系，目标是预测尾实体。比如：

$$
q=(\text{阿尔茨海默病},\ \text{能治疗},\ ?)
$$

对于这个查询，系统会在图中搜索一批路径集合 $\mathcal{P}$。每条路径 $p$ 都会提取出一个特征 $f(p)$。特征可以理解为“把一条路径压缩成可计算信号”，例如路径长度、中间实体类型、关系链模式、规则匹配分。

最终得分常写成：

$$
s(q)=\sum_{p\in\mathcal{P}} w_p f(p)
$$

这里：

- $w_p$ 是路径权重，可以理解为“这条路径被模型采信的程度”
- $f(p)$ 是路径特征值，可以理解为“这条路径自身提供了多少支持”
- $s(q)$ 是对某个候选答案的总支持分

对白话一点的理解：每条路径都是一块拼图，最后结果不是由一块拼图决定，而是所有拼图加权叠加出来。

但这个定义有几个边界必须说清。

| 项目 | 含义 | 典型风险 |
|---|---|---|
| 路径 | 从头实体到候选尾实体的多跳链 | 数量爆炸，搜索成本高 |
| 特征 | 对路径的结构化表示 | 设计不当会丢失关键信息 |
| 权重 | 模型对路径的采信程度 | 高权重不等于真因果 |
| 解释 | 人能复核的证据链 | 可能只是“看起来合理” |

可解释性的边界主要有三类。

第一，路径搜索空间很大。图越大、关系类型越多，可能路径就越多。即使限制到 3 跳或 4 跳，候选路径仍可能成千上万。

第二，注意力容易漂移。注意力可以理解为“模型当前把关注分配到哪里”。如果训练数据有噪声，或者多个相似路径共存，注意力可能落在一个表面合理、实际不关键的路径上。

第三，可解释性和准确率之间常常要做交换。强约束的路径模型更透明，但表达能力可能不如完全自由的黑盒嵌入模型。

## 核心机制与推导

路径推理可视化通常由三个步骤组成：路径生成、路径打分、反事实验证。

先看路径生成器。路径生成器可以理解为“在图里找可能有用的逻辑链条的模块”。它并不直接给出最终答案，而是给出候选解释。常见写法是：

$$
P(z=p\mid q)
$$

意思是在查询 $q$ 下，生成器选择路径 $p$ 的概率。这里的 $z$ 是潜变量，可以理解为“当前采用了哪条解释路径”。

这样做的意义在于：模型不是盲目遍历全部路径，而是学会优先采样更可能有解释价值的路径。若再结合规则模板，例如“疾病 → 基因/靶点 → 药物”，生成过程就会更可控。

然后是路径打分。对每条候选路径 $p$，模型计算两个东西：

1. 路径本身提供多少证据，即 $f(p)$
2. 这条路径应分到多少权重，即 $w_p$

于是总分仍然是：

$$
s(q)=\sum_{p\in\mathcal{P}} w_p f(p)
$$

注意一个关键点：$w_p$ 表示可信度，不等同于“这条路径一定正确”。它更像“在当前模型和当前样本里，这条路径被当作主要证据的程度”。

玩具例子可以用两个路径说明。查询 `(A, 能治疗, ?)`：

| 路径 | 路径特征 $f(p)$ | 权重 $w_p$ | 贡献 $w_p f(p)$ |
|---|---:|---:|---:|
| `A -> 靶点X -> 药物Y` | 1.00 | 0.78 | 0.78 |
| `A -> 靶点Z -> 药物W` | 1.00 | 0.32 | 0.32 |

如果候选答案是 `药物Y`，第一条路径显然贡献更大。可视化时，界面通常会把这条链画得更粗、更亮，或者把中间节点 `靶点X` 标成高贡献节点。

但仅凭高权重还不够，所以需要反事实。反事实可以理解为“把某个因素拿掉，再看结果会不会变”。对路径解释来说，最常见的量是：

$$
\Delta s = s_{\text{full}} - s_{\text{without } p}
$$

其中：

- $s_{\text{full}}$ 是使用全部路径时的得分
- $s_{\text{without } p}$ 是删掉路径 $p$ 后的得分
- $\Delta s$ 越大，说明路径 $p$ 对结果越关键

继续上面的例子。若完整得分是 0.91，删除 `A -> 靶点X -> 药物Y` 后降到 0.38，那么：

$$
\Delta s = 0.91 - 0.38 = 0.53
$$

这个掉分很大，说明这条路径不只是“被看见”，而是“真正支撑了预测”。

可以把整个流程压缩成一个简化示意：

```text
查询 q=(h,r,?)
   ↓
路径生成器 P(z=p|q)
   ↓
候选路径集合 P
   ↓
路径特征 f(p) + 权重 w_p
   ↓
总分 s(q)=Σ w_p f(p)
   ↓
删除高权路径做反事实验证
   ↓
输出：答案 + 解释路径 + 掉分证据
```

真实工程例子常见于药物再利用。药物再利用就是“给旧药找新用途”。系统不是只输出“某药可能治疗某病”，还会同步给出类似下面的解释链：

`疾病D -> 相关靶点T -> 被药物M作用`

专家复核时，往往先看高权重中间节点 `T` 是否合理，再看删除该路径后候选药物分数是否大幅下降。如果两者都成立，这条路径更适合进入人工审查或监管材料。

## 代码实现

下面给一个可运行的简化 Python 例子，只演示“路径打分 + 反事实掉分 + 可视化文本输出”。它不是完整训练代码，但足够说明核心机制。

```python
from dataclasses import dataclass

@dataclass
class Path:
    name: str
    feature: float
    weight: float

def score(paths):
    return sum(p.feature * p.weight for p in paths)

def counterfactual_drop(paths, remove_name):
    full = score(paths)
    kept = [p for p in paths if p.name != remove_name]
    without = score(kept)
    return full, without, full - without

def render_bar(weight, width=20):
    n = int(weight * width)
    return "#" * n + "-" * (width - n)

paths = [
    Path("A->靶点X->药物Y", feature=1.0, weight=0.78),
    Path("A->靶点Z->药物W", feature=1.0, weight=0.32),
]

full, without_main, delta = counterfactual_drop(paths, "A->靶点X->药物Y")

print("完整得分:", round(full, 2))
print("删除主路径后:", round(without_main, 2))
print("掉分:", round(delta, 2))
print()

for p in paths:
    contrib = p.feature * p.weight
    print(f"{p.name:16} weight={p.weight:.2f} contrib={contrib:.2f} {render_bar(p.weight)}")

assert round(full, 2) == 1.10
assert round(without_main, 2) == 0.32
assert round(delta, 2) == 0.78
assert delta > 0.5
```

如果把它映射到工程系统，通常会拆成三个模块：

| 输入 | 模块 | 输出 |
|---|---|---|
| 查询 `(h,r,?)` | 路径生成器 | 候选路径集合 |
| 候选路径 | 推理预测器 | 路径得分、总分 |
| 路径得分 | 可视化模块 | 路径图、权重条、反事实掉分 |

一个更接近真实项目的流程是：

1. 从图数据库或内存图中取出候选多跳路径。
2. 用规则模板过滤明显无意义的路径。
3. 对每条路径计算特征，例如关系序列、节点类型、先验规则匹配度。
4. 用模型给出 $w_p$ 和 $f(p)$。
5. 绘制解释界面，把高权重路径和高贡献中间节点高亮。
6. 对前几条主路径做删除实验，输出 $\Delta s$。

如果是博客站点中的类比式实现，也可以把路径对象当成 `posts.json` 里的记录来读，再渲染为“路径名、权重、掉分”的前端列表。核心不是数据来源，而是“路径作为一等对象被展示和验证”。

一个简单的文本可视化效果如下：

```text
A->靶点X->药物Y   0.78  ################----
A->靶点Z->药物W   0.32  ######--------------
```

这种展示对初级工程师很重要，因为它把抽象的注意力矩阵，变成了可读的证据链和强弱排序。

## 工程权衡与常见坑

路径推理可解释，不代表天然可靠。工程里最容易踩的坑，恰恰来自“看起来有解释”。

| 坑 | 原因 | 规避方式 |
|---|---|---|
| 只看注意力权重 | 注意力可能受噪声和训练随机性影响 | 必须配合反事实删去实验 |
| 路径空间过大 | 图很稠密，多跳组合爆炸 | 限制跳数、规则模板、采样生成 |
| 高权路径很漂亮但不关键 | 路径与结果相关，但不一定决定结果 | 计算 $\Delta s$ 验证 |
| 规则过强导致漏召回 | 约束太死，很多真路径进不来 | 分层规则，保留探索支路 |
| 可解释模型精度下降 | 表达能力被路径结构限制 | 在业务指标中明确透明度预算 |

最常见误区是把“高权重”直接当“真解释”。这是不够的。因为模型可能把注意力集中在一条视觉上整洁、统计上常见、但实际并非主因的路径上。只有当删掉它后，预测结果明显变差，解释才更接近因果支撑。

另一个难点是稀疏。图谱中的很多真实推理链很长、很少见，直接搜索很难命中。这时常会引入强化学习或变分路径生成。强化学习可以理解为“通过奖励信号学会走更有价值路径的搜索策略”；变分生成可以理解为“通过概率近似，在巨大路径空间里挑更可能有用的那部分”。但这两类方法若没有领域知识约束，仍会陷入低效探索。

真实工程里，药物图谱、金融风控图谱、供应链图谱都面临同一个问题：业务方希望“有解释”，但不会接受解释把系统拖慢 10 倍，或者把准确率打掉太多。所以评测不能只看 AUC、Hits@K 这类预测指标，还要加上：

- 平均解释路径长度
- 高权路径人工通过率
- 反事实掉分稳定性
- 单次查询解释耗时

## 替代方案与适用边界

不是所有场景都值得上路径解释。

最简单的替代方案，是直接用 embedding 打分。embedding 可以理解为“把实体和关系压缩成向量，让模型在向量空间里比较相似性”。它通常速度快、效果强，但解释很弱。你能知道“这个答案分高”，却很难知道“它依赖哪条逻辑链”。

另一类替代方案，是纯规则系统。纯规则系统完全透明，但覆盖率和泛化能力往往不够，规则维护成本也高。

因此常见比较如下：

| 方案 | 准确率上限 | 可解释性 | 合规/审计友好度 | 适用场景 |
|---|---|---|---|---|
| 纯 embedding 黑盒 | 高 | 低 | 低 | 推荐、召回、低监管场景 |
| 纯规则推理 | 中低 | 很高 | 很高 | 强规则、小规模专家系统 |
| 路径 + 权重解释 | 中高 | 高 | 高 | 医疗、金融、审计、科研辅助 |

对于“零基础到初级工程师”，可以用一句话记住选择标准：

- 如果业务只关心命中率，黑盒模型常常更省事。
- 如果业务要求“把证据给我看”，路径推理更合适。
- 如果业务还要求“证明这条证据真的影响了结论”，就必须加入反事实验证。

药物再利用是很典型的适用场景。因为这里不仅要预测，还要把候选药物和疾病之间的中间靶点、关系链、文献证据交给专家审查。相反，在海量广告粗排里，单次请求延迟极紧，路径可解释模型往往不划算。

因此，透明度提升通常要接受一个现实：准确率可能小幅下降，系统复杂度会上升，解释模块本身也要评估和维护。这不是缺点，而是工程上的显式交换。
