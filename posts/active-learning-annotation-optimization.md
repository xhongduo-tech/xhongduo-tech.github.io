## 核心结论

主动学习是一种标注优化方法：它不追求把所有未标注数据都标完，而是在每一轮从未标注池里挑出最值得人工标注的样本，用更少标注成本逼近全量标注效果。

一句话定义：主动学习就是“模型先看未标注数据，再主动请求人工标注最有信息量的样本”。

在 `NLP` 场景里，主动学习最常见的价值不是替代人工，而是把有限人工预算集中到高价值样本上。典型任务包括文本分类、`NER` 和序列标注。`NER` 是命名实体识别，目标是在文本中找出人名、机构名、产品名、时间、地点等实体片段。

一个新手版例子是：有 10 万条客服日志，但只能标 1000 条。随机标注会直接抽 1000 条；主动学习会先用少量种子数据训练一个初始模型，再找出模型最拿不准、最能覆盖新表达模式的日志交给人工标注。

| 方法 | 选样策略 | 标注效率 | 风险 | 典型任务 |
|---|---|---:|---|---|
| 随机标注 | 从未标注池随机抽样 | 中等或偏低 | 可能浪费预算在简单样本上 | 基线实验、小数据集 |
| 主动学习 | 按不确定性、多样性或混合分数选样 | 通常更高 | 分数不稳定时可能选偏 | 文本分类、`NER`、序列标注 |
| 全量标注 | 所有样本都人工标注 | 信息最完整但成本最高 | 时间和预算压力大 | 高风险、小规模任务 |

主动学习的核心判断是：标注预算是有限资源，样本之间的信息价值不同。工程目标不是“少标一定更好”，而是“在同样标注预算下，让模型更快学到关键边界和数据分布”。

## 问题定义与边界

主动学习解决的问题是：在标注预算受限时，如何从大量未标注数据中选择下一批最值得标注的样本。

基本对象如下：

| 符号 | 含义 | 白话解释 |
|---|---|---|
| `L` | 已标注数据集 | 已经有人给出正确标签的数据 |
| `U` | 未标注数据池 | 还没有人工标签的数据 |
| `b` | 每轮标注预算 | 每一轮最多能送去人工标注的样本数 |
| `θ` | 模型参数 | 当前模型学到的内部权重 |
| `φ(x)` | 样本表示 | 把文本 `x` 编码成向量后的结果 |
| `S` | 本轮选择样本 | 准备送给人工标注的一批数据 |

一个标准流程是：

1. 先随机或分层抽取少量样本，形成种子标注集 `L`。
2. 用 `L` 训练初始模型 `θ`。
3. 让模型对未标注池 `U` 做预测。
4. 根据预测概率和文本向量计算每个样本的价值分数。
5. 选择 `b` 条样本交给人工标注。
6. 把新标注样本加入 `L`，从 `U` 删除。
7. 重新训练模型，进入下一轮。

这说明主动学习是循环优化，不是一次性算法。它优化的是标注顺序和标注对象，不是自动消灭标注成本。

| 场景 | 是否适合主动学习 | 原因 |
|---|---|---|
| 标注昂贵，未标注数据很多 | 适合 | 每条标注都要优先产生价值 |
| 标签体系相对清晰 | 适合 | 模型不确定性才有稳定含义 |
| 文本分类、`NER`、意图识别 | 适合 | 能用模型概率和文本向量选样 |
| 标注非常便宜，数据很少 | 不一定适合 | 直接全量标注更简单 |
| 标签标准经常变化 | 不适合或收益低 | 模型上一轮学到的边界很快失效 |
| 初始模型完全不可用 | 不适合直接使用 | 不确定性分数可能没有参考价值 |

边界要说清楚：主动学习不保证每一轮指标都提升。尤其在冷启动阶段，模型还没有学到稳定边界，所谓“最不确定样本”可能只是随机噪声。因此工程里通常先做一批随机或分层种子标注，再进入主动学习循环。

## 核心机制与推导

主动学习最常见的两类机制是不确定性采样和多样性采样。

不确定性采样的思想是：模型越拿不准的样本，越可能帮助模型修正决策边界。决策边界是模型区分类别的分界区域，比如“退款咨询”和“投诉升级”之间的模糊地带。

设模型对样本 `x` 的类别概率为 $p_\theta(y|x)$，常见不确定性分数是熵：

$$
u(x) = H(p_\theta(y|x)) = -\sum_y p_\theta(y|x)\log p_\theta(y|x)
$$

熵越大，概率分布越平均，模型越不确定。另一种更简单的写法是：

$$
u(x) = 1 - \max_y p_\theta(y|x)
$$

如果是二分类，模型输出如下：

| 样本 | 预测概率 | 直观判断 |
|---|---|---|
| `x1` | `(0.51, 0.49)` | 非常不确定 |
| `x2` | `(0.90, 0.10)` | 很确定 |
| `x3` | `(0.60, 0.40)` | 比较不确定 |

只看不确定性时，`x1` 最应该先标，因为它最接近分类边界。

但只看不确定性会有问题：模型可能连续挑到很多噪声样本、异常样本，或者同一类模板句。噪声样本是指标签不清、文本残缺、含义混乱的数据。它们会让模型困惑，但不一定有训练价值。

多样性采样的目标是覆盖数据空间。数据空间可以理解为：把每条文本变成向量后，它们在向量空间里的分布。距离近表示语义相似，距离远表示表达模式不同。

`core-set` 是一种典型多样性方法，它把选样写成覆盖优化：

$$
S^* = \arg\min_{|S|=b} \max_{x\in U} \min_{s\in L\cup S} d(\phi(x), \phi(s))
$$

这条公式的意思是：选择一批样本 `S`，让未标注池里的每个样本都尽量靠近已标注数据或新选择样本。`d` 是距离函数，比如欧氏距离或余弦距离。

一个玩具例子是：`x1`、`x2` 都和已有标注样本很近，`x3` 离已标注集很远。即使 `x3` 不是最不确定的样本，它也可能值得优先标注，因为它代表了一个新区域。

工程里通常混合两类分数：

$$
score(x) = \lambda u(x) + (1-\lambda)d(x)
$$

其中 `u(x)` 表示不确定性，`d(x)` 表示多样性或离已标注集的距离，$\lambda$ 控制两者权重。$\lambda$ 越大，越偏向“挑最难的”；$\lambda$ 越小，越偏向“挑最不一样的”。

流程可以写成：

```text
未标注文本 U
    ↓
模型预测概率 pθ(y|x)
    ↓
计算不确定性 u(x)
    ↓
提取文本向量 φ(x)
    ↓
计算多样性 d(x)
    ↓
合成 score(x)
    ↓
排序 / 聚类 / 去重
    ↓
选择 b 条样本
    ↓
人工标注
    ↓
更新 L 并重新训练
```

所以，新手可以把主动学习理解为两个选择器一起工作：一个负责挑“模型最难判断的样本”，一个负责挑“和已有数据最不一样的样本”。

## 代码实现

实现主动学习时，代码里没有神奇优化器，核心就是四步：训练模型、计算分数、排序选样、更新数据池。

下面是一个可运行的最小 Python 例子。它用二分类概率计算不确定性，用样本到已标注集的最小距离表示多样性，然后混合选出一批样本。

```python
import math

def entropy(prob):
    return -sum(p * math.log(p) for p in prob if p > 0)

def euclidean(a, b):
    return math.sqrt(sum((x - y) ** 2 for x, y in zip(a, b)))

def min_distance_to_labeled(x_vec, labeled_vecs):
    return min(euclidean(x_vec, v) for v in labeled_vecs)

def normalize(values):
    lo, hi = min(values), max(values)
    if hi == lo:
        return [0.0 for _ in values]
    return [(v - lo) / (hi - lo) for v in values]

def select_batch(unlabeled, labeled_vecs, budget=2, lam=0.6):
    uncertainties = [entropy(item["prob"]) for item in unlabeled]
    distances = [min_distance_to_labeled(item["vec"], labeled_vecs) for item in unlabeled]

    u_norm = normalize(uncertainties)
    d_norm = normalize(distances)

    scored = []
    for item, u, d in zip(unlabeled, u_norm, d_norm):
        score = lam * u + (1 - lam) * d
        scored.append((score, item["id"]))

    scored.sort(reverse=True)
    return [item_id for _, item_id in scored[:budget]]

labeled_vecs = [
    [0.0, 0.0],
    [1.0, 1.0],
]

unlabeled = [
    {"id": "x1", "prob": [0.51, 0.49], "vec": [0.1, 0.1]},
    {"id": "x2", "prob": [0.90, 0.10], "vec": [0.2, 0.2]},
    {"id": "x3", "prob": [0.60, 0.40], "vec": [5.0, 5.0]},
]

selected = select_batch(unlabeled, labeled_vecs, budget=2, lam=0.6)

assert "x1" in selected
assert "x3" in selected
assert len(selected) == 2

print(selected)
```

这段代码体现了一个真实系统的骨架：

| 函数 | 工程含义 |
|---|---|
| `train_model(L)` | 用已标注数据训练模型 |
| `score_unlabeled(U, model)` | 给未标注样本计算预测概率和向量 |
| `select_batch(U, scores, budget)` | 根据分数选择本轮样本 |
| `update_pool(L, U, labeled_batch)` | 更新已标注集和未标注池 |

真实工程例子：做客服日志文本分类时，可以先随机标注 500 条，训练一个初始分类器；然后对 10 万条未标注日志计算分类概率和句向量；先按熵筛出前 5000 条高不确定样本，再用句向量聚类去重，最后选 500 条交给标注团队。下一轮把这 500 条加入训练集，再重复同样流程。

`NER` 任务和文本分类略有不同。文本分类通常是句子级标签，比如“退款”“咨询”“投诉”。`NER` 是 token 级标签，token 是文本切分后的最小处理单元，比如一个字、一个词或一个子词。因此 `NER` 的不确定性常按 token 概率聚合到句子级，例如取平均熵、最大熵，或统计低置信 token 的数量。

## 工程权衡与常见坑

主动学习在工程上不是单一分数就够了。只用不确定性，容易挑到噪声、异常点或大量重复边界样本；只用多样性，又可能挑到代表性强但模型已经会的简单样本。因此生产系统通常会混合不确定性、多样性、密度、去重和类别覆盖约束。

密度约束是指优先选择周围有很多相似样本的数据，避免把预算浪费在孤立异常点上。类别覆盖约束是指尽量让每一轮样本覆盖不同预测类别，避免只服务多数类。

客服日志标注就是一个典型例子。如果只按熵选样，系统可能连续选出很多同类模板句，比如“我要退款”“申请退款”“退款怎么操作”。这些句子对模型边界有帮助，但重复太多会浪费预算。如果只按聚类覆盖，系统可能选到很多表达正常、模型已经能准确分类的样本，学习收益也有限。更合理的做法是：不确定性排序 + 句向量聚类去重 + 类别覆盖约束。

| 常见坑 | 现象 | 规避方式 |
|---|---|---|
| 噪声样本 | 模型最不确定的样本可能本身不可标 | 加密度约束，人工标注时允许丢弃低质量样本 |
| 重复样本 | 一批样本语义高度相似 | 句向量去重、聚类后每簇限额 |
| 冷启动弱 | 初始模型分数不稳定 | 先做随机或分层种子标注 |
| 类别偏置 | 多数类样本被反复选中 | 按预测类别分层采样 |
| 批量冗余 | 同一轮内部样本互相重复 | 使用 `core-set` 或最大最小距离选择 |
| 标注标准漂移 | 不同标注员理解不同 | 维护标注指南和一致性抽检 |

还有一个容易忽略的问题：主动学习会改变训练数据分布。随机采样得到的数据更接近原始分布，而主动学习会故意选择“困难样本”或“稀有区域”。这有利于提升学习效率，但评估集不能也用同样策略构造。评估集应该独立、稳定，并尽量接近真实线上分布，否则指标会失真。

## 替代方案与适用边界

主动学习不是所有场景的最优解。如果标注便宜、数据规模不大、任务风险高，直接全量标注可能更简单可靠。如果标签体系还没稳定，应该先定义标签标准，而不是急着做主动学习。

| 方法 | 优点 | 缺点 | 适用条件 | 风险 |
|---|---|---|---|---|
| 随机抽样 | 实现简单，分布自然 | 样本效率不高 | 基线、小规模数据、初始种子集 | 可能抽不到关键边界样本 |
| 不确定性采样 | 快速找到模型薄弱点 | 容易选到噪声和重复样本 | 模型概率较可靠 | 过度关注边界异常点 |
| 多样性采样 | 覆盖数据空间 | 不一定选择困难样本 | 数据分布复杂、重复样本多 | 标到很多简单样本 |
| 混合策略 | 平衡难度和覆盖 | 参数和流程更复杂 | 标注贵、样本多、任务稳定 | 需要持续监控选样质量 |
| 规则筛选 | 可解释、容易控范围 | 依赖人工规则 | 先缩小候选集再精标 | 规则偏差会影响覆盖 |

一个现实选择是：先随机抽样建立基线，再引入主动学习。不要一开始就把系统做复杂。基线能回答两个问题：当前任务是否能学起来，标注体系是否稳定。如果随机 1000 条训练出的模型已经达到业务要求，主动学习未必值得投入。

规则筛选也经常和主动学习配合使用。例如在法律合同 `NER` 中，先用关键词、正则或文档结构筛出可能包含金额、期限、主体名称的句子，再在候选集中做主动学习。这样可以先缩小未标注池，降低后续模型选样压力。

何时不用主动学习：

| 情况 | 更合适的做法 |
|---|---|
| 数据只有几百条 | 全量标注 |
| 标注单价很低 | 随机标注后直接训练 |
| 标签定义频繁变化 | 先稳定标注规范 |
| 初始模型完全不可用 | 先做种子集和基线模型 |
| 线上分布剧烈变化 | 先做数据监控和分布分层 |

主动学习的适用边界可以总结为一句话：当未标注数据很多、人工标注昂贵、标签体系稳定、模型表示基本可用时，它最有价值。结合 `BERT` 或其他预训练语言模型后，主动学习通常更容易生效，因为预训练模型已经提供了较好的文本表示，选样策略不必从零理解语义。
