## 核心结论

强化学习里最基础的一层，不是“怎么训练神经网络”，而是先把决策问题写成一个数学对象：马尔可夫决策过程，简称 MDP。马尔可夫的白话解释是“下一步只看当前，不用回头翻全部历史”。一个 MDP 用五元组 $(S, A, P, R, \gamma)$ 描述：

| 符号 | 名称 | 白话解释 |
| --- | --- | --- |
| $S$ | 状态集合 | 系统此刻处于什么情况 |
| $A$ | 动作集合 | 智能体现在能做什么 |
| $P$ | 转移概率 | 做完动作后，环境会怎样变化 |
| $R$ | 奖励函数 | 这一步给多少反馈 |
| $\gamma$ | 折扣因子 | 未来奖励现在还值多少 |

策略 $\pi(a \mid s)$ 是“在状态 $s$ 下选动作 $a$ 的概率分布”。如果把智能体比作一个决策程序，策略就是它的决策规则。

价值函数回答的问题是：“如果从现在开始按这个策略一直走，未来总共大概能拿到多少奖励？”  
状态价值函数写作：

$$
V^\pi(s)=\mathbb{E}_\pi\left[\sum_{t=0}^{\infty}\gamma^t r_{t+1}\mid s_0=s\right]
$$

动作价值函数写作：

$$
Q^\pi(s,a)=\mathbb{E}_\pi\left[\sum_{t=0}^{\infty}\gamma^t r_{t+1}\mid s_0=s,a_0=a\right]
$$

两点最重要：

1. MDP 定义了“问题是什么”。
2. 策略和价值函数定义了“智能体如何做、做了以后值不值”。

先看一个最小玩具例子。只有一个状态 $s_1$、一个动作 $a_1$：

- $S=\{s_1\}$
- $A=\{a_1\}$
- $P(s_1\mid s_1,a_1)=1$
- $R(s_1,a_1)=5$
- $\gamma=0.9$
- $\pi(a_1\mid s_1)=1$

流程可以写成：

```text
s1 --执行 a1--> 奖励 5，回到 s1
```

于是未来回报是：

$$
V^\pi(s_1)=5+0.9\times 5+0.9^2\times 5+\cdots
$$

这是等比数列，和为：

$$
V^\pi(s_1)=\frac{5}{1-0.9}=50
$$

因为这里只有一个动作，所以：

$$
Q^\pi(s_1,a_1)=50
$$

这就是最基础的结论：价值函数本质上是在算“未来折扣奖励之和”，Bellman 方程只是把这个和写成递归形式。

## 问题定义与边界

MDP 适合描述“分步决策、每步有反馈、下一步受当前状态和动作影响”的问题。游戏、机器人控制、推荐系统的排序决策，都能落到这个框架里。

但它有边界。它默认当前状态已经包含了做决策需要的全部信息。如果现实里你只能看到不完整观测，比如机器人摄像头被遮挡，当前画面不足以判断真实位置，那么严格说更接近 POMDP，而不是普通 MDP。

机器人导航是一个典型例子。假设把房间离散成 $4\times4$ 的格子，每个格子是一个状态，机器人每次可以执行四个动作：上、下、左、右。于是：

- 状态 $S$：16 个格子位置
- 动作 $A$：$\{$上，下，左，右$\}$
- 转移 $P$：如果地面无噪声，向右基本到右边格子；如果轮子打滑，就可能以 0.8 概率向右、0.1 概率原地、0.1 概率偏上
- 奖励 $R$：走一步记 $-1$，到目标点记 $+10$，撞墙记 $-3$
- 折扣 $\gamma$：比如 0.95，表示更重视长期到达目标，而不是只看眼前一步

一个简化格子图可以写成：

```text
(0,0) -> (0,1) -> (0,2) -> Goal
  |        |        |
(1,0) -> (1,1) -> (1,2)
```

箭头不是说只能这样走，而是说明“状态之间存在转移关系”。

这里有两个常见误解。

第一，状态不是“程序里随便定义一个变量”。状态必须足够完整，能支持马尔可夫性质。比如导航任务里只记录“机器人朝向”，却不记录“位置”，那通常不够。

第二，策略不是必须确定的。策略可以是随机的。  
例如在某个格子里：

$$
\pi(\text{右} \mid s)=0.7,\quad \pi(\text{上} \mid s)=0.2,\quad \pi(\text{下} \mid s)=0.1
$$

这表示策略会按概率采样动作。随机策略的白话解释是“不是每次都死板选同一个动作”，这样能保留探索空间。

如果任务是离线分析、环境模型已知，MDP 这套定义足够用。  
如果任务是部分可观测、高维连续状态、奖励很稀疏，MDP 仍然是理论起点，但实际工程通常要再加函数逼近、采样估计和稳定化手段。

## 核心机制与推导

价值函数的核心不是“累加奖励”，而是“在策略和环境概率共同作用下，对未来回报取期望”。

先看状态价值函数：

$$
V^\pi(s)=\mathbb{E}_\pi[G_t \mid s_t=s]
$$

其中 $G_t$ 叫回报，白话解释是“从第 $t$ 步开始往后，把未来奖励按折扣加总”：

$$
G_t=r_{t+1}+\gamma r_{t+2}+\gamma^2 r_{t+3}+\cdots
$$

把第一步单独拆出来：

$$
G_t=r_{t+1}+\gamma G_{t+1}
$$

因此：

$$
V^\pi(s)=\mathbb{E}_\pi[r_{t+1}+\gamma G_{t+1}\mid s_t=s]
$$

又因为 $G_{t+1}$ 的期望就是下一状态的价值，所以得到 Bellman 期望方程：

$$
V^\pi(s)=\sum_a \pi(a\mid s)\left[R(s,a)+\gamma \sum_{s'} P(s'\mid s,a)V^\pi(s')\right]
$$

这条式子可以顺着“状态 $\to$ 动作 $\to$ 奖励和下一状态”去理解：

```text
当前状态 s
  -> 按策略选动作 a
  -> 环境给奖励 R(s,a)
  -> 转移到 s'
  -> 后续价值是 Vπ(s')
```

动作价值函数同理，只是它固定了第一步动作：

$$
Q^\pi(s,a)=R(s,a)+\gamma \sum_{s'}P(s'\mid s,a)\sum_{a'}\pi(a'\mid s')Q^\pi(s',a')
$$

这条式子比 $V^\pi$ 多了一层对下一步动作 $a'$ 的求和，因为在到达 $s'$ 之后，还是要继续按策略选动作。

回到前面的单状态单动作例子：

$$
V^\pi(s_1)=R(s_1,a_1)+\gamma V^\pi(s_1)=5+0.9V^\pi(s_1)
$$

移项得到：

$$
0.1V^\pi(s_1)=5
$$

所以：

$$
V^\pi(s_1)=50
$$

同理：

$$
Q^\pi(s_1,a_1)=5+0.9Q^\pi(s_1,a_1)=50
$$

这个例子虽然极小，但已经把 Bellman 递归的本质说明白了：  
当前价值 = 当前一步奖励 + 折扣后的未来价值。

真实工程例子可以看自动驾驶中的局部决策。假设系统每 100ms 决策一次，状态包含车速、车道位置、前车距离、周边障碍物摘要；动作是“加速、保持、减速、变道”；奖励综合了安全、舒适、通行效率。此时：

- 策略决定当前状态下各动作概率
- $Q^\pi(s,a)$ 评估“此刻执行某动作，长期值不值”
- Bellman 递归把“一步效果”连接到“长期后果”

这也是为什么 RL 不只是监督学习换个损失函数。监督学习通常直接学“输入对应标签”，而 RL 要处理动作会改变未来数据分布，当前决策会影响后续状态和奖励。

## 代码实现

如果环境模型 $P,R$ 已知，最直接的方法是做策略评估，也就是在固定策略 $\pi$ 下求 $V^\pi$。这通常用动态规划完成。

下面是一个可运行的 Python 版本。它实现了前面的单状态单动作 MDP，并用迭代方式逼近价值函数。

```python
def policy_evaluation(states, actions, policy, transitions, gamma, tol=1e-10, max_iter=10000):
    """
    states: 状态列表
    actions: 动作列表
    policy: dict[state][action] -> probability
    transitions: dict[(state, action)] -> list of (prob, next_state, reward)
    gamma: 折扣因子
    return: dict[state] -> V_pi(state)
    """
    V = {s: 0.0 for s in states}

    for _ in range(max_iter):
        delta = 0.0
        new_V = V.copy()

        for s in states:
            value = 0.0
            for a in actions:
                pi = policy[s].get(a, 0.0)
                if pi == 0.0:
                    continue

                expected_return = 0.0
                for prob, next_state, reward in transitions[(s, a)]:
                    expected_return += prob * (reward + gamma * V[next_state])

                value += pi * expected_return

            new_V[s] = value
            delta = max(delta, abs(new_V[s] - V[s]))

        V = new_V
        if delta < tol:
            break

    return V


states = ["s1"]
actions = ["a1"]
policy = {
    "s1": {"a1": 1.0}
}
transitions = {
    ("s1", "a1"): [(1.0, "s1", 5.0)]
}

V = policy_evaluation(states, actions, policy, transitions, gamma=0.9)
assert abs(V["s1"] - 50.0) < 1e-6
print(V)
```

这段代码对应的就是 Bellman 方程：

$$
V^\pi(s)=\sum_a \pi(a\mid s)\sum_{s',r}p(s',r\mid s,a)\left[r+\gamma V^\pi(s')\right]
$$

新手最容易卡住的是“公式看懂了，循环怎么写”。可以直接映射：

| 数学对象 | 代码结构 |
| --- | --- |
| $\sum_s$ | `for s in states` |
| $\sum_a \pi(a\mid s)$ | `for a in actions` 后乘 `pi` |
| $\sum_{s'} P(s'\mid s,a)$ | 遍历 `transitions[(s, a)]` |
| $r+\gamma V(s')$ | `reward + gamma * V[next_state]` |

如果策略是确定性的，比如“在状态 `s` 只走右边”，可以直接把概率写成 1 和 0。  
如果策略是随机的，就把概率分布填进 `policy[s]`。

再给一个更接近真实工程的伪代码，展示如何对多个状态做同步更新：

```python
for each iteration:
    for each state s:
        new_v = 0
        for each action a:
            new_v += pi(a|s) * sum_over_next_state(
                P(s'|s,a) * (R(s,a,s') + gamma * V[s'])
            )
        V_new[s] = new_v
    V = V_new
```

如果环境模型未知，工程里就不能直接枚举 $P$ 和 $R$，而是改用采样轨迹估计价值，这时会转向 Monte Carlo、TD(0)、SARSA 或 Q-learning。

## 工程权衡与常见坑

理论上 Bellman 方程很干净，工程上却不稳定。原因是：你在用“当前估计值”去定义“未来估计值”，递归一旦带噪声，就可能放大误差。

最常见的问题有这些：

| 问题 | 现象 | 原因 | 常见处理 |
| --- | --- | --- | --- |
| $\gamma$ 过大甚至等于 1 | 价值很大或不收敛 | 无限期任务里未来权重过高 | 用 $\gamma<1$，或设置终止状态 |
| 奖励尺度过大 | 训练震荡，梯度爆炸 | Bellman 目标数值太大 | 奖励裁剪、归一化 |
| 稀疏奖励 | 学很久都没信号 | 大多数轨迹回报接近 0 | 奖励塑形、课程学习 |
| 自举误差累积 | 值函数漂移 | 用估计去更新估计 | 目标网络、双重估计 |
| 样本相关性强 | 更新方向抖动 | 连续轨迹高度相关 | 经验回放、打乱采样 |

机器人导航里有一个典型坑。假设每步奖励是 +100/-100 这种大数，$\gamma=0.999$，任务还没有明确终止，那么价值会非常大。因为：

$$
V \approx r + \gamma r + \gamma^2 r + \cdots
$$

如果 $r$ 的量级本来就大，而 $\gamma$ 又非常接近 1，后面的和会被拉得很高。这不一定数学上发散，但会让数值优化非常难做。

一个更稳的做法是：

- 把每步奖励缩放到较小区间，比如 $[-1,1]$
- 把 $\gamma$ 设成 0.95、0.99 这类常见值
- 明确 episode 终止条件
- 如果用了神经网络 critic，再加目标网络和经验回放

训练循环里稳定化手段通常插在这里：

```text
采样状态转移
-> 写入 replay buffer
-> 随机抽样 batch
-> 用 target network 计算 Bellman 目标
-> 更新 value / Q 网络
-> 周期性同步 target network
```

这里“目标网络”的白话解释是“单独维护一个更新更慢的参考副本”，避免你每一步都追着自己刚改过的值跑。  
“经验回放”的白话解释是“把过去样本存起来，打散后再训练”，减少样本之间的强相关。

还有一个常见坑是把 $V^\pi$、$Q^\pi$ 和最优价值 $V^*, Q^*$ 混在一起。  
带 $\pi$ 的价值函数是在“固定某个策略”下定义的；带星号的是“所有策略里最优的那个值”。基础阶段先把策略评估看明白，再谈控制问题更稳。

## 替代方案与适用边界

当环境模型已知时，动态规划最直接；当模型未知时，就要靠采样估计。常见替代方法可以这样比较：

| 方法 | 更新频率 | 方差/偏差 | 是否需要完整回合 | 适用场景 |
| --- | --- | --- | --- | --- |
| Monte Carlo | 回合结束后更新 | 方差高，偏差低 | 需要 | 回合短、终止明确 |
| TD(0) | 每一步更新 | 方差低，偏差高 | 不需要 | 在线学习、持续任务 |
| SARSA | 每一步更新 | 较稳，偏保守 | 不需要 | 关心执行中探索风险 |
| Q-learning | 每一步更新 | 偏向最优，可能更激进 | 不需要 | 离策略控制、离线复用数据 |
| Actor-Critic | 每一步更新 | 折中 | 不需要 | 高维状态、策略参数化 |

先看 Monte Carlo 和 TD(0) 的区别。

Monte Carlo 的思路是：等整条轨迹跑完，再用真实总回报更新。  
例如导航任务里，机器人从起点走到终点一共 20 步，最终回报算出来以后，再回头更新每个经过状态的价值。

TD(0) 的思路是：不等走完整局，只看一步：

$$
V(s_t)\leftarrow V(s_t)+\alpha\left[r_{t+1}+\gamma V(s_{t+1})-V(s_t)\right]
$$

这里的 TD 误差，白话解释是“新观察到的一步结果，和原先估计之间的差”。

导航任务里：

- Monte Carlo：到终点后再统一复盘，信号真实，但等得久
- TD(0)：每走一步就更新，反应快，但用了当前估计值做目标，会带来偏差

确定性策略和随机策略也有边界差异。

- 确定性策略：每个状态只选一个动作。优点是执行稳定，适合部署阶段。
- 随机策略：每个状态输出动作分布。优点是能探索，适合训练初期或多峰动作空间。

在真实工程里，如果你已经知道动作基本唯一，比如工业控制里某些安全约束很强的场景，确定性策略更自然。  
如果任务需要持续探索，比如推荐系统要试新内容、机器人要在未知地图里搜索，随机策略通常更合适。

Actor-Critic 适合什么时候上？  
当状态空间很大，无法给每个状态单独存一个表格值函数时，就要用函数逼近。Actor 负责学策略，Critic 负责学价值。Critic 提供更低方差的学习信号，Actor 再据此改策略。这类方法是从基础 MDP、策略、价值函数自然延伸出来的，不是另一套理论。
