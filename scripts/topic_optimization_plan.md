## 二、主题生成策略

### 四维覆盖原则
每个技术主题从四个维度展开：
1. **原理深度** - 数学推导、理论证明、直觉解释
2. **代码实现** - 从零实现、框架使用、生产代码
3. **工程实践** - 踩坑经验、性能调优、最佳实践
4. **前沿进展** - 最新改进、替代方案、研究趋势

### 主题粒度原则
- 每个主题聚焦一个技术点
- 可以拆分的不合并（如：Adam/AdamW 分开）
- 从不同角度可以重复覆盖
- 前沿模型每个写一篇

### 专家级质量门槛（新增）
- `brief` 必须给出可验证技术信息：公式、复杂度、协议约束、系统指标、实验结论至少一项。
- `depth_hint` 必须是可执行任务，包含动作与产出（如“推导/实现/对比/压测 + 指标”）。
- `tags[0]` 必须落在 18 个一级分类白名单中，不允许新增自由一级标签。
- `slug` 全库唯一；标题允许相近但不允许语义完全重复。
- 涉及时效主题（如“趋势/最新/年度展望”）必须写明时间范围与失效策略。

## 五、生成脚本使用

### 批量生成新主题
```bash
# 生成指定领域的主题
python3 scripts/generate_topics.py --domain "模型架构" --count 100

# 生成所有领域的平衡主题
python3 scripts/generate_topics.py --balance

# 更新现有主题的 depth_hint
python3 scripts/update_depth_hints.py
```

### 主题质量控制（硬门禁）
- `brief` 最低 60 字，且必须包含可验证技术点（公式/复杂度/协议/指标/实验至少一项）。
- `depth_hint` 最低 22 字，必须包含执行动作和可交付结果。
- `slug` 必须全局唯一；禁止语义重复题目仅改措辞。
- `tags[0]` 必须在 18 个一级分类白名单中。
- 趋势/年度类主题必须含时间锚点和复审日期。

### 快速审计命令（建议每次批量生成后执行）
```bash
# slug 重复
jq -r '.[].slug' scripts/topic_queue.json | sort | uniq -d

# 一级标签分布
jq -r 'group_by(.tags[0])[] | "\(.[0].tags[0])\t\(length)"' scripts/topic_queue.json | sort

# 质量信号：brief 过短
jq -r '[.[] | select((.brief|length)<40)] | length' scripts/topic_queue.json

# 质量信号：depth_hint 过短
jq -r '[.[] | select((.depth_hint|length)<18)] | length' scripts/topic_queue.json
```
