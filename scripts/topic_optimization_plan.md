## 二、主题生成策略

### 四维覆盖原则
每个技术主题从四个维度展开：
1. **原理深度** - 数学推导、理论证明、直觉解释
2. **代码实现** - 从零实现、框架使用、生产代码
3. **工程实践** - 踩坑经验、性能调优、最佳实践
4. **前沿进展** - 最新改进、替代方案、研究趋势

### 主题粒度原则
- 每个主题聚焦一个技术点
- 可以拆分的不合并（如：Adam/AdamW 分开）
- 从不同角度可以重复覆盖
- 前沿模型每个写一篇

## 四、生成脚本使用

### 批量生成新主题
```bash
# 生成指定领域的主题
python3 scripts/generate_topics.py --domain "模型架构" --count 100

# 生成所有领域的平衡主题
python3 scripts/generate_topics.py --balance

# 更新现有主题的 depth_hint
python3 scripts/update_depth_hints.py
```

### 主题质量控制
- 每个 brief 必须包含技术要点
- 每个 depth_hint 必须具体可执行
- slug 必须唯一且有意义
- tags 第一个是主分类
